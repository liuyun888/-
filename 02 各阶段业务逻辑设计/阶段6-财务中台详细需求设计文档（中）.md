## 二、结算管理模块

### 2.1 结算单生成

#### 2.1.1 功能概述

系统根据对账完成的保单数据，按照业务员/团队维度自动汇总应发佣金，生成结算单。结算单是发放佣金的依据，须经过审核后才能进入打款流程。

#### 2.1.2 功能入口

PC管理后台 → 财务中台 → 结算管理 → 结算单管理，进入结算单列表页。

#### 2.1.3 结算单列表页

筛选条件：
- 结算月份（月份选择器，默认上月）
- 业务员姓名/工号（文本输入）
- 结算单号（文本输入）
- 结算状态（下拉：待审核/审核中/审核通过/审核驳回/已打款/已作废）

列表字段：

| 字段 | 说明 |
|------|------|
| 结算单号 | 格式：SET+年月+4位序号，如SET2025010001 |
| 业务员姓名 | |
| 工号 | |
| 所属团队 | |
| 结算月份 | YYYY-MM |
| 保单数量 | 本结算单包含的保单笔数 |
| 应结佣金 | 税前佣金总额 |
| 个税金额 | 应代扣个税 |
| 实发金额 | 应结佣金 - 个税金额 |
| 结算状态 | 带颜色状态标签 |
| 生成方式 | 自动生成/手动生成 |
| 创建时间 | |
| 操作 | 查看 / 提交审核 / 作废（条件显示） |

页面右上角提供 **"生成结算单"** 按钮和 **"批量导出"** 按钮。

#### 2.1.4 自动生成结算单

**触发时机**：每月5日凌晨2点，系统定时任务（XXL-Job）自动执行上月结算单生成。

**生成逻辑（后端）：**

1. 查询对账状态为"已完成"且结算月份 = 上月的所有导入批次。
2. 从 `finance_import_detail`中查询 match_status IN (1,2)（已匹配）的保单明细。
3. 按 `system_order_id` 关联系统订单，获取业务员ID、保单号、佣金金额等信息。
4. 按业务员维度聚合：
   ```sql
   SELECT 
     agent_id, agent_name, agent_no, team_id,
     COUNT(*) AS policy_count,
     SUM(commission_amount) AS total_commission
   FROM (已匹配保单明细 JOIN 系统订单)
   WHERE reconcile_month = '上月'
   GROUP BY agent_id
   ```
5. 对每个业务员调用个税计算服务（见3.1节），得到应扣个税。
6. 逐一插入 `finance_settlement`（结算单主表）和 `finance_settlement_detail`（结算单明细表）。
7. 若某业务员本月应结佣金为0，不生成结算单。
8. 生成完毕后，在系统通知中推送"XX月结算单已生成，共XX条，请及时提交审核"。

**手动生成（"生成结算单"按钮）：**

点击后弹出对话框：

| 字段 | 类型 | 是否必填 | 说明 |
|------|------|---------|------|
| 结算月份 | 月份选择器 | 必填 | 不能选择未来月份 |
| 业务员范围 | 单选 | 必填 | 全部业务员 / 指定业务员（多选） |
| 指定业务员 | 多选下拉 | 条件必填 | 选择"指定业务员"时必填 |
| 是否覆盖已有结算单 | 开关 | 必填 | 默认否，若该业务员本月已有结算单且选择否，则跳过 |

点击确认后，后台异步执行，生成逻辑与自动生成相同，完成后提示"已生成XX条结算单"。

### 2.2 结算审核

#### 2.2.1 功能概述

结算单生成后，需经财务主管审核通过才能进入打款流程。使用Activiti工作流实现多级审核，支持配置一级或二级审核。

#### 2.2.2 提交审核

**入口**：结算单列表中，状态为"待审核"的记录，点击"提交审核"按钮（也可以批量勾选批量提交）。

**后端处理**：
1. 校验结算单状态必须为0（待审核）。
2. 校验结算单的业务员银行卡信息是否已维护（从业务员档案获取），若未配置银行卡，则提示"请先维护业务员银行卡信息后再提交"，阻止提交。
3. 启动Activiti流程：`financeSettlementApprove`流程定义，传入variables: `{settlementId, agentName, totalCommission, actualAmount}`。
4. 更新结算单状态为1（审核中），记录流程实例号到approve_no字段。
5. 向审核人推送待办通知。

#### 2.2.3 审核页面

审核人（财务主管角色）在PC后台的"我的待办"中看到结算审核任务，点击进入审核详情页。

**审核详情页包含：**

**基本信息区：**
- 结算单号、业务员、结算月份、生成时间

**金额信息区：**
- 保单数量、应结佣金（税前）、代扣个税、实发金额

**保单明细表（分页，每页20条）：**
- 保单号、投保人、产品名称、保费、佣金率、佣金金额、对账月份

**审核操作区（底部固定）：**

| 字段 | 类型 | 是否必填 | 说明 |
|------|------|---------|------|
| 审核意见 | 单选 | 必填 | 通过 / 驳回 |
| 审核备注 | 文本域 | 驳回时必填 | 最多500字 |

点击"确认"按钮提交审核结果：
- 通过：Activiti complete task，若还有下一级审核则流转至下一级；若无则结算单状态改为2（审核通过），推送通知给财务人员"可以进行打款操作"。
- 驳回：Activiti终止流程，结算单状态改为3（审核驳回），推送通知给操作人"结算单XXX被驳回，原因：xxx"。

被驳回的结算单，操作人可点击"修改后重新提交"（跳转至结算单编辑页，允许手动调整佣金金额后重新提交审核）。

#### 2.2.4 审核记录查看

结算单详情页底部展示完整的审核流水记录（审核人、操作、时间、备注）。

### 2.3 发票管理

#### 2.3.1 功能概述

结算单审核通过后，系统支持向业务员（若其为个体工商户或公司主体）开具电子发票，或记录业务员向平台提交的发票信息（平台代为申报）。

#### 2.3.2 功能入口

PC管理后台 → 财务中台 → 结算管理 → 发票管理。

#### 2.3.3 发票管理列表页

筛选条件：发票申请月份、发票状态（待申请/申请中/已开具/已作废）、业务员、结算单号。

列表字段：

| 字段 | 说明 |
|------|------|
| 发票申请号 | 格式：INV+年月日+序号 |
| 关联结算单号 | 对应的结算单 |
| 发票类型 | 增值税专用发票/增值税普通发票/收据 |
| 开票主体 | 发票抬头（公司名称或个人） |
| 含税金额 | 开票金额 |
| 税率 | 增值税税率 |
| 不含税金额 | 含税金额/(1+税率) |
| 税额 | 含税金额 - 不含税金额 |
| 发票状态 | |
| 发票文件 | 电子发票PDF链接 |
| 操作 | 查看 / 上传发票 / 作废 |

#### 2.3.4 申请开票流程

**入口**：在结算单列表，状态为"审核通过"的结算单，点击"申请开票"。

**弹出开票申请对话框：**

| 字段 | 类型 | 是否必填 | 说明 |
|------|------|---------|------|
| 关联结算单 | 只读 | - | 自动带入结算单号和金额 |
| 发票类型 | 单选 | 必填 | 增值税专用发票 / 增值税普通发票 |
| 开票抬头 | 文本 | 必填 | 公司名称或个人姓名 |
| 统一社会信用代码 | 文本 | 专票必填 | 15/18位 |
| 纳税人识别号 | 文本 | 专票必填 | |
| 开票金额（含税） | 数字 | 必填 | 默认带入结算单实发金额，可调整 |
| 税率 | 下拉 | 必填 | 1%/3%/6%/9%/13%，默认6% |
| 开票内容 | 文本 | 必填 | 默认"代理服务费" |
| 收票邮箱 | 文本 | 必填 | 用于接收电子发票 |

提交后：
1. 校验金额不得超过对应结算单实发金额。
2. 生成发票申请记录，状态"待开具"。
3. 若接入了发票API（如百旺、航信），则调用API自动开票，异步获取发票文件URL更新记录。
4. 若未接入API，则工单推送给财务人员手动处理，财务人员开完票后通过"上传发票"功能上传PDF。

**上传发票：**
- 点击"上传发票"按钮，弹出文件上传框，支持PDF，最大5MB，上传后存OSS，更新发票文件路径，状态改为"已开具"。

#### 2.3.5 发票作废

点击"作废"按钮，弹出确认框，输入作废原因（必填），确认后：
1. 发票状态改为"已作废"，记录作废时间和原因。
2. 对应结算单的发票状态重置为"未开票"，允许重新申请。
3. 若已通过API开票，同步调用发票API的冲红接口。

### 2.4 打款管理

#### 2.4.1 功能概述

审核通过的结算单，由财务人员发起打款操作，支持批量打款，系统生成银行批量付款文件或调用银行API，完成佣金实际划拨。

#### 2.4.2 功能入口

PC管理后台 → 财务中台 → 结算管理 → 打款管理。

#### 2.4.3 打款列表页

筛选条件：结算月份、业务员、打款状态（待打款/打款中/已打款/打款失败）、打款批次号。

列表字段：

| 字段 | 说明 |
|------|------|
| 打款批次号 | 格式：PAY+年月日+序号 |
| 结算单号 | |
| 业务员姓名/工号 | |
| 结算月份 | |
| 实发金额 | 本次打款金额 |
| 收款银行 | 业务员银行卡所在银行 |
| 收款卡号 | 脱敏展示，如 622202****1234 |
| 打款状态 | |
| 打款时间 | |
| 操作 | 查看回单 / 重试（失败时显示） |

#### 2.4.4 发起打款操作

**步骤一：勾选结算单**

在结算单列表（筛选状态=审核通过），勾选需要打款的结算单（可多选），点击顶部"发起打款"按钮。

**步骤二：确认打款信息**

弹出打款确认对话框，展示：
- 本次打款总笔数：XX笔
- 本次打款总金额：¥XX.XX
- 明细列表：每行展示 结算单号 / 业务员 / 实发金额 / 收款卡号（脱敏）

底部操作：
| 字段 | 类型 | 是否必填 | 说明 |
|------|------|---------|------|
| 打款方式 | 单选 | 必填 | 批量文件导出（手动上传银行系统）/ API自动打款 |
| 付款账户 | 下拉 | 必填 | 公司付款银行账户，来源于系统配置 |
| 备注 | 文本 | 非必填 | |

**步骤三：确认执行**

点击"确认打款"按钮：

**若选择"批量文件导出"：**
1. 生成打款批次记录（status=打款中）。
2. 生成银行批量付款Excel文件（格式遵循平安银行/招商银行等标准格式），文件存OSS。
3. 弹出"文件已生成，请下载后上传至银行系统"提示，提供下载链接。
4. 页面出现"确认已完成上传"按钮，点击后将打款状态改为"已打款"（人工确认模式）。

**若选择"API自动打款"：**
1. 生成打款批次记录（status=打款中）。
2. 调用银行API批量转账接口（使用银行SDK，每笔参数：付款账号、收款账号、收款户名、金额、摘要）。
3. API异步回调处理：
   - 成功回调：更新结算单status=4（已打款），记录paid_time，更新打款明细状态。
   - 失败回调：更新打款明细状态=打款失败，记录失败原因，发送告警通知财务人员。

## 三、税务管理模块

### 3.1 个税计算

#### 3.1.1 业务背景

保险中介平台在向业务员支付佣金时，需依法代扣代缴个人所得税。佣金所得属于"劳务报酬所得"，适用预扣率表。系统根据每次应发金额自动计算应扣税额。

#### 3.1.2 个税计算规则

**劳务报酬所得预扣率表（2024年有效）：**

| 每次收入额 | 预扣率 | 速算扣除数 |
|-----------|-------|----------|
| 不超过20,000元 | 20% | 0 |
| 超过20,000元不超过50,000元 | 30% | 2,000 |
| 超过50,000元 | 40% | 7,000 |

**每次收入额计算规则：**
- 每次收入不超过4,000元：收入额 = 收入 - 800元（定额扣除）
- 每次收入超过4,000元：收入额 = 收入 × (1 - 20%)（按比例扣除）

**应扣税额 = 收入额 × 预扣率 - 速算扣除数**

**示例：**
- 本月应结佣金 = 30,000元
- 收入额 = 30,000 × (1 - 20%) = 24,000元
- 预扣率 = 30%，速算扣除数 = 2,000
- 应扣税额 = 24,000 × 30% - 2,000 = 5,200元
- 实发金额 = 30,000 - 5,200 = 24,800元

### 3.2 税务申报

#### 3.2.1 功能概述

每月完成代扣税款后，系统生成符合税务局要求的申报数据文件，供财务人员下载后上传至自然人电子税务局（扣缴端）完成申报。

#### 3.2.2 功能入口

PC管理后台 → 财务中台 → 税务管理 → 税务申报。

#### 3.2.3 申报列表页

筛选：申报月份、申报状态（待申报/申报中/已申报）。

列表字段：

| 字段 | 说明 |
|------|------|
| 申报批次号 | TAX+年月+序号 |
| 申报月份 | YYYY-MM |
| 申报人数 | 代扣人数 |
| 代扣税额合计 | |
| 申报状态 | |
| 申报文件 | 下载链接 |
| 操作 | 生成申报文件 / 标记已申报 / 查看明细 |

#### 3.2.4 生成申报文件

点击"生成申报文件"按钮：
1. 弹出确认框，展示本月参与申报的人数和代扣税额合计。
2. 选择申报月份（默认上月），确认。
3. 后端执行：
   - 查询该月所有status=4（已打款）的结算单对应的tax_record记录。
   - 生成两份文件：
     - **申报汇总表**（Excel）：扣缴义务人信息 + 汇总税额
     - **申报明细表**（Excel/固定格式）：每个业务员一行，字段包括：姓名、身份证号、所得项目（劳务报酬所得）、本期收入、本期税额
   - 文件格式严格按照《个人所得税扣缴申报管理办法》附表格式生成（EasyExcel模板填充）。
   - 文件存OSS，生成下载链接，有效期7天。
4. 记录申报批次，状态改为"申报中"。

**标记已申报：**
财务人员完成手动上传税务局后，回到列表点击"标记已申报"，录入：
- 申报日期（日期选择器，必填）
- 申报凭证号（文本，非必填）
- 上传申报截图（图片，非必填）

确认后状态改为"已申报"，记录时间和凭证信息。
