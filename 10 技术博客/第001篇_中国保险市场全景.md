## 一、保险的本质——三个核心概念

> 💡 **为什么要先讲这个？**  
> 很多工程师接到保险项目，直接就开始建表写接口。结果做到一半发现：为什么保费不是一个固定数？为什么同一款产品不同人的价格不一样？为什么有人投保会被"拒"？  
> 这一切的根源，都在保险的三个本质概念里。

---

### 1.1 大数法则（Law of Large Numbers）

**通俗解释**：一个人今年会不会出车祸，谁也不知道。但如果你观察 100 万个司机，今年出车祸的比例是可以精确预测的。

**数学表述**：

```
当样本量 n → ∞ 时：
样本均值 X̄ → 总体均值 μ（概率收敛）

即：大量独立同分布的随机事件，其平均结果趋近于期望值
```

**在保险中的应用**：

```
某城市有 100 万辆私家车
历史数据：每年出险率 = 8%（即每年约 8 万辆发生事故）
平均每次事故赔付 = 5,000 元

则：
  理论年赔付总额 = 8万辆 × 5,000元 = 4亿元
  理论人均保费（纯保费）= 4亿元 ÷ 100万辆 = 400元/年

保险公司只需收取 400 元保费（加上费用和利润附加），
就能覆盖所有人的风险损失。
```

**对系统设计的三个启示**：

- **为什么要"核保"**？→ 过滤高风险个体，防止"逆向选择"（只有生病的人才买医疗险，导致实际出险率远高于预测值，保险公司亏损）
- **为什么新险种定价很难**？→ 样本量不足，大数法则不成立，精算误差大，所以新产品初期保费通常偏高
- **为什么系统要存储大量历史数据**？→ 保险公司依赖历史数据不断修正精算模型，这些数据是核心资产

---

### 1.2 风险转移（Risk Transfer）

**保险的核心交易逻辑**：

```
投保人 ──（支付保费）──→ 保险公司
投保人 ←──（承担风险）─── 保险公司

本质：
投保人用一笔确定的小额损失（保费），
换取了对一个不确定的大额损失（赔偿）的保障。
```

**风险转移的三个必要条件**：

| 条件 | 解释 | 违反则... |
|------|------|-----------|
| 风险必须是不确定的 | 已知必然发生的损失不能投保 | 已经确诊才想买重疾险 → 拒保 |
| 风险必须是可计量的 | 损失必须能用货币衡量 | 精神痛苦一般不在赔偿范围内 |
| 风险必须是意外的 | 故意造成的损失不赔 | 故意纵火 → 拒赔 |

**对系统设计的三个启示**：

- **健康告知功能存在的原因**：识别"既往症"（已经发生的风险），排除"带病投保"
- **等待期设计的原因**：防止"冲着理赔来"的投保行为（买了保险立刻出险）
- **核保逻辑的原因**：识别高风险个体，差异化定价或直接拒保

---

### 1.3 精算定价（Actuarial Pricing）

精算师（Actuary）是保险公司的"定价大脑"，负责回答一个核心问题：**这个保险产品，应该收多少钱？**

**基本定价公式**：

```
保费（Premium）= 纯保费（Pure Premium）+ 附加保费（Loading）

纯保费 = 预期赔付成本
       = 出险概率 × 平均赔付金额

附加保费 = 运营费用 + 利润 + 安全边际
```

**以百万医疗险为例，用 Java 代码演示定价逻辑**：

```java
import java.math.BigDecimal;
import java.math.RoundingMode;

/**
 * 医疗险精算定价逻辑（教学示例）
 *
 * 说明：真实精算模型比这复杂得多，此处仅演示核心计算思路。
 * 包名使用通用示例命名，不对应任何真实项目。
 */
public class MedicalInsurancePricingDemo {

    public static void main(String[] args) {
        // 示例：计算一位 35 岁男性的百万医疗险保费
        InsuredProfile profile = new InsuredProfile();
        profile.age = 35;
        profile.gender = "M";
        profile.occupation = "office";   // 白领
        profile.region = "tier1";        // 一线城市

        BigDecimal premium = calculatePremium(profile);
        System.out.println("预估年保费：" + premium + " 元");
        // 输出：预估年保费：416.00 元
    }

    static BigDecimal calculatePremium(InsuredProfile profile) {
        // ① 基础发病率（来自精算生命表，此处简化为示例值）
        //    35岁男性一年内需要住院的概率约 2.5%
        double baseMorbidityRate = 0.025;

        // ② 职业系数（高危职业风险更高）
        double occupationFactor = getOccupationFactor(profile.occupation);

        // ③ 地区系数（各地医疗费用水平不同）
        double regionFactor = getRegionFactor(profile.region);

        // ④ 平均住院费用（来自医院历史数据统计，示例值）
        BigDecimal avgClaimAmount = new BigDecimal("12500");

        // ⑤ 纯保费 = 发病率 × 职业系数 × 地区系数 × 平均赔付额
        BigDecimal purePremium = avgClaimAmount
                .multiply(BigDecimal.valueOf(baseMorbidityRate))
                .multiply(BigDecimal.valueOf(occupationFactor))
                .multiply(BigDecimal.valueOf(regionFactor));
        // 12500 × 0.025 × 1.0 × 1.2 = 375 元

        // ⑥ 附加费用率（通常 20%~35%，含运营成本和利润）
        BigDecimal loadingRate = new BigDecimal("0.10");

        // ⑦ 最终保费 = 纯保费 ÷ (1 - 附加费用率)
        BigDecimal finalPremium = purePremium
                .divide(BigDecimal.ONE.subtract(loadingRate), 2, RoundingMode.CEILING);
        // 375 ÷ 0.90 ≈ 416 元

        return finalPremium;
    }

    static double getOccupationFactor(String occupation) {
        switch (occupation) {
            case "office": return 1.0;  // 白领，标准风险
            case "labor":  return 1.3;  // 体力劳动，风险较高
            case "hazard": return 1.8;  // 高危职业（采矿/建筑）
            default:       return 1.0;
        }
    }

    static double getRegionFactor(String region) {
        switch (region) {
            case "tier1": return 1.2;   // 一线城市医疗费用更贵
            case "tier2": return 1.0;   // 二线城市，标准系数
            case "tier3": return 0.85;  // 三线及以下
            default:      return 1.0;
        }
    }

    static class InsuredProfile {
        int age;
        String gender;
        String occupation;
        String region;
    }
}
```

**对系统设计的三个启示**：

- 保险产品的保费不是一个固定值，而是**动态计算**的 → 系统需要**费率引擎**
- 不同人的保费不同（性别、年龄、职业、地区等因素）→ 数据库不能只存一个"单价"字段
- 费率表会随时更新 → 需要**版本管理**，历史保单的保费必须锁定在投保时的费率版本

## 二、中国保险市场规模——数字说话

### 2.1 三大核心指标定义

在保险行业，衡量市场发展水平有三个标准指标，工作中经常用到：

| 指标名称 | 计算公式 | 含义 | 单位 |
|---------|---------|------|------|
| **保费收入** | 当年承保的所有保费总额 | 衡量市场绝对规模 | 元 |
| **保险密度** | 年保费收入 ÷ 总人口 | 人均花多少钱买保险 | 元/人·年 |
| **保险深度** | 年保费收入 ÷ GDP × 100% | 保险在经济中的渗透率 | % |

### 2.2 中国保险市场现状数据

```
【2023年中国保险业关键数据】

总保费收入：约 5.1 万亿元（位居全球第二）

│   ├── 寿险：      约 2.4 万亿元
```

### 2.3 市场规模对系统设计的意义

理解市场规模，有助于做出正确的技术选型决策：

```
问题：我们的系统需要多大的并发量？

推算逻辑：
全国保险代理人 ≈ 300 万人（活跃从业者）
假设平台服务 1 万名代理人（0.33% 市场份额）
工作日高峰期（上午10点）同时在线比例 ≈ 20%
→ 约 2,000 并发用户

报价操作 TPS 估算：
2,000人 × 平均每人每分钟操作 0.5 次 ÷ 60秒 ≈ 17 TPS

结论：
保险中介平台不是高并发场景（不需要像电商秒杀一样扛万级TPS）
但对数据准确性要求极高（保费计算错误 → 潜在法律纠纷）
设计重心：数据一致性 > 极限吞吐量
```

## 三、监管体系——每行代码都有监管依据

> 🚨 **特别提示**  
> 在金融行业做系统，不了解监管等于盲人骑马。很多看起来"奇怪"的产品设计决策（"为什么要录像？""为什么要等10天？"），背后都有明确的监管规定。理解这些规定，才能写出有说服力的技术方案。

### 4.1 保险产品总体分类

```
保险产品
├── 人身险（以"人"为保险标的）
│   ├── 寿险（4种）
│   ├── 健康险（3种）
│   └── 意外险
│
├── 财产险（以"财产/责任"为保险标的）
│   ├── 车险（交强险 + 商业车险）
│   ├── 企业财产险
│   ├── 责任险
│   ├── 农业险
│   └── 其他财产险
│

```

### 4.2 人身险详解

#### 4.2.1 寿险（Life Insurance）

寿险以被保险人的"生死"为保险事件：

```
【寿险四大类型对比】

① 普通寿险
   ├── 定期寿险：在规定期间内身故赔付，期满不返还保费
   │   示例：30年定期寿险，保额100万，年缴约2,000元
   │   系统特点：定价简单，保单有效期固定，不涉及账户价值管理
   │
   └── 终身寿险：无论何时身故均赔付
       示例：终身寿险，保额100万，年缴约8,000元
       系统特点：保单永不到期，需要管理"现金价值"字段

② 分红保险
   特点：保费固定，每年有可能分红（不保证，视保司经营情况）
   系统特点：
   ├── 每年须计算并发放"红利"
   ├── 系统需存储每位客户的分红历史记录
   └── 销售时必须明确告知"红利不保证"（监管强制要求）

③ 万能保险（Universal Life）
   特点：
   ├── 保费灵活（在最低缴费要求以上，可随时追加）
   ├── 账户价值每月结算（有最低保证利率，通常1.75%/年）
   └── 保额可申请调整
   系统特点（最复杂的险种之一）：
   ├── 需要维护每人的"个人账户"
   ├── 每月计算利息并写入账户
   └── 保单价值 = 已缴保费 - 风险保费 - 费用 + 累计利息

④ 投资连结保险（投连险）
   特点：
   ├── 保费与投资基金挂钩，保单价值随市场波动
   ├── 不保证本金，高风险高收益
   └── 监管要求最严（销售必须双录）
   系统特点：
   ├── 需要每日同步基金净值
   ├── 每日更新保单账户价值
   └── 销售合规成本最高
```

#### 4.2.2 健康险（Health Insurance）

```
【健康险主要类型】

① 百万医疗险（最常见）
   保障：住院及手术医疗费用报销（通常100万~600万限额）
   关键参数：
   ├── 免赔额：通常1万元（超出部分才报销）
   ├── 报销比例：通常80%~100%
   └── 续保条件：保险公司有权拒绝续保（消费者最大痛点）
   系统要点：需管理免赔额、报销比例、续保状态等多个维度

② 重大疾病保险（重疾险）
   保障：确诊规定重大疾病（通常100种以上），一次性现金赔付
   关键参数：
   ├── 等待期：通常90~180天（等待期内患病不赔）
   ├── 疾病分级：
   │   ├── 轻症：赔付保额的20%~30%（如原位癌）
   │   ├── 中症：赔付保额的50%~60%
   │   └── 重症：赔付保额的100%（如恶性肿瘤、心梗、脑中风）
   └── 多次赔付：部分产品允许轻症/重症分别赔付多次
   系统要点：疾病分类码表管理、等待期校验、多次赔付状态机

③ 护理险（Long-term Care Insurance）
   背景：国家大力推广的政策性险种，为长期护理费用提供保障
   特点：与年龄高度相关，老龄化加剧推动需求增长
```

#### 4.3.2 商业车险费率计算（工程师必须理解的公式）

**2020年"二次费改"后的定价公式**：

```
商业车险保费 = 基准纯风险保费 × NCD系数 × 自主定价系数

其中：
  基准纯风险保费 → 根据车辆品牌/型号/价格，查行业费率表
  NCD系数       → 根据历史出险记录，有折扣或上浮
  自主定价系数  → 各保险公司自行制定（范围 0.65 ~ 1.35）
```

**完整的 Java 实现示例**：

```java
import java.math.BigDecimal;
import java.math.RoundingMode;

/**
 * 商业车险保费计算器（教学示例）
 *
 * 注意：真实保司系统的计算逻辑更复杂，此处演示核心公式。
 */
public class CommercialCarInsuranceCalculator {

    public static void main(String[] args) {
        // 示例：2020年某款B级车，连续3年未出险，某保司自主系数0.80
        CarInfo car = new CarInfo();
        car.brandModel = "某B级轿车 2020款";
        car.vehiclePrice = 200000;   // 车辆购置价 20万

        ClaimHistory history = new ClaimHistory();
        history.continuousNcdYears = 3;  // 连续3年未出险
        history.claimsLastYear = 0;      // 上一年未出险

        double insurerAutonomyFactor = 0.80; // 保司自主系数

        CommercialCarInsuranceCalculator calculator =
                new CommercialCarInsuranceCalculator();
        PremiumResult result = calculator.calculate(car, history, insurerAutonomyFactor);

        System.out.println("=== 商业车险保费计算结果 ===");
        System.out.println("车辆：" + car.brandModel);
        System.out.printf("NCD 系数：%.2f%n", result.ncdFactor);
        System.out.printf("自主系数：%.2f%n", insurerAutonomyFactor);
        System.out.println("---");
        System.out.println("车损险保费：¥" + result.vehicleDamagePremium);
        System.out.println("三者险保费（100万限额）：¥" + result.thirdPartyPremium);
        System.out.println("商业险合计：¥" + result.totalCommercial);
        System.out.println("交强险（固定）：¥" + result.compulsoryPremium);
        System.out.println("===========================");
        System.out.println("应付总保费：¥" + result.grandTotal);
    }

    public PremiumResult calculate(CarInfo car,
                                   ClaimHistory history,
                                   double autonomyFactor) {
        PremiumResult result = new PremiumResult();

        // ① 计算 NCD 系数
        result.ncdFactor = calculateNcdFactor(history.continuousNcdYears,
                                              history.claimsLastYear);

        // ② 查基准纯风险保费（简化：按车价比例估算）
        BigDecimal baseVehicleDamage = calculateBaseRate(car.vehiclePrice, "vehicle_damage");
        BigDecimal baseThirdParty    = calculateBaseRate(car.vehiclePrice, "third_party_100w");

        // ③ 各险种保费 = 基准 × NCD系数 × 自主系数
        result.vehicleDamagePremium = applyFactors(baseVehicleDamage,
                result.ncdFactor, autonomyFactor);
        result.thirdPartyPremium = applyFactors(baseThirdParty,
                result.ncdFactor, autonomyFactor);

        // ④ 交强险固定费率（家用6座以下私家车）
        result.compulsoryPremium = new BigDecimal("950.00");

        // ⑤ 汇总
        result.totalCommercial = result.vehicleDamagePremium
                .add(result.thirdPartyPremium);
        result.grandTotal = result.totalCommercial
                .add(result.compulsoryPremium);

        return result;
    }

    /**
     * NCD 系数（无赔款优待系数）计算规则
     *
     * 连续无出险年数  → NCD系数（享受折扣）
     *   0年（新车）  → 1.00
     *   1年无出险    → 0.90（九折）
     *   2年无出险    → 0.80
     *   3年无出险    → 0.70
     *   4年无出险    → 0.60
     *   5年及以上    → 0.50（最低五折）
     *
     * 上一年有出险   → 上浮（惩罚）
     *   出险1次      → 1.00（取消所有优惠）
     *   出险2次      → 1.25（上浮25%）
     *   出险3次及以上 → 1.50（上浮50%）
     */
    private double calculateNcdFactor(int continuousNcdYears, int claimsLastYear) {
        if (claimsLastYear >= 3) return 1.50;
        if (claimsLastYear == 2) return 1.25;
        if (claimsLastYear == 1) return 1.00;
        switch (continuousNcdYears) {
            case 0:  return 1.00;
            case 1:  return 0.90;
            case 2:  return 0.80;
            case 3:  return 0.70;
            case 4:  return 0.60;
            default: return 0.50;
        }
    }

    /** 简化版基准保费估算（真实系统查费率表） */
    private BigDecimal calculateBaseRate(int vehiclePrice, String coverageType) {
        double rate;
        switch (coverageType) {
            case "vehicle_damage":
                rate = vehiclePrice * 0.009;  // 车损险基准费率约为车价的 0.9%
                break;
            case "third_party_100w":
                rate = 1500;  // 三者险100万限额，与车价关联较弱，此处固定示例值
                break;
            default:
                rate = 0;
        }
        return BigDecimal.valueOf(rate).setScale(2, RoundingMode.HALF_UP);
    }

    private BigDecimal applyFactors(BigDecimal base, double ncd, double autonomy) {
        return base
                .multiply(BigDecimal.valueOf(ncd))
                .multiply(BigDecimal.valueOf(autonomy))
                .setScale(2, RoundingMode.CEILING);
    }

    // ---- 数据模型 ----

    static class CarInfo {
        String brandModel;
        int vehiclePrice;
    }

    static class ClaimHistory {
        int continuousNcdYears;
        int claimsLastYear;
    }

    static class PremiumResult {
        double ncdFactor;
        BigDecimal vehicleDamagePremium;
        BigDecimal thirdPartyPremium;
        BigDecimal compulsoryPremium;
        BigDecimal totalCommercial;
        BigDecimal grandTotal;
    }
}

车辆：某B级轿车 2020款
NCD 系数：0.70
自主系数：0.80
---
车损险保费：¥1008.00
三者险保费（100万限额）：¥840.00
商业险合计：¥1848.00
交强险（固定）：¥950.00
===========================
应付总保费：¥2798.00
*/
```

### 4.4 再保险（Reinsurance）——了解概念即可

```
直接保险关系：
  投保人 ──→ 保险公司（直保公司）

再保险关系：
  直保公司 ──→ 再保险公司

用途：
  保险公司承保了超大风险（如某工厂价值10亿元），
  单家公司无力承担全部风险，
  将部分风险"分保"给再保险公司共担。

类比：再保险 ≈ "保险公司的保险"

与保险中介平台的关系：
  中介平台直接服务终端投保人和中介机构，
  再保险层面的业务对平台系统影响很小，
  但理解这个概念有助于解释"保险公司为什么能承保超大金额风险"。
```

```java
import java.util.*;

/**
 * 简化版保险需求诊断与推荐（教学示例）
 *
 * 功能：根据用户基本画像，输出优先级排序的保险推荐列表。
 * 说明：生产环境通常使用 Drools 规则引擎或 LLM，此处以纯 Java 演示逻辑。
 */
public class InsuranceRecommendationDemo {

    public static void main(String[] args) {
        // 构造用户画像：35岁男性，已婚有孩子，年收入20万，有车，尚无健康险
        UserProfile user = new UserProfile();
        user.age = 35;
        user.gender = "M";
        user.married = true;
        user.hasChildren = true;
        user.annualIncomeTenThousand = 20;
        user.hasHealthInsurance = false;
        user.hasCar = true;
        user.occupationRisk = "low";   // 低风险职业（白领）

        InsuranceRecommendationDemo advisor = new InsuranceRecommendationDemo();
        List<Recommendation> result = advisor.recommend(user);

        System.out.println("╔══════════════════════════════════════════════╗");
        System.out.println("║           个人保险配置建议                    ║");
        System.out.println("╠══════════════════════════════════════════════╣");
        for (Recommendation rec : result) {
            System.out.printf("║ 优先级 %d │ %s%n", rec.priority, rec.productType);
            System.out.printf("║  → %s%n", rec.reason);
            System.out.println("╠══════════════════════════════════════════════╣");
        }
        System.out.println("╚══════════════════════════════════════════════╝");
    }

    public List<Recommendation> recommend(UserProfile user) {
        List<Recommendation> list = new ArrayList<>();

        // 规则1：无健康险 → 百万医疗险是基础，最优先推荐
        if (!user.hasHealthInsurance) {
            list.add(new Recommendation(1, "百万医疗险",
                    "医疗险是保障体系的基础，可报销住院费用，年保费约300~600元，性价比极高"));
        }

        // 规则2：家庭顶梁柱（已婚或有子女）+ 年龄<55 → 定期寿险
        if ((user.married || user.hasChildren) && user.age < 55) {
            list.add(new Recommendation(1, "定期寿险",
                    "您是家庭经济支柱，万一不幸身故，寿险赔偿可保障家人生活不受影响"));
        }

        // 规则3：25~55岁 → 重疾险
        if (user.age >= 25 && user.age <= 55) {
            int priority = user.hasHealthInsurance ? 2 : 1;
            list.add(new Recommendation(priority, "重大疾病险",
                    "大病确诊后一次性赔付，可弥补治疗期间的收入损失，建议保额为年收入的5倍"));
        }

        // 规则4：有车 → 商业车险
        if (user.hasCar) {
            list.add(new Recommendation(2, "商业车险（三者险≥100万）",
                    "建议三者险保额选100万以上，超出交强险限额的赔偿由商业险覆盖"));
        }

        // 规则5：高危职业 → 专属意外险提示
        if ("high".equals(user.occupationRisk)) {
            list.add(new Recommendation(1, "高危职业专属意外险",
                    "您的职业属于高风险类别，普通意外险可能不承保，须选择专属产品"));
        }

        // 按优先级升序排列（1最高）
        list.sort(Comparator.comparingInt(r -> r.priority));
        return list;
    }

    static class UserProfile {
        int age;
        String gender;
        boolean married;
        boolean hasChildren;
        int annualIncomeTenThousand;
        boolean hasHealthInsurance;
        boolean hasCar;
        String occupationRisk;   // "low" / "medium" / "high"
    }

    static class Recommendation {
        int priority;
        String productType;
        String reason;

        Recommendation(int priority, String productType, String reason) {
            this.priority = priority;
            this.productType = productType;
            this.reason = reason;
        }
    }
}
```

### 8.3 险种枚举代码示例

```java
/**
 * 保险险种类型枚举（通用教学示例）
 *
 * 设计建议：
 * 1. 用 code 字段存入数据库（稳定不变），name 用于前端展示
 * 2. isLifeInsurance() 判断是否路由到寿险微服务处理
 * 3. requiresDualRecording() 判断是否触发合规双录流程
 */
public enum InsuranceTypeEnum {

    // ── 人身险·寿险 ──────────────────────────────────
    LIFE_TERM(          "定期寿险",   "LIFE_01"),
    LIFE_WHOLE(         "终身寿险",   "LIFE_02"),
    LIFE_PARTICIPATING( "分红险",     "LIFE_03"),
    LIFE_UNIVERSAL(     "万能险",     "LIFE_04"),
    LIFE_INVESTMENT(    "投连险",     "LIFE_05"),

    // ── 人身险·健康险 ────────────────────────────────
    HEALTH_MEDICAL(     "百万医疗险", "HEALTH_01"),
    HEALTH_CRITICAL(    "重大疾病险", "HEALTH_02"),
    HEALTH_CARE(        "护理险",     "HEALTH_03"),

    // ── 人身险·意外险 ────────────────────────────────
    ACCIDENT(           "意外险",     "ACC_01"),

    // ── 财产险·车险 ──────────────────────────────────
    CAR_COMPULSORY(     "交强险",     "CAR_01"),
    CAR_COMMERCIAL(     "商业车险",   "CAR_02"),

    // ── 财产险·非车 ──────────────────────────────────
    PROPERTY_ENTERPRISE("企业财产险", "PROP_01"),
    LIABILITY_PUBLIC(   "公众责任险", "LIAB_01"),
    LIABILITY_EMPLOYER( "雇主责任险", "LIAB_02"),
    LIABILITY_PRODUCT(  "产品责任险", "LIAB_03"),
    AGRICULTURAL(       "农业险",     "AGR_01");

    private final String name;
    private final String code;

    InsuranceTypeEnum(String name, String code) {
        this.name = name;
        this.code = code;
    }

    public String getName() { return name; }
    public String getCode() { return code; }

    /**
     * 是否属于寿险类型
     * 寿险保单生命周期长（30年），路由到独立的寿险微服务处理
     */
    public boolean isLifeInsurance() {
        return code.startsWith("LIFE_");
    }

    /**
     * 是否必须触发销售双录
     *
     * 依据：《保险销售行为可回溯管理暂行办法》
     * - 分红险、万能险、投连险属于"新型人身险"，强制双录
     * - 其他险种满足"老人/大额保费"条件时同样触发双录（见第006篇）
     */
    public boolean requiresDualRecording() {
        return this == LIFE_PARTICIPATING
            || this == LIFE_UNIVERSAL
            || this == LIFE_INVESTMENT;
    }

    /** 是否为财产险 */
    public boolean isPropertyInsurance() {
        return code.startsWith("CAR_")
            || code.startsWith("PROP_")
            || code.startsWith("LIAB_")
            || code.startsWith("AGR_");
    }
}
```

| 术语 | 英文全称 | 解释 |

| 投保人 | Policyholder | 与保险公司签订合同、缴纳保费的人（可与被保险人不同） |

| 保费 | Premium | 投保人支付给保险公司的费用 |
| 纯保费 | Pure Premium | 保费中用于赔付的部分（不含费用利润） |
| 附加费 | Loading | 保费中用于运营费用和利润的部分 |
| 免赔额 | Deductible | 出险后投保人须自行承担的损失金额 |

| 拒保 | Declination | 保险公司拒绝为某标的提供保险 |
| 加费 | Rated-up | 因高风险因素，在标准保费基础上上浮收取 |
| 理赔 | Claims | 发生保险事故后，向保险公司申请赔偿的全流程 |
| FYC | First Year Commission | 首年佣金（新单出单后按保费比例发放） |
| RYC | Renewal Year Commission | 续期佣金（长险每年续保后持续发放） |
| FYP | First Year Premium | 首年年化保费（衡量新增业务量的核心指标） |
| NCD | No Claim Discount | 无赔款优待系数（车险专用，未出险则享受折扣） |
| 双录 | Dual Recording | 保险销售过程中的同步录音+录像（监管强制要求） |
| 报行合一 | — | 实际手续费不得超过向监管报备的费率（2023年起严格执行） |
| 继续率 | Persistency Rate | 第N年仍有效的保单占比（衡量保单留存质量） |
| 保全 | Policy Alteration | 保单生效后对合同内容的变更（如变更受益人、地址） |
| InsurTech | Insurance Technology | 保险科技，技术驱动的保险行业创新 |
| RAG | Retrieval-Augmented Generation | 检索增强生成，AI智能客服的核心技术架构 |

## 附录B：数据库设计关键原则（提前预习）

在后续章节会详细讲解建表规范，这里先给出与本篇知识直接相关的几条原则：

```sql
-- 原则1：保险产品表必须包含险种分类字段
-- 险种分类直接决定路由逻辑、合规要求和展示模板

CREATE TABLE ins_product (
    id              BIGINT        PRIMARY KEY COMMENT '产品ID',
    product_name    VARCHAR(100)  NOT NULL    COMMENT '产品名称',
    insurance_type  VARCHAR(20)   NOT NULL    COMMENT '险种分类码（对应InsuranceTypeEnum.code）',
    -- 险种类型决定：
    --   是否走寿险微服务
    --   是否触发双录
    --   使用哪张扩展表
    insurer_code    VARCHAR(20)   NOT NULL    COMMENT '承保保司编码',
    premium_rule    JSON                      COMMENT '保费规则（费率表引用/计算参数）',

    -- yudao-cloud 框架标准审计字段（所有业务表必须包含）
    creator         VARCHAR(64)   DEFAULT ''  COMMENT '创建者',
    updater         VARCHAR(64)   DEFAULT ''  COMMENT '更新者',
    create_time     DATETIME      NOT NULL    DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME      NOT NULL    DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted         TINYINT(1)    NOT NULL    DEFAULT 0  COMMENT '逻辑删除（0未删/1已删）',
    tenant_id       BIGINT        NOT NULL    DEFAULT 0  COMMENT '租户ID（0=平台预设数据）'
) COMMENT = '保险产品主表';

-- 原则2：主表 + 险种扩展表（避免字段膨胀）
-- 不同险种的特有字段差异巨大，不要全部塞入同一张主表

-- 车险专属扩展表
CREATE TABLE ins_policy_car_ext (
    policy_id       BIGINT        PRIMARY KEY COMMENT '关联保单ID',
    plate_number    VARCHAR(20)               COMMENT '车牌号',
    vin_code        VARCHAR(50)               COMMENT '车架号（VIN）',
    engine_code     VARCHAR(50)               COMMENT '发动机号',
    vehicle_brand   VARCHAR(50)               COMMENT '车辆品牌',
    vehicle_model   VARCHAR(100)              COMMENT '车型名称',
    ncd_years       TINYINT                   COMMENT '连续无赔款年数',
    ncd_factor      DECIMAL(4,2)              COMMENT 'NCD系数（如0.70）'
) COMMENT = '车险保单扩展表';

-- 寿险专属扩展表（建议放在独立的寿险微服务 Schema 中）
CREATE TABLE ins_policy_life_ext (
    policy_id           BIGINT        PRIMARY KEY COMMENT '关联保单ID',
    payment_period      TINYINT                   COMMENT '缴费期限（年）',
    insurance_period    TINYINT                   COMMENT '保险期限（年，999=终身）',
    cash_value          DECIMAL(15,2)             COMMENT '当前现金价值',
    policy_loan_amount  DECIMAL(15,2) DEFAULT 0   COMMENT '已贷款金额',
    visit_status        TINYINT                   COMMENT '回访状态（1待回访/2已完成/3无法联系）',
    visit_deadline      DATE                      COMMENT '回访截止日期（承保后10个工作日内）'
) COMMENT = '寿险保单扩展表';
```
