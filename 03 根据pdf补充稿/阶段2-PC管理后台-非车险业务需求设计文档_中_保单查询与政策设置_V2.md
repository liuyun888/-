## 一、非车保单查询列表

> **对应 PDF：`89_非车保单管理非车保单查询目录`（目录）**
> **对应排期**：非车保单查询列表，前端1.5天 + 后端1天

### 1.1 入口

导航：【非车】→【保单管理】→【保单查询】，默认加载近1个月（具体默认范围由【系统设置-保单设置】控制）的非车保单列表。

---

### 1.2 查询条件

查询区域支持**用户自定义显示哪些筛选条件**（详见「查询条件设置」章节），以下为全量可选条件：

**固定条件（灰色，不可移除）：**
- 日期类型（签单日期/支付日期/起保日期）+ 日期区间（起止日期选择器）
- 保险公司（下拉多选）
- 保单状态（下拉多选）

**可选条件（用户可选择展示，来自排期表92字段说明）：**
- 保(批)单号（文本，支持逗号分隔多个保单号，功能开关由保单设置控制）
- 业务员（下拉搜索）
- 险种（下拉树形，多级联动）
- 产品名称（文本模糊匹配）
- 工号描述
- 业务员登录名
- 渠道名称
- 保单来源
- 出单来源
- 出单员
- 出单机构
- 被保险人姓名
- 投保人姓名
- 投保人证件号
- 上游结算状态（下拉：已结算/未结算）
- 共号（是/否）
- 经办人

点击【查询】按钮触发查询；点击【重置】清空所有条件恢复默认。

---

### 1.3 查询列表

列表支持**用户自定义展示字段**（详见「查询列表字段设置」章节），以下为全量字段（40+）：

**固定列（不可移除）：** 保险公司、保(批)单号、操作（编辑/删除/批改历史等）

**可选列（部分常用）：** 业务员、起保日期、保险止期、险种类型、产品名称、全保费、净保费、上游手续费比例(%)、上游手续费金额、下游手续费比例(%)、下游手续费金额、保单状态、签单日期、支付日期、投保人名称、被保人姓名、工号名称、渠道名称、业务来源、录入方式、上游结算状态、机构名称、车牌号、车架号等。

**列表底部汇总行（固定展示）：**
- 总保费合计 = SUM(全保费)
- 净保费合计 = SUM(净保费)
- 上游手续费合计 = SUM(上游手续费金额)
- 下游手续费合计 = SUM(下游手续费金额)

> 注：汇总为当前查询条件下所有数据的合计，不只是当前分页。后端在查询列表时额外执行一条 `SUM` 聚合 SQL。

---

### 1.4 导出

点击【导出】按钮，将当前查询条件下的全量数据导出为 Excel，导出字段与当前列表展示字段一致（或按所选导出模板）。后端使用 EasyExcel 异步导出，导出任务完成后提示下载。

## 二、查询条件设置（个人自定义）

> **对应 PDF：`90_非车保单管理非车保单查询查询条件设置`**
> **对应排期**：非车保单查询-查询条件设置，前端1天 + 后端0.5天

### 2.1 入口

在保单查询页顶部，点击【筛选设置】按钮，弹出配置弹窗。

### 2.2 弹窗交互

> 来自 PDF 90：在右侧勾选需要的条件，显示在左侧即为添加成功；左侧已添加蓝色的条件，再次点击可移除；灰色条件是固定的不可修改；鼠标点中选项上下移动可调整顺序。

弹窗分为左右两列：
- **右侧「可添加条件」**：展示所有可选筛选条件，每项有勾选框。
- **左侧「已启用条件」**：展示已勾选的条件，支持上下**拖拽调整排列顺序**；**灰色固定条件不可移除**。

操作逻辑：
1. 在右侧勾选某条件 → 该条件出现在左侧已启用列表末尾。
2. 在左侧已启用列表中，点击已启用的**蓝色**条件 → 移除（同时右侧取消勾选）。
3. 在左侧拖拽条件卡片上下移动 → 调整查询条件显示顺序。
4. 点击【确定】保存当前配置；点击【取消】不保存。

### 2.3 后端逻辑

- 配置按 `用户ID + 模块标识（non_vehicle_query）` 存储到 `sys_user_config` 表（或专用的用户偏好表）。
- 字段格式：JSON 数组，记录已启用的条件字段名及排序，例如：`[{"fieldCode":"policy_no","sort":1},{"fieldCode":"salesperson","sort":2}]`
- 每次进入保单查询页时，从该表加载用户配置，动态渲染查询条件区域。
- 若用户未配置，使用系统默认配置（由管理员在保单设置中定义）。

## 四、非车批量更新信息

> **对应 PDF：`92_非车保单管理非车批量更新信息`**
> **对应排期**：非车批量更新信息，前端1天 + 后端1天

### 4.1 入口

在保单查询列表，**勾选多条保单**（Checkbox全选或逐条勾选） → 点击【批量更新信息】按钮。

### 4.2 操作流程

> 来自 PDF 92：点击【非车】-【保单查询】-【批量更新信息】-【下载导入模板】

#### Step 1：下载导入模板

点击弹窗中的【下载导入模板】，下载 Excel 模板（包含两个 Sheet）：
- **sheet1**：数据录入区，固定第一列为「保单号」（**必填，A列**），从 B 列起为可修改的字段列，表头名称即为字段名（如「业务员」「非车产品」「上游结算方式」「下游结算方式」等）。
- **sheet2**：字段参考说明（来自 PDF 92：sheet2 导入示例说明是相关字段参考），列出各字段的枚举值及填写规范。

#### Step 2：填写要修改的数据

> 来自 PDF 92：A 列保单号：填写要修改数据的对应保单号；B 列及以后填写要修改的保单数据；C 列产品：如非车保单号重复时必填，其他可不填。

- **A 列「保单号」**：填写要修改的保单的保(批)单号（**必填**）。
- **B 列及之后**：只填需要修改的字段列，不修改的列可留空（后端只更新有值的字段）。
- **C 列「产品名称」**：若同一保单号存在多条记录（相同保单号不同产品场景），**必填**，用于精确定位要修改的保单行。

**实际操作示例（来自 PDF 92）：**
想修改保单的录入方式，打开 sheet2 找到录入方式对应字段为「录入方式」，打开 sheet1 把该表头名称修改为「录入方式」，下面填写修改后的内容（如「自动录单」），如需修改多个字段可在 C 列后依次添加。

| A-保单号 | B-业务员 | C-非车产品 | D-录入方式 |
|---------|----------|-----------|----|
| P2024001 | 张三 | | 自动录单 |

#### Step 3：上传文件

点击【选择要导入的数据文件】或拖拽上传 Excel。

> **重要**：来自 PDF 92 第3条——此按钮（「允许覆盖」开关）**一定是打开状态才会覆盖系统原有数据**。弹窗中有此开关，默认关闭，必须开启后才生效。

#### Step 4：执行更新

> 来自 PDF 92：上传后点击「去任务列表」，状态显示成功则说明导入成功；状态显示已取消则说明导入失败，点击下载文件拉到最后查看「校验结果」，根据校验结果修改原表格内容重新上传。

- 状态显示「成功」：更新成功。
- 状态显示「已取消/失败」：点击【下载文件】，拉到最后一列查看「校验结果」，根据错误提示修改 Excel 后重新上传。

### 4.3 后端校验规则

1. A列保单号必须在系统中存在，否则报错「保单不存在」。
2. **已结算/已配政策保单的手续费字段保护**：若保单的上游已完成结算（`upstream_settle_status = 已结算`），则本次更新中的「上游手续费比例」「上游手续费金额」字段**忽略不更新**，并在结果中提示「已结算保单费率不予更新」（来自排期表技术难点）。
3. 每次更新操作写入操作审计日志表 `sys_operation_log`：记录操作人、操作时间、操作类型（批量更新）、涉及保单ID列表、更新前/后字段快照（JSON）。

> **对应 PDF：`95_非车政策设置新增政策批次`**
> **对应排期**：非车政策-新增政策批次，前端2天 + 后端2天

### 5.1 入口

导航：【非车】→【政策管理】→【非车政策】→点击【新增政策批次】按钮，弹出政策配置页（全屏弹窗或独立页面）。

---

### 5.2 政策基本配置

> 来自 PDF 95：政策批次可以自定义；保险公司选择对应的保险公司；日期类型选择签单日期或者支付日期；状态"正常"是政策点击保存后立即生效，"待确认"是政策保存以后是编辑状态。

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 政策批次名称 * | 文本 | 是 | 自定义，如「2024年财产险Q1政策」 |
| 保险公司 * | 下拉选择 | 是 | 选择后，产品方案和承保条件联动 |
| 日期类型 * | 单选：签单日期 / 支付日期 | 是 | 决定政策匹配时用哪个日期字段 |
| 状态 * | 单选：正常 / 待确认 | 是 | 「正常」= 保存后**立即生效**；「待确认」= 草稿，不匹配保单 |
| 生效日期起 * | 日期 | 是 | 政策有效期开始 |
| 生效日期止 * | 日期 | 是 | 政策有效期结束，必须 > 生效日期起 |
| 投保区域 | 下拉（省市多选） | 否 | 未选则适用全部区域；**注：未知区域适用于未上牌车辆**（来自 PDF 95） |
| 适用工号 | 多选下拉 | 否 | **必须先在车险-报价出单-出单工号-编辑-工号范围中选择「非车」或「车险及非车」的工号，才会显示此工号**（来自 PDF 95）；工号名称对应的是非车保单信息中的工号名称 |
| 适用机构 | 多选树形选择 | 否 | 仅归属所选机构的保单可匹配此政策 |
| 备注 | 文本 | 否 | 可自定义 |

---

### 5.3 产品点位配置（核心部分）

政策配置表格，每行代表一个产品维度的点位配置：

> 来自 PDF 95：产品名称点击编辑图标，选择产品名称（若未找到产品名称，可在系统设置-产品管理-系统产品查询；若系统产品查询不到，可以在自定义产品-新增产品名称即可）。

| 列名 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 产品名称 * | 点击编辑图标弹出选择器 | 是 | 选择产品。若产品不在列表中：①先到系统产品查询；②若无则到自定义产品新增后返回选择 |
| 产品方案 | 下拉（依赖保险公司） | 否 | 选定保司后加载对应产品方案列表；若没有对应的产品方案，也是在产品管理里面新增（来自 PDF 95） |
| 承保条件 | 多选勾选框 | 否 | **选政策条件之前，要先选上保司**（来自 PDF 95）；按照原始政策文件勾选 |
| 价税分离 | 开关 | 否 | **按钮开启**：按净保费计算手续费；**按钮关闭**：按总保费（全保费）计算手续费（来自 PDF 95） |
| 上游手续费比例(%) * | 数字 | 是 | 保险公司给付的手续费比例 |
| 下游手续费比例(%) | 数字 | 否 | 给到业务员/机构的手续费比例（通用点位） |
| 操作 | 按钮 | — | 点击【加号】图标新增一行政策；点击【垃圾桶】图标删除本行政策（来自 PDF 95） |

**下游独立设置（可选）：**
- 点击每行的【设置下游】，可为特定组织部门或业务员单独配置不同的下游比例（覆盖通用点位）。
- 来自 PDF 95：可以单独给组织部门或业务员设置点位，没有单独要求都是通用点位。
- 格式：选择组织/业务员 + 填写独立下游比例。

### 5.4 保存逻辑

1. **前端校验**：政策批次名称、保险公司、日期类型、生效日期不为空；生效日期止 > 生效日期起；至少配置一行产品点位；上游比例为有效数字。
2. **后端校验**：
   - 检查是否存在生效日期区间重叠且保险公司相同的「正常」状态政策（允许用户选择是否强制保存）。
   - 状态=「正常」时，保存后立即在保单政策匹配中生效。
3. **入库**：
   - 政策批次写入 `ins_non_vehicle_policy_batch` 表。
   - 每行产品点位写入 `ins_non_vehicle_policy_item` 表。
   - 下游独立设置写入 `ins_non_vehicle_policy_downstream` 表。

---

### 5.5 政策匹配逻辑（后端核心）

每次非车保单保存时，触发政策匹配（异步或同步均可）：

```
匹配条件：
1. tenant_id = 当前租户
2. 保险公司ID = 保单.保险公司ID
3. 产品名称 = 保单.产品名称（或产品ID）
4. 状态 = 正常（status = 1）
5. 日期匹配：
   - 若政策日期类型=签单日期：政策生效日期起 <= 保单.签单日期 <= 政策生效日期止
   - 若政策日期类型=支付日期：政策生效日期起 <= 保单.支付日期 <= 政策生效日期止
6. 工号匹配：政策.适用工号 包含 保单.工号名称，或政策未设置适用工号（全部适用）
7. 机构匹配：政策.适用机构 包含 保单.归属机构，或政策未设置适用机构
8. 区域匹配：政策.投保区域 包含 保单.region，或政策未设置区域

多条匹配结果：取最新创建的政策批次（或生效日期最精确的批次）。
```

匹配成功 → 更新 `ins_non_vehicle_policy.policy_batch_id`，并回填 `upstream_rate`、`downstream_rate`。
匹配失败 → `policy_batch_id = null`，手续费保持手工录入值。

### 5.6 数据库设计

#### `ins_non_vehicle_policy_batch`（政策批次表）

```sql
CREATE TABLE `ins_non_vehicle_policy_batch` (
  `id`                    BIGINT       NOT NULL AUTO_INCREMENT,
  `tenant_id`             BIGINT       NOT NULL,
  `batch_name`            VARCHAR(200) NOT NULL COMMENT '政策批次名称',
  `insurance_company_id`  BIGINT       NOT NULL COMMENT '保险公司ID',
  `date_type`             TINYINT      NOT NULL COMMENT '日期类型：1-签单日期 2-支付日期',
  `status`                TINYINT      NOT NULL COMMENT '状态：1-正常 2-待确认 3-停用',
  `effect_start_date`     DATE         NOT NULL COMMENT '生效日期起',
  `effect_end_date`       DATE         NOT NULL COMMENT '生效日期止',
  `applicable_work_nos`   JSON         COMMENT '适用工号列表，JSON数组',
  `applicable_org_ids`    JSON         COMMENT '适用机构ID列表，JSON数组',
  `applicable_regions`    JSON         COMMENT '适用投保区域，JSON数组',
  `remark`                VARCHAR(500) COMMENT '备注',
  `creator`               BIGINT,
  `create_time`           DATETIME     DEFAULT CURRENT_TIMESTAMP,
  `updater`               BIGINT,
  `update_time`           DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `deleted`               TINYINT(1)   DEFAULT 0,
  PRIMARY KEY (`id`),
  KEY `idx_tenant_company_status` (`tenant_id`, `insurance_company_id`, `status`)
) ENGINE=InnoDB COMMENT='非车险政策批次表';
```

#### `ins_non_vehicle_policy_item`（政策产品点位表）

```sql
CREATE TABLE `ins_non_vehicle_policy_item` (
  `id`                  BIGINT       NOT NULL AUTO_INCREMENT,
  `batch_id`            BIGINT       NOT NULL COMMENT '政策批次ID',
  `product_id`          BIGINT       COMMENT '产品ID',
  `product_name`        VARCHAR(200) NOT NULL COMMENT '产品名称',
  `product_plan`        VARCHAR(200) COMMENT '产品方案',
  `underwrite_cond`     JSON         COMMENT '承保条件（选中项）',
  `is_tax_separate`     TINYINT(1)   DEFAULT 0 COMMENT '价税分离：0-否(按全保费计算) 1-是(按净保费计算)',
  `upstream_rate`       DECIMAL(8,4) NOT NULL COMMENT '上游手续费比例(%)',
  `downstream_rate`     DECIMAL(8,4) COMMENT '通用下游手续费比例(%)',
  `sort`                INT          DEFAULT 0 COMMENT '排序',
  PRIMARY KEY (`id`),
  KEY `idx_batch_id` (`batch_id`)
) ENGINE=InnoDB COMMENT='非车险政策产品点位配置表';
```

#### `ins_non_vehicle_policy_downstream`（下游独立点位表）

CREATE TABLE `ins_non_vehicle_policy_downstream` (

## 六、非车政策其他操作

> **对应 PDF：`96_非车政策设置其他操作`**
> **对应排期**：非车政策其他操作（编辑/复制/停用/删除），前端1天 + 后端1天

### 6.1 政策列表

【非车】→【政策管理】→【非车政策】展示政策批次列表，字段：政策批次名称、保险公司、生效日期、状态、创建时间、操作列。

### 6.2 各操作说明

> 来自 PDF 96：复制-可以复制该政策，生成一版相同内容的新政策；编辑-对该政策内容进行编辑；详情-查看政策信息，不可编辑；删除-即删除该政策；停用-点击停用，该政策即为失效（**如需编辑已生效政策，需要先停用，再编辑即可**）。

#### 编辑
- 点击【编辑】，进入与新增相同的编辑表单页。
- **重要（来自 PDF 96）**：如需编辑已生效政策，**需要先停用，再编辑即可**。
- 保存后对**此后匹配到该政策的新保单**生效，不影响已匹配的历史保单。

#### 复制
- 点击【复制】，系统自动创建一份相同内容的新政策批次，状态默认为「待确认」（草稿）。
- 新政策批次名称自动追加「-副本」，生效日期保留原日期（用户需手动修改）。

#### 停用
- 点击【停用】，弹出确认框：「确认停用该政策？停用后新保单将不再匹配此政策。」
- 确认后将 `status` 更新为 `3-停用`。
- **停用后**，该政策不参与新保单的政策匹配；历史已匹配的保单不受影响（`policy_batch_id` 保留）。

#### 删除
- 点击【删除】，即删除该政策（来自 PDF 96）。
- **仅「待确认」（草稿）状态**的政策可删除（软删除，`deleted=1`）。
- 「正常」或「停用」状态的政策**不可删除**，点击删除时提示「已生效政策不可删除，如需停用请使用停用功能」。

#### 详情
- 点击【详情】，展示政策只读信息（来自 PDF 96：查看政策信息，不可编辑），同时在下方附带「匹配此政策的保单列表」，展示历史上所有通过该政策批次匹配到手续费的保单（分页展示，字段：保单号、业务员、签单日期、产品名称等）。

### 6.3 状态流转

```
待确认（草稿）→【保存为正常】→ 正常（生效中）→【停用】→ 停用
待确认（草稿）→【删除】→ 已删除（软删除）
正常（生效中）→【先停用，再编辑保存】→ 正常（内容更新，重新生效）
停用 →【编辑保存为正常】→ 正常（重新生效）
```
