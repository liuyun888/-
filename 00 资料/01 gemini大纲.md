为了满足您每月都有阶段性成果展示的需求，我为您规划了一份为期六个月的**保险中介数字化平台开发路线图与需求文档摘要**。这份规划参考了优保联的系统集成度 、手回科技的多渠道分销模式 、以及水星保的智能规划逻辑 。



------

### 第一阶段：核心作业引擎与B端基石（第1-2月）

**目标：** 实现业务员最迫切的报价与组织管理功能，形成闭环作业能力。

#### 第1个月：车险极速报价引擎（MVP版本）

- **B端核心需求：**

  - **OCR智能识别**：支持行驶证、身份证拍照识别，自动带入车辆信息 。

    

    

  - **一键比价系统**：对接主流保司API，实现多保司报价同屏对比，展示保费明细与地区政策。

  - **续保提醒工作台**：自动抓取未来60天到期客户，支持业务员一键发起续保询价 。

    

    

- **成果展示：** 演示从拍照到出具多保司对比报价单的全流程，耗时控制在3分钟内。

#### 第2个月：非车险中枢与“基本法”规则引擎

- **管理后台核心需求：**

  - **基本法配置器**：支持配置职级架构（推荐人、主管、部经理）、晋升考核指标以及FYC（首年佣金）、RYC（续期佣金）发放比例 。

    

    

  - **佣金分摊逻辑**：针对分红险等特定产品，内置“首年支付不超总佣金70%、剩余5年平均分摊”的合规计算模型 。

    

    

  - **产品库管理**：非车险（意外、医疗、重疾）的产品协议存储、保障责任标签化管理。

- **成果展示：** 演示修改一个职级的津贴比例后，系统自动计算出不同层级业务员的预估收益。

------

### 第二阶段：C端商城与智能拓客工具（第3-4月）

**目标：** 构建面向客户的触点，通过专业化工具提升转化率。

#### 第3个月：C端保险商城与数字化名片

- **C端商城需求：**

  - **货架系统**：参照“小雨伞”模式，提供分类清晰、支持多维度筛选的人身险/财产险商城 。

    

    

  - **投保闭环**：集成智能健康告知（跳题逻辑）与保司支付接口。

- **B端营销工具：**

  - **数字化名片**：业务员个人微店，支持挂载产品推荐海报，海报自带专属二维码，实现扫码获客追踪 。

    

    

- **成果展示：** 业务员分享一张带有二维码的海报到朋友圈，客户扫码进入商城并完成一笔简单意外险的投保。

#### 第4个月：AI家庭保障规划系统

- **核心逻辑：**

  - **风险测评引擎**：参照水星保，基于标普资产配置理论构建算法，通过家庭成员年龄、收入、负债输出测评报告 。

    

    

  - **智能方案生成**：利用 LaTeX 模型计算保障缺口：

    $$Gap = Need_{protection} - (Asset_{liquid} + SocialSecurity + Insurance_{current})$$

    自动匹配商城内产品生成全家保障方案。

- **成果展示：** 模拟输入一个四口之家的财务数据，系统在10秒内输出一份20页的深度 PDF 计划书。

------

### 第三阶段：合规管控与财务闭环（第5-6月）

**目标：** 解决中介机构的监管合规与大规模财务结费痛点。

#### 第5个月：智能双录与全流程合规系统

- **合规需求：**
  - **远程/智能双录**：内置 AI 质检能力，在录制过程中实时识别业务员是否完整朗读条款，识别率需满足合规要求。
  - **可回溯管理**：记录 C 端用户点击健康告知、阅读条款的操作轨迹，确保证据链完整。
- **成果展示：** 演示在双录过程中，如果业务员漏读了重要免责声明，系统立即自动弹窗阻断并提醒重读。

#### 第6个月：人业财一体化财务中心

- **后端核心需求：**

  - **自动化对账**：支持批量导入保司结算流水，利用算法实现保单级自动对账，标示出漏报、错报单据 。

    

    

  - **业务结算表生成**：一键生成符合税务要求的结算详单，涵盖客户名称、险种、保单号、佣金计算方法等全要素 。

    

    

  - **BI经营看板**：实时分析全司 FYP（首年年化保费）、继续率、人力留存等核心经营指标 。

    

    

- **成果展示：** 导入一份包含万条数据的保司对账单，系统在30秒内完成自动匹配，并生成待发佣金汇总表。

------

### 建议的技术保障与实施重点

1. **接口标准化**：鉴于要对接 80+ 保司，建议从第1月起就构建统一的 API 适配层，避免后期因保司接口变动导致的大规模重构 。

   

   

2. **数据安全**：保险数据涉及用户隐私，需按照“保险中介技术服务商安全 convention 1.0”进行加密存储，并在第5月前完成等保备案 。

   

   

3. **合规先导**：在开发基本法逻辑时，必须严格遵守“报行合一”政策，确保系统计算的手续费率与保司备案的一致 。
