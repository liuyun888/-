## 1. 监管报表

> **对应 PDF**：`159_寿险报表管理监管报表.pdf`
> **排期表工时**：前端 2天 + 后端 2天（含业绩报表合并工时）

### 1.1 功能概述

菜单路径：寿险 → 报表管理 → 监管报表

按保险监管局要求格式生成寿险保单数据报表。对应操作手册 PDF 159。

### 1.2 查询条件（来自 PDF 159 §一）

> 📌 PDF 原文：「可根据机构选择、保险公司、日期类型、开始时间、结束时间进行精准查询」

**查询条件**：
- 机构选择
- 保险公司
- 日期类型（起保日期/签单日期/生效日期，枚举选择）
- 开始时间
- 结束时间

### 1.3 关键指标说明（来自 PDF 159 §二）

> 📌 PDF 原文：「新单保费金额（累计）：已开票的新单的保费金额」「对应个险保单模块新单保单的总保费」

**核心指标**：
- **新单保费金额（累计）**：已开票的新单保费金额，对应个险保单模块新单保单的总保费
- 报表字段包括：保单号、险种代码、产品代码、保险公司名称、保额、年度保费、被保人证件号（脱敏）、被保人姓名（部分脱敏）、起保日期、到期日期、缴费方式、保单状态

### 1.4 生成与导出逻辑

1. 点击【生成报表】→ 后端按保监局标准格式（需运维配置模板）聚合数据
2. 生成 Excel（监管格式，列名和编码固定，不可自定义）
3. 存入 `ins_life_regulatory_report`（report_type, period, file_url, create_time, status）
4. 支持按**月度/季度/年度**生成

### 1.5 提交记录管理

报表生成后，可手动在系统标记提交状态：
- 点击【标记已提交】→ UPDATE status='SUBMITTED'，submit_time=now()
- 列表展示历史报表：报表类型、期间、生成时间、提交状态、文件下载链接

## 3. 继续率报表

> **对应 PDF**：`161_寿险报表管理继续率报表.pdf`
> **排期表工时**：前端 2天 + 后端 2天（含保单结算/产品占比合并工时）

### 3.1 功能概述

菜单路径：寿险 → 报表管理 → 继续率报表

统计寿险行业核心 KPI——保单在第 13、25、37、49 个月时续缴的比例。对应操作手册 PDF 161。

### 3.2 继续率维度（来自 PDF 161 §一）

> 📌 PDF 原文：「代理人、机构、组织、保险公司维度的继续率报表支持查看 R13、R25、R37、R49 的保单明细」

**支持的维度**：代理人、机构、组织、保险公司

**支持的继续率节点**：
- **R13**：13个月继续率
- **R25**：25个月继续率
- **R37**：37个月继续率
- **R49**：49个月继续率

### 3.3 明细查询与导出（来自 PDF 161 §二）

> 📌 PDF 原文：「在明细列表，支持筛选和导出保单明细数据，点击【导出】可自动跳转到导出任务列表进行下载」

- 点击某个维度/节点，展开该节点下的**保单明细列表**
- 明细列表支持进一步筛选
- 点击【导出】自动跳转到**导出任务列表**进行下载

### 3.4 继续率计算逻辑（业务逻辑）

**R13 计算公式**：
```
R13 = 满足条件的保单数（起保后第13个月仍有效且已续缴）
      ÷ 起保后第13个月到期应续缴的保单总数
      × 100%
```

**SQL 实现思路**：
1. 取基准月往前推 12 个月，找该月起保的保单为分母（应续缴保单）
2. 在分母保单中，找到其第 13 次缴费记录已存在（`ins_life_renewal_track` 中该期 status=已缴费）的为分子
3. 继续率 = 分子 / 分母 × 100%

R25/R37/R49 类同，偏移月数依次为 24/36/48。

### 6.5 后端实现：Redis热更新

配置保存到 `ins_life_h5_config` 表（key-value 结构）。

**热更新实现**：
1. 保存配置 → UPDATE DB → 主动写入/刷新 Redis：`HSET life:h5:config:{key} value`
2. H5 前端每次请求配置时，先读 Redis；Redis miss 则读 DB 并回写（TTL=24小时）
3. 配置更新时调用 `redisTemplate.delete("life:h5:config:*")` 主动清除相关 key，保证实时生效

## 10. 寿险产品管理

> **对应 PDF**：`170_寿险产品管理寿险产品管理.pdf`
> **排期表工时**：含在合作保司管理工时中

### 10.1 功能概述

菜单路径：产品 → 寿险产品管理

新增/编辑/上下架寿险产品，管理费率表、佣金配置、健康告知问卷绑定，并授权产品到机构。对应操作手册 PDF 170。

### 10.2 产品列表与新增

点击【新增产品】，弹出产品录入弹窗（字段较复杂，分 Tab 展示）：

#### Tab 1：基本信息

| 字段名 | 必填 | 说明 |
|---|---|---|
| 险种 | ★ | 枚举：寿险/重疾/医疗/意外/年金/万能险 |
| 产品代码 | ★ | 保司分配的产品代码，同一保司内唯一 |
| 产品名称 | ★ | |
| 保险公司 | ★ | 下拉 |
| 保障期限类型 | ★ | 枚举：定期（X年）/终身 |
| 保障期限（年） | 条件必填 | 保障期限类型=定期时必填 |
| 缴费方式（可多选） | ★ | 趸缴/年缴/半年缴/季缴/月缴，支持多选 |
| 最低保额（元） | 选填 | |
| 最高保额（元） | 选填 | |
| 产品状态 | ★ | 枚举：上架/下架，新建默认下架 |
| 产品简介 | 选填 | 富文本 |
| 产品说明书 | 选填 | PDF附件上传，OSS存储 |

#### Tab 2：佣金配置

- 配置首年佣金率（%）和续年佣金率（%）（产品层级预设费率，可被政策层级覆盖）
- 支持按缴费方式配置不同佣金率（表格形式，每行一种缴费方式）

#### Tab 3：健康告知问卷

- 下拉多选，从已维护的健告问卷模板（`ins_questionnaire_template`）中绑定该产品适用的健康告知问卷
- 一个产品可绑定多份问卷（主问卷+特殊问卷）

#### Tab 4：费率表

- 费率表为产品定价依据，字段：年龄段、性别、保障期、保费（元/万元保额）
- 支持 EasyExcel 导入：点击【导入费率表】→ 下载模板 → 填写 → 上传
- 上传后预览前 20 行确认 → 确认无误点【导入】→ INSERT `ins_life_product_rate`（product_id, age_min, age_max, gender, coverage_term, premium_per_unit）

### 10.3 产品授权到机构

产品新建后默认不对任何机构开放，需单独授权：
1. 在产品列表点击【机构授权】
2. 弹出机构树选择框，勾选有权销售该产品的机构（支持批量勾选）
3. 保存 INSERT `ins_life_product_auth`（product_id, org_id, auth_time, operator_id）

取消授权：删除对应 `ins_life_product_auth` 记录。

### 10.4 产品上下架

**上架操作**：
- 点击【上架】→ UPDATE `ins_life_product.status = 'ON_SALE'`
- **校验**：产品必须已配置费率表且已授权至少一个机构，否则提示「请先维护费率表并授权机构」

**下架操作**：
- 点击【下架】→ UPDATE status = 'OFF_SALE'
- 下架后在业务员 App 端不可选用该产品，不影响已有保单
