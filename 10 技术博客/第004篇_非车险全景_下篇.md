## 第五章：责任险（Liability Insurance）

责任险承保的是"被保险人依法应承担的民事赔偿责任"，这是一个与其他险种截然不同的逻辑——不是赔被保险人自己的财产损失，而是赔被保险人对第三方造成的损害。

### 5.1 责任险的底层逻辑

```
责任险赔付的基本逻辑：

被保险人（A）──造成损害──▶ 第三方（B）
       │                        │
       │                        │ 向A索赔
       │◀───────────────────────┘
       │
       │ 向保险公司索赔（A 向保险公司要求赔偿给 B 的费用）
       ▼
保险公司 ──赔付──▶ A（用于支付给 B）

核心：赔的是"A 对 B 造成的损失"，不是 A 自己的损失
```

> 📌 **与财产险的根本区别**：
> - 财产险：车撞坏了，赔**自己的车**（直接损失）
> - 责任险：车撞伤了路人，赔**路人的医疗费 + 误工费**（对他人的赔偿责任）

---

### 5.2 公众责任险（Public Liability Insurance）

#### 适用场景

公众责任险面向**所有对公众开放**的场所经营者：

```
典型承保客户：
  ├── 商场、超市、酒店、餐厅
  ├── 游乐场、健身房、游泳馆
  ├── 工厂（厂区对外访客）
  ├── 物业公司（管辖区域内的第三方伤亡）
  └── 活动举办方（演唱会、展览会）
```

#### 保障内容

```
公众责任险承保范围：

被保险人（场所经营者）因以下原因导致第三方（公众）：
  ├── 人身伤亡（含医疗费、误工费、残疾赔偿金、死亡赔偿金）
  └── 财产损失

常见除外责任：
  ├── 被保险人故意行为
  ├── 战争、核辐射
  ├── 雇员工伤（需投雇主责任险）
  └── 污染责任（需单独投保环境污染责任险）
```

#### 典型赔付案例

```
案例 1：商场滑倒事故
  场景：顾客在商场地面湿滑处滑倒骨折，起诉商场索赔
  赔付：医疗费 3 万 + 误工费 0.5 万 + 营养费 0.3 万 = 3.8 万
  由公众责任险赔付给商场，用于支付给顾客

案例 2：餐厅食物中毒
  场景：50名顾客集体食物中毒，集体起诉
  赔付：每人平均医疗费 5000 元 × 50人 = 25 万
  注意：单一事故赔偿限额和累计赔偿限额同时适用
```

### 5.3 雇主责任险（Employer's Liability Insurance）

雇主责任险是企业合规运营的"必备险"，弥补工伤保险的不足。

#### 为什么有了工伤保险还要买雇主责任险？

```
工伤保险（社会保险）的局限性：
  ├── 赔付标准固定，通常低于实际损失
  ├── 覆盖范围有限（上下班途中认定争议多）
  └── 处理周期长（从申请到赔付可能需要数月）

雇主责任险的补充价值：
  ├── 可以赔付超出工伤保险的部分（差额补偿）
  ├── 赔付速度快（商业险，不依赖政府审批）
  └── 包含法律费用（劳动争议诉讼的律师费）
```

#### 承保范围与主要条款

```
雇主责任险承保范围：

被保险人（雇主）的员工在工作期间发生以下情况：
  ├── 意外伤害导致的身故
  ├── 意外伤害导致的残疾
  ├── 职业病（列明的）
  └── 相关法律费用

关键条款参数：
  ├── 每人每次事故赔偿限额：如 50万/人
  ├── 累计赔偿限额：如 500万（所有员工全年合计上限）
  ├── 法律费用限额：如 10万/案
  └── 适用员工范围（正式员工/外包/临时工的区别）
```

### 5.4 产品责任险（Product Liability Insurance）

面向生产商、销售商，承保因产品缺陷造成消费者损失的赔偿责任。

#### 典型适用行业

```
必须或强烈建议投保产品责任险的行业：
  ├── 食品饮料生产企业（食品安全事故高发）
  ├── 医疗器械制造商（FDA/NMPA认证产品）
  ├── 儿童用品制造商（玩具、奶粉、童车等）
  ├── 电气设备生产商（家电、汽车配件等）
  └── 出口企业（目标国可能要求提供产品责任险证明）
```

#### 产品责任险的触发条件

```
产品责任险赔付的4个要件：

1. 产品存在缺陷（设计缺陷、制造缺陷、警示缺陷）
2. 消费者遭受人身伤亡或财产损失
3. 缺陷与损失之间存在因果关系
4. 被保险人（生产商/销售商）对损失承担法律责任

举例：
  某食品企业生产的果汁被查出含有不达标防腐剂
  导致 100 名消费者食物中毒
  保险公司赔付：
    ├── 消费者医疗费用
    ├── 召回费用（如附加产品召回险）
    └── 诉讼费用
```

---

### 5.5 三类责任险对比速览

| 维度 | 公众责任险 | 雇主责任险 | 产品责任险 |
|------|----------|----------|----------|
| **受害方** | 进入场所的公众 | 雇主的员工 | 购买产品的消费者 |
| **触发场景** | 场所内事故 | 工作中事故 | 产品缺陷导致损害 |
| **赔付内容** | 人身伤亡 + 财产损失 | 工伤 + 职业病 | 人身伤亡 + 财产损失 + 召回 |
| **主要客户** | 商业场所经营者 | 所有雇用员工的企业 | 制造商 / 销售商 |
| **与工伤险关系** | 无直接关联 | 互补关系 | 无直接关联 |
| **年保费范围** | 1,000~50,000 元 | 5,000~200,000 元 | 5,000~500,000 元 |
| **系统复杂度** | 中等 | 中等 | 较高（产品批次管理） |

## 第六章：产品差异化挑战——工程师的真正难题

### 6.1 问题的本质

现在我们来面对一个残酷的现实：**同样是"百万医疗险"，平安、人保、友邦的条款可能有 50 处差异**。

```
以百万医疗险为例，不同保司的差异点：

免赔额设置：
  保司A：10,000元（固定）
  保司B：5,000元（固定）
  保司C：0元（特定就医方式）/ 10,000元（普通就医方式）

续保条款：
  保司A：保证续保6年（确诊疾病不影响续保）
  保司B：保证续保至60岁
  保司C：非保证续保（每年审核，可能被拒）

就医限制：
  保司A：限定合作医院（约200家三甲医院）
  保司B：不限医院（全国公立二级及以上）
  保司C：仅限公立医院（私立医院不赔）

自费药报销比例：
  保司A：60%
  保司B：0%（不报销）
  保司C：100%（无限制）

特殊责任：
  保司A：含质子重离子治疗（保额内赔付）
  保司B：含海外就医（赔付额度100万）
  保司C：含CAR-T细胞治疗（额外保额50万）
```

### 6.2 朴素方案的致命缺陷

很多初级开发者面对上述差异，会想到一个简单粗暴的方案：

```sql
-- ❌ 错误方案：为每个产品创建一张独立表
CREATE TABLE product_ping_an_medical (
    product_id BIGINT,
    deductible DECIMAL(12,2),
    renewal_type VARCHAR(20),
    hospital_limit VARCHAR(200),
    self_pay_rate DECIMAL(5,2),
    proton_therapy BIT,
    -- 平安特有的30个字段...
);

CREATE TABLE product_renjia_medical (
    product_id BIGINT,
    -- 人保特有的35个字段...
);

-- 问题：有100个产品就要建100张表，这是反模式！
```

**另一个常见错误方案**：

```sql
-- ❌ 错误方案：把所有字段堆到一张大表
CREATE TABLE insurance_product (
    id BIGINT,
    product_name VARCHAR(200),
    
    -- 意外险字段
    occupation_max_category INT,
    accidental_death_amount DECIMAL(15,2),
    
    -- 健康险字段
    deductible_amount DECIMAL(12,2),
    reimbursement_rate DECIMAL(5,2),
    
    -- 企财险字段
    property_coverage_type VARCHAR(20),
    business_interruption_days INT,
    
    -- 责任险字段
    per_accident_limit DECIMAL(15,2),
    aggregate_limit DECIMAL(15,2),
    
    -- ...100个字段，大量字段对特定险种为NULL
);
-- 问题：表无法扩展，且充满NULL值，维护噩梦
```

### 6.3 正确方案：原子化责任配置

**原子化责任（Atomized Coverage）** 是产品中台设计的核心思想：

> 将一个复杂的保险产品拆解为最小颗粒度的"责任单元"（Coverage Item），每个责任单元独立配置，产品由责任单元组合而成。

```
产品 = 责任清单 + 每个责任的具体参数

以百万医疗险为例：

产品：某某百万医疗险
  └─ 责任列表：
      ├─ 责任1：住院医疗费用（Coverage: HOSPITALIZATION_EXPENSE）
      │    ├─ 参数：deductible_amount = 10000
      │    ├─ 参数：reimbursement_rate_insured = 100
      │    ├─ 参数：reimbursement_rate_self = 60
      │    └─ 参数：annual_limit = 6000000
      │
      ├─ 责任2：特殊门诊（Coverage: SPECIAL_OUTPATIENT）
      │    ├─ 参数：包含类型 = [化疗, 透析, 门诊手术]
      │    └─ 参数：annual_limit = 6000000（与住院合并计算）
      │
      └─ 责任3：质子重离子治疗（Coverage: PROTON_THERAPY）
           ├─ 参数：benefit_amount = 1000000（另给100万）
           └─ 参数：waiting_period_days = 180
```

## 第七章：产品中台完整数据库设计

### 7.1 设计原则

```
产品中台数据库设计原则：

1. 主表存通用字段
   ├── 产品基本信息（名称、险种类型、保险公司、状态）
   └── 销售配置（最低/最高保费、可投年龄、渠道）

2. 扩展表存险种特有字段
   ├── ins_product_accident_ext（意外险扩展）
   ├── ins_product_health_medical_ext（医疗险扩展）
   ├── ins_product_health_critical_ext（重疾险扩展）
   └── ins_product_property_ext（企财险扩展）

3. 责任配置表存"原子化责任"
   ├── ins_product_coverage（责任定义）
   └── ins_product_coverage_param（责任参数）

4. 核保规则表存差异化核保逻辑
   └── ins_product_underwriting_rule（核保规则）
```

```sql
    `premium_calc_type` TINYINT         DEFAULT 1 COMMENT '保费计算方式：1-固定费率 2-费率表查询 3-公式计算 4-调用保司API',

    `highlight_points`  VARCHAR(500)    DEFAULT '' COMMENT '产品亮点（JSON数组，用于展示）',
    `cover_image_url`   VARCHAR(500)    DEFAULT '' COMMENT '产品封面图URL',
    
    -- 审计字段（项目规范必备）
    `creator`           VARCHAR(64)     NOT NULL DEFAULT '' COMMENT '创建者',
    `create_time`       DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updater`           VARCHAR(64)     NOT NULL DEFAULT '' COMMENT '更新者',
    `update_time`       DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    `tenant_id`         BIGINT          NOT NULL DEFAULT 0 COMMENT '租户ID（0=系统预置）',
    
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_product_code` (`product_code`),
    KEY `idx_ins_type` (`ins_type`, `ins_sub_type`),
    KEY `idx_insurer_id` (`insurer_id`),
    KEY `idx_status` (`status`),
    KEY `idx_tenant_id` (`tenant_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='保险产品主表';
```

```java
package com.example.insurance.product.enums;

import lombok.Getter;
import lombok.RequiredArgsConstructor;

/**
 * 险种类型枚举
 * 覆盖意外险、健康险、企财险、责任险等非车险主要类型
 */
@Getter
@RequiredArgsConstructor

    // ===== 人身险 =====
    ACCIDENT(1, "意外险", "accident"),
    HEALTH_MEDICAL(21, "医疗险", "health_medical"),
    HEALTH_CRITICAL(22, "重疾险", "health_critical"),
    HEALTH_NURSING(23, "护理险", "health_nursing"),
    LIFE(3, "寿险", "life"),

    // ===== 财产险 =====
    PROPERTY(4, "企业财产险", "property"),
    MOTOR(5, "车险", "motor"),

    // ===== 责任险 =====
    LIABILITY_PUBLIC(61, "公众责任险", "liability_public"),
    LIABILITY_EMPLOYER(62, "雇主责任险", "liability_employer"),
    LIABILITY_PRODUCT(63, "产品责任险", "liability_product");

    /** 险种代码 */
    private final int code;
    /** 险种名称 */
    private final String name;
    /** 扩展表后缀（用于动态关联扩展表） */
    private final String extTableSuffix;

    /**
     * 根据 code 获取枚举
     */
    public static InsuranceTypeEnum of(int code) {
        for (InsuranceTypeEnum e : values()) {
            if (e.code == code) return e;
        }
        throw new IllegalArgumentException("未知险种代码: " + code);
    }

    /**
     * 是否为人身险类
     */
    public boolean isPersonal() {
        return code == 1 || (code >= 21 && code <= 29) || code == 3;
    }

    /**
     * 是否需要健康告知
     */
    public boolean requiresHealthDeclaration() {
        return this == HEALTH_MEDICAL || this == HEALTH_CRITICAL || this == HEALTH_NURSING;
    }

    /**
     * 是否支持团体投保
     */
    public boolean supportsGroupPolicy() {
        return this == ACCIDENT || this == LIABILITY_EMPLOYER;
    }
}
```

```java
package com.example.insurance.product.enums;

/**

    CATEGORY_1(1, "1类职业", "一般办公室人员、教师、医生（非手术）、IT程序员等", 1.00, true),
    CATEGORY_2(2, "2类职业", "销售外勤、驾驶员（轻型）、厨师、理发师等", 1.20, true),

    CATEGORY_4(4, "4类职业", "消防员、高空作业（20m内）、电工、化工厂工人", 2.00, true),
    CATEGORY_5(5, "5类职业", "高空作业（20m以上）、矿山开采、爆破工人", 3.00, false),
    CATEGORY_6(6, "6类职业", "潜水员、职业赛车手、特种兵等极高风险职业", null, false);

    private final int categoryCode;
    private final String categoryName;
    private final String typicalOccupations;
    /**
     * 相对于1类职业的费率系数（null = 通常拒保）
     */
    private final Double rateCoefficient;
    /**
     * 是否可通过标准意外险产品投保
     */
    private final boolean standardInsurable;

    /**
     * 判断该职业类别是否满足产品要求的最高接受类别
     *
     * @param productMaxCategory 产品可接受的最高职业类别
     * @return true=可以投保
     */
    public boolean isAcceptableFor(int productMaxCategory) {
        return this.standardInsurable && this.categoryCode <= productMaxCategory;
    }

    /**
     * 计算基础保费的费率系数
     * 不可投保时抛出异常
     */
    public double getRateCoefficientOrThrow() {
        if (rateCoefficient == null) {
            throw new UnsupportedOperationException(
                String.format("[%s] 属于极高风险职业（%s类），无法通过标准渠道投保意外险",
                    categoryName, categoryCode)
            );
        }
        return rateCoefficient;
    }
}
```

```java
        private LocalDate insuredBirthDate;
        /** 职业类别（意外险必填） */
        private Integer occupationCategory;
        /** 产品配置的最高可承保职业类别 */
        private Integer productMaxOccupationCategory;
        /** 产品配置的最低投保年龄 */
        private Integer productMinAge;
        /** 产品配置的最高投保年龄 */
        private Integer productMaxAge;
        /** 健康告知结果：是否有任何健康异常（健康险必填） */
        private Boolean hasHealthIssue;
        /** 申请保额 */
        private BigDecimal coverageAmount;
        /** 产品可承保的最高保额 */
        private BigDecimal productMaxCoverage;
    }

    /**
     * 核保结论枚举
     */
    public enum UnderwritingDecision {
        /** 自动核保通过，可立即出单 */
        AUTO_APPROVED,
        /** 需要人工核保，进入审核队列 */
        MANUAL_REVIEW_REQUIRED,
        /** 附加除外责任后可承保（特定情况不保） */
        APPROVED_WITH_EXCLUSION,
        /** 拒保 */
        DECLINED
    }

    /**
     * 核保结果
     */
    @Data
    public static class UnderwritingResult {
        private UnderwritingDecision decision;
        /** 加费系数（1.0 = 标准费率，1.5 = 加收50%） */
        private BigDecimal loadingFactor = BigDecimal.ONE;
        /** 核保说明 */
        private String message;
        /** 触发的规则列表（用于日志和调试） */
        private List<String> triggeredRules = new ArrayList<>();

        public static UnderwritingResult autoApprove() {
            UnderwritingResult result = new UnderwritingResult();
            result.setDecision(UnderwritingDecision.AUTO_APPROVED);
            result.setMessage("核保通过，可以出单");
            return result;
        }

        public static UnderwritingResult decline(String reason) {
            UnderwritingResult result = new UnderwritingResult();
            result.setDecision(UnderwritingDecision.DECLINED);
            result.setMessage(reason);
            return result;
        }

        public static UnderwritingResult manualReview(String reason) {
            UnderwritingResult result = new UnderwritingResult();
            result.setDecision(UnderwritingDecision.MANUAL_REVIEW_REQUIRED);
            result.setMessage(reason);
            return result;
        }
    }

    /**
     * 执行核保
     *
     * @param request 核保请求
     * @return 核保结论
     */
    public UnderwritingResult underwrite(UnderwritingRequest request) {
        log.info("[核保引擎] 开始核保，险种={}, 产品ID={}",
                request.getInsuranceType(), request.getProductId());

        UnderwritingResult result;

        // 根据险种类型分发到对应的核保逻辑
        result = switch (request.getInsuranceType()) {
            case ACCIDENT -> underwriteAccident(request);
            case HEALTH_MEDICAL -> underwriteHealthMedical(request);
            case HEALTH_CRITICAL -> underwriteHealthCritical(request);
            case PROPERTY, LIABILITY_PUBLIC, LIABILITY_EMPLOYER, LIABILITY_PRODUCT ->
                underwritePropertyAndLiability(request);
            default -> {
                log.warn("[核保引擎] 暂不支持该险种的自动核保: {}", request.getInsuranceType());
                yield UnderwritingResult.manualReview("该险种需人工核保");
            }
        };

        log.info("[核保引擎] 核保完成，结论={}，原因={}",
                result.getDecision(), result.getMessage());
        return result;
    }

    /**
     * 意外险核保逻辑
     */
    private UnderwritingResult underwriteAccident(UnderwritingRequest request) {
        List<String> rules = new ArrayList<>();

        // 规则1：年龄校验
        int age = calculateAge(request.getInsuredBirthDate());
        if (age < request.getProductMinAge() || age > request.getProductMaxAge()) {
            return UnderwritingResult.decline(
                String.format("投保年龄 %d 岁，超出产品承保范围（%d-%d岁）",
                    age, request.getProductMinAge(), request.getProductMaxAge())
            );
        }
        rules.add("年龄校验通过: " + age + "岁");

        // 规则2：职业类别校验（意外险最核心的核保规则）
        if (request.getOccupationCategory() == null) {
            return UnderwritingResult.decline("意外险必须填写职业类别");
        }

        try {
            OccupationCategoryEnum occupation =
                getOccupationCategory(request.getOccupationCategory());

            if (!occupation.isAcceptableFor(request.getProductMaxOccupationCategory())) {
                return UnderwritingResult.decline(
                    String.format("职业类别为%s，超出本产品最高可承保职业类别（%d类），无法投保",
                        occupation.getCategoryName(),
                        request.getProductMaxOccupationCategory())
                );
            }
            rules.add("职业类别校验通过: " + occupation.getCategoryName());

            // 规则3：高职业风险加费
            double loadingCoeff = occupation.getRateCoefficientOrThrow();
            UnderwritingResult result = UnderwritingResult.autoApprove();
            result.setLoadingFactor(BigDecimal.valueOf(loadingCoeff));
            result.setTriggeredRules(rules);

            if (loadingCoeff > 1.0) {
                result.setMessage(String.format("核保通过，因职业风险等级较高，保费系数为 %.2f", loadingCoeff));
            }
            return result;

        } catch (UnsupportedOperationException e) {
            return UnderwritingResult.decline(e.getMessage());
        }
    }

    /**
     * 医疗险核保逻辑
     */
    private UnderwritingResult underwriteHealthMedical(UnderwritingRequest request) {
        // 规则1：年龄校验
        int age = calculateAge(request.getInsuredBirthDate());
        if (age < request.getProductMinAge() || age > request.getProductMaxAge()) {
            return UnderwritingResult.decline(
                String.format("投保年龄 %d 岁，超出产品承保范围（%d-%d岁）", age,
                    request.getProductMinAge(), request.getProductMaxAge())
            );
        }

        // 规则2：健康告知校验（关键！）
        if (Boolean.TRUE.equals(request.getHasHealthIssue())) {
            // 有健康问题，转人工核保（可能被拒保或附加除外责任）
            return UnderwritingResult.manualReview(
                "健康告知中存在异常，需人工核保评估是否可以承保及是否需要附加除外责任"
            );
        }

        // 规则3：高龄人群需要人工复核
        if (age >= 60) {
            UnderwritingResult result = UnderwritingResult.manualReview(
                "被保险人年龄 >= 60岁，需人工复核"
            );
            result.getTriggeredRules().add("高龄人工复核规则");
            return result;
        }

        return UnderwritingResult.autoApprove();
    }

    /**
     * 重疾险核保逻辑
     */
    private UnderwritingResult underwriteHealthCritical(UnderwritingRequest request) {
        // 规则1：年龄校验
        int age = calculateAge(request.getInsuredBirthDate());
        if (age < request.getProductMinAge() || age > request.getProductMaxAge()) {
            return UnderwritingResult.decline(
                String.format("投保年龄 %d 岁不在承保范围内", age)
            );
        }

        // 规则2：重疾险必须完成健康告知
        if (request.getHasHealthIssue() == null) {
            return UnderwritingResult.decline("重疾险必须完成健康告知问卷");
        }

        if (Boolean.TRUE.equals(request.getHasHealthIssue())) {
            return UnderwritingResult.manualReview(
                "健康告知异常，重疾险需人工核保（可能被拒保）"
            );
        }

        // 规则3：高保额重疾险需要人工复核
        if (request.getCoverageAmount() != null &&
            request.getCoverageAmount().compareTo(new BigDecimal("5000000")) > 0) {
            return UnderwritingResult.manualReview(
                "申请保额超过 500万，需人工复核"
            );
        }

        return UnderwritingResult.autoApprove();
    }

    /**
     * 企财险 & 责任险核保逻辑
     * 企财险和责任险通常需要勘察/资料审核，直接转人工核保
     */
    private UnderwritingResult underwritePropertyAndLiability(UnderwritingRequest request) {
        // 小额标准化产品可以自动核保
        if (request.getCoverageAmount() != null &&
            request.getCoverageAmount().compareTo(new BigDecimal("500000")) <= 0) {
            return UnderwritingResult.autoApprove();
        }

        // 大额或复杂企财险/责任险需要人工核保
        return UnderwritingResult.manualReview(
            "企财险/责任险需提交营业执照、财产清单等资料，进入人工核保流程"
        );
    }

    /**
     * 计算年龄（精确到周岁）
     */
    private int calculateAge(LocalDate birthDate) {
        if (birthDate == null) {
            throw new IllegalArgumentException("出生日期不能为空");
        }
        return Period.between(birthDate, LocalDate.now()).getYears();
    }

    /**
     * 获取职业类别枚举
     */
    private OccupationCategoryEnum getOccupationCategory(int code) {
        for (OccupationCategoryEnum e : OccupationCategoryEnum.values()) {
            if (e.getCategoryCode() == code) return e;
        }
        throw new IllegalArgumentException("未知职业类别代码: " + code);
    }
}
```

### 8.5 健康告知存证服务

```java
package com.example.insurance.product.service;

import lombok.Data;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;

import java.security.MessageDigest;
import java.security.NoSuchAlgorithmException;
import java.time.LocalDateTime;
import java.util.HexFormat;
import java.util.List;

/**
 * 健康告知存证服务
 *
 * <p>监管要求：
 * 1. 不能预填答案，客户必须主动回答每一道题
 * 2. 记录每道题的停留时长（证明客户已阅读）
 * 3. 记录所有修改行为（从YES改为NO，需重点关注）
 * 4. 存证数据保留至保单终止后5年
 * </p>
 */
@Slf4j
@Service
public class HealthDeclarationService {

    /**
     * 健康告知答题记录 DTO
     */
    @Data
    public static class HealthAnswerDTO {
        /** 问题编码 */
        private String questionCode;
        /** 问题内容（快照，防止问题内容被修改后存证失效） */
        private String questionContent;
        /** 答案：true=是(有该情况) / false=否 */
        private Boolean answer;
        /** 在该题上停留的秒数 */
        private Integer dwellSeconds;
        /** 是否从YES修改为NO */
        private Boolean changedFromYesToNo;
    }

    /**
     * 健康告知合规校验结果
     */
    @Data
    public static class DeclarationValidationResult {
        /** 是否通过合规校验 */
        private boolean valid;
        /** 不合规原因 */
        private String failReason;
        /** 是否需要人工核保（有健康异常时） */
        private boolean requireManualUnderwriting;
        /** 异常问题数量 */
        private int abnormalAnswerCount;
    }

    /**
     * 保存健康告知答题记录
     *
     * @param applicationId 投保申请ID
     * @param insuredIdNumber 被保险人身份证号（仅存哈希，不存明文）
     * @param answers 答题记录列表
     * @return 合规校验结果
     */
    public DeclarationValidationResult saveAndValidate(
            Long applicationId,
            String insuredIdNumber,
            List<HealthAnswerDTO> answers) {

        // 1. 合规校验
        DeclarationValidationResult validation = validate(answers);

        // 2. 计算身份证号哈希（不在数据库存明文）
        String idHash = sha256(insuredIdNumber);

        // 3. 批量保存存证记录（实际项目中此处写入数据库）
        for (HealthAnswerDTO answer : answers) {
            log.info("[健康告知存证] 申请ID={}, 身份证哈希={}, 问题={}, 答案={}, 停留={}s, 是否修改={}",
                    applicationId, idHash, answer.getQuestionCode(),
                    answer.getAnswer() ? "是" : "否",
                    answer.getDwellSeconds(),
                    answer.getChangedFromYesToNo());
            // 实际项目：this.declarationTraceMapper.insert(buildTraceRecord(...));
        }

        // 4. 返回校验结果
        return validation;
    }

    /**
     * 健康告知合规校验
     *
     * <p>校验规则：
     * 1. 所有问题必须有答案（不能跳过）
     * 2. 每道题停留时间 >= 最低秒数（防止快速滑过）
     * 3. 统计有健康异常的答案数量
     * </p>
     */
    private DeclarationValidationResult validate(List<HealthAnswerDTO> answers) {
        DeclarationValidationResult result = new DeclarationValidationResult();
        result.setValid(true);

        int abnormalCount = 0;

        for (HealthAnswerDTO answer : answers) {
            // 规则1：不能有未作答的问题
            if (answer.getAnswer() == null) {
                result.setValid(false);
                result.setFailReason("问题 [" + answer.getQuestionCode() + "] 未作答");
                return result;
            }

            // 规则2：停留时间过短（< 2秒）视为未认真阅读，触发告警
            if (answer.getDwellSeconds() != null && answer.getDwellSeconds() < 2) {
                log.warn("[健康告知] 问题 {} 停留时间仅 {} 秒，可能未认真阅读",
                        answer.getQuestionCode(), answer.getDwellSeconds());
                // 注意：这里仅告警，不拒绝。监管暂无强制最低秒数规定。
                // 但此记录在理赔纠纷中可作为"是否认真阅读"的参考证据。
            }

            // 规则3：统计健康异常答案
            if (Boolean.TRUE.equals(answer.getAnswer())) {
                abnormalCount++;
            }
        }

        result.setAbnormalAnswerCount(abnormalCount);
        // 有任何健康异常答案，需要人工核保
        result.setRequireManualUnderwriting(abnormalCount > 0);

        return result;
    }

    /**
     * 计算 SHA-256 哈希（用于身份证号脱敏存证）
     */
    private String sha256(String input) {
        try {
            MessageDigest digest = MessageDigest.getInstance("SHA-256");
            byte[] hash = digest.digest(input.getBytes());
            return HexFormat.of().formatHex(hash);
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException("SHA-256 不可用", e);
        }
    }
}
```

### 9.1 非车险产品特征对比总览

| 险种 | 保障对象 | 核保核心 | 赔付模式 | 系统难点 |
|------|---------|---------|---------|---------|
| **个人意外险** | 被保险人人身 | 职业类别 | 一次性给付 | 职业分类管理 |
| **团体意外险** | N名员工人身 | 无健康告知 | 一次性给付 | 名单批量管理 |
| **百万医疗险** | 医疗费用 | 健康告知 + 年龄 | 报销型 | 免赔额计算、续保管理 |
| **重疾险** | 特定疾病 | 严格健康告知 | 确诊即给付 | 多次赔付状态机 |
| **企财险** | 企业财产 | 财产评估 | 损失补偿 | 保额确定、BI计算 |
| **公众责任险** | 对第三方责任 | 场所风险 | 赔偿实际损失 | 赔偿限额管理 |
| **雇主责任险** | 对员工责任 | 人员规模 | 赔偿实际损失 | 员工变更管理 |

### 9.2 产品中台设计要点回顾

```
产品中台设计三层结构：

第一层：ins_product 主表（通用字段）
  │ 保存所有险种共有的信息
  │ 一张表，所有险种共用

第二层：险种扩展表（差异化字段）
  ├── ins_product_accident_ext（意外险）
  ├── ins_product_health_medical_ext（医疗险）
  ├── ins_product_health_critical_ext（重疾险）
  ├── ins_product_property_ext（企财险）
  └── ins_product_liability_ext（责任险）
  
  注意：一张扩展表对应一个 ins_product 记录（1:1关系）

第三层：原子化责任配置（极致差异化）
  ├── ins_product_coverage（责任定义）
  └── ins_product_coverage_param（责任参数）
  
  注意：此层用于处理同一险种内不同产品的细粒度差异
  当第二层扩展表的字段仍不足以表达差异时，使用此层
```

## 📎 附录 C：下篇课后练习题答案

**练习题1（上篇）**：
- "心脏病晕倒"部分：心脏病是内因引起，**不符合**意外险要素，意外险**不赔**。
- "被车辆撞伤"部分：撞伤是外来、突发、非本意的伤害，**符合**意外险要素，意外险**赔付**车辆撞伤造成的损失。

**练习题2（上篇）**：
- 实际价值 500 万，保额 400 万，不足额投保比例 = 400/500 = 80%
- 实际损失 300 万，理赔金额 = 300 万 × 80% = **240 万**

**练习题4（上篇）**：
- 意外险不设等待期的原因：意外是突发的、不可预测的事件，投保人无法通过"等待"来规避风险（即不存在逆向选择问题）。而健康险设等待期，是防止已知患病的人立即投保后申请理赔（逆向选择）。
