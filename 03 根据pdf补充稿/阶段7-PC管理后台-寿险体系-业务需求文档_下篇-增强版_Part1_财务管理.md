## 1. 上游结算统计

> **对应 PDF**：`151_寿险财务管理上游结算统计.pdf`
> **排期表工时**：前端 2天 + 后端 3天（含上游保单结算合并工时）

### 1.1 功能概述

菜单路径：寿险 → 财务管理 → 上游结算统计

汇总展示各保司**应付佣金**（系统按政策计算的应收金额）与**实付佣金**（保司实际结算到账金额）的差异，是财务对账的总览视图。对应操作手册 PDF 151。

### 1.2 查询条件（来自 PDF 151 操作手册）

根据操作手册，支持如下精准查询条件：
- **机构**（机构选择下拉，支持多选）
- **保险公司**（多选）
- **开始时间**（日期选择器）
- **结束时间**（日期选择器）

> 📌 PDF 原文：「可根据机构、保险公司、开始时间、结束时间进行精准查询」

### 1.3 列表展示字段

| 字段名 | 说明 |
|---|---|
| 保险公司 | |
| 机构 | |
| 统计月份 | 格式：YYYY-MM |
| 应收佣金（元） | 系统按上游政策计算的应收金额，聚合自 `ins_life_upstream_policy_calc` |
| 实收佣金（元） | 保司已结算到账金额，聚合自 `ins_life_settlement` |
| 差异金额 | 应收 - 实收，正数=少收，负数=多收 |
| 差异率（%） | 差异金额 / 应收佣金 × 100% |

### 1.4 导出功能

点击【导出】按钮 → 自动跳转到**导出任务列表**，在导出任务列表中下载查看。

> 📌 PDF 原文：「点击导出即跳转到导出任务列表中下载查看即可」

**导出文件格式**：Excel（EasyExcel 异步生成），导出字段包含列表所有字段及明细。

> 📌 PDF 151 附有下载的表格内容示例，字段包括：保司名称、机构、月份、应收金额、实收金额、差额等。

### 1.5 后端实现要点

- 数据来源：聚合 `ins_life_settlement`（实收）与 `ins_life_upstream_policy_calc`（应收）两张表，按保司 + 月份分组
- 导出走异步任务，生成后写入 `ins_export_task` 表，轮询状态完成后提供下载链接
- 金额计算使用 BigDecimal，保留2位小数，差异率四舍五入保留4位

## 2. 上游保单结算

> **对应 PDF**：`152_寿险财务管理上游保单结算.pdf`
> **排期表工时**：含在上游结算统计工时中（前端 2天 + 后端 3天）

### 2.1 功能概述

菜单路径：寿险 → 财务管理 → 上游保单结算

按保单维度逐条确认保司佣金到账，是寿险财务收款的核心操作。对应操作手册 PDF 152。

### 2.2 一、上游对账计算（来自 PDF 152 §一）

#### 2.2.1 对账操作

> 📌 PDF 原文：「选择计算日期和保险公司，点击【开始计算】，提示计算成功即结算完成」

操作流程：
1. 选择**计算日期**（通常选当月或上月）
2. 选择**保险公司**
3. 点击【开始计算】按钮
4. 后端逻辑：
   - 查询指定时间段内该保司下的寿险保单
   - 按上游政策计算每张保单的应收佣金
   - 结果写入 `ins_life_upstream_calc`（policy_id, insurer_id, calc_date, calc_amount, status='PENDING'）
5. 系统提示「计算成功」

#### 2.2.2 对账查询

> 📌 PDF 原文：「根据计算日期和保险公司进行对账 ID 查询」

展示已计算完成的对账 ID 列表，可按计算日期 + 保险公司精准筛选。

### 2.3 二、上游保单对账（来自 PDF 152 §二）

#### 2.3.1 导入对账（导入保司结算单）

> 📌 PDF 原文：「点击【导入】输入对应保险公司，点击【模板下载】下载相应保司的对账模板」

操作流程：
1. 点击【导入】
2. 选择对应**保险公司**
3. 点击【模板下载】，下载该保司专属对账模板（不同保司字段不同，从 `ins_insurer_import_field_mapping` 获取字段映射）
4. 填写对账模板后上传 Excel
5. 后端解析：
   - 按保单号精确匹配 `ins_policy_life`
   - 将保司实付金额写入 `ins_life_settlement`（policy_id, insurer_id, settle_amount, settle_date, source='IMPORT', status='CONFIRMED'）
   - 匹配失败的行（保单号不存在）记录到异常列表
6. 展示导入结果：成功 N 条、失败 N 条（含保单号和失败原因）

> 📌 PDF 原文：「1.6 这是下载的表格内容」—— 模板字段包括：保单号、保司合同号、险种、年度保费、应付佣金率、应付佣金额等。

#### 2.3.2 根据保险公司、保单号等精准查询

> 📌 PDF 原文：「根据保险公司、保单号等信息精准查询到此保单的对账记录」「点击【导出】按钮跳转到导出任务列表进行下载」

- 支持按保险公司、保单号筛选单条记录的对账明细
- 可点击【导出】跳转导出任务列表下载

### 2.4 三、对账明细（来自 PDF 152 §三）

> 📌 PDF 原文：「根据机构、保险公司、对账区间、对账状态、收款状态进行精准查询」

**查询条件**：机构、保险公司、对账区间（开始/结束日期）、对账状态（待对账/已对账/差异）、收款状态（未到账/已到账）

### 2.5 开票操作（来自 PDF 152 §开票）

> 📌 PDF 原文：「勾选对账明细点击【开票】填好开票信息后点击开票即可」

操作流程：勾选已核对无误的对账明细 → 点击【开票】→ 填写开票信息（发票抬头/税号/金额/开票日期）→ 确认提交

#### 查看开票内容

> 📌 PDF 原文：「点击【查看】按钮可查看开票状态和开票内容」「点击【下载】可下载此对账明细内容」

- 每条明细支持点击【查看】查看开票状态和内容
- 点击【下载】可下载该条对账明细详情 Excel

### 2.6 收款操作（来自 PDF 152 §收款）

> 📌 PDF 原文：「已开票的对账明细点击【收款】，选择【收款日期】，点击【确定】即可完成收款操作」

操作流程：
1. 已开票的对账明细 → 点击【收款】
2. 选择**收款日期**
3. 点击【确定】
4. 后端：UPDATE status='SETTLED'；发送 MQ 消息通知佣金系统从池中扣减对应金额用于下游结算

### 2.7 差异处理

保司结算金额 ≠ 系统计算应收金额时，该条记录标记为「金额差异」。财务人员在差异列表中选择：
- **接受保司金额**：以保司实付金额为准更新记录
- **标注为差异待处理**：写入 `ins_life_reconcile_diff` 等待进一步核查

## 3. 机构计算

> **对应 PDF**：`153_寿险财务管理机构计算.pdf`
> **排期表工时**：前端 1.5天 + 后端 2.5天（含机构对账合并工时）

### 3.1 功能概述

菜单路径：寿险 → 财务管理 → 机构计算

将已到账的上游佣金，按寿险下游政策（机构费率/直销政策）分配到各机构和业务员。对应操作手册 PDF 153。

### 3.2 触发方式

**手动触发**：点击【开始计算】→ 选择计算月份和机构范围 → 确认 → 后端异步执行

**定时自动触发（XXL-Job）**：每月指定日期（从系统配置 `life.commission.calc.day` 读取）自动执行上月数据的机构计算

### 3.3 计算入口说明（结合 PDF 153 操作手册）

根据操作手册 PDF 153，机构计算页面提供以下筛选条件：
- 机构
- 保险公司
- 保单号
- 产品代码
- **结算月份**

> 📌 PDF 153 与 PDF 156（薪资计算）操作界面相似，均通过结算月份筛选数据。

### 3.4 计算逻辑

`InsLifeFinanceService.calcMachineCommission()` 方法：

1. 读取指定月份已到账的佣金记录（`ins_life_settlement.status='SETTLED'`）
2. 对每条保单记录：
   - 根据 policy 的 `insurer_id + product_id + agent_id + org_id` 匹配有效的下游政策
   - 按政策档位和比例基数（标保/保费）计算该业务员/机构应得佣金
   - 若有折标系数，先折算标保：`标保 = 年度保费 × 折标系数`
   - 写入 `ins_life_commission_calc`（policy_id, agent_id, org_id, calc_month, fyc_amount, ryc_amount, calc_policy_id, status='PENDING'）
3. 计算结果进入**待审核队列**，展示在列表供财务审核

### 3.5 计算结果审核

列表展示待审核的计算结果，字段：机构名称、业务员、险种、保司、计算月份、FYC金额、RYC金额、计算状态。

财务确认无误后点击【审核通过】→ UPDATE status='APPROVED'，进入待发放状态。

## 4. 机构对账

> **对应 PDF**：`154_寿险财务管理机构对账.pdf`
> **排期表工时**：含在机构计算工时中

### 4.1 功能概述

菜单路径：寿险 → 财务管理 → 机构对账

核对机构实际收到的佣金与系统计算应得佣金的差异。对应操作手册 PDF 154。

### 4.2 结算操作

1. 勾选需要对账的机构 → 点击【结算】按钮
2. 系统聚合该机构已审核通过的计算记录（`ins_life_commission_calc.status='APPROVED'`），生成结算报告
3. 提示「结算成功」后，记录更新为 `status='SETTLED'`
4. 点击【明细】可下载该机构的结算明细 Excel

**明细导出字段**：业务员、保单号、险种、保司、FYC、RYC、合计

### 4.3 对账差异标注

若机构反馈实际收款与系统计算有差异，在差异记录中标注原因：
- **保司扣减**：保司主动扣减了某些保单的佣金
- **政策变更**：核保后政策调整导致
- **结算周期差**：跨期结算导致统计口径不一致

差异标注写入 `ins_life_reconcile_diff`（calc_id, diff_amount, diff_reason, note, operator_id）。

## 5. 代理个税查询

> **对应 PDF**：`155_寿险财务管理代理个税查询.pdf`
> **排期表工时**：前端 2天 + 后端 2.5天（含薪资计算合并工时）

### 5.1 功能概述

菜单路径：寿险 → 财务管理 → 代理个税查询

汇总业务员当月寿险佣金收入，按累计预扣预缴法计算代扣个人所得税。对应操作手册 PDF 155。

### 5.2 一、个税报表（来自 PDF 155 §一）

> 📌 PDF 原文：「可根据机构选择、组织名称、薪酬月份等信息进行精准查询」「点击【导出】可自动跳转到导出任务列表进行下载」

**查询条件**：机构选择、组织名称、薪酬月份

**操作**：点击【导出】自动跳转到导出任务列表进行下载

### 5.3 二、结算记录（来自 PDF 155 §二）

> 📌 PDF 原文：「可根据机构选择、组织名称、薪酬月份等信息进行精准查询」

**查询条件**：机构选择、组织名称、薪酬月份（与个税报表查询条件相同，展示对应月份的结算汇总数据）

### 5.4 三、个税计算（来自 PDF 155 §三）

> 📌 PDF 原文：「点击【计算月份】选择需要计算的月份后点击【计算】即可」「可根据计算月份精准查询」

操作步骤：
1. 点击【计算月份】，选择需要计算的月份
2. 点击【计算】按钮，系统执行个税预扣计算
3. 计算完成后，可根据计算月份精准查询结果

### 5.5 个税计算逻辑（业务逻辑）

**累计预扣法（适用代理人佣金）**：

```
收入额 = 当月佣金收入 × (1 - 20%)  // 减除费用20%
应纳税所得额 = 收入额 - 减除费用（800元或20%择高）
应纳税额 = 应纳税所得额 × 适用税率 - 速算扣除数
当月预扣税额 = 本年累计应纳税额 - 已累计预扣税额
```

适用税率表（劳务报酬）：

| 应纳税所得额 | 税率 | 速算扣除数 |
|---|---|---|
| ≤20000元 | 20% | 0 |
| 20001-50000元 | 30% | 2000 |
| >50000元 | 40% | 7000 |

### 5.6 个税申报数据导出

点击【导出申报数据】→ 生成符合税务总局标准格式的 CSV/Excel（格式参照当年个税申报文件模板）。

### 5.7 完税证明生成

点击某业务员行的【生成完税证明】→ 后端使用 PDF 模板（Jasper Report 或 iText）填充数据 → 生成 PDF 存到 OSS → 返回下载链接。

## 6. 保单薪资计算

> **对应 PDF**：`156_寿险财务管理保单薪资计算.pdf`
> **排期表工时**：含在代理个税查询工时中

### 6.1 功能概述

菜单路径：寿险 → 财务管理 → 保单薪资计算

将寿险佣金与管理津贴、加扣款合并计算，生成业务员工资单，并推送给业务员。对应操作手册 PDF 156。

### 6.2 查询与计算（来自 PDF 156 §一）

> 📌 PDF 原文：「可根据机构、保险公司、保单号、产品代码、结算月份等条件进行计算」

**查询/筛选条件**：机构、保险公司、保单号、产品代码、结算月份

### 6.3 导入功能（来自 PDF 156 §二、§三）

> 📌 PDF 原文：「点击【导入】可下载导入模版，填写导入模版后进行导入」「导入成功后界面就可以查询到此记录了，可点击【删除】按钮删除此记录」

操作流程：
1. 点击【导入】
2. 下载**导入模版**（含字段说明）
3. 填写导入模版后进行上传导入
4. 导入成功后，列表可查询到该记录
5. 支持点击【删除】按钮删除某条记录

### 6.4 工资组成

```
工资 = FYC（首年佣金）+ RYC（续年佣金）+ 管理津贴 + 加款 - 扣款 - 代扣个税
```

- **FYC（First Year Commission）**：新单首年佣金，从 `ins_life_commission_calc.fyc_amount` 聚合
- **RYC（Renewal Year Commission）**：续年佣金，从 `ins_life_commission_calc.ryc_amount` 聚合
- **管理津贴**：机构负责人因下属业绩获得的管理费
- **加扣款**：从 `ins_life_upstream_deduct` 表读取
- **代扣个税**：从代理个税计算结果读取

### 6.5 工资单生成与推送

1. 选择工资所属月份 → 点击【计算工资】→ 后端聚合数据，生成 `ins_life_salary` 记录（status='DRAFT'）
2. 财务审核无误 → 点击【确认发放】→ UPDATE status='APPROVED'
3. 审核通过后自动推送工资条：
   - **站内信推送**：「XX月工资单已出，实发金额：XXX元，请查收」
   - **短信推送（可选）**：调用 SMS 服务（需在系统配置中开启 `life.salary.push.sms`）

## 7. 上游加扣管理

> **对应 PDF**：`157_寿险财务管理上游加扣管理.pdf`
> **排期表工时**：前端 1天 + 后端 2天

### 7.1 功能概述

菜单路径：寿险 → 财务管理 → 上游加扣款管理

记录保司对平台的额外加款（奖励政策）或扣款（退单扣点/投诉扣点）。对应操作手册 PDF 157。

### 7.2 一、新增加扣款（来自 PDF 157 §一）

> 📌 PDF 原文：「点击【新增】依次填写所属机构、所属保司等标星信息，其他内容选填，填写完成后点击【确定】即可」

点击【新增】弹出录入弹窗，**必填字段**（标红标星）：

| 字段名 | 必填 | 说明 |
|---|---|---|
| 所属机构 | ★ | 下拉选择 |
| 所属保司 | ★ | 下拉选择保险公司 |
| 类型 | ★ | 枚举：加款/扣款 |
| 金额 | ★ | 正数，方向由类型字段区分 |
| 发生日期 | ★ | 日期选择 |
| 原因说明 | ★ | 文本，最多500字 |
| 关联保单号 | 选填 | 若与特定保单相关则填写 |
| 附件 | 选填 | 上传保司通知文件/协议 |

填写完成后点击【确定】即可保存。

保存逻辑：INSERT `ins_life_upstream_deduct`（insurer_id, org_id, policy_id, type, amount, occur_date, reason, attachment_url, status='PENDING', operator_id）

### 7.3 二、加扣款查询（来自 PDF 157 §二）

> 📌 PDF 原文：「可根据机构、保险公司等筛选条件精准查到加扣款信息」

**查询条件**：机构、保险公司（其他可选筛选条件：类型、发生日期区间、状态）

### 7.4 三、编辑和删除（来自 PDF 157 §三）

> 📌 PDF 原文：「可以对此加扣款信息进行【编辑】和【删除】操作」「点击【编辑】可对已建立的加扣款信息进行修改（2023.12.24更新）」

**编辑**：点击【编辑】按钮，在弹窗中修改加扣款信息，保存后生效（注意：已审批通过且已参与结算的记录不可编辑）

**删除**：点击【删除】按钮，二次确认后删除记录

### 7.5 批量导入

点击【批量导入】→ 下载加扣款模板 → 填写 → 上传，EasyExcel 解析批量 INSERT，每条记录进入审批流。

### 7.6 审批流

新增/编辑加扣款进入审批流（`LIFE_DEDUCT_APPROVE`）：
- 待审批 → 财务主管审批通过 → status='APPROVED'
- 审批通过后，该加扣款参与当月上游结算计算

**反审批（撤回）**：仅当月未结算时可反审批，将 status 回退到 PENDING。
