## 1. 上游结算统计

### 1.1 功能概述

菜单：寿险 → 财务管理 → 上游结算统计

汇总展示各保司**应付佣金**（系统按政策计算的应收金额）与**实付佣金**（保司实际结算到账的金额）的差异，是财务对账的总览视图。

### 1.2 查询与展示

筛选条件：机构（多选）、保险公司（多选）、开始时间、结束时间。

列表字段：

| 字段名 | 说明 |
|---|---|
| 保险公司 | |
| 机构 | |
| 统计月份 | 格式：YYYY-MM |
| 应收佣金（元） | 系统按上游政策计算的应收金额，聚合自 `ins_life_upstream_policy_calc` |
| 实收佣金（元） | 保司已结算到账金额，聚合自 `ins_life_settlement` |
| 差异金额 | 应收 - 实收，正数=少收，负数=多收 |
| 差异率（%） | 差异金额 / 应收佣金 × 100% |

点击【导出】：异步 EasyExcel 导出，下载到本地。导出字段包含列表所有字段加明细。

## 2. 上游保单结算

### 2.1 功能概述

菜单：寿险 → 财务管理 → 上游保单结算

按保单维度逐条确认保司佣金到账，是寿险财务收款的核心操作。

### 2.2 操作流程

#### 2.2.1 对账计算（自动）

1. 点击【对账计算】Tab
2. 选择计算日期（通常选当月或上月）和保险公司
3. 点击【开始计算】按钮
4. 后端逻辑：
   - 查询指定时间段内，该保司下的寿险保单，按上游政策计算每张保单的应收佣金
   - 结果写入 `ins_life_upstream_calc`（policy_id, insurer_id, calc_date, calc_amount, calc_basis, status='PENDING'）
   - 提示"计算成功"，跳转到对账列表

对账查询：按计算日期和保险公司查询 `ins_life_upstream_calc` 记录，展示对账ID列表。

#### 2.2.2 上游保单对账（导入结算单）

1. 点击【上游保单对账】Tab → 点击【导入】
2. 选择保险公司 → 点击【模板下载】，下载该保司的对账模板（模板按保司差异化，从 `ins_insurer_import_field_mapping` 获取字段映射）
3. 填写后上传 Excel
4. 后端解析：
   - 按保单号精确匹配 `ins_policy_life`
   - 将保司实付金额写入 `ins_life_settlement`（policy_id, insurer_id, settle_amount, settle_date, source='IMPORT', status='CONFIRMED'）
   - 匹配失败的行（保单号不存在）记录到异常列表
5. 展示导入结果：成功N条、失败N条（含保单号和失败原因）

**差异处理：**
- 保司结算金额 ≠ 系统计算应收金额时，该条记录标记为"金额差异"
- 财务人员在差异列表中：选择"接受保司金额"或"标注为差异待处理"

#### 2.2.3 确认到账

已导入的结算记录，状态=`CONFIRMED` 后，佣金金额流入**佣金池**（`ins_commission_pool`），可用于下游分配发放。

具体操作：勾选已核对无误的结算记录 → 点击【确认到账】→ UPDATE status='SETTLED'；发送 MQ 消息通知佣金系统从池中扣减对应金额用于下游结算。

### 2.3 对账明细查询

筛选：机构、保险公司、对账区间、对账状态（待对账/已对账/差异）、收款状态（未到账/已到账）。

每条记录支持点击【明细】下载该条对账单详情 Excel。

## 3. 机构计算

### 3.1 功能概述

菜单：寿险 → 财务管理 → 机构计算

将已到账的上游佣金，按寿险下游政策（机构费率/直销政策）分配到各机构和业务员。

### 3.2 触发方式

**手动触发：** 点击【开始计算】→ 选择计算月份和机构范围 → 确认 → 后端异步执行

**定时自动触发（XXL-Job）：** 每月指定日期（从系统配置读取）自动执行上月数据的机构计算

### 3.3 计算逻辑

`InsLifeFinanceService.calcMachineCommission()` 方法：

1. 读取指定月份已到账的佣金记录（`ins_life_settlement.status='SETTLED'`）
2. 对每条保单记录：
   - 根据 policy 的 `insurer_id + product_id + agent_id + org_id` 匹配有效的下游政策
   - 按政策档位和比例基数（标保/保费）计算该业务员/机构应得佣金
   - 若有折标系数，先折算标保：`标保 = 年度保费 × 折标系数`
   - 写入 `ins_life_commission_calc`（policy_id, agent_id, org_id, calc_month, fyc_amount, ryc_amount, calc_policy_id, status='PENDING'）
3. 计算结果进入**待审核队列**，展示在列表供财务审核

### 3.4 计算结果审核

列表展示待审核的计算结果，字段：机构名称、业务员、险种、保司、计算月份、FYC金额、RYC金额、计算状态。

财务确认无误后点击【审核通过】→ UPDATE status='APPROVED'，进入待发放状态。

## 4. 机构对账

### 4.1 功能概述

菜单：寿险 → 财务管理 → 机构对账

核对机构实际收到的佣金（机构对账单）与系统计算应得佣金的差异。

### 4.2 操作流程

1. 勾选需要对账的机构 → 点击【结算】按钮
2. 系统聚合该机构已审核通过的计算记录（`ins_life_commission_calc.status='APPROVED'`），生成结算报告
3. 提示"结算成功"后，记录更新为 `status='SETTLED'`
4. 点击【明细】可下载该机构的结算明细 Excel（字段：业务员、保单号、险种、保司、FYC、RYC、合计）

### 4.3 对账差异标注

若机构反馈实际收款与系统计算有差异，在差异记录中标注原因：
- 保司扣减（保司主动扣减了某些保单的佣金）
- 政策变更（核保后政策调整导致）
- 结算周期差（跨期结算导致统计口径不一致）

差异标注写入 `ins_life_reconcile_diff`（calc_id, diff_amount, diff_reason, note, operator_id）。

## 5. 代理个税查询

### 5.1 功能概述

菜单：寿险 → 财务管理 → 代理个税查询

汇总业务员当月寿险佣金收入，按**累计预扣预缴法**计算代扣个人所得税（适用于劳务报酬/佣金类）。

### 5.2 个税计算逻辑

**累计预扣法（适用代理人佣金）：**

```
收入额 = 当月佣金收入 × (1 - 20%)  // 减除费用20%
应纳税所得额 = 收入额 - 减除费用（800元或20%择高）
应纳税额 = 应纳税所得额 × 适用税率 - 速算扣除数
当月预扣税额 = 本年累计应纳税额 - 已累计预扣税额
```

适用税率表（劳务报酬）：
| 应纳税所得额 | 税率 | 速算扣除数 |
|---|---|---|
| ≤20000 | 20% | 0 |
| 20001-50000 | 30% | 2000 |
| >50000 | 40% | 7000 |

**专项附加扣除：** 若业务员在系统中录入了专项附加扣除信息（子女教育/大病医疗等），在计算时一并扣除。

### 5.3 查询界面

筛选：税款所属月份、机构、业务员。

列表字段：业务员、所属机构、当月佣金收入、减除费用、应纳税所得额、税率、速算扣除数、当月代扣税额、累计代扣税额。

### 5.4 个税申报数据导出

点击【导出申报数据】→ 生成符合税务总局标准格式的 CSV/Excel（具体格式参照当年最新的个税申报文件模板，需运维配置模板文件路径）。

### 5.5 完税证明生成

点击某业务员行的【生成完税证明】→ 后端使用 PDF 模板（Jasper Report 或 iText）填充数据 → 生成 PDF 存到 OSS → 返回下载链接。

## 6. 薪资计算

### 6.1 功能概述

菜单：寿险 → 财务管理 → 保单薪资计算

将寿险佣金与管理津贴、加扣款合并计算，生成业务员工资单，并推送给业务员。

### 6.2 工资组成

```
工资 = FYC（首年佣金）+ RYC（续年佣金）+ 管理津贴 + 加款 - 扣款 - 代扣个税
```

- **FYC（First Year Commission）：** 新单首年佣金，从 `ins_life_commission_calc.fyc_amount` 聚合
- **RYC（Renewal Year Commission）：** 续年佣金，从 `ins_life_commission_calc.ryc_amount` 聚合
- **管理津贴：** 机构负责人因下属业绩获得的管理费，单独配置
- **加扣款：** 从 `ins_life_upstream_deduct` 表读取（见第7节）
- **代扣个税：** 从第5节计算结果读取

### 6.3 工资单生成操作

1. 选择工资所属月份 → 点击【计算工资】→ 后端聚合上述数据，生成 `ins_life_salary`（agent_id, salary_month, fyc, ryc, mgmt_allowance, add_amount, deduct_amount, tax, net_salary, status='DRAFT'）
2. 财务审核工资单数据无误 → 点击【确认发放】→ UPDATE status='APPROVED'
3. 审核通过后自动推送工资条：
   - **站内信推送：** 格式"XX月工资单已出，实发金额：XXX元，请查收"
   - **短信推送（可选）：** 调用 SMS 服务发送（需在系统配置中开启短信推送开关）

### 6.4 工资单查询

业务员可在 App 端查询自己的历史工资单（本需求为 PC 管理端，管理员查询所有人）。

筛选：业务员、工资月份、状态（草稿/已审核/已发放）。支持导出工资明细 Excel。

## 7. 上游加扣管理

### 7.1 功能概述

菜单：寿险 → 财务管理 → 上游加扣管理

记录保司对平台的额外加款（奖励政策/协议奖励）或扣款（退单扣点/投诉扣点），这些加扣款影响当期上游结算金额。

### 7.2 新增加扣款

点击【新增】，弹出录入弹窗：

| 字段名 | 必填 | 说明 |
|---|---|---|
| 类型 | ★ | 枚举：加款/扣款 |
| 保险公司 | ★ | 下拉 |
| 关联保单号 | 选填 | 若与特定保单相关则填写，可追溯来源 |
| 金额 | ★ | 正数，加款和扣款统一填正数，类型字段区分方向 |
| 发生日期 | ★ | 日期选择 |
| 原因说明 | ★ | 文本，最多500字 |
| 附件 | 选填 | 上传保司通知文件/协议 |

保存 INSERT `ins_life_upstream_deduct`（insurer_id, policy_id, type, amount, occur_date, reason, attachment_url, status='PENDING', operator_id）

### 7.3 审批流

新增加扣款进入审批流（`LIFE_DEDUCT_APPROVE`）：
- 待审批 → 财务主管审批通过 → status='APPROVED'
- 审批通过后，该加扣款参与当月上游结算计算

**反审批（撤回）：** 仅当月未结算时可反审批，将 status 回退到 PENDING。

### 7.4 批量导入

点击【批量导入】→ 下载加扣款模板 → 填写 → 上传，EasyExcel 解析批量 INSERT，每条记录进入审批流。

### 8.2 业绩报表

菜单：寿险 → 报表管理 → 业绩报表

#### 8.2.1 查询维度

支持按如下维度聚合分析（多维下钻）：

| 维度 | 说明 |
|---|---|
| 机构 | 机构树层级下钻 |
| 业务员 | 按个人汇总 |
| 险种 | 寿险/重疾/医疗/意外/年金 |
| 保险公司 | 各合作保司 |
| 时间 | 月/季/年 |

- **环比增长（%）：** 与上月对比

#### 8.2.3 展示方式

- 汇总表格：按所选维度展示各指标
- 柱状图/折线图：趋势展示（echarts）
- 导出 Excel：同步导出（数据量小）或异步（数据量大）

## 9. 继续率报表

### 9.1 功能概述

菜单：寿险 → 报表管理 → 继续率报表

继续率是寿险行业核心KPI，统计保单在第13个月、25个月时续缴的比例。

### 9.2 继续率计算逻辑

**13个月继续率（N13）：**

```sql
N13 = 
  满足条件的保单数（起保后第13个月仍有效且已续缴）
  ÷ 
  起保后第13个月到期应续缴的保单总数
  × 100%
```

**25个月继续率（N25）类同。**

**SQL实现思路：**
1. 取基准月（如本月）往前推12个月，找该月起保的保单为分母（应续缴保单）
2. 在分母保单中，找到其第13次缴费记录已存在（`ins_life_renewal_track` 中该期`status=已缴费`）的为分子
3. 继续率 = 分子 / 分母 × 100%

### 9.3 报表展示

筛选：险种、保险公司、业务员、机构、统计月份。

列表字段：险种、保司/业务员/机构（按维度）、13月到期件数、13月实缴件数、13月继续率（%）、25月到期件数、25月实缴件数、25月继续率（%）。

趋势折线图：展示近12个月的继续率趋势，若当月继续率 < 85%（可配置阈值）则数据标红。

## 11. 寿险H5后台配置

### 11.1 功能概述

菜单：寿险 → 系统管理 → H5后台配置

为寿险业务的 H5 投保页面（微信/App 端）提供后台内容配置，配置变更实时生效（不需重新发布前端）。

### 11.2 可配置项

| 配置项 | 类型 | 说明 |
|---|---|---|
| 主题色 | 颜色选择器 | HEX颜色值，影响H5页面主色调 |
| Banner图 | 图片上传 | OSS存储，建议尺寸750×300px，最多3张轮播 |
| 联系方式 | 文本 | 客服电话/微信，展示在H5页面底部 |
| 免责声明 | 富文本编辑器 | 投保前必读的免责声明内容 |
| 投保流程步骤 | 列表 | 配置H5投保流程各步骤（哪些步骤必填/可选），支持拖拽排序 |
| 险种H5字段 | 每个险种独立配置 | 该险种在H5投保时展示哪些字段（从产品字段定义中勾选） |

### 11.3 后端处理与热更新

配置保存到 `ins_life_h5_config` 表（key-value 结构，config_key, config_value, insurer_id/product_type, update_time）。

**Redis热更新实现：**
1. 保存配置 → UPDATE DB → 主动写入/刷新 Redis：`HSET life:h5:config:{key} value`
2. H5前端每次请求配置时，先读 Redis；Redis miss 则读 DB 并回写 Redis（TTL=24小时）
3. 配置更新时调用 `redisTemplate.delete("life:h5:config:*")` 主动清除相关 key，保证实时生效

### 13.2 寿险产品管理

菜单：产品 → 寿险产品管理

#### 13.2.1 产品列表与新增

点击【新增产品】，弹出产品录入弹窗（字段较复杂，分Tab展示）：

**基本信息 Tab：**

| 字段名 | 必填 | 说明 |
|---|---|---|
| 险种 | ★ | 枚举：寿险/重疾/医疗/意外/年金/万能险 |
| 产品代码 | ★ | 保司分配的产品代码，在同一保司内唯一 |
| 产品名称 | ★ | |
| 保险公司 | ★ | 下拉 |
| 保障期限类型 | ★ | 枚举：定期（X年）/终身 |
| 保障期限（年） | 条件必填 | 保障期限类型=定期时必填 |
| 缴费方式（可多选） | ★ | 趸缴/年缴/半年缴/季缴/月缴，支持多选 |
| 最低保额（元） | 选填 | |
| 最高保额（元） | 选填 | |
| 产品状态 | ★ | 枚举：上架/下架，新建默认下架 |
| 产品简介 | 选填 | 富文本 |
| 产品说明书 | 选填 | PDF附件上传，OSS存储 |

**佣金配置 Tab：**
- 配置首年佣金率（%）和续年佣金率（%）（此处为产品层级的预设费率，可被政策层级覆盖）
- 支持按缴费方式配置不同佣金率（表格形式，每行一种缴费方式）

**健康告知问卷 Tab：**
- 下拉多选，从已维护的健告问卷模板（`ins_questionnaire_template`）中绑定该产品适用的健康告知问卷
- 一个产品可绑定多份问卷（主问卷+特殊问卷）

**费率表 Tab：**
- 费率表为产品定价依据，字段：年龄段、性别、保障期、保费（元/万元保额）
- 支持 EasyExcel 导入（点击【导入费率表】），下载模板 → 填写 → 上传
- 上传后预览前20行确认 → 确认无误点【导入】→ INSERT `ins_life_product_rate`（product_id, age_min, age_max, gender, coverage_term, premium_per_unit）

#### 13.2.2 产品授权到机构

产品新建后默认不对任何机构开放，需单独授权：
1. 在产品列表点击【机构授权】
2. 弹出机构树选择框，勾选有权销售该产品的机构（支持批量勾选）
3. 保存 INSERT `ins_life_product_auth`（product_id, org_id, auth_time, operator_id）

取消授权：删除对应 `ins_life_product_auth` 记录。

#### 13.2.3 产品上下架

- 点击【上架】：UPDATE `ins_life_product.status = 'ON_SALE'`；校验：产品必须已配置费率表且已授权至少一个机构，否则提示错误
- 点击【下架】：UPDATE status = 'OFF_SALE'；下架后在业务员 App 端不可选用该产品

## 附录：接口规范说明

所有接口路径前缀：`/admin-api/ins/life/`

对应 Controller 类：
- 保单管理：`AdminInsLifePolicyController`（ins-order 模块）
- 回访/保全：`AdminInsLifeVisitController`（ins-order 模块）
- 孤儿单：`AdminInsLifeOrphanController`（ins-order 模块）
- 续期跟踪：`AdminInsLifeRenewalController`（ins-life 模块）
- 数据回传：`AdminInsLifeDataReturnController`（ins-life 模块）
- 财务管理：`AdminInsLifeFinanceController`（ins-life 模块）
- 报表管理：`AdminInsLifeReportController`（ins-life 模块）
- 政策管理：`AdminInsLifePolicyController`（ins-life 模块，注意与保单管理同名但在不同模块）
- 系统管理：`AdminInsLifeSystemController`（ins-life 模块）
- 产品管理：`AdminInsLifeProductController`（ins-life 模块）

**认证：** 所有 admin 接口需要在 Header 中携带 `Authorization: Bearer {token}`，通过 Gateway 统一鉴权，未登录返回 401，无权限返回 403。

**返回格式统一：**
```json
{
  "code": 0,
  "data": { ... },
  "msg": "success"
}
```
错误时 code 非0，msg 返回错误描述。
