# 阶段8 · C端商城 · 寿险投保 业务需求设计文档（上）

> **文档范围**：寿险商城入口（产品列表、产品详情、免费计划书）
>
> **所属模块**：`intermediary-module-ins-product`（产品中台）、`intermediary-module-ins-life`（寿险专属中台）
>
> **前端工程**：`intermediary-ui-consumer-uniapp`（C端消费者小程序）
>
> **接口路径前缀**：`/app-api/ins/life/`

---

## 一、总体流程说明

C端寿险入口复用商城基础框架，在商城首页新增"寿险"独立 Tab。用户通过产品列表→产品详情→计划书/立即投保的路径进入投保流程。本文档（上篇）覆盖商城入口三个功能点；中篇覆盖投保流程四个步骤；下篇覆盖C端保单管理与续期提醒。

## 二、功能点详细业务逻辑

### 2.1 寿险产品列表（C端）

#### 2.1.1 页面入口

商城首页底部 Tab 栏或顶部导航增加「寿险」Tab，点击后进入寿险产品列表页。该页面复用非车险 C 端产品列表组件，扩展寿险专有字段。

#### 2.1.2 列表展示逻辑

页面加载时调用 `GET /app-api/ins/life/product/page`，默认分页 10 条，按热销权重降序排列。

每张产品卡片展示以下信息：

- 险种图标（重疾险 / 寿险 / 医疗险 / 意外险 / 储蓄险，来自产品配置的 `category_icon` 字段）
- 产品名称
- 亮点文案（最多显示 3 条，来自 `ins_life_product.highlight_list` JSON 字段，超出截断）
- 参考保费（展示文案，如"30岁男性，20年期，月缴 ¥328 起"）
- 保障期限（如"保至终身 / 保至70岁"）
- 热销标签（`is_hot = 1` 时显示橙色"热销"角标）

#### 2.1.3 筛选条件

列表页顶部提供筛选栏，筛选参数传入接口 query 中：

| 筛选字段 | 说明 | 入参字段名 |
|---|---|---|
| 险种分类 | 重疾险/寿险/医疗险/意外险/储蓄险 | `category_code` |
| 年龄段 | 0-17岁/18-40岁/41-60岁/60岁以上 | `age_range` |
| 缴费期 | 趸缴/10年/20年/30年/终身 | `payment_period` |
| 保障额度 | 10万以下/10-50万/50-100万/100万以上 | `coverage_amount_range` |

用户选择筛选条件后，实时刷新列表（防抖 300ms）。筛选条件可多选，多个条件之间取交集。

#### 2.1.4 后端接口逻辑

接口 `GET /app-api/ins/life/product/page` 处理逻辑：

1. 查询 `ins_life_product` 表，`status = 1`（上架状态），`deleted = 0`。
2. 根据 `category_code`、`age_range`、`payment_period`、`coverage_amount_range` 拼接动态 WHERE 条件。
3. 年龄段筛选：转换为最小投保年龄 `min_insure_age` 和最大投保年龄 `max_insure_age` 的范围过滤。
4. 参考保费字段 `reference_premium_desc` 直接取产品配置的描述文字，不实时计算。
5. 返回分页结果，包含 `product_id`、`product_name`、`category_code`、`category_icon`、`highlight_list`、`reference_premium_desc`、`coverage_period`、`is_hot`。
6. 将热销产品（`is_hot = 1`）放在列表前排，其次按 `sort_weight` 降序。
7. 结果缓存至 Redis，Key：`ins:life:product:list:{category_code}:{page}`，TTL 5 分钟（产品配置变更时主动清除该缓存）。

#### 2.1.5 数据库表

**`ins_life_product`**（已在工程结构中定义，补充关键字段说明）：

```sql
CREATE TABLE `ins_life_product` (
  `id`                     BIGINT       NOT NULL AUTO_INCREMENT COMMENT '产品ID',
  `product_name`           VARCHAR(100) NOT NULL COMMENT '产品名称',
  `insurer_id`             BIGINT       NOT NULL COMMENT '保险公司ID',
  `category_code`          VARCHAR(20)  NOT NULL COMMENT '险种分类：CRITICAL/LIFE/MEDICAL/ACCIDENT/SAVING',
  `category_icon`          VARCHAR(255) COMMENT '险种图标URL',
  `highlight_list`         JSON         COMMENT '亮点文案，最多3条，JSON数组',
  `reference_premium_desc` VARCHAR(200) COMMENT '参考保费描述文字',
  `coverage_period`        VARCHAR(50)  COMMENT '保障期限描述，如"保至终身"',
  `is_hot`                 TINYINT      NOT NULL DEFAULT 0 COMMENT '是否热销：0否1是',
  `sort_weight`            INT          NOT NULL DEFAULT 0 COMMENT '排序权重，越大越靠前',
  `min_insure_age`         INT          NOT NULL DEFAULT 0 COMMENT '最小投保年龄（岁）',
  `max_insure_age`         INT          NOT NULL DEFAULT 70 COMMENT '最大投保年龄（岁）',
  `payment_period_list`    JSON         COMMENT '支持的缴费期，JSON数组，如["趸缴","10年","20年"]',
  `min_coverage_amount`    DECIMAL(15,2) COMMENT '最低保障额度（元）',
  `max_coverage_amount`    DECIMAL(15,2) COMMENT '最高保障额度（元）',
  `health_notice_json`     JSON         COMMENT '健康告知问题配置，JSON结构见下文',
  `underwriting_rules`     JSON         COMMENT '智能核保规则，JSON结构',
  `product_detail_json`    JSON         COMMENT '产品详情页各Tab内容（保障责任/费率表/投保须知/常见问题）',
  `dual_record_required`   TINYINT      NOT NULL DEFAULT 0 COMMENT '是否需要合规双录：0否1是',
  `status`                 TINYINT      NOT NULL DEFAULT 0 COMMENT '状态：0下架 1上架 2停售',
  `creator`                VARCHAR(64)  COMMENT '创建人',
  `create_time`            DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updater`                VARCHAR(64)  COMMENT '更新人',
  `update_time`            DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `deleted`                TINYINT      NOT NULL DEFAULT 0,
  PRIMARY KEY (`id`),
  INDEX `idx_category_code` (`category_code`),
  INDEX `idx_status` (`status`)
) ENGINE=InnoDB COMMENT='寿险产品表';
```

### 2.2 寿险产品详情（C端）

#### 2.2.1 页面入口

点击产品列表中任意产品卡片，进入产品详情页，URL 携带 `product_id`。

#### 2.2.2 页面结构

详情页分为以下区域：

**顶部固定区域**：产品名称、保险公司名称/Logo、险种标签。

**Tab 内容区**（滑动切换，默认展示"保障责任"Tab）：

| Tab名 | 内容来源字段 | 展示方式 |
|---|---|---|
| 保障责任 | `product_detail_json.coverage_items` | 列表，每项含责任名称+保障金额+说明文字 |
| 费率表 | `product_detail_json.rate_table` | 表格，行为年龄，列为保障额度×缴费期的对应保费 |
| 投保须知 | `product_detail_json.purchase_notice` | 富文本/HTML渲染 |
| 常见问题 | `product_detail_json.faq_list` | 手风琴展开/收起，每项含问题+答案 |

**底部固定双按钮**：

- 「免费计划」按钮（灰色辅助色）：点击跳转至 2.3 免费计划书功能
- 「立即投保」按钮（主题色）：点击进入投保流程（跳转至中篇 健康告知问卷页面）

**右侧悬浮操作**：

- 收藏按钮：点击调用 `POST /app-api/ins/life/product/favorite`，入参 `{product_id}`。已收藏状态显示实心红心图标，再次点击取消收藏（逻辑删除 `ins_member_favorite` 记录）。
- 评价入口：点击后展示用户评价弹窗（调用 `GET /app-api/ins/life/product/{id}/reviews`，分页10条，按时间降序）。
- 分享按钮：点击后弹出分享选项（微信好友/朋友圈），生成的分享链接包含当前登录业务员的追踪参数（`agent_id`），若用户通过分享链接访问并最终投保，则该保单关联该业务员作为归因来源。

#### 2.2.3 后端接口逻辑

接口 `GET /app-api/ins/life/product/{id}`：

1. 查询 `ins_life_product`，`id = {id}` 且 `status = 1`，否则返回"产品不存在或已下架"。
2. 查询当前登录用户 `member_id` 是否在 `ins_member_favorite` 中有该产品的收藏记录，返回 `is_favorite` 字段。
3. 解析 `product_detail_json` 返回各Tab内容。
4. 从请求 Header 或 Query 中解析 `agent_id`（业务员分享追踪参数），写入用户 Session / Redis（Key：`ins:trace:member:{member_id}`，TTL 30 天），供后续投保时归因使用。
5. 返回完整产品详情 VO（含 `is_favorite`、`insurer_name`、`insurer_logo`、各Tab内容）。

**评价接口** `GET /app-api/ins/life/product/{id}/reviews`：查询 `ins_life_product_review` 表，按 `create_time` 降序分页返回，字段包含昵称（脱敏，仅显示第一个字+**）、评分（1-5星）、评价内容、评价时间。

#### 2.2.4 费率表渲染说明

`rate_table` JSON 结构定义：

```json
{
  "headers": ["年龄", "保额50万/年缴20年", "保额100万/年缴20年"],
  "rows": [
    {"age": 20, "values": [2880, 5520]},
    {"age": 25, "values": [3120, 5980]},
    {"age": 30, "values": [3680, 7020]}
  ],
  "unit": "元/年"
}
```

前端根据此结构渲染为滚动表格，列数量根据 `headers` 动态生成，不超过 5 列时全部展示，超过 5 列时横向滚动。

### 2.3 寿险免费计划书（C端）

#### 2.3.1 页面入口

在产品详情页点击「免费计划」按钮，或在商城首页悬浮入口点击，弹出计划书输入弹窗（半屏蒙层）。

#### 2.3.2 表单字段与校验

弹窗内包含以下输入项（全部必填，无"必填"以外的可选项）：

| 字段 | 类型 | 校验规则 |
|---|---|---|
| 年龄 | 数字输入 | 必填，18-65整数，超出范围提示"请输入18-65岁" |
| 性别 | 单选（男/女） | 必填 |
| 期望保额 | 数字输入（单位：万元） | 必填，10-500整数，提示"10万-500万" |
| 邮箱 | 文本输入 | 选填，若填写则校验邮箱格式（正则：标准邮箱）；填写邮箱则可接收 PDF 计划书 |

点击「查看保障方案」按钮后，前端校验通过则调用接口，按钮变为加载状态（禁用防重复提交）。

#### 2.3.3 前端展示逻辑（简版试算）

接口调用成功后，弹窗内容切换展示简化版保障规划结果：

- 顶部显示"根据您的情况，推荐以下方案"
- 展示 2-3 个推荐产品卡片（产品名、月缴保费、核心保障亮点）
- 底部显示"已为您发送完整计划书至邮箱"（仅当用户填写了邮箱时显示）
- 「立即投保」按钮指向推荐产品详情页

#### 2.3.4 后端接口逻辑

接口 `POST /app-api/ins/life/plan/request`，入参：

```json
{
  "age": 30,
  "gender": "M",
  "expected_coverage": 100,
  "email": "user@example.com",
  "source_product_id": 12345
}
```

后端处理步骤：

1. **参数校验**：age 范围 18-65，gender 枚举值 M/F，expected_coverage 范围 10-500。
2. **简版推荐计算**：根据 age、gender、expected_coverage 查询 `ins_life_product` 表，筛选满足条件的产品（`min_insure_age <= age <= max_insure_age`，`min_coverage_amount <= expected_coverage * 10000 <= max_coverage_amount`，`status = 1`），按 `sort_weight` 取前3条。
3. **立即返回**推荐产品列表给前端（同步返回，不等待 PDF 生成）。
4. **潜在客户记录**：将输入信息写入 `ins_life_plan_lead` 表（见下方表结构），`agent_id` 从 Redis 归因缓存中取（Key：`ins:trace:member:{member_id}`）。若用户未登录，则以 `open_id` 或设备唯一标识记录。
5. **异步触发 PDF 生成**：若用户填写了邮箱，向 RocketMQ 发送 `ins_life_plan_pdf_generate` 消息，消息体包含 `{lead_id, age, gender, expected_coverage, email, recommended_product_ids}`。
6. **PDF 生成消费者**（`InsLifePlanPdfConsumer`）：从 AI 保障规划模块（`intermediary-module-ins-ai`）的 Feign 接口调用 `InsAiPlanApi.generatePlanPdf()`，生成 PDF 文件上传至 OSS，更新 `ins_life_plan_lead.pdf_url` 字段，状态改为"已生成"，同时调用邮件服务发送邮件、调用站内信服务推送消息给用户。

**`ins_life_plan_lead`** 表结构：

```sql
CREATE TABLE `ins_life_plan_lead` (
  `id`                BIGINT       NOT NULL AUTO_INCREMENT,
  `member_id`         BIGINT       COMMENT '用户ID（已登录）',
  `open_id`           VARCHAR(64)  COMMENT '微信OpenID（未登录）',
  `age`               INT          NOT NULL COMMENT '年龄',
  `gender`            CHAR(1)      NOT NULL COMMENT '性别：M男/F女',
  `expected_coverage` INT          NOT NULL COMMENT '期望保额（万元）',
  `email`             VARCHAR(100) COMMENT '邮箱',
  `agent_id`          BIGINT       COMMENT '归因业务员ID',
  `source_product_id` BIGINT       COMMENT '来源产品ID',
  `pdf_url`           VARCHAR(500) COMMENT 'PDF计划书OSS地址',
  `pdf_status`        TINYINT      NOT NULL DEFAULT 0 COMMENT 'PDF状态：0待生成1生成中2已生成3失败',
  `create_time`       DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `deleted`           TINYINT      NOT NULL DEFAULT 0,
  PRIMARY KEY (`id`),
  INDEX `idx_member_id` (`member_id`),
  INDEX `idx_agent_id` (`agent_id`)
) ENGINE=InnoDB COMMENT='寿险免费计划书潜在客户记录';
```

#### 2.3.5 归因业务员通知

PDF 生成完成后，若存在 `agent_id`，同时向该业务员推送一条"潜在客户通知"（极光推送），内容为"您的客户[昵称]申请了[产品名]的保障方案，可主动跟进"，并在业务员 App 的"客户线索"列表中新增一条记录。
