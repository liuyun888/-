## 1. 续期跟踪

> **PDF 参考：** `PDF-128`、`PDF-129`、`PDF-130`、`PDF-131`

### 1.1 功能概述

**菜单路径：** 寿险 → 续期管理 → 续期跟踪

**子菜单（含 PDF-128 目录结构）：**
- 续期跟踪（含续期查询、续期撤回、批量导入）
- 续期缴费

续期跟踪用于记录保单续期操作的具体情况，核心功能是：查询续期跟踪明细、记录跟进、导出、撤回、修改。

### 1.2 续期跟踪列表查询

> **PDF 参考：** `PDF-130`（续期跟踪-续期查询）

**列表数据来源：** 查询 `ins_policy_life` 中 `policy_status = ACTIVE` 且根据缴费方式计算出下次缴费日期在指定范围内的保单。

**续期周期计算逻辑（`InsLifeRenewalService.calcNextDueDate`）：**
- 年缴：`start_date + N年`（N = 当前是第几年）
- 半年缴：`start_date + N*6个月`
- 季缴：`start_date + N*3个月`
- 月缴：`start_date + N月`

**默认展示范围：** 未来60天内到期的保单（可通过筛选条件调整）。

**筛选条件：**

| 筛选项 | 类型 | 说明 |
|---|---|---|
| 录入业务员 | 下拉搜索 | 按录入人筛选 |
| 险种 | 下拉多选 | |
| 保险公司 | 下拉多选 | |
| 缴费到期日期区间 | 日期范围选择器 | |
| 跟进状态 | 枚举 | 未跟进/跟进中/已缴费/已拒缴/放弃 |

**列表字段：** 保单号、被保人、险种、保司、年度保费、缴费方式、下次缴费日、距到期天数、跟进状态、最后跟进时间、操作。

**操作按钮（每行）：**
- 【跟进】：录入跟进记录（详见 1.3）
- 【修改】：对续期操作完成的保单进行修改（标星为必填项）（PDF-130）
- 【撤回】：将续期操作完成的保单进行单条撤回（PDF-130）

**页面级批量操作：**
- 【批量撤回】：勾选多张保单后可批量撤回（PDF-130，左侧勾选保单后点批量撤回）
- 【导出】：筛选出要导出的续期明细后点击导出（PDF-130）
- 【批量导入】：批量导入续期需求（PDF-131，详见 1.4）

### 1.3 续期跟进记录录入

点击列表某行的【跟进】按钮，弹出跟进记录录入弹窗：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 联系时间 | ★必填 | 默认当前时间 |
| 联系方式 | ★必填 | 枚举：电话/微信/短信/上门 |
| 联系结果 | ★必填 | 枚举：无人接听/已联系-准备续缴/已联系-犹豫中/已联系-明确拒缴/已缴费 |
| 备注 | 选填 | 最多500字 |
| 下次跟进计划 | 选填 | 日期时间 |

**保存后端逻辑：**
1. INSERT `ins_life_renewal_track`（policy_id, contact_time, contact_type, contact_result, note, next_plan_time, operator_id）
2. UPDATE `ins_life_renewal.last_follow_time` 和 `follow_status`
3. 若联系结果=已缴费，则更新 `follow_status = PAID`

### 1.4 批量导入续期需求

> **PDF 参考：** `PDF-131`（续期跟踪-需求批量导入）

**操作流程（PDF-131 详述）：**
1. 点击【批量导入】→ 点击【下载模版】，下载系统模版
2. 填写导入模板数据
   - 注意：模板前 9 行不能删除或修改
   - 带 ※ 的列是必填项
   - 其他项目按模板格式填写
3. 填好模板后，点击【批量导入】→ 选取文件进行导入
4. 导入有问题时会弹出报错窗口，点击导出查看校验结果，根据提示修改原表格后再次导入
5. 提示"导入成功"后，即可在系统查到导入的续期信息

**模板关键字段（结合 PDF-131）：**

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 保单号 | ★必填 | 与系统已有保单匹配 |
| 续期状态 | ★必填 | 如：已缴费/未缴费 |
| 实缴金额 | 选填 | 实际缴纳保费 |
| 实缴日期 | 选填 | 实际缴费日期 |

**后端逻辑：**
- EasyExcel 解析，按保单号匹配更新 `ins_life_renewal_track` 记录
- 每批最多500条，超出分批处理
- 生成导入报告（成功条数/失败条数/错误明细）
- 导入失败可导出错误明细进行核查

### 1.5 续期修改

> **PDF 参考：** `PDF-130`

点击列表行【修改】按钮，进入续期记录编辑页面：
- 标星字段为必填项
- 修改后点击【保存】生效
- UPDATE `ins_life_renewal_track` 对应记录

### 1.6 续期撤回

> **PDF 参考：** `PDF-130`

**单条撤回：** 点击列表行【撤回】按钮，确认弹窗后将该续期操作状态回滚。

**批量撤回：** 勾选左侧多张保单 → 点击【批量撤回】→ 系统批量撤销选中的续期操作。

**撤回后端逻辑：**
- UPDATE `ins_life_renewal_track.status = CANCELED`
- 清空关联的跟进状态变更（视业务规则）

### 1.7 续期跟踪导出

> **PDF 参考：** `PDF-130`

筛选出要导出的续期明细后，点击【导出】按钮：
- EasyExcel 异步导出
- 支持导出当前筛选结果的全部续期跟踪明细
- 导出文件包含：保单号、被保人、险种、保司、年度保费、缴费方式、下次缴费日、跟进状态、最后跟进时间等字段

## 2. 数据回传

> **PDF 参考：** `PDF-135`（数据回传-目录）、`PDF-136`（数据回传-交互平台）、`PDF-137`（数据回传-数据回传）

### 2.1 功能概述

**菜单路径：** 寿险 → 数据回传

**子菜单（PDF-135 目录）：**
- 交互平台：从保司同步回来的保单数据，支持筛选查看和批量修改
- 数据回传：将系统数据回传给保司或监管，支持手动触发和查看回传结果

### 2.2 交互平台（保司同步数据管理）

> **PDF 参考：** `PDF-136`（数据回传-交互平台）

**菜单路径：** 寿险 → 数据回传 → 交互平台

**注意：** 交互平台的核心功能是查看从保司**同步过来**的保单数据（即保司→平台方向），而非向外回传。

#### 2.2.1 交互平台查询

从交互平台同步系统的保单，支持筛选查看。

**筛选条件：**

| 筛选项 | 类型 |
|---|---|
| 保单号 | 输入框，精确或模糊匹配 |
| 保险公司 | 下拉多选 |
| 同步时间范围 | 日期范围选择器 |
| 数据状态 | 枚举：同步成功/同步失败/待处理 |

**列表字段：** 保单号、被保人、险种、保司、同步时间、数据状态、操作。

#### 2.2.2 批量修改同步数据

> **PDF 参考：** `PDF-136`

**操作流程：**
1. 点击【批量修改】→ 下载批量修改模版
2. 按照系统模板提示填写需要修改的保单信息（可在 Sheet2 查看保险公司代码）
3. 填写保存完成后，点击【导入文件】，选择填好的模板表格进行导入
4. 导入有问题时会弹出报错窗口，点击"下载校验文件"进行检查
5. 根据最后一行提示修改原表格，再次导入，导入成功即修改完成

**模板字段说明：**
- A列：保单号（必填，与系统已有保单匹配）
- B列及后续：需要修改的保单数据字段
- Sheet2：保险公司代码参考表

**后端逻辑：**
- EasyExcel 解析，按保单号匹配更新同步数据
- 如导入数量较多，系统会统一处理，处理完后在查询列表查看

### 2.3 数据回传（平台→保司方向）

> **PDF 参考：** `PDF-137`（数据回传-数据回传）

**菜单路径：** 寿险 → 数据回传 → 数据回传

#### 2.3.1 回传数据查看

通过保单号在保司回传搜索中搜索到回传记录，点击【查看】可以：
- 查看回传内容详情
- 查看失败原因，根据失败原因具体排查修改（PDF-137）

**筛选条件：**

| 筛选项 | 类型 |
|---|---|
| 保单号 | 输入框 |
| 保险公司 | 下拉选择 |
| 回传时间范围 | 日期范围选择器 |
| 回传状态 | 枚举：成功/失败/部分失败/待处理 |

**列表字段：** 保单号、保险公司、回传数据类型、回传时间、总数、成功数、失败数、状态、操作。

#### 2.3.2 手动触发回传

> **PDF 参考：** `PDF-137`

**操作步骤：**
1. 选择保司、数据类型、回传时间段（起止日期）
2. 点击【开始回传】按钮
3. 后端异步执行（RocketMQ 异步，避免超时）：
   - 从 `ins_policy_life` 查询指定时间段内的保单数据
   - 按对应保司的配置格式进行数据转换（适配器模式：`InsLifeDataReturnAdapter` 接口，每个保司有独立实现类）
   - API方式：HTTP POST 到保司接口
   - SFTP方式：生成文件，SFTP上传
   - 记录回传结果到 `ins_life_data_return_log`（insurer_id, data_type, date_range, total_count, success_count, fail_count, status, error_detail, return_time）
4. 完成后发站内信通知操作人结果

#### 2.3.3 手动再次同步回传（补传）

> **PDF 参考：** `PDF-137`

针对某次失败的回传记录：
1. 修改保单数据中导致失败的字段
2. 点击【重试】/【再次同步】按钮，手动操作再次同步
3. 系统重新执行该批次回传

#### 2.3.4 查看失败详情

对于"失败"或"部分失败"的回传记录，点击【查看详情】：
- 展示每条失败数据的保单号、错误字段、错误原因（从保司返回的错误信息解析）
- 支持下载失败数据明细 Excel
- 根据失败原因指导用户修正数据后重试

### 2.4 保司接口配置（系统配置级）

为每家保司配置数据回传接口，存储到 `ins_life_data_return_config` 表：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 保险公司 | ★必填 | 下拉选择已配置的寿险合作保司 |
| 传输方式 | ★必填 | 枚举：API接口/SFTP文件传输 |
| 接口地址 | 条件必填 | 传输方式=API时必填，HTTP/HTTPS URL |
| SFTP地址 | 条件必填 | 传输方式=SFTP时必填，含IP/端口/路径 |
| 认证方式 | ★必填 | 枚举：API-KEY/用户名密码/证书 |
| AppID/用户名 | ★必填 | |
| AppKey/密码 | ★必填 | 加密存储（AES-256） |
| 数据格式 | ★必填 | 枚举：JSON/XML/CSV/固定格式Excel |
| 回传数据类型 | ★必填 | 多选：保单数据/续期数据/理赔数据 |
| 是否启用 | ★必填 | 开关，默认关闭 |

点击【测试连接】：后端尝试连接对应接口（API则发探测请求，SFTP则尝试建立连接），返回"连接成功"或"连接失败（含错误原因）"。

## 3. 理赔管理

> **PDF 参考：** `PDF-138`（寿险-理赔管理）

### 3.1 功能概述

**菜单路径：** 寿险 → 理赔管理

记录和跟进寿险理赔案件全生命周期。

**页面主要操作（PDF-138）：**
- 点击【新增】录入新理赔记录
- 通过保险公司、机构选择等筛选条件精准筛选理赔信息
- 在查询列表进行：查询、修改、查看明细

### 3.2 新增理赔案件

> **PDF 参考：** `PDF-138`

**操作路径：** 点击【寿险】→【理赔管理】→【新增】

填写所有信息后点击保存，保存成功后可在查询列表根据保险公司、机构选择等筛选条件查询。

**录入字段：**

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 保单号 | ★必填 | 输入后联查带出：被保人姓名、险种、保司、保额 |
| 出险时间 | ★必填 | 日期时间选择器 |
| 出险类型 | ★必填 | 枚举：身故/全残/重大疾病/轻症/住院/意外 |
| 出险经过 | ★必填 | 文本，最多2000字 |
| 赔案号 | 选填 | 保司分配的理赔流水号，可后续补录 |
| 已提交材料 | 选填 | 多选枚举：死亡证明/医疗报告/身份证/银行卡/其他 |
| 负责业务员 | ★必填 | 默认保单归属业务员，可修改 |

**保存后端逻辑：**
- INSERT `ins_claim_record`（policy_id, claim_no, accident_time, accident_type, accident_desc, case_no, submitted_docs, agent_id, status='PROCESSING', create_time）

### 3.3 案件查询与列表

> **PDF 参考：** `PDF-138`

**筛选条件：**

| 筛选项 | 类型 |
|---|---|
| 保险公司 | 下拉多选 |
| 机构选择 | 机构树下拉 |
| 保单号 | 输入框 |
| 被保人姓名 | 输入框，模糊匹配 |
| 案件状态 | 枚举：处理中/已结案/已拒赔 |
| 出险类型 | 枚举多选 |
| 出险时间范围 | 日期范围选择器 |

**列表操作（PDF-138）：**
- 【查询】：按条件筛选
- 【修改】：对已录入的理赔案件进行信息修改
- 【查看明细】：进入案件详情页，查看跟进记录、赔付结果等完整信息

### 3.4 案件跟进记录

在案件详情页，点击【添加跟进】录入跟进信息：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 跟进时间 | ★必填 | 默认当前时间 |
| 处理进展 | ★必填 | 文本描述 |
| 保司补充材料要求 | 选填 | 记录保司要求补交的材料清单 |
| 附件 | 选填 | 上传相关材料，OSS存储 |

**保存后端逻辑：**
- INSERT `ins_claim_follow_record`（claim_id, follow_time, progress, doc_require, attachment_urls, operator_id）

### 3.5 赔付结果录入

案件跟进到结案时，点击【录入赔付结果】：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 赔付金额 | ★必填 | BigDecimal，精度2位小数 |
| 赔付日期 | ★必填 | 日期 |
| 赔付账号 | ★必填 | 收款银行卡/账号信息 |
| 赔付结果说明 | 选填 | 文本 |

**保存后端逻辑：**
- UPDATE `ins_claim_record.status = SETTLED`、`claim_amount`、`claim_date`
- 若出险类型=身故，联动保单状态变更为 `LAPSE`（失效），并发站内信给业务员

### 3.6 理赔统计看板

- 总案件数（本月/年）
- 平均处理天数（从开案到结案天数平均值）
- 总赔付金额
- 拒赔率

以上指标按险种/保司可下钻，SQL聚合计算。
