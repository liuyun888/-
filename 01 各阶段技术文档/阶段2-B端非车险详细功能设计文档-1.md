## 一、模块概述

### 1.1 业务背景

非车险业务(寿险、健康险、意外险等)是保险中介公司的核心利润来源,具有件均保费高、佣金比例高、客户价值高的特点。本模块旨在为业务员提供一整套非车险展业工具,包括:

- **产品库管理**: 浏览、搜索、对比非车险产品
- **保费试算**: 快速计算保费和利益演示
- **计划书制作**: 生成专业的保障方案计划书
- **客户管理CRM**: 管理客户档案和跟进记录
- **订单管理**: 跟踪订单状态
- **业绩统计**: 可视化业绩数据
- **佣金管理**: 查看佣金明细和提现

## 三、模块一: 产品库管理

### 3.1 业务逻辑说明

#### 3.1.1 产品列表展示逻辑

**核心业务需求:**
- 业务员需要快速浏览所有在售产品
- 支持按保险公司、产品分类、保障期限等维度筛选
- 支持按销量、佣金比例、保费高低排序
- 突出显示热销产品和高佣产品

**数据加载策略:**
1. 默认加载前20条产品(在售状态)
2. 按销量降序排列
3. 优先展示热销标签(销量>100件)
4. 标记高佣产品(佣金率>60%)

**筛选联动逻辑:**
- 选择保险公司后,产品分类自动过滤该公司的分类
- 选择产品分类后,保障期限选项自动调整为该分类支持的期限
- 年龄范围筛选会自动过滤不符合投保年龄的产品

#### 3.1.2 产品详情展示逻辑

**信息组织结构:**
```
产品详情页 = {
  基础信息区: {
    产品名称、保险公司、分类、投保年龄、保额范围
  },
  产品亮点区: {
    3-5个核心卖点,图标+文字形式展示
  },
  保障责任区: {
    主险责任、附加险责任、特色保障
    按重要性排序,高频责任前置
  },
  费率说明区: {
    费率表、缴费示例、保费试算入口
  },
  投保须知区: {
    健康告知要点、免责条款、理赔流程
  },
  常见问答区: {
    FAQ列表,支持折叠展开
  },
  投保案例区: {
    典型案例,包含客户画像、方案配置、理赔实例
  }
}
```

**数据加载优化:**
- 基础信息从MySQL直接读取
- 详细信息(保障责任、FAQ等)首次访问时从Redis缓存读取
- 缓存失效后从MySQL加载并重建缓存
- 缓存过期时间: 1小时

#### 3.1.3 产品对比逻辑

**对比维度设计:**
```
对比表格 = {
  横轴: 产品A、产品B、产品C (最多4个产品)
  纵轴: {
    基础信息: [保险公司、产品类型、保障期限、缴费期限]
    投保条件: [投保年龄、保额范围、职业限制、健康要求]
    保障责任: [重疾保额、轻症保额、中症保额、身故保障、特色保障]
    费率对比: [30岁50万保额年缴保费、35岁50万保额年缴保费]
    增值服务: [就医绿通、质子重离子、特药服务]
    佣金信息: [首年佣金率、续年佣金率]
  }
}
```

**差异高亮规则:**
- 数值型字段: 最优值高亮绿色,最差值灰色
- 布尔型字段: 有则显示√,无则显示-
- 责任覆盖: 独有责任标记红色星号
- 保费对比: 最低保费显示"最优"标签

**对比算法实现:**
1. 提取所有产品的coverage_detail字段
2. 遍历JSON结构,提取共同责任和差异责任
3. 构建对比矩阵数据结构
4. 前端渲染时应用差异高亮样式

#### 3.1.4 产品搜索逻辑

**搜索场景:**
- 场景1: 客户说"我想买重疾险" → 搜索分类
- 场景2: 客户说"平安的产品" → 搜索保险公司
- 场景3: 客户说"保到70岁的" → 搜索保障期限
- 场景4: 客户说"给孩子买" → 搜索投保年龄0-17岁
- 场景5: 客户说"带身故责任" → 全文搜索保障责任

**ElasticSearch索引设计:**
```json
{
  "mappings": {
    "properties": {
      "productId": {"type": "long"},
      "productName": {"type": "text", "analyzer": "ik_max_word"},
      "productCode": {"type": "keyword"},
      "companyName": {"type": "text", "analyzer": "ik_smart"},
      "categoryName": {"type": "keyword"},
      "highlights": {"type": "text", "analyzer": "ik_max_word"},
      "coverageText": {"type": "text", "analyzer": "ik_max_word"},
      "minAge": {"type": "integer"},
      "maxAge": {"type": "integer"},
      "coveragePeriod": {"type": "keyword"},
      "paymentPeriod": {"type": "keyword"},
      "salesCount": {"type": "integer"},
      "commissionRate": {"type": "float"}
    }
  }
}
```

**搜索权重配置:**
- 产品名称匹配: 权重5.0
- 保险公司匹配: 权重3.0
- 产品亮点匹配: 权重2.0
- 保障责任匹配: 权重1.5
- 分类名称匹配: 权重1.0

**搜索结果排序:**
1. 优先按相关性得分排序
2. 相关性相同时,按销量降序
3. 销量相同时,按佣金率降序
4. 支持用户手动切换排序维度

### 3.2 功能实现要点

#### 3.2.1 产品数据同步

**同步触发时机:**
- 产品信息变更时(新增/修改/下架)
- 每日凌晨2点全量同步一次
- 保障责任等复杂字段变更时增量同步

**同步流程:**
```
1. 监听产品表变更事件(Canal或Binlog监听)
2. 提取变更的产品ID
3. 查询产品完整信息
4. 构建ES文档
5. 调用ES批量更新API
6. 更新Redis缓存
7. 记录同步日志
```

#### 3.2.2 产品推荐算法

**推荐场景:**
- 产品详情页"相似产品推荐"
- 客户画像匹配产品推荐
- 销售线索智能推荐

**推荐策略:**
```
协同过滤推荐 = {
  基于产品: 同一客户购买过的其他产品
  基于用户: 相似客户购买的产品
}

内容推荐 = {
  相同分类产品
  相同保险公司产品
  价格区间相近产品
}

规则推荐 = {
  if (客户年龄 < 30 && 无保障) → 推荐定期重疾险
  if (客户年龄 30-45 && 有家庭) → 推荐终身重疾险+定期寿险
  if (客户年龄 > 45) → 推荐医疗险+防癌险
  if (客户职业类别 >= 4) → 推荐意外险
}
```

**推荐权重计算:**
```
最终得分 = 协同过滤得分 * 0.4 + 内容推荐得分 * 0.3 + 规则推荐得分 * 0.3
```

#### 3.2.3 产品收藏功能

## 四、模块二: 保费试算

### 4.1 业务逻辑说明

#### 4.1.1 试算场景分析

**典型试算场景:**
```
场景1: 快速试算
  客户问: "30岁买50万重疾险多少钱?"
  业务员: 选择产品 → 输入年龄30、保额50万 → 立即得出年缴保费
  
场景2: 方案对比
  客户问: "交10年和交20年,哪个更划算?"
  业务员: 设置两组缴费期限 → 对比总保费和年缴压力 → 给出建议

场景3: 保额测算
  客户问: "我预算1万元,能买多少保额?"
  业务员: 输入预算金额 → 反算保额 → 推荐合适方案

场景4: 利益演示
  客户问: "如果买了这个产品,未来能拿多少钱?"
  业务员: 生成现金价值表 → 演示年金领取 → 计算IRR
```

#### 4.1.2 保费计算引擎设计

**计算公式管理:**
- 每个产品配置Groovy脚本作为计算公式
- 脚本支持复杂的费率表查询和计算逻辑
- 支持动态参数传入(年龄、性别、保额、缴费期等)

**费率表结构:**
```json
{
  "rateTable": {
    "male": {
      "30": {"10年": 0.0065, "20年": 0.0045, "终身": 0.0035},
      "31": {"10年": 0.0068, "20年": 0.0048, "终身": 0.0038}
    },
    "female": {
      "30": {"10年": 0.0055, "20年": 0.0038, "终身": 0.0030},
      "31": {"10年": 0.0058, "20年": 0.0041, "终身": 0.0033}
    }
  }
}
```

**Groovy脚本示例逻辑:**
```
1. 根据被保人性别选择费率表(male/female)
2. 根据投保年龄查询对应年龄的费率
3. 根据缴费期限获取费率系数
4. 计算公式: 年缴保费 = 保额 × 费率系数
5. 如有附加险,累加附加险保费
6. 应用优惠折扣(如有)
7. 返回最终保费
```

**计算性能优化:**
- 相同参数的试算结果缓存5分钟
- 缓存Key: `quote:{productId}:{age}:{gender}:{amount}:{period}`
- 费率表预加载到Redis,避免每次查询MySQL
- Groovy脚本预编译缓存,避免重复解析

#### 4.1.3 利益演示逻辑

**现金价值计算:**
```
现金价值(第N年) = {
  如果是储蓄型产品:
    累计已缴保费 + 累计投资收益 - 管理费用 - 风险保费
  如果是消费型产品:
    通常为0或极低
}
```

**年金领取演示:**
```
年金型产品利益演示 = {
  生存总利益: 累计领取金额
  身故保险金: 未领完的现金价值
  IRR计算: 内部收益率
  演示假设: 低档2.5%、中档4.5%、高档6.0%
}
```

**IRR计算算法:**
```
IRR(内部收益率) = 使得NPV=0的贴现率

NPV(净现值) = Σ(CFt / (1+r)^t)
  CFt = 第t年的现金流(负数为保费支出,正数为领取金额)
  r = 贴现率
  t = 年份

使用牛顿迭代法求解IRR:
1. 初始猜测r0 = 0.05
2. 计算NPV(r0)
3. 计算NPV的导数
4. 更新r1 = r0 - NPV(r0) / NPV'(r0)
5. 重复步骤2-4直到收敛(误差<0.0001)
```

#### 4.1.4 智能推荐逻辑

**保额推荐规则:**
```
if (客户年收入 < 10万) {
  推荐重疾保额 = 年收入 × 3-5倍
  推荐寿险保额 = 年收入 × 5-10倍
}
else if (客户年收入 10-30万) {
  推荐重疾保额 = 50万
  推荐寿险保额 = 100-150万
}
else {
  推荐重疾保额 = 100万
  推荐寿险保额 = 300-500万
}

if (有房贷) {
  寿险保额 += 房贷余额
}

if (有车贷) {
  寿险保额 += 车贷余额
}

if (有子女 && 子女年龄 < 18) {
  寿险保额 += 子女抚养费(至18岁)
}
```

**缴费期限推荐:**
```
if (客户年龄 < 35 && 收入稳定) {
  推荐缴费期: 20年或30年
  理由: 拉长缴费期,降低年缴压力,保费豁免更有价值
}
else if (客户年龄 35-50 && 收入较高) {
  推荐缴费期: 10年或15年
  理由: 快速缴费完成,尽早锁定保障
}
else {
  推荐缴费期: 趸交或5年
  理由: 临近退休,尽快完成缴费
}
```

### 7.1 业务逻辑说明

#### 7.1.1 订单状态流转

**状态流转图:**
```
待支付(1) → 核保中(2) → 已承保(3)
              ↓
           已取消(4)
```

**流转规则:**
- 待支付: 客户下单后初始状态

**状态变更触发:**
- 支付回调 → 待支付变为核保中
- 核保通过 → 核保中变为已承保

**催单场景:**

### 7.2 功能实现要点

#### 7.2.1 订单编号生成规则

**编号规则:** `ORD + yyyyMMdd + 8位流水号`

**示例:** ORD202502150001234

**生成逻辑:**

#### 7.2.2 佣金计算

**计算时机:** 订单状态变为"已承保"时

**计算公式:**
```
FYC(首年佣金) = 首年保费 × 佣金比例
RYC(续年佣金) = 续年保费 × 佣金比例(通常低于FYC)

示例:
某重疾险,年缴保费10000元,缴费20年
FYC佣金比例: 60%
RYC佣金比例: 5%

首年佣金 = 10000 × 0.6 = 6000元
续年佣金 = 10000 × 0.05 = 500元/年 × 19年 = 9500元
总佣金 = 6000 + 9500 = 15500元
```

**佣金记录:**
- 承保时生成一条FYC佣金记录
- 每年自动生成RYC佣金记录(定时任务)

#### 7.2.3 订单导出

**导出字段:**

## 九、模块七: 佣金管理

### 9.1 业务逻辑说明

#### 9.1.1 佣金计算规则

**FYC(首年佣金):**
```
计算时机: 订单承保后
计算公式: 首年保费 × 产品佣金比例
结算时间: 承保后次月15日结算
```

**RYC(续年佣金):**
```
计算时机: 每年保单续期后
计算公式: 续期保费 × 续期佣金比例
结算时间: 续期后次月15日结算
```

**佣金调整:**
- 退保: 扣除已发放的FYC佣金
- 减额: 按比例扣减佣金
- 复效: 补发佣金

#### 9.1.2 提现流程

**提现条件:**
- 已结算佣金 >= 100元
- 银行卡已绑定
- 实名认证通过

**提现流程:**
```
1. 业务员发起提现申请
2. 系统校验可提现金额
3. 锁定佣金(settlement_status = 2)
4. 生成提现记录(status = 1 审核中)
5. 财务审核
6. 审核通过后调用银行打款接口
7. 打款成功更新状态(status = 3 已打款)
8. 打款失败解锁佣金,提现失败
```

**提现限制:**
- 单次最低提现: 100元
- 单次最高提现: 50000元
- 每月最多提现: 3次
- 提现手续费: 0(公司承担)

#### 9.1.3 佣金对账

**对账周期:** 每月1-5日对账上月佣金

**对账内容:**
- 核对订单数量
- 核对保费总额
- 核对佣金金额
- 核对已提现金额
- 核对余额

**差异处理:**
- 发现差异记录对账差异表
- 人工核实差异原因
- 调整佣金数据

### 9.2 功能实现要点

#### 9.2.1 银行卡信息加密

**加密字段:**
- 银行卡号
- 持卡人姓名
- 预留手机号

**加密算法:** AES-256

**加密逻辑:**
```
加密: encryptedBankAccount = AES.encrypt(bankAccount, secretKey)
解密: bankAccount = AES.decrypt(encryptedBankAccount, secretKey)
```

**密钥管理:**
- 密钥存储在配置中心
- 定期轮换密钥
- 密钥分级管理(开发/测试/生产)

#### 9.2.2 提现审核工作流

**审核角色:**
- 一级审核: 财务专员
- 二级审核: 财务经理(金额>10000元)

**审核规则:**
- 小额提现(<10000): 一级审核即可
- 大额提现(>=10000): 需二级审核
- 异常提现(频繁、大额): 风控审核

#### 9.2.3 银行打款接口对接

**打款方式:**
- 银行卡: 调用银行代付接口
- 支付宝: 调用支付宝转账接口
- 微信: 调用微信企业付款接口

**打款流程:**
```
1. 构建打款请求参数
2. 调用银行接口
3. 获取打款流水号
4. 轮询打款状态
5. 更新提现记录状态
6. 记录打款凭证
```
