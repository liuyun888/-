## 一、商城首页模块(续)

### 2.2 分类导航

**开发工时**: 前端0.5天 + 后端0.5天

#### 2.2.1 功能概述

在首页展示保险产品的分类导航,帮助用户快速找到所需险种,提升用户浏览效率。

#### 2.2.2 核心业务逻辑

**分类结构设计**:
1. **一级分类**(主分类):
   - 车险: 交强险、商业车险
   - 健康险: 医疗险、重疾险、防癌险
   - 寿险: 定期寿险、终身寿险、增额终身寿
   - 意外险: 综合意外险、旅游险、运动险
   - 财产险: 家财险、雇主责任险
   - 其他: 宠物险、高端医疗等
   
2. **二级分类**(子分类):
   - 车险 → 交强险、商业车险(车损、三者、座位险等)
   - 健康险 → 百万医疗、小额医疗、重疾险单次赔付、重疾险多次赔付
   - 每个一级分类下设3-8个二级分类

3. **分类图标**:
   - 每个一级分类配置图标
   - 图标尺寸: 80*80px
   - 支持自定义上传或选择系统图标库

**前端展示样式**:
1. **宫格布局**:
   - 首页显示8-10个一级分类
   - 每行4个,共2-3行
   - 图标+文字排版
   
2. **点击交互**:
   - 点击一级分类进入对应产品列表页
   - 传递参数: categoryId或categoryCode
   - 筛选该分类下的所有产品

3. **更多分类**:
   - 如分类超过10个,显示"更多"入口
   - 点击进入全部分类页面
   - 支持搜索分类功能

**动态配置**:
- 后台可配置分类的名称、图标、排序、状态
- 关联的产品范围
- 是否在首页显示
- 是否热门分类(打上角标)

### 2.3 热销推荐

**开发工时**: 前端1天 + 后端1天

#### 2.3.1 功能概述

在首页展示热销保险产品,通过销量、人气、评价等维度筛选优质产品,引导用户购买。

#### 2.3.2 核心业务逻辑

**热销产品筛选规则**:
1. **基础条件**:
   - 产品状态: 已上架
   - 库存状态: 有库存或不限库存
   - 销售时间: 在有效期内
   
2. **排序权重计算**:
   ```
   热度分 = (销量权重 * 0.4) + (点击权重 * 0.2) + (收藏权重 * 0.15) 
            + (评分权重 * 0.15) + (人工权重 * 0.1)
   
   销量权重 = 近30天销量 / 该分类最高销量 * 100
   点击权重 = 近7天点击 / 该分类最高点击 * 100
   收藏权重 = 总收藏数 / 该分类最高收藏 * 100
   评分权重 = 产品评分 / 5 * 100
   人工权重 = 后台配置的推荐权重(0-100)
   ```

3. **展示数量**: 首页显示6-8个热销产品,每日凌晨2点更新热销榜单

4. **产品信息展示**:
   - 产品图片(主图)、产品名称、保险公司LOGO
   - 价格: 起投价/年、销量: "已售XX份"
   - 标签: 热销、新品、限时优惠等、评分: 5星评价

**人工干预机制**:
1. 后台可置顶特定产品到热销榜
2. 可屏蔽某些产品不出现在热销榜
3. 新品冷启动: 新上架产品给予额外权重

**榜单更新策略**:
1. **定时更新**: 每日凌晨2点全量计算,计算近30天数据,更新Redis缓存
2. **准实时更新**: 每小时增量更新销量,避免排序频繁变动
3. **缓存策略**: 热销榜单缓存6小时,产品详情缓存1小时

#### 2.3.5 技术实现要点

1. **热度分计算任务**: 使用XXL-Job定时任务框架,每日凌晨2点执行,分批处理,避免数据库压力

2. **缓存设计**: key: `home:hot:products`,存储结构: List<ProductVO>,TTL: 6小时,缓存穿透: 空结果也缓存

3. **性能优化**: 热销榜单预生成不实时计算,产品详情关联查询优化,图片使用CDN支持WebP格式

4. **数据统计**: 
   - 销量统计: 从订单表聚合
   - 点击统计: 从埋点日志聚合
   - 收藏统计: 从收藏表count
   - 评分统计: 从评价表avg

5. **AB测试支持**: 支持配置多套推荐策略,不同用户群体展示不同榜单,统计各策略的转化效果

#### 2.3.6 注意事项

- 新产品需要冷启动策略,避免永远上不了榜
- 价格展示需明确标注"起"或"最低",避免误导
- 销量造假检测: 异常增长预警
- 评分过滤: 剔除刷分评价
- 下架产品及时从榜单移除

### 2.4 爆款专区

**开发工时**: 前端0.5天 + 后端0.5天

#### 2.4.1 功能概述

展示限时特惠、秒杀、拼团等营销活动产品,营造紧迫感,刺激用户快速下单。

#### 2.4.2 核心业务逻辑

**爆款专区类型**:
1. **限时秒杀**: 每日固定时段(如10点、14点、20点),特价商品限量供应,倒计时显示剩余时间
2. **拼团优惠**: 2人团、3人团等,成团享优惠价,显示已成团数量
3. **新人专享**: 仅新注册用户可购买,超低价引流产品,限购1份
4. **今日特惠**: 每日推荐1-2款特价产品,限时24小时,库存有限

**活动配置**:
1. 活动基本信息: 活动名称、活动类型(秒杀/拼团/特惠)、活动时间(开始-结束)、活动状态(未开始/进行中/已结束)
2. 产品配置: 关联产品、活动价格、库存数量(可选)、限购数量
3. 规则配置: 参与条件(新人/全员)、拼团人数要求、成团有效期

**前端展示**:
1. **秒杀专区**: 当前场次+下一场次预告,倒计时组件,进度条显示已抢百分比
2. **拼团专区**: 拼团列表,显示差X人成团,一键参团功能
3. **特惠专区**: 横向滑动卡片,显示原价、活动价、折扣,库存预警(库存<10时显示"仅剩X份")

#### 2.4.5 技术实现要点

1. **秒杀库存扣减**: 使用Redis原子操作decrby,key: `seckill:stock:{activityId}:{productId}`,扣减成功再创建订单,15分钟未支付库存回滚

2. **拼团成团检测**: 每次加入拼团后检查是否满足成团条件,成团后异步通知所有成员,过期未成团自动标记失败,定时任务检查过期拼团

3. **倒计时实现**: 服务端返回结束时间戳,前端计算剩余秒数,每秒更新显示,结束后自动刷新页面

3. **索引设计**: 常用排序字段建立索引,复合索引优化
4. **缓存策略**: 热门排序结果缓存5分钟

1. **Elasticsearch集成**: 产品数据同步到ES,使用ik_max_word分词器,multi_match查询多个字段
2. **搜索评分**: 使用function_score组合相关性分数和业务分数
3. **搜索建议**: 使用ES的completion suggester,拼音搜索使用pinyin分词器
4. **热门搜索统计**: 实时搜索记录到消息队列,定时任务聚合统计,每日更新热门榜单

### 3.4 产品卡片展示

**开发工时**: 前端1天 + 后端0.5天

#### 3.4.1 功能概述

以卡片形式展示产品核心信息,用户可快速浏览产品特点,点击进入详情页。

#### 3.4.2 核心业务逻辑

**卡片信息元素**:
1. 基础信息: 产品主图、产品名称、保险公司LOGO和名称
2. 价格信息: 起售价、价格区间、原价和活动价
3. 保障亮点: 最高3个核心亮点
4. 标签: 左上角标签、底部标签,最多显示3个
5. 社会化信息: 销量、评分、收藏按钮

**卡片样式**:
1. 列表模式: 图文混排,左图右文或上图下文
2. 宫格模式: 2列宫格布局,适合移动端

**交互功能**:
1. 点击卡片跳转产品详情页
2. 点击收藏按钮收藏/取消收藏
3. 长按卡片显示快捷菜单

#### 3.4.3 技术实现要点

1. **图片懒加载**: 使用Intersection Observer API
2. **标签优先级**: 活动标签 > 热销标签 > 新品标签
3. **收藏状态**: 登录用户查询收藏表获取收藏状态
4. **价格显示策略**: 有活动价显示原价和活动价,无活动价显示起售价
5. **骨架屏**: 首次加载显示骨架屏

### 3.5 下拉加载更多

**开发工时**: 前端0.5天 + 后端0.5天

#### 3.5.1 功能概述

支持下拉刷新和上拉加载更多功能,用户可无限滚动浏览产品列表。

#### 3.5.2 核心业务逻辑

**下拉刷新**: 用户在列表顶部下拉,显示"下拉刷新"提示,释放后触发刷新,重新加载第一页数据

**上拉加载**: 用户滚动到列表底部,自动触发加载下一页,显示"加载中...",加载完成追加到列表末尾,无更多数据显示"没有更多了"

**加载状态**: loading(加载中)、success(加载成功)、noMore(没有更多数据)、error(加载失败)

#### 3.5.3 技术实现要点

1. **分页参数**: page从1开始,pageSize建议20条
2. **滚动监听**: 监听scroll事件,判断距离底部<100px时触发加载,防抖处理
3. **状态管理**: 维护list、page、pageSize、total、loading、finished、error
4. **数据追加**: 上拉加载追加数据,下拉刷新替换数据,去重处理
5. **缓存策略**: 已加载数据缓存在内存,刷新或筛选条件变化清空缓存
