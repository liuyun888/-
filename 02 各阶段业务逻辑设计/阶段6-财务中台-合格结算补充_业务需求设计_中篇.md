## FN-03 合格-车险导出模板配置

### 3.1 功能入口

菜单路径（两个入口均可操作）：
- 入口1（保单查询页）：**车险 → 保单管理 → 保单查询 → 【导出】→ 【自定义表头】**
- 入口2（系统设置页，仅管理员）：**财务 → 财务配置 → 导出模板管理 → 车险模板**

### 3.2 模板列表页

#### 3.2.1 页面元素

- 顶部筛选：模板名称（文本模糊搜索）、保险公司（下拉）、状态（启用/禁用）、创建时间范围
- 列表列：模板ID、模板名称、适用保险公司、创建人、创建时间、最后修改时间、状态、操作（编辑/复制/删除/预览/下载样本）
- 右上角按钮：【新增模板】

#### 3.2.2 权限说明

- **配置权限**（新增/编辑/删除模板）：角色 `ADMIN` 或 `FINANCE_MANAGER`
- **使用权限**（在保单查询页选择模板导出）：角色 `FINANCE`、`INNER_STAFF`、`ADMIN`

### 3.3 新增/编辑模板弹窗

#### 3.3.1 弹窗字段

| 字段 | 类型 | 是否必填 | 校验规则 |
|------|------|---------|---------|
| 模板名称 | 文本输入 | ✅ | 最长50字，同类型（车险）下不允许重名 |
| 授权组织 | 树形选择器 | 选填 | 勾选组织后仅该组织成员可在导出时看到此模板；不勾选则全员可见 |
| 适用保险公司 | 下拉多选 | 选填 | 不选则适用所有保司 |
| 模板备注 | 文本域 | 选填 | 最长200字 |
| 字段配置区 | 双栏穿梭框 | ✅（至少选1个字段） | 见3.3.2 |

#### 3.3.2 字段配置区（穿梭框）

穿梭框分为**左侧"可选字段"**和**右侧"已选字段"**，右侧支持拖拽排序。

**可选字段按分组展示（勾选后移入右侧）：**

> 以下为合格品类车险导出模板的推荐标准字段集，对应操作手册250号配置：

**基本信息组：**
- 保险公司、保险类型、业务类型、保（批）单号、涉农业务、互联网业务、业务员名称、渠道名称、签单日期、起保日期、结束日期

**车辆信息组：**
- 发动机号、车牌号、车架号

**被保人信息组：**
- 被保人姓名

**投保人信息组：**
- 投保人姓名

**保单费用组：**
- 车船税、总保费、总净保费

**上游手续费组：**
- 上游保单手续费（%）、上游保单手续费（元）

**下游手续费组（可选）：**
- 下游保单手续费（%）、下游保单手续费（元）

**结算状态组：**
- 上游对账状态、上游结算状态、对账状态（下游）、下游结算状态

**险种组：**
- 交强险、商业险（含子险种：车损险、三者险、座位险、盗抢险、玻璃险、自燃险等），支持全选

> 字段分组数据由后端接口 `/finance/export-template/car/fields` 返回，字段定义存储在 `ins_export_field_config` 配置表中，便于后续扩展。

#### 3.3.3 字段排序

右侧已选字段支持通过拖拽（Sortable.js）调整列顺序，顺序即为Excel导出时的列顺序。

也提供"上移"、"下移"按钮供辅助操作。

#### 3.3.4 保存逻辑

点击【确定】后：

1. 后端校验：模板名称唯一性（同一品类下）、字段列表至少1项。
2. 写库：

```sql
-- 模板主表
INSERT INTO ins_export_template (
    template_code, template_name, template_type, 
    insurer_codes, org_ids, remark,
    status, creator, create_time
) VALUES (
    UUID(), #{name}, 'CAR_INSURANCE',
    #{insurerCodes}, #{orgIds}, #{remark},
    1, #{operator}, NOW()
);

-- 字段明细表（按顺序）
INSERT INTO ins_export_template_field (
    template_id, field_code, field_name, field_order, is_required
) VALUES (...);
```

3. 返回成功，刷新列表。

### 3.4 预览功能

点击列表中某模板的【预览】按钮：
- 弹出模态框，以表格形式展示该模板的**列头预览**（列名按顺序横向排列，每行展示最多一行示例数据）。
- 提供【下载样本】按钮，生成仅含表头（无数据）的xlsx文件，文件名：`模板名称_样本_${yyyyMMdd}.xlsx`。

### 3.5 使用模板导出保单

#### 3.5.1 入口

在**车险 → 保单管理 → 保单查询**页面：
1. 通过筛选条件查询到目标保单列表
2. 点击【导出】下拉，选择【自定义表头】
3. 弹出对话框

#### 3.5.2 导出对话框

| 字段 | 类型 | 说明 |
|------|------|------|
| 选择模板 | 下拉单选 | 拉取当前用户有权限的模板列表；默认显示上次使用的模板 |
| 导出范围 | Radio | "当前查询结果（全部）" / "仅勾选记录" |
| 文件名 | 文本输入 | 默认值：`车险保单_批次号_yyyyMMdd`，可修改 |

点击【确认导出】后：

#### 3.5.3 后端导出逻辑

1. 获取模板配置（字段列表及顺序）。
2. 按查询条件或勾选保单号列表查询 `insurance_policy` 及关联表，组装数据。
3. 使用 **EasyExcel 动态表头**生成Excel：

```java
// 动态表头生成示例
List<List<String>> headList = templateFields.stream()
    .map(f -> Collections.singletonList(f.getFieldName()))
    .collect(Collectors.toList());

EasyExcel.write(outputStream)
    .head(headList)
    .sheet("车险保单")
    .doWrite(dataList);
```

4. 文件名含：模板名称 + 批次号 + 导出日期。
5. **写导出历史记录** `ins_export_history`：记录操作人、模板ID、导出时间、导出条数、文件名，便于查询历史导出记录。

#### 3.5.4 历史导出记录查询

在【导出】下拉菜单中增加【历史导出记录】入口，展示：
- 导出时间、操作人、模板名称、导出条数、状态（生成中/已完成/失败）
- 提供【重新下载】按钮（文件保留N天，超期后按钮置灰）

## FN-04 合格-非车险导出模板配置

### 4.1 功能入口

- 入口1：**非车 → 系统设置 → 模板设置 → 【新增模板】**
- 入口2：**财务 → 财务配置 → 导出模板管理 → 非车模板**

### 4.2 与车险导出模板的功能差异

非车险导出模板与车险模板功能框架完全一致（同一套代码，通过 `template_type` 字段区分），主要差异在于：

| 差异点 | 车险模板 | 非车险模板 |
|--------|---------|-----------|
| 字段分组 | 含车辆信息（车牌号、车架号、发动机号） | 无车辆信息；有被保标的相关字段 |
| 险种分组 | 按交强/商业子险种 | 按险种大类区分配置 |
| 适用的险种大类 | — | 财产险、工程险、责任险、农险等（可多选） |
| 入口菜单 | 车险-保单查询 | 非车-保单查询 |

### 4.3 非车险模板字段集

> 对应操作手册251号，以下为合格品类非车险标准字段集：

**基本信息组：**
- 保险公司、标的类型、标的名称、保（批）单号、险种编码、险种、互联网业务、涉农业务、保单状态、签单日期、起保日期、保险止期、渠道名称、业务员

**被保人信息组：**
- 被保人姓名

**投保人信息组：**
- 投保人名称

**保单费用组：**
- 全保费、净保费

**上游手续费组：**
- 上游保单手续费比例（%）、上游保单手续费金额（元）

**下游手续费组（选填）：**
- 下游保单手续费比例（%）、下游保单手续费金额（元）

**结算状态组：**
- 上游结算状态、下游结算状态

**险种大类专有字段（按险种大类动态显示）：**

| 险种大类 | 专有字段 |
|---------|---------|
| 财产险 | 财产类别、保障金额、财产地址 |
| 工程险 | 工程名称、工程地址、承包方式 |
| 责任险 | 责任类型、赔偿限额 |
| 农险 | 农作物种类、种植/养殖面积 |
| 货运险 | 货物名称、运输方式、起运地、目的地 |

> 这些专有字段在穿梭框中属于"扩展字段"分组，模板管理员可根据需要选配。

### 4.4 权限分离（非车险导出模板特殊要求）

非车险模板权限与车险相同，但需特别强调：

| 操作 | 有权限的角色 |
|------|------------|
| 新增/编辑/删除模板配置 | `ADMIN`、`FINANCE_MANAGER` |
| 在保单查询页使用模板导出 | `FINANCE`、`INNER_STAFF`、`ADMIN` |
| 查看历史导出记录 | `FINANCE`、`INNER_STAFF`、`ADMIN`（仅自己的记录）；`FINANCE_MANAGER` 可查所有人记录 |

### 4.5 新增非车险模板弹窗特有字段

在车险模板弹窗基础上，增加：

| 字段 | 类型 | 说明 |
|------|------|------|
| 适用险种大类 | 多选复选框 | 财产险/工程险/责任险/农险/货运险/其他；不选则适用所有非车险种 |

当用户在"适用险种大类"中选择了具体险种时，穿梭框右侧的已选字段会提示："当前模板适用 XX 险，建议添加以下专属字段：[字段名1、字段名2]"（智能推荐，可忽略）。

## 数据库设计（FN-03 & FN-04）

### 导出模板主表

```sql
CREATE TABLE `ins_export_template` (
  `id`              BIGINT       NOT NULL AUTO_INCREMENT,
  `template_code`   VARCHAR(64)  NOT NULL COMMENT '模板编码（UUID）',
  `template_name`   VARCHAR(100) NOT NULL COMMENT '模板名称',
  `template_type`   VARCHAR(32)  NOT NULL COMMENT '模板类型（CAR_INSURANCE/NON_CAR_INSURANCE）',
  `insurer_codes`   VARCHAR(500) DEFAULT NULL COMMENT '适用保司编码，逗号分隔；NULL=全部',
  `org_ids`         VARCHAR(500) DEFAULT NULL COMMENT '授权组织ID，逗号分隔；NULL=全部',
  `insurance_categories` VARCHAR(200) DEFAULT NULL COMMENT '适用险种大类（非车专用），逗号分隔',
  `remark`          VARCHAR(500) DEFAULT NULL,
  `status`          TINYINT(1)   NOT NULL DEFAULT 1 COMMENT '状态（1启用 0禁用）',
  `version`         INT          NOT NULL DEFAULT 1 COMMENT '版本号',
  `creator`         BIGINT       NOT NULL,
  `create_time`     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updater`         BIGINT       DEFAULT NULL,
  `update_time`     DATETIME     DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP,
  `deleted`         TINYINT(1)   NOT NULL DEFAULT 0,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_type_name` (`template_type`, `template_name`, `deleted`),
  KEY `idx_template_type` (`template_type`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='导出模板主表';
```

### 导出模板字段表

```sql
CREATE TABLE `ins_export_template_field` (
  `id`            BIGINT       NOT NULL AUTO_INCREMENT,
  `template_id`   BIGINT       NOT NULL COMMENT '关联模板ID',
  `field_code`    VARCHAR(64)  NOT NULL COMMENT '字段编码（与导出数据映射用）',
  `field_name`    VARCHAR(100) NOT NULL COMMENT '列头名称（显示在Excel）',
  `field_order`   INT          NOT NULL DEFAULT 0 COMMENT '列顺序（从1开始）',
  `is_required`   TINYINT(1)   NOT NULL DEFAULT 0 COMMENT '是否必填字段（不可从模板移除）',
  `format_rule`   VARCHAR(200) DEFAULT NULL COMMENT '格式化规则（日期格式/数字精度等）',
  `create_time`   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `deleted`       TINYINT(1)   NOT NULL DEFAULT 0,
  PRIMARY KEY (`id`),
  KEY `idx_template_id` (`template_id`),
  KEY `uk_template_field` (`template_id`, `field_code`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='导出模板字段配置';
```

### 导出历史记录表

```sql
CREATE TABLE `ins_export_history` (
  `id`            BIGINT       NOT NULL AUTO_INCREMENT,
  `template_id`   BIGINT       NOT NULL COMMENT '使用的模板ID',
  `template_name` VARCHAR(100) NOT NULL COMMENT '模板名称快照',
  `template_type` VARCHAR(32)  NOT NULL,
  `file_name`     VARCHAR(200) NOT NULL COMMENT '导出文件名',
  `file_path`     VARCHAR(500) DEFAULT NULL COMMENT '文件存储路径（OSS路径）',
  `export_count`  INT          NOT NULL DEFAULT 0 COMMENT '导出条数',
  `query_params`  TEXT         DEFAULT NULL COMMENT '导出时的查询条件快照（JSON）',
  `status`        VARCHAR(20)  NOT NULL DEFAULT 'PROCESSING' COMMENT '状态（PROCESSING/SUCCESS/FAILED）',
  `error_msg`     VARCHAR(500) DEFAULT NULL COMMENT '失败原因',
  `expire_time`   DATETIME     DEFAULT NULL COMMENT '文件过期时间',
  `operator`      BIGINT       NOT NULL COMMENT '操作人',
  `create_time`   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time`   DATETIME     DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP,
  `deleted`       TINYINT(1)   NOT NULL DEFAULT 0,
  PRIMARY KEY (`id`),
  KEY `idx_operator_time` (`operator`, `create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='保单导出历史记录';
```

## 技术实现要点

### EasyExcel 动态表头生成

```java
/**
 * 动态表头导出核心代码思路
 */
public void export(HttpServletResponse response, Long templateId, List<Long> policyIds) {
    // 1. 获取模板字段（按 field_order 排序）
    List<ExportTemplateField> fields = templateFieldMapper.selectByTemplateId(templateId);
    
    // 2. 构建动态表头（List<List<String>>，每个List<String>对应一列）
    List<List<String>> headList = fields.stream()
        .map(f -> Collections.singletonList(f.getFieldName()))
        .collect(Collectors.toList());
    
    // 3. 查询数据（按字段映射反射取值）
    List<InsurancePolicy> policies = policyMapper.selectByIds(policyIds);
    List<List<Object>> dataList = buildDataRows(policies, fields);
    
    // 4. 写出
    String fileName = URLEncoder.encode(buildFileName(templateId), "UTF-8");
    response.setHeader("Content-Disposition", "attachment;filename=" + fileName + ".xlsx");
    EasyExcel.write(response.getOutputStream())
        .head(headList)
        .sheet("保单数据")
        .doWrite(dataList);
    
    // 5. 异步记录导出历史
    exportHistoryService.saveAsync(templateId, fileName, policyIds.size(), operatorId);
}
```

### 模板版本管理

每次编辑模板时，`version` 字段 +1。历史导出记录中保存 `template_name`（快照），即使后续模板被修改也不影响历史记录显示。

若模板被删除（软删除），已有导出历史记录保留，只是下载链接可能过期。
