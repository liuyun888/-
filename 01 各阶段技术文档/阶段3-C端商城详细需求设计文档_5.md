## 五、车险投保模块

车险投保是C端商城的核心业务,支持用户在线完成车险报价、投保、核保和支付全流程。

### 5.1 车辆信息录入

**开发工时**: 前端2天 + 后端2天

#### 5.1.1 功能概述

提供车辆信息录入功能,支持车牌号自动识别、VIN码解析、车辆品牌型号选择,为报价投保奠定基础。

#### 5.1.2 核心业务逻辑

**录入方式**:

1. **车牌号录入**:
   - 支持7位和8位车牌(新能源车)
   - 格式校验:省份简称+字母+5位字母数字
   - 自动识别车牌类型:蓝牌、黄牌、绿牌
   - 关联查询历史投保记录

2. **VIN码识别**:
   - 用户拍照上传行驶证
   - OCR识别VIN码(17位字符)
   - 调用车辆信息库API解析
   - 自动填充:品牌、车系、车型、发动机号、车架号

3. **品牌型号选择**:
   - 三级联动:品牌→车系→车型→年款
   - 支持搜索品牌和车型
   - 常用品牌置顶显示

**必填信息**:
- 车牌号、车辆类型(家用/营运)、初登日期
- 发动机号、车架号(VIN)、品牌型号、座位数
- 车主姓名、车主身份证号、车主手机号

**数据校验规则**:
- VIN码校验:17位字符,不含I、O、Q
- 初登日期:不能晚于当前,不能早于车型上市年份
- 车牌号与车辆类型匹配
- 车主信息实名认证

#### 5.1.3 数据库设计

**用户车辆信息表 (tb_user_vehicle)**:

返回:vinCode, licensePlate, ownerName, registerDate
```

**根据VIN解析车辆信息**
```
GET /api/vehicle/parseVin?vinCode={vinCode}
返回:brand, series, model, modelYear
```

**保存车辆信息**
```
POST /api/vehicle/save
参数:车辆所有信息
```

#### 5.1.5 技术实现要点

1. OCR识别:腾讯云OCR行驶证识别API,识别率≥95%
2. VIN码解析:对接中保研车型库API
3. 车型库同步:每月从中保研同步最新车型
4. 身份证校验:调用公安部实名认证接口
5. 数据缓存:车型库Redis缓存24小时

#### 5.1.6 注意事项

- 车牌号和VIN码需脱敏存储
- 车主身份证号加密存储
- 支持新能源车8位车牌
- 过户车需特别标注
- 营运车辆保费远高于家用车

### 5.2 险种选择

**开发工时**: 前端2天 + 后端2天

#### 5.2.1 功能概述

提供车险险种选择,包括交强险、商业险主险、附加险,支持智能推荐方案和保额自定义。

#### 5.2.2 核心业务逻辑

**险种分类**:

1. **交强险**(必选):
   - 国家强制购买
   - 保费固定(6座以下家用车950元/年)
   - 根据出险次数浮动(-30%~+30%)

2. **商业险主险**:
   - 车损险:赔付己方车辆损失
   - 三者险:保额可选20万/50万/100万/150万/200万/300万
   - 车上人员险:1-10万/座
   - 全车盗抢险:新车或高价值车推荐

3. **商业险附加险**:
   - 不计免赔险:强烈推荐
   - 涉水险:南方多雨地区
   - 玻璃单独破碎险:豪华车
   - 自燃险:老旧车(5年以上)
   - 划痕险:新车
   - 无法找到第三方特约险

**智能推荐方案**:

1. **基础保障型**(2000-3000元/年):
   - 交强险+三者险50万+车损险
   - 适用:老旧车、驾龄长

2. **经济实用型**(3500-5000元/年,推荐):
   - 交强险+三者险100万+车损险+不计免赔+车上人员险
   - 适用:家用代步车、新手司机

3. **全面保障型**(6000-10000元/年):
   - 交强险+三者险200万+车损险+所有常用附加险
   - 适用:新车、豪华车

**保额计算逻辑**:
- 车损险保额=车辆实际价值=购置价×(1-使用年限×10%)
- 三者险保额:一线城市建议100万以上

#### 5.2.3 数据库设计

**车险险种配置表 (tb_car_insurance_type)**:

| description | TEXT | - | NULL | 险种说明 |

| is_required | TINYINT | - | DEFAULT 0 | 是否必选 |
| recommend_level | TINYINT | - | DEFAULT 0 | 推荐级别:0-5 |
| create_time | DATETIME | - | NOT NULL | 创建时间 |

**索引**:
- PRIMARY KEY: id
- UNIQUE INDEX: type_code

#### 5.2.4 接口设计

**获取险种列表**

返回:所有可选险种
```

**获取推荐方案**
```
GET /api/car-insurance/recommendPlan?vehicleId={vehicleId}
返回:3个推荐方案
```

**保存用户方案**
```

参数:vehicleId, planType, insuranceItems
```

#### 5.2.5 技术实现要点

1. 险种配置管理:后台灵活配置险种
2. 智能推荐算法:基于车辆属性、用户画像
3. 保费预估:实时计算保费变化
4. 方案对比:最多3个方案并排对比

#### 5.2.6 注意事项

- 交强险必选,前端置灰
- 附加险依赖主险
- 三者险保额建议≥50万
- 车损险保额不能超车辆实际价值
- 提供险种详情弹窗

### 5.3 在线报价

**开发工时**: 前端3天 + 后端5天

#### 5.3.1 功能概述

对接多家保险公司API,实现并发报价,提供多家保险公司的车险报价对比。

#### 5.3.2 核心业务逻辑

**报价流程**:

1. **并发请求**:
   - 同时向3-5家保险公司发起报价请求
   - 使用线程池异步调用
   - 超时时间:单家10秒,总计15秒

2. **报价参数**:
   - 车辆信息:VIN、车牌、品牌型号
   - 险种方案:选择的险种列表和保额
   - 投保人信息
   - 历史出险记录

3. **报价计算**:
   ```
   总保费 = Σ(各险种基础保费) × NCD系数 × 出险系数 
            × 车型系数 × 地区系数 × 渠道系数 + 交强险
   ```

4. **报价结果展示**:
   - 保险公司名称、LOGO
   - 总保费(突出显示)
   - 保费明细拆分
   - 优惠信息
   - 服务承诺

**报价有效期**:
- 报价结果有效期24小时
- Redis缓存:key=`quote:result:{vehicleId}:{planId}`

### 5.5 核保流程

**开发工时**: 前端1天 + 后端3天

#### 5.5.1 功能概述

将投保订单提交至保险公司核保,核保通过后生成保单号和电子保单。

#### 5.5.2 核心业务逻辑

**核保提交流程**:

1. **订单校验**:
   - 检查订单状态(待提交核保)
   - 校验必填信息完整性
   - 校验报价是否过期(24小时)

2. **核保请求**:
   - 调用保险公司核保接口
   - 提交投保信息+车辆信息+险种方案
   - 同步或异步返回结果

3. **核保结果**:
   - **通过**:保存保单号,订单状态→待支付
   - **拒绝**:记录拒保原因,订单状态→核保拒绝
   - **处理中**:轮询查询结果(最多10次,间隔5秒)

**核保异常处理**:

1. **网络超时**:重试3次,间隔2秒
2. **业务拒绝**常见原因:
   - 车辆信息不符
   - 车辆改装或非标
   - 上年出险次数过多(≥5次)
   - 车辆用途与申报不符

**核保数据留存**:
- 核保请求报文
- 核保响应报文
- 核保时间和耗时

### 6.3 健康告知

**开发工时**: 前端3天 + 后端3天

#### 6.3.1 功能概述

健康险、寿险投保前必须完成健康告知,如实告知被保人健康状况,保险公司根据告知结果决定是否承保。

#### 6.3.2 核心业务逻辑

**健康告知原则**:
- **如实告知义务**:法律要求必须如实回答
- **隐瞒或虚假告知**:理赔时可能拒赔
- **有限告知**:只需回答问卷中的问题

**告知问卷结构**:

1. **基础健康问题**:
   - 过去2年是否住院/手术
   - 是否患有高血压、糖尿病等常见病
   - 是否有家族遗传病史

2. **疾病类问题**(按险种):
   - 重疾险:心脑血管、癌症相关
   - 医疗险:既往病史、用药情况
   - 寿险:危重疾病、残疾情况

3. **生活习惯问题**:
   - 是否吸烟、饮酒
   - 是否从事高危运动

4. **体检异常问题**:
   - 近期体检异常指标
   - 肝功能、肾功能、血糖等

**智能问卷**:
- **动态问题**:回答"是"触发追问
- **问题解释**:每个问题提供详细说明
- 不确定时引导咨询客服

**告知结果处理**:

1. **全部回答"否"**:
   - 标准体承保
   - 正常费率
   - 直接进入下一步

2. **部分回答"是"**:
   - 触发人工核保
   - 可能结果:
     - 标准体承保
     - 加费承保(保费上浮10%-50%)
     - 除外承保(特定疾病不赔)
     - 延期承保
     - 拒保

3. **智能核保**:
   - 常见疾病自动核保
   - 如:乙肝小三阳→肝功能正常→标准体
   - 快速给出核保结论

**健康告知材料**:
部分情况需上传:
- 体检报告、病历记录
- 出院小结、诊断证明
- 用药清单

**告知书签署**:
- 用户阅读健康告知说明
- 勾选"已如实告知"
- 电子签名确认

#### 9.1.1 功能概述

#### 9.1.2 核心业务逻辑

**理赔申请前提**:

- 在理赔时效内(事故发生后2年内)

**出险信息录入**:

1. **基础信息**:选择理赔保单,出险人姓名、身份证,与被保人关系

3. **就诊信息**(医疗险):就诊医院名称、等级、科室、诊断结果、治疗方案、住院天数
4. **损失情况**:医疗费用总额、已报销金额、申请理赔金额、收款账户

**理赔材料清单**:

**医疗险理赔材料**:

- ✅ 身份证、银行卡

**理赔金额计算**:

意外险赔付:
- 意外医疗:实报实销
- 意外伤残:按伤残等级赔付(10%-100%)
- 意外身故:赔付保额100%

**预理赔评估**:
- 系统根据上传材料自动评估
- 预估赔付金额范围
- 提示缺失材料

**提交理赔**:
- 完成信息填写和材料上传
- 系统校验完整性
- 生成理赔案件号
- 推送至保险公司系统

### 9.2 理赔进度

**开发工时**: 前端2天 + 后端2天

#### 9.2.1 功能概述

实时展示理赔审核进度,包括受理、审核、核赔、支付各环节状态,支持消息推送。

#### 9.2.2 核心业务逻辑

**理赔流程节点**:

1. **已提交**(提交当天):用户提交理赔,等待保险公司受理
2. **已受理**(1-2工作日):保险公司受理案件,分配理赔专员,材料审核中
3. **审核中**(3-5工作日):理赔员审核材料,核查真实性,计算理赔金额
   - 可能状态:审核通过→核赔,需补充材料,审核拒绝→拒赔
4. **需补充材料**:材料不全或不清晰,列明缺失材料,用户补充后重新审核(2-3工作日)
5. **核赔审批**(1-3工作日):
   - 小额(≤5000):自动审批
   - 中额(5000-5万):主管审批
   - 大额(>5万):总监审批
6. **待支付**(1-2工作日):审批通过,财务准备付款
7. **理赔完成**:理赔款已转账,用户收到款项,案件结案
8. **拒赔**:不符合理赔条件,说明拒赔原因,支持申诉

**进度展示方式**:
- 时间轴展示:纵向时间轴
- 已完成节点:绿色✅
- 当前节点:蓝色进行中
- 未完成节点:灰色⭕

**进度通知**:
- 短信通知:已受理、需补充材料、审核通过、理赔到账
- 站内消息:每个节点变更
- 微信通知:模板消息推送

**理赔时效**:
- 小额案件(≤5000):7个工作日
- 普通案件(5000-5万):15个工作日
- 复杂案件(>5万):30个工作日

### 10.5 积分商城

**开发工时**: 前端3天 + 后端3天

#### 10.5.1 功能概述

用户通过投保、理赔、推荐等行为获得积分,积分可兑换保险产品、优惠券、实物商品、增值服务,提升用户活跃度和粘性。

#### 10.5.2 核心业务逻辑

**积分获取**:
1. **投保获取**:完成投保=保费×0.1%积分,如5000元保费=500积分,首次投保额外奖励1000积分
2. **推荐获取**:推荐好友注册500积分,好友完成投保1000积分,最多10人/月
3. **任务获取**:签到每日10积分,完善资料500积分,绑定微信200积分,实名认证300积分
4. **活动获取**:节日活动翻倍,抽奖获得积分,问卷调查积分

**积分规则**:
- 1元保费=0.1积分
- 积分有效期:2年
- 过期自动清零
- 不可转让

**兑换商品**:

1. **保险产品**:
   - 短期意外险:500积分兑换
   - 健康险优惠券:1000积分=100元优惠券
   - 车险续保券:2000积分=200元续保优惠

2. **实物商品**:
   - 生活用品:500-2000积分
   - 数码产品:5000-20000积分
   - 京东卡/话费充值:1000积分起

3. **增值服务**:
   - 健康咨询次数:500积分=1次
   - 法律咨询次数:800积分=1次
   - 体检套餐折扣:1500积分=8折券

**兑换流程**:
1. 浏览商品:按分类浏览或搜索
2. 选择商品:查看详情、兑换价格
3. 确认兑换:填写收货地址(实物商品)
4. 扣减积分:积分扣减,生成兑换订单
5. 配送/发放:
   - 实物商品:发货配送
   - 优惠券:自动发放到账户
   - 增值服务:次数增加

**积分明细**:
- 积分收入:投保、任务、活动
- 积分支出:兑换商品
- 积分过期:到期清零记录
- 支持筛选和导出

**积分抽奖**:
- 定期举办积分抽奖活动
- 消耗100积分抽奖1次
- 奖品:积分、优惠券、实物
