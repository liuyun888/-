## 一、支付模块

支付模块是保险购买流程的关键环节,需要对接多种支付渠道,确保支付安全、高效、稳定。

### 7.1 支付方式选择

**开发工时**: 前端1天 + 后端1.5天

#### 7.1.1 功能概述

提供多种支付方式供用户选择,包括第三方支付、银行卡支付、分期支付等,满足不同用户的支付需求。

#### 7.1.2 核心业务逻辑

**支付方式类型**:

1. **第三方支付**:
   - 微信支付:微信APP支付、H5支付、小程序支付
   - 支付宝支付:支付宝APP支付、H5支付
   - 云闪付:银联快捷支付
   - 支持扫码支付、跳转支付

2. **银行卡支付**:
   - 储蓄卡快捷支付
   - 信用卡快捷支付
   - 网银支付
   - 支持主流银行(工商、建设、农业、中国等)

3. **分期支付**:
   - 花呗分期:3期、6期、12期
   - 信用卡分期:3期、6期、12期、24期
   - 显示每期金额和手续费

4. **余额支付**(企业用户):
   - 账户余额抵扣
   - 余额不足组合支付
   - 充值入口

5. **线下支付**(大额企业):
   - 银行转账
   - 对公转账
   - 上传付款凭证

**支付方式展示**:
1. 根据订单金额推荐支付方式
2. 显示支付方式图标和名称
3. 标注优惠信息(如:微信支付立减5元)
4. 禁用不可用的支付方式

**支付限额**:
1. 单笔限额:不同支付方式不同限额
   - 微信支付:单笔5万
   - 支付宝:单笔20万
   - 银行卡:根据银行设置
2. 日累计限额:防风控
3. 月累计限额:部分支付方式

**支付优惠**:
1. 支付方式优惠(微信立减、支付宝红包)
2. 银行卡优惠(指定银行优惠)
3. 分期免息活动
4. 优惠叠加规则

#### 7.1.3 数据库设计

**支付方式配置表 (tb_payment_method)**:

|--------|------|------|------|------|

| method_code | VARCHAR | 50 | UNIQUE, NOT NULL | 支付方式代码 |
| method_name | VARCHAR | 50 | NOT NULL | 支付方式名称 |
| method_type | TINYINT | - | NOT NULL | 类型:1第三方 2银行卡 3分期 4余额 5线下 |
| payment_channel | VARCHAR | 50 | NOT NULL | 支付渠道:WECHAT/ALIPAY/UNIONPAY |
| icon_url | VARCHAR | 255 | NULL | 图标URL |

| max_amount | BIGINT | - | NULL | 最大金额(分) |

| monthly_limit | BIGINT | - | NULL | 月限额(分) |

| id | BIGINT | - | PK, AUTO_INCREMENT | 主键 |

**索引**:
- PRIMARY KEY: id
- INDEX: (user_id, deleted)
- INDEX: (user_id, is_default)

#### 7.1.4 接口设计

**获取可用支付方式接口**

```http
GET /api/payment/methods?orderNo={orderNo}
Authorization: Bearer {token}
```

**请求参数**:
- orderNo: 订单号

**响应内容**:
- 可用支付方式列表
- 每种方式的限额、手续费、优惠信息
- 推荐支付方式
- 用户绑定的支付方式

**选择支付方式接口**

```http
POST /api/payment/selectMethod
Authorization: Bearer {token}
Content-Type: application/json
```

**请求参数**:
- orderNo: 订单号
- methodCode: 支付方式代码
- installmentPeriod: 分期期数(分期支付时)

**响应内容**:
- 支付金额明细
- 手续费
- 实付金额

#### 7.1.5 技术实现要点

1. **支付方式过滤**:
   - 根据订单金额过滤
   - 根据用户类型过滤(个人/企业)
   - 根据支付场景过滤(APP/H5/PC)
   - 根据支付状态过滤(维护中/正常)

2. **支付限额校验**:
   - 实时查询用户支付额度
   - 单笔限额检查
   - 日累计限额检查
   - 月累计限额检查

3. **分期计算**:
   - 分期总额 = 订单金额
   - 每期金额 = 总额 / 期数
   - 手续费 = 总额 × 费率
   - 首期可能不同(含手续费)

4. **支付优惠计算**:
   - 读取支付方式优惠配置
   - 计算优惠金额
   - 叠加订单优惠
   - 更新实付金额

5. **支付方式记忆**:
   - 记录用户最近使用的支付方式
   - 下次支付自动选中
   - 提升支付效率

#### 7.1.6 注意事项

- 支付方式配置需灵活,支持动态上下架
- 分期支付需明确告知手续费
- 支付限额需实时同步支付平台规则
- 支付优惠需防止羊毛党
- 银行卡信息需脱敏展示
- 支付方式故障需及时切换备用方案

### 7.2 支付密码

**开发工时**: 前端0.5天 + 后端1天

#### 7.2.1 功能概述

为保护用户资金安全,支付时需要输入支付密码或进行生物识别认证,防止未授权支付。

#### 7.2.2 核心业务逻辑

**支付密码设置**:

1. **首次设置**:
   - 用户注册后引导设置
   - 6位数字密码
   - 两次输入确认
   - 不能与登录密码相同
   - 不能使用简单密码(123456、000000等)

2. **修改密码**:
   - 需验证旧密码
   - 或通过短信验证码重置
   - 设置新密码
   - 密码修改后通知用户

3. **忘记密码**:
   - 短信验证码验证
   - 重新设置密码
   - 记录重置日志

**支付验证方式**:

1. **支付密码验证**:
   - 输入6位支付密码
   - 支持数字键盘
   - 密码掩码显示
   - 错误3次锁定30分钟

2. **生物识别**(辅助验证):
   - 指纹识别
   - 面容识别(Face ID)
   - 仅作为辅助,重要支付仍需密码

3. **短信验证码**(高额支付):
   - 单笔超过5000元
   - 发送短信验证码
   - 密码+验证码双重验证

**支付免密**:
1. 小额免密:单笔200元以下
2. 用户可自行设置免密额度
3. 免密支付仍需设备认证
4. 可随时关闭免密

**密码安全**:
1. 密码加密存储(BCrypt)
2. 传输加密(HTTPS + RSA)
3. 密码错误次数限制
4. 异常支付风控拦截

#### 7.2.3 数据库设计

**支付密码表 (tb_payment_password)**:

| 字段名 | 类型 | 长度 | 约束 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | - | PK, AUTO_INCREMENT | 主键 |
| user_id | BIGINT | - | UNIQUE, NOT NULL | 用户ID |
| password_hash | VARCHAR | 200 | NOT NULL | 密码哈希 |
| password_salt | VARCHAR | 50 | NOT NULL | 密码盐值 |
| is_set | TINYINT | - | DEFAULT 0 | 是否已设置 |
| password_level | TINYINT | - | DEFAULT 1 | 密码强度:1弱 2中 3强 |
| error_count | INT | - | DEFAULT 0 | 错误次数 |
| lock_until | DATETIME | - | NULL | 锁定截止时间 |
| last_modify_time | DATETIME | - | NULL | 最后修改时间 |
| create_time | DATETIME | - | NOT NULL | 创建时间 |
| update_time | DATETIME | - | NULL | 更新时间 |

**索引**:
- PRIMARY KEY: id
- UNIQUE INDEX: user_id

**支付验证记录表 (tb_payment_verification)**:

| 字段名 | 类型 | 长度 | 约束 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | - | PK, AUTO_INCREMENT | 主键 |
| user_id | BIGINT | - | NOT NULL | 用户ID |
| order_no | VARCHAR | 32 | NOT NULL | 订单号 |
| verify_type | TINYINT | - | NOT NULL | 验证类型:1密码 2指纹 3面容 4短信 |
| verify_result | TINYINT | - | NOT NULL | 结果:1成功 2失败 |
| fail_reason | VARCHAR | 100 | NULL | 失败原因 |
| client_ip | VARCHAR | 50 | NULL | 客户端IP |
| client_device | VARCHAR | 200 | NULL | 客户端设备 |
| create_time | DATETIME | - | NOT NULL | 创建时间 |

**索引**:
- PRIMARY KEY: id
- INDEX: (user_id, create_time)
- INDEX: (order_no)

#### 7.2.4 接口设计

**设置支付密码接口**

```http
POST /api/payment/password/set
Authorization: Bearer {token}
Content-Type: application/json
```

**请求参数**:
- password: 支付密码(前端RSA加密)
- confirmPassword: 确认密码
- smsCode: 短信验证码

**修改支付密码接口**

```http
POST /api/payment/password/modify
Authorization: Bearer {token}
Content-Type: application/json
```

**请求参数**:
- oldPassword: 旧密码(前端RSA加密)
- newPassword: 新密码
- smsCode: 短信验证码

**忘记密码接口**

```http
POST /api/payment/password/reset
Authorization: Bearer {token}
Content-Type: application/json
```

**请求参数**:
- mobile: 手机号
- smsCode: 短信验证码
- newPassword: 新密码

**支付密码验证接口**

```http
POST /api/payment/password/verify
Authorization: Bearer {token}
Content-Type: application/json
```

**请求参数**:
- orderNo: 订单号
- password: 支付密码(前端RSA加密)
- verifyType: 验证类型(1密码 2生物识别)

**响应内容**:
- verified: 是否验证通过
- remainAttempts: 剩余尝试次数(失败时)
- lockUntil: 锁定截止时间(锁定时)

#### 7.2.5 技术实现要点

1. **密码加密**:
   - 前端使用RSA公钥加密传输
   - 后端解密后使用BCrypt加密存储
   - 加盐哈希,每个用户盐值不同
   - 密码不可逆加密

2. **密码强度校验**:
   - 不能全部相同(111111、666666)
   - 不能连续递增/递减(123456、654321)
   - 不能与手机号、生日相同
   - 建议使用复杂密码

3. **错误次数限制**:
   - Redis记录错误次数
   - key: `pay:pwd:error:{userId}`
   - 错误3次锁定30分钟
   - 锁定期间禁止尝试

4. **生物识别集成**:
   - iOS: Touch ID / Face ID
   - Android: Fingerprint API
   - 仅本地验证,不上传生物特征
   - 验证成功后调用密码验证接口

5. **免密支付**:
   - 用户设置免密额度
   - 存储在用户配置表
   - 支付时校验金额
   - 超过额度强制密码验证

6. **风控拦截**:
   - 异常设备支付拦截
   - 异地支付拦截
   - 高频支付拦截
   - 大额支付人工审核

#### 7.2.6 注意事项

- 支付密码安全等级高于登录密码
- 密码必须加密传输和存储
- 错误次数限制防暴力破解
- 生物识别仅作为辅助验证
- 免密支付需用户主动开启
- 密码修改需短信验证
- 支付验证日志完整记录

### 7.3 支付结果

**开发工时**: 前端1天 + 后端1天

#### 7.3.1 功能概述

展示支付结果,包括支付成功、支付失败、支付处理中等状态,提供后续操作引导。

#### 7.3.2 核心业务逻辑

**支付状态**:

1. **支付成功**:
   - 展示成功图标和文案
   - 显示订单号、支付金额
   - 显示支付方式和时间
   - 提示保单生效时间
   - 查看保单按钮
   - 返回首页按钮

2. **支付失败**:
   - 展示失败图标和文案
   - 显示失败原因
   - 重新支付按钮
   - 更换支付方式按钮
   - 联系客服按钮
   - 取消订单按钮

3. **支付处理中**:
   - 展示处理中动画
   - 提示正在确认支付结果
   - 轮询查询支付状态
   - 超时处理(5分钟)

4. **支付超时**:
   - 提示支付超时
   - 查询支付结果按钮
   - 重新支付按钮
   - 订单详情按钮

**支付结果查询**:
1. 支付成功后主动查询保单状态
2. 轮询查询(最多5次,间隔3秒)
3. 保单未生成提示等待
4. 支持手动刷新

**支付失败原因**:
1. 余额不足
2. 银行卡限额
3. 密码错误
4. 支付超时
5. 系统异常
6. 风控拦截

**支付通知**:
1. 短信通知支付结果
2. 微信/APP推送通知
3. 邮件发送电子保单
4. 站内消息通知

#### 7.3.3 数据库设计

复用前面的支付记录表(tb_payment_record)和订单表(tb_insurance_order、tb_car_insurance_order)。

#### 7.3.4 接口设计

**查询支付结果接口**

```http
GET /api/payment/result/{paymentNo}
Authorization: Bearer {token}
```

**请求参数**:
- paymentNo: 支付单号

**响应内容**:
- payStatus: 支付状态(1待支付 2成功 3失败 4处理中)
- payAmount: 支付金额
- payMethod: 支付方式
- payTime: 支付时间
- policyNo: 保单号(支付成功时)
- failReason: 失败原因(支付失败时)
- orderInfo: 订单信息

**查询保单状态接口**

```http
GET /api/policy/status/{orderNo}
Authorization: Bearer {token}
```

**响应内容**:
- policyNo: 保单号
- policyStatus: 保单状态(1生效中 2未生效 3已失效)
- effectiveDate: 生效日期
- expireDate: 到期日期

#### 7.3.5 技术实现要点

1. **支付结果轮询**:
   - 前端轮询查询支付结果
   - 间隔3秒,最多5次
   - 轮询期间显示处理中动画
   - 超时后提示手动查询

2. **支付回调处理**:
   - 接收支付平台异步通知
   - 验证签名和金额
   - 更新订单和支付状态
   - 通知保险公司出单
   - 发送用户通知

3. **幂等性保证**:
   - 支付回调可能重复
   - 使用分布式锁
   - 检查订单状态
   - 已处理的回调直接返回成功

4. **保单查询**:
   - 支付成功后查询保单
   - 保单未生成轮询等待
   - 最多等待5分钟
   - 超时提示联系客服

5. **消息通知**:
   - 支付成功发送短信
   - 保单生效发送电子保单
   - 使用消息队列异步发送
   - 发送失败重试机制

#### 7.3.6 注意事项

- 支付结果以后端为准,前端可能不准确
- 支付回调需验签,防止伪造
- 保单生成需时间,不是即时的
- 支付失败需明确告知原因
- 支付超时不代表支付失败
- 重复支付需防止
- 支付成功后不可取消订单

### 7.4 退款处理

**开发工时**: 前端1天 + 后端1.5天

#### 7.4.1 功能概述

处理用户退保、撤单等场景的退款需求,支持全额退款和部分退款,自动或人工审核后原路退回。

#### 7.4.2 核心业务逻辑

**退款场景**:

1. **犹豫期退保**:
   - 保单生效后15天内
   - 无理由全额退款
   - 自动审核通过
   - 扣除工本费10元

2. **正常退保**:
   - 保单生效超过犹豫期
   - 退还现金价值
   - 需人工审核
   - 扣除手续费和已承保天数保费

3. **订单取消**:
   - 未支付或支付后未生效
   - 全额退款
   - 自动审核

4. **理赔后退保**:
   - 已发生理赔
   - 部分产品不可退保
   - 或扣除已赔付金额

**退款流程**:

1. **申请退款**:
   - 用户发起退款申请
   - 填写退款原因
   - 上传退保申请书(部分需要)
   - 提交审核

2. **退款审核**:
   - 自动审核:犹豫期退保、订单取消
   - 人工审核:正常退保、争议退款
   - 审核通过/拒绝
   - 通知用户审核结果

3. **退款计算**:
   - 犹豫期:实付金额 - 工本费
   - 正常退保:现金价值 - 手续费
   - 订单取消:实付金额

4. **退款执行**:
   - 调用支付平台退款接口
   - 原路退回支付账户
   - 更新订单退款状态
   - 发送退款通知

5. **退款到账**:
   - 微信/支付宝:1-3个工作日
   - 银行卡:3-7个工作日
   - 支持查询退款进度

**退款规则**:

1. **退款时效**:
   - 犹豫期退保:15天内
   - 正常退保:随时可申请
   - 订单取消:未生效前

2. **退款限制**:
   - 已理赔不可退(部分产品)
   - 退保后不可恢复
   - 每个保单只能退一次

3. **退款金额**:
   - 犹豫期:实付金额 - 工本费(10元)
   - 正常退保:现金价值
   - 现金价值 = 保费 × (1 - 已承保天数/保障期天数) - 手续费

   - 定时任务查询退款状态
