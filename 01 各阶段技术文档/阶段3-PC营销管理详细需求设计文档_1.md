## 二、内容管理模块(续)

### 2.4 视频管理(完整版)

#### 2.4.3 业务逻辑

**视频上传流程**:
1. 前端选择视频文件进行校验:
   - 支持格式:MP4、AVI、FLV、MOV、WMV
   - 文件大小限制:单个文件不超过2GB
   - 时长限制:建议不超过60分钟
2. 获取阿里云VOD上传凭证:
   - 调用阿里云SDK获取uploadAuth和uploadAddress
   - 凭证有效期3000秒
   - 包含上传地址和签名信息
3. 前端直传到阿里云VOD:
   - 使用阿里云上传SDK
   - 支持断点续传
   - 实时显示上传进度
   - 上传成功返回VideoId
4. 保存视频基础信息:
   - 标题、分类、描述必填
   - 封面图:可手动上传或自动截取视频第5秒画面
   - 保存video_id到数据库
   - transcode_status设置为0(待转码)
5. 异步转码处理:
   - 提交转码任务到阿里云VOD
   - 转码模板:标清(480P)、高清(720P)、超清(1080P)
   - 转码完成后阿里云回调通知
   - 更新transcode_status为2,保存play_url
6. 转码回调处理:
   - 接收阿里云回调通知
   - 验证回调签名
   - 更新视频转码状态和播放地址
   - 提取视频时长、分辨率、文件大小等元数据
   - 发送通知给上传者

**视频管理操作**:
1. 视频编辑:
   - 可修改标题、描述、分类、封面图
   - 不可修改video_id和播放地址
   - 修改后记录操作日志
2. 封面图管理:
   - 支持手动上传封面图
   - 支持从视频中截取任意时间点的画面作为封面
   - 封面图建议尺寸:16:9比例,最小800x450px
   - 自动生成缩略图(400x225px)用于列表展示
3. 视频加密:
   - 使用阿里云私有加密
   - 防盗链设置:Referer白名单、IP黑白名单、URL鉴权
   - 播放器获取加密播放凭证才能播放
   - 凭证有效期30分钟,过期需重新获取
4. 视频预览:
   - 后台管理员可直接预览
   - 使用阿里云播放器SDK
   - 支持倍速播放、画质切换
   - 显示播放进度、时长等信息

**视频列表查询**:
1. 筛选条件:
   - 分类:单选或多选
   - 转码状态:待转码、转码中、转码成功、转码失败
   - 上架状态:已上架、已下架
   - 关键词:搜索标题、描述
   - 上传时间范围
2. 排序方式:
   - 按上传时间倒序(默认)
   - 按播放量倒序
   - 按点赞数倒序
   - 按时长正序
3. 列表展示字段:
   - 封面缩略图(点击可预览)
   - 标题、时长、文件大小
   - 分类、上传者、上传时间
   - 转码状态、上架状态
   - 播放量、点赞数
   - 操作按钮:编辑、删除、上下架、预览

**转码失败处理**:
1. 系统定时检测转码失败的视频
2. 自动重试机制:
   - 转码失败后自动重试3次
   - 重试间隔:5分钟、15分钟、30分钟
   - 3次重试后仍失败,发送告警通知
3. 失败原因分析:
   - 视频文件损坏
   - 格式不支持
   - 文件过大
   - 网络超时
4. 管理员可手动重新提交转码任务

**视频删除**:
1. 删除前校验:
   - 检查是否有活动或文章引用该视频
   - 已上架视频需先下架
2. 删除操作:
   - 逻辑删除:设置deleted=1
   - 同步删除阿里云VOD中的视频文件
   - 保留数据库记录用于数据分析
3. 物理删除:
   - 管理员权限可执行物理删除
   - 彻底删除数据库记录和云端文件
   - 不可恢复,需二次确认

**播放统计**:
1. 播放量统计:
   - C端用户点击播放时调用统计接口
   - Redis实时计数,每小时同步到MySQL
   - 防刷机制:同一用户对同一视频10分钟内只计数1次
   - 记录播放时长、完播率等数据
2. 点赞统计:
   - 用户点赞后实时+1
   - 取消点赞实时-1
   - 记录点赞用户ID防重复
3. 播放明细记录:
   - 记录到cms_video_play_log表
   - 包含用户ID、视频ID、播放时长、完播率、播放时间
   - 用于数据分析和用户行为追踪

**分类管理**:
1. 视频分类采用一级分类,不支持多级
2. 常见分类:
   - 产品介绍
   - 理赔指南
   - 保险课堂
   - 案例分享
   - 活动宣传
3. 分类操作:
   - 新增分类:填写名称、图标、排序
   - 编辑分类:修改名称、图标、排序
   - 删除分类:检查是否有视频关联,有则不允许删除
   - 启用/禁用:禁用后该分类下的视频不展示

#### 3.1.3 业务逻辑

**活动创建流程**:
1. 基础信息设置:
   - 活动名称:2-100字符,必填
   - 活动类型:单选必填
   - 活动时间:开始时间必须早于结束时间,不能晚于当前时间7天前
   - 封面图:必传,建议尺寸750x400px
   - 轮播图:可选,最多5张
   - 活动描述:富文本编辑,支持图文混排
2. 活动规则配置(根据活动类型不同):
   - **新人礼**:
     - 新用户定义:注册时间<30天且未下单
     - 奖励内容:优惠券包(可选多张)+积分
     - 领取条件:完成实名认证
   - **满减活动**:
     - 满减阶梯:可设置多档,如满100减10,满200减25
     - 是否可叠加使用优惠券
     - 是否包邮
   - **折扣活动**:
     - 折扣力度:1-99折
     - 折扣上限:最多优惠金额
   - **赠品活动**:
     - 赠品列表:选择赠品及数量
     - 触发条件:购买指定产品或满指定金额
   - **拼团活动**:
     - 成团人数:2-10人
     - 拼团价格
     - 开团有效期:24-72小时
     - 模拟成团:开启后系统自动补足人数
   - **秒杀活动**:
     - 秒杀价格
     - 秒杀库存
     - 每人限购数量
     - 秒杀时段:可设置多个时段
3. 目标用户设置:
   - 全部用户:不限制
   - 新用户:注册时间<30天
   - 老用户:注册时间>=30天且有过订单
   - 指定用户:上传用户ID列表或选择标签分组
4. 产品范围设置:
   - 全部产品:活动规则适用所有产品
   - 指定分类:选择产品分类,可多选
   - 指定产品:搜索添加产品,可批量导入
   - 排除产品:设置不参与活动的产品
5. 参与限制设置:
   - 不限:用户可无限次参与
   - 限制次数:每人可参与N次
   - 限制频率:每天/每周/每月限N次
6. 其他配置:
   - 关联优惠券:活动期间自动发放优惠券
   - 赠送积分:完成活动后赠送积分
   - 分享奖励:分享活动给好友可获得额外奖励
7. 保存草稿或提交审核

**活动审核流程**:
1. 活动创建者提交审核:
   - status从0(草稿)变为1(待审核)
   - audit_status设置为0(待审核)
   - 系统发送待办通知给审核员
2. 审核员进入审核页面:
   - 查看活动详细信息
   - 预览活动页面效果
   - 检查规则配置是否合理
   - 验证产品库存是否充足
   - 评估活动成本和预期收益
3. 审核决策:
   - **通过**:
     - audit_status设置为1
     - status设置为2(已通过)
     - 发送通知给创建者
     - 活动进入待生效状态
   - **驳回**:
     - audit_status设置为2
     - status回到0(草稿)
     - 填写驳回原因(必填)
     - 发送通知给创建者,告知驳回原因
4. 审核记录:
   - 保存审核人、审核时间、审核意见
   - 支持审核历史查询
5. 重新提交:
   - 驳回后可修改并重新提交
   - 重新生成审核流程

**活动生效与结束**:
1. 自动生效:
   - 定时任务每分钟扫描
   - 当前时间>=start_time且status=2,自动设置status=3(进行中)
   - 发送活动开始通知
2. 自动结束:
   - 当前时间>=end_time,自动设置status=4(已结束)
   - 停止新用户参与
   - 已参与但未使用的奖励设置过期时间
3. 手动下架:
   - 管理员可手动下架进行中的活动
   - status设置为5(已下架)
   - 需填写下架原因
   - 发送通知给已参与用户
4. 活动延期:
   - 活动进行中可申请延长结束时间
   - 需重新审核
   - 审核通过后更新end_time

**活动列表查询**:
1. 筛选条件:
   - 活动类型:多选
   - 活动状态:草稿、待审核、进行中、已结束、已下架
   - 创建时间范围
   - 活动时间范围
   - 创建人
   - 关键词:搜索活动名称、活动编号
2. 排序:
   - 按创建时间倒序(默认)
   - 按开始时间倒序
   - 按参与人数倒序
   - 按订单金额倒序
3. 列表展示:
   - 封面图缩略图
   - 活动名称、活动编号
   - 活动类型、活动时间
   - 目标用户、产品范围
   - 状态标识(不同颜色)
   - 浏览量、参与人数、订单数、订单金额
   - 操作按钮:查看、编辑、删除、审核、上下架、数据统计

**活动编辑**:
1. 草稿状态:可任意修改
2. 待审核状态:可撤回后修改
3. 已通过状态:
   - 未开始:可修改活动时间、规则配置
   - 已开始:仅可修改结束时间、排序
   - 不可修改核心规则(如折扣力度、满减金额等)
4. 进行中状态:
   - 仅可修改描述、排序
   - 不可修改规则和时间
5. 已结束/已下架:不可修改

**活动删除**:
1. 删除限制:
   - 草稿和已下架状态可删除
   - 进行中和已结束状态不可删除
   - 有参与记录的活动不可物理删除
2. 删除操作:
   - 逻辑删除:设置deleted=1
   - 级联处理:活动相关的参与记录保留
3. 删除前确认:
   - 二次确认弹窗
   - 显示活动基本信息和统计数据

**活动参与流程**:
1. C端用户进入活动页面:
   - 检查活动状态是否进行中
   - 检查用户是否符合目标用户条件
   - 检查用户参与次数是否超限
   - 显示活动详情和规则
2. 用户点击参与:
   - 验证用户登录状态
   - 验证活动资格
   - 验证库存(秒杀、拼团等)
   - 创建参与记录
3. 发放奖励:
   - 优惠券:自动发放到用户账户
   - 积分:异步发放,记录积分明细
   - 赠品:生成赠品订单
4. 参与后处理:
   - 更新活动统计数据(参与人数+1)
   - 发送参与成功通知
   - 记录用户行为日志

**活动数据统计**:
1. 实时统计指标:
   - 浏览量:PV和UV
   - 参与人数:去重用户数
   - 参与次数:总参与次数
   - 转化率:参与人数/浏览人数
   - 订单数:活动期间产生的订单数
   - 订单金额:活动期间订单总金额
   - ROI:订单金额/活动成本
2. 分时统计:
   - 按小时统计:适用于秒杀活动
   - 按天统计:适用于长期活动
   - 生成折线图展示趋势
3. 用户分析:
   - 新老用户占比
   - 用户地域分布
   - 用户来源渠道
   - 用户设备类型
4. 产品分析:
   - 参与活动的产品销量排行
   - 产品转化率
   - 产品贡献金额
5. 数据导出:
   - 支持导出Excel报表
   - 包含明细数据和汇总数据
   - 可选择时间范围和统计维度

#### 4.1.3 业务逻辑

**优惠券创建**:
1. 基础信息设置:
   - 优惠券名称:2-100字符
   - 优惠券编号:自动生成,格式:CPN+时间戳+随机数
   - 优惠券类型:单选必填
2. 优惠设置:
   - **满减券**:
     - 满减金额:如满100减20
     - 使用门槛:满X元可用
   - **折扣券**:
     - 折扣率:1-99折,支持一位小数
     - 最高优惠:如最高优惠50元
     - 使用门槛:可选
   - **兑换券**:
     - 兑换商品:选择可兑换的商品
     - 兑换数量:默认1件
   - **立减券**:
     - 立减金额:直接抵扣
     - 无门槛限制
3. 发行设置:
   - 发行总量:
     - 限量:输入具体数量
     - 不限量:选择-1
   - 领取方式:
     - 手动领取:用户主动在优惠券中心领取
     - 自动发放:满足条件自动发放(如新用户注册)
     - 活动发放:参与活动后发放
     - 兑换码:生成兑换码供用户兑换
   - 每人限领:
     - 不限:选择-1
     - 限制次数:输入具体次数
4. 适用范围设置:
   - 产品范围:
     - 全部产品:所有产品可用
     - 指定分类:选择产品分类,支持多选
     - 指定产品:搜索添加产品
   - 用户范围:
     - 全部用户
     - 新用户:注册时间<30天
     - 指定用户:上传用户ID列表
     - 标签用户:选择用户标签
5. 有效期设置:
   - 固定日期:
     - 开始时间和结束时间
     - 可使用时间范围(可选)
   - 相对日期:
     - 领取后N天内有效
     - 例如:领取后7天内可用
6. 高级设置:
   - 是否可叠加:允许与其他优惠券同时使用
   - 使用说明:详细的使用规则和注意事项
   - 展示位置:优惠券中心、活动页、产品详情页
7. 兑换码生成(如选择兑换码发放):
   - 生成数量:与发行总量一致或指定数量
   - 兑换码规则:8-16位字母数字组合
   - 批量生成并导出Excel

**优惠券审核**(可选):
1. 如设置审核流程:
   - 创建后提交审核
   - 审核员审核优惠力度是否合理
   - 审核通过后方可发放
2. 直接发放:
   - 无需审核,创建后直接生效
   - 适用于常规优惠券

**优惠券发放**:
1. **手动领取**:
   - C端展示优惠券列表
   - 用户点击领取按钮
   - 校验:
     - 是否已达领取上限(总量和个人限领)
     - 是否符合用户范围
     - 是否在领取有效期内
   - 创建用户优惠券记录
   - 生成唯一优惠券码
   - 发送领取成功通知
2. **自动发放**:
   - 触发条件:
     - 用户注册
     - 用户首单
     - 用户生日
     - 消费满额
   - 系统自动检测并发放
   - 无需用户操作
3. **活动发放**:
   - 用户参与活动后触发
   - 活动结束时批量发放
   - 与活动记录关联
4. **兑换码发放**:
   - 用户输入兑换码
   - 校验兑换码有效性
   - 校验是否已使用
   - 绑定用户并创建优惠券记录
   - 标记兑换码已使用

**优惠券使用**:
1. 用户下单时选择优惠券:
   - 展示可用优惠券列表
   - 筛选适用当前订单的优惠券
   - 按优惠力度排序
2. 优惠券校验:
   - 检查优惠券状态是否为未使用
   - 检查是否在有效期内
   - 检查订单金额是否满足使用门槛
   - 检查订单产品是否在适用范围
   - 检查用户使用次数是否超限
3. 优惠计算:
   - 满减券:订单金额-优惠金额
   - 折扣券:订单金额*折扣率,不超过最高优惠
   - 立减券:订单金额-立减金额
4. 优惠券锁定:
   - 提交订单时锁定优惠券
   - status设置为4(已锁定)
   - 15分钟未支付自动解锁
5. 优惠券核销:
   - 订单支付成功后核销
   - status设置为2(已使用)
   - 记录use_time和order_id
   - 优惠券use_count+1
6. 退款处理:
   - 订单退款后优惠券返还
   - status从2变回1(未使用)
   - 延长有效期(原有效期+7天)

**优惠券列表查询**:
1. 筛选条件:
   - 优惠券类型
   - 发放方式
   - 状态:未开始、进行中、已结束、已下架
   - 创建时间范围
   - 关键词:优惠券名称、编号
2. 排序:
   - 按创建时间倒序
   - 按领取数量倒序
   - 按使用数量倒序
3. 列表展示:
   - 优惠券名称、编号
   - 优惠券类型、优惠值
   - 发行总量、已领取、已使用
   - 领取率、使用率
   - 有效期
   - 状态
   - 操作:查看、编辑、删除、下架、统计

**优惠券编辑**:
1. 未开始状态:可任意修改
2. 进行中状态:
   - 可修改:结束时间、使用说明
   - 不可修改:优惠金额、使用门槛、适用范围
3. 已结束状态:不可修改
4. 特殊操作:
   - 追加发行量:增加total_count
   - 延长有效期:修改end_time或valid_days

**优惠券下架**:
1. 下架操作:
   - 设置status=3
   - 停止新用户领取
   - 已领取未使用的券不受影响
2. 强制失效:
   - 将所有未使用的券设置为已过期
   - 不可恢复,需二次确认

**优惠券统计**:
1. 基础统计:
   - 发行总量
   - 已领取数量、领取率
   - 已使用数量、使用率
   - 剩余数量
2. 趋势分析:
   - 按日统计领取量和使用量
   - 生成折线图
3. 用户分析:
   - 领取用户列表
   - 用户分布:新老用户、地域分布
   - 用户行为:领取后未使用、已使用
4. 订单分析:
   - 使用该券的订单列表
   - 订单金额统计
   - 平均优惠金额
   - 优惠券带来的GMV
5. 数据导出:
   - 导出领取明细
   - 导出使用明细
   - 导出兑换码列表

**优惠券提醒**:
1. 即将过期提醒:
   - 优惠券过期前3天发送提醒
   - 推送方式:站内信、短信、微信模板消息
2. 新券到账提醒:
   - 用户领取或获得优惠券后发送通知
   - 展示优惠券详情和使用方式

| create_time | DATETIME | 是 | 创建时间 |
| update_time | DATETIME | 是 | 更新时间 |

#### 5.1.3 业务逻辑

**积分规则配置**:
1. **获取规则**:
   - **注册奖励**:
     - 新用户注册赠送积分
     - 设置奖励积分数:如100积分
     - 仅限首次注册,每人限1次
     - 可设置完成实名认证后发放
   - **签到奖励**:
     - 每日签到获得积分
     - 基础积分:如每天10积分
     - 连续签到额外奖励:
       - 连续7天+20积分
       - 连续30天+100积分
     - 每天限1次
   - **消费奖励**:
     - 下单支付成功后赠送积分
     - 计算规则:订单金额*积分比例
     - 比例设置:如每消费1元=1积分
     - 订单完成后发放(不可退款)
     - 部分退款按比例扣减积分
   - **分享奖励**:
     - 分享产品或活动给好友
     - 好友注册/下单后获得积分
     - 每天限制次数:如每天最多5次
     - 防刷机制:同一设备不重复计算
   - **评价奖励**:
     - 订单完成后评价产品
     - 文字评价:10积分
     - 图片评价:20积分
     - 视频评价:50积分
     - 每个订单限1次
   - **完善资料**:
     - 首次完善个人信息:50积分
     - 绑定手机号:20积分
     - 实名认证:30积分
     - 每项限1次

2. **消费规则**:
   - **商品兑换**:
     - 使用积分兑换实物商品
     - 扣减对应积分
   - **优惠券兑换**:
     - 使用积分兑换优惠券
     - 不同券值对应不同积分
   - **现金抵扣**:
     - 下单时积分抵扣现金
     - 抵扣比例:如100积分=1元
     - 最高抵扣比例:订单金额的10%
   - **积分过期**:
     - 按规则设置的有效期自动过期
     - 即将过期积分提前提醒

3. **限制设置**:
   - 不限:该规则可无限次触发
   - 每天限制:每天最多触发N次
   - 每周限制:每周最多触发N次
   - 每月限制:每月最多触发N次
   - 总次数限制:累计最多触发N次

4. **条件配置**:
   - 用户条件:新用户、老用户、会员等级
   - 金额条件:订单金额满X元
   - 产品条件:购买指定产品或分类
   - 时间条件:特定时间段内有效

**积分发放流程**:
1. 触发事件:
   - 用户完成某个行为(注册、签到、下单等)
   - 系统检测到事件触发
2. 规则匹配:
   - 根据事件类型查询对应规则
   - 检查规则是否启用
   - 检查用户是否符合条件
3. 次数校验:
   - 查询用户今日/本周/本月/累计触发次数
   - 判断是否超过限制
4. 积分计算:
   - 根据规则配置计算积分值
   - 如消费类:订单金额*比例
5. 积分发放:
   - 创建积分记录
   - 更新用户积分余额:available_point+积分值
   - 更新累计积分:total_point+积分值
   - 设置有效期:valid_end_time
6. 发送通知:
   - 站内信通知用户积分到账
   - 显示积分来源和数量

**积分冻结与解冻**:
1. **冻结场景**:
   - 订单退款:冻结该订单赠送的积分
   - 风险控制:异常账号冻结积分
   - 活动规则:活动赠送的积分需订单完成后解冻
2. **冻结操作**:
   - 扣减available_point
   - 增加frozen_point
   - 创建冻结记录,状态=4
3. **解冻操作**:
   - 扣减frozen_point
   - 增加available_point
   - 更新记录状态为1(有效)
4. **自动解冻**:
   - 订单完成后自动解冻
   - 定时任务每小时检测
   - 超过冻结期自动解冻

**积分过期处理**:
1. 过期规则:
   - 按积分记录的valid_end_time判断
   - 先进先出原则:最早获得的积分先过期
2. 过期提醒:
   - 积分过期前7天推送提醒
   - 过期前1天再次提醒
   - 显示即将过期的积分数量
3. 过期执行:
   - 定时任务每天凌晨2点执行
   - 查询当天过期的积分记录
   - 扣减用户available_point
   - 增加用户expire_point
   - 更新记录状态为3(已过期)
   - 发送过期通知

**积分兑换流程**:
1. 用户选择兑换品:
   - 浏览积分商城
   - 查看兑换品详情
   - 确认所需积分和库存
2. 提交兑换:
   - 检查用户可用积分是否充足
   - 检查兑换品库存是否充足
   - 检查用户兑换次数是否超限
   - 实物商品需选择收货地址
3. 扣减积分:
   - 扣减用户available_point
   - 增加used_point
   - 创建消费记录,状态=2(已使用)
4. 创建兑换记录:
   - 保存兑换品信息
   - 记录消耗积分
   - 状态设为1(待发货)
5. 发货处理:
   - **实物商品**:
     - 运营人员填写快递信息
     - 更新状态为2(已发货)
     - 发送发货通知
   - **虚拟商品**(优惠券、话费):
     - 系统自动发放
     - 更新状态为3(已完成)
     - 发送领取通知
6. 确认收货:
   - 用户收货后点击确认
   - 或发货后7天自动确认
   - 更新状态为3(已完成)

**积分等级体系**:
1. 等级划分:
   - 普通会员:0-999积分
   - 银卡会员:1000-4999积分
   - 金卡会员:5000-19999积分
   - 钻石会员:20000积分以上
2. 等级权益:
   - 普通会员:基础积分获取
   - 银卡会员:积分获取1.2倍,专属优惠券
   - 金卡会员:积分获取1.5倍,优先客服,生日礼包
   - 钻石会员:积分获取2倍,专属客服,年度大礼包
3. 等级计算:
   - 基于累计积分(total_point)
   - 每日凌晨自动更新等级
   - 等级不降级(仅升级)
4. 等级展示:
   - 用户中心显示等级徽章
   - 列表展示等级标识
   - 享受对应等级权益

**积分统计分析**:
1. 发放统计:
   - 按规则统计发放数量
   - 按时间维度统计
   - 生成发放趋势图
2. 消费统计:
   - 按兑换品统计消费数量
   - 统计兑换率
   - 分析热门兑换品
3. 用户统计:
   - 积分排行榜
   - 用户等级分布
   - 活跃用户分析
4. 异常监控:
   - 单日积分异常增长告警
   - 批量兑换行为监控
   - 疑似刷积分行为标记

   - 不同地域的用户行为差异

1. **基础画像**:
   - 性别分布

   - 新用户:注册30天内
   - 活跃用户:近30天有访问
   - 沉睡用户:30-90天未访问
   - 流失用户:90天以上未访问
4. **RFM模型**:
   - R(Recency):最近一次消费时间
   - F(Frequency):消费频次
   - M(Monetary):消费金额
   - 用户价值分群:重要价值用户、重要发展用户、重要保持用户、重要挽留用户、一般价值用户、一般发展用户、一般保持用户、一般挽留用户

**转化漏斗分析**:
1. **购买转化漏斗**:
   - 访问首页 → 浏览产品 → 加入购物车 → 提交订单 → 完成支付
   - 统计每个环节的用户数和转化率
   - 识别流失最严重的环节
   - 分析流失原因
2. **注册转化漏斗**:
   - 访问注册页 → 填写信息 → 验证手机 → 完成注册
   - 优化注册流程
3. **活动转化漏斗**:
   - 浏览活动 → 参与活动 → 下单 → 支付
   - 评估活动吸引力和转化能力

**用户留存分析**:
1. **留存率计算**:
   - 次日留存:第2天回访用户/首日新增用户
   - 7日留存:第7天回访用户/首日新增用户
   - 30日留存:第30天回访用户/首日新增用户
2. **留存曲线**:
