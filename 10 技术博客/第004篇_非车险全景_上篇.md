## 🗺️ 非车险产品地图

在正式学习之前，先建立全局视图。非车险（Non-Motor Insurance）是除车险以外所有财产险和人身险产品的总称。

```
非车险
├── 人身险类
│   ├── 意外险（Accident Insurance）
│   │   ├── 个人意外险
│   │   └── 团体意外险
│   ├── 健康险（Health Insurance）
│   │   ├── 医疗险（百万医疗、住院险）
│   │   ├── 重疾险（Critical Illness）
│   │   └── 护理险（Long-term Care）
│   └── 寿险（Life Insurance）—— 第005篇专讲
│
└── 财产险类
    ├── 企业财产险（Commercial Property）
    │   ├── 财产一切险
    │   ├── 综合险
    │   └── 业务中断险（BI）
    ├── 责任险（Liability Insurance）
    │   ├── 公众责任险
    │   ├── 雇主责任险
    │   └── 产品责任险
    └── 特殊险种
        ├── 货运险
        ├── 工程险
        └── 农业险
```

> 💡 **为什么非车险是"利润高地"？**
>
> 车险经过多次费改，保险公司竞争激烈，中介佣金率普遍被压缩到 8%~15%。而非车险，尤其是企业险和责任险，佣金率可达 **20%~40%**，且产品复杂、客户粘性强、续保率高。对于中介机构而言，非车险才是真正的利润来源。

## 第一章：意外险（Accident Insurance）

### 1.1 意外险的本质定义

**意外险**承保被保险人因"意外伤害"导致的身故、残疾或医疗费用。

理解意外险，首先要搞清楚什么是**"意外"**。保险合同中的意外，必须同时满足以下 4 个要素：

| 要素 | 含义 | 反例（不构成意外） |
|------|------|----------------|
| 外来的 | 伤害来自外部原因 | 疾病引起的突然死亡（心脏病发作） |
| 突发的 | 事件是突然发生的 | 职业病（长期积累） |
| 非本意的 | 不是被保险人故意为之 | 自杀、自伤 |
| 非疾病的 | 非身体内在原因 | 内脏出血、脑出血 |

> 📌 **典型案例**：被保险人骑自行车摔跤，导致腿骨骨折 → 符合意外的4要素，意外险赔付。
>
> **反例**：被保险人在运动中突发脑梗死亡 → 虽然突然，但原因是疾病（内因），意外险**不赔**，但医疗险可能赔。

---

### 1.2 意外险产品结构：四大责任

一张完整的意外险保单，通常包含以下责任组合：

```
意外险保单结构
│
├── 主险：意外身故/残疾
│   ├── 意外身故：被保险人因意外死亡，赔付保额（如100万）
│   └── 意外残疾：按残疾等级（1-10级）赔付保额的百分比
│       ├── 一级残疾（最重）：赔付保额 × 100%
│       ├── 二级残疾：赔付保额 × 90%
│       ├── ...
│       └── 十级残疾（最轻）：赔付保额 × 10%
│
├── 附加险①：意外医疗
│   ├── 报销因意外就医产生的医疗费用
│   ├── 通常设有免赔额（如100元）
│   └── 报销比例：如100%（社保内）/ 80%（自费）
│
└── 附加险②：意外住院津贴
    ├── 因意外住院，每天赔付固定金额（如200元/天）
    └── 不需要凭票报销，直接按天数计算
```

#### 残疾等级与赔付比例（重点！）

2014年保监会发布的《人身保险伤残评定标准》将残疾分为1-10级：

```
一级残疾：双眼失明、双手截肢、双足截肢等 → 赔100%保额
二级残疾：一眼失明+一手截肢等              → 赔90%保额
三级残疾：一手腕以上截肢等                  → 赔80%保额
四级残疾：                                  → 赔70%保额
五级残疾：                                  → 赔60%保额
六级残疾：                                  → 赔50%保额
七级残疾：                                  → 赔40%保额
八级残疾：                                  → 赔30%保额
九级残疾：                                  → 赔20%保额
十级残疾：部分手指截肢、轻度功能障碍等      → 赔10%保额
```

> 📌 **开发提示**：在系统中设计理赔模块时，残疾等级是一个**枚举类型**，每个等级对应固定的赔付比例。这个比例是行业标准，不应该由用户自行配置。

### 1.3 职业分类：意外险核保的核心逻辑

**为什么要做职业分类？**

不同职业面临的意外风险天差地别：坐办公室的程序员和在高楼外墙作业的架子工，意外死亡概率相差可能达 **10 倍以上**。

保险公司将职业分为 **1-6 类**（部分公司分到 7 类）：

```
职业类别  风险等级  代表职业                           可投保意外险类型
─────────────────────────────────────────────────────────────────────
1类       最低      教师、医生（非外科手术）、           可投保所有意外险
          风险      行政人员、IT程序员、会计师           保费最低

2类       较低      销售人员（外勤）、驾驶员（轻型）     可投保大部分意外险
          风险      摄影师、厨师、理发师                 保费略高

3类       中等      重型货车司机、搬运工、               部分意外险可投保
          风险      建筑工地一般工人（地面）              保费较高

4类       较高      消防员、高空作业（20米以内）          限制条件多
          风险      电工、化工厂工人                     需要特殊产品

5类       高风险    高空作业（20米以上）、               大部分产品不接受
                   矿山开采工人、爆破工人

6类       极高      潜水员、军人（作战）、               通常无法通过普通
          风险      赛车手、职业运动员（高危）            渠道投保
```

> ⚠️ **重要规则**：
> - 1-3 类职业：大多数意外险均可投保
> - 4 类职业：需要专门的"高危职业意外险"，保费更高
> - 5-6 类职业：通常被**拒保**，或需要通过团体意外险渠道

### 1.4 职业类别与系统设计

职业分类直接影响系统中的核保逻辑和产品配置：

```java
/**
 * 职业类别枚举
 * 用于意外险核保校验
 */
public enum OccupationCategory {
    
    CATEGORY_1(1, "1类职业 - 一般办公室人员", 1.0),
    CATEGORY_2(2, "2类职业 - 外勤/驾驶员等", 1.2),
    CATEGORY_3(3, "3类职业 - 建筑/搬运等", 1.5),
    CATEGORY_4(4, "4类职业 - 高空作业/电工等", 2.0),
    CATEGORY_5(5, "5类职业 - 矿山/爆破等", 3.0),
    CATEGORY_6(6, "6类职业 - 极高风险", null); // 通常拒保

    private final int code;
    private final String description;
    /**
     * 相对1类的费率系数，null表示不可投保
     */
    private final Double rateCoefficient;
    
    OccupationCategory(int code, String description, Double rateCoefficient) {
        this.code = code;
        this.description = description;
        this.rateCoefficient = rateCoefficient;
    }
    
    /**
     * 判断该职业类别是否可以投保指定意外险产品
     * @param maxAcceptCategory 产品可接受的最高职业类别
     */
    public boolean isAcceptable(int maxAcceptCategory) {
        return this.code <= maxAcceptCategory && this.rateCoefficient != null;
    }
}
```

### 1.5 典型产品对比：互联网意外险 vs 团体意外险

这两类产品在实际业务中差异极大，理解这个差异对系统设计至关重要：

| 对比维度 | 互联网个人意外险 | 团体意外险 |
|--------|--------------|-----------|
| **代表产品** | 年交 99 元的"综合意外险" | 企业员工团意险 |
| **投保方式** | C 端自助购买，个人投保 | B 端企业统一为员工投保 |
| **保障金额** | 身故 20~50 万，医疗 1~3 万 | 身故 50~200 万，医疗 5~30 万 |
| **价格** | 年缴 99~399 元 | 按人头收费，百人企业年费 5~50 万 |
| **健康告知** | 简单（通常 3~5 条） | 简化（团险通常免健康告知） |
| **续保** | 每年独立续保，费率可能变化 | 企业每年批量续保，协议价格 |
| **核保效率** | 即时出单（规则引擎） | 需要报批，2~5 个工作日 |
| **佣金率** | 中介佣金 10~20% | 中介佣金 20~35% |
| **系统复杂度** | 标准化程度高，系统易实现 | 被保险人名单管理复杂 |

> 💡 **对产品中台的启示**：
>
> 同样是"意外险"，个人险和团险的**投保主体不同**（个人 vs 企业）、**被保险人数量不同**（1人 vs N人）、**核保流程不同**。这意味着在数据库设计时，我们必须区分**投保人**和**被保险人**，且保单需要支持**1:N**的关系（一张团体保单对应多个被保险人）。

### 1.6 意外险投保流程（带系统节点）

```
[C端/B端 APP]
      │
      ▼
1. 选择产品 → 系统加载产品配置（保障内容、价格、可投职业类别）
      │
      ▼
2. 填写职业 → 系统根据职业分类校验该职业是否可投保该产品
      │          ├─ 不可投保 → 提示"该职业不在承保范围"，推荐其他产品
      │          └─ 可投保 → 计算费率系数，更新保费展示
      ▼
3. 填写被保险人信息
      │  ├── 姓名、身份证号、手机
      │  └── 如是团险：上传被保险人名单Excel
      ▼
4. 确认保障内容 & 保费 → 展示"免责条款"（监管要求明显标注）
      │
      ▼
5. 支付 → 支付成功后异步通知
      │
      ▼
6. 核保 → 规则引擎执行核保逻辑（职业类别、年龄限制、重复投保检查）
      │    ├─ 自动核保通过 → 出单，发送电子保单
      │    └─ 转人工核保 → 进入待审队列
      ▼
7. 出单 → 生成保单号，通知被保险人
```

## 第二章：健康险（Health Insurance）

健康险是非车险中**最复杂**的险种，也是当前市场增速最快的品类。健康险的复杂性体现在：条款差异极大、核保逻辑复杂（健康告知）、赔付计算有多种模式。

### 2.1 健康险产品分类

```
健康险
│
├── 医疗险（报销型）
│   ├── 百万医疗险（主流）
│   ├── 住院医疗险
│   └── 门诊医疗险（含药险、齿科险）
│
├── 疾病险（给付型）
│   ├── 重大疾病险（重疾险）
│   │   ├── 纯消费型（保定期）
│   │   └── 储蓄型（保终身）
│   └── 轻症/中症险（附加在重疾险上）
│
└── 护理险
    ├── 商业护理险
    └── 长期护理保险（政策险，部分城市试点）
```

---

### 2.2 百万医疗险：深度解析

百万医疗险是目前C端购买量最大的健康险产品，以"每年数百元、保额百万"为核心卖点。

#### 核心条款要素

```
产品名称：某某百万医疗险（示例）

保险期间：1年期（每年续保）

保障范围：
  ✅ 住院医疗费用
  ✅ 特殊门诊（肿瘤放化疗、肾透析、门诊手术）
  ✅ 住院前后门急诊（通常为住院前7天、出院后30天）
  
免赔额：10,000 元（也叫"绝对免赔额"）
  → 理解：每次赔付，前1万元自付，超过1万元的部分才由保险公司报销
  
报销比例：
  ├── 社保内费用：100%报销
  └── 自费/自选药：60% ~ 100%（各产品不同）

保额：600万（每次住院最高赔付600万）

续保条件（关键！）：
  ├── 保证续保：投保后患病，只要按时缴费，保险公司必须续保（不能拒保涨价）
  └── 非保证续保：保险公司可以在续保时拒绝高风险客户或提高费率（实际多数产品如此）
```

> ⚠️ **免赔额是理解百万医疗的关键**
>
> 以 10,000 元免赔额为例：
> - 若本次住院花费 **8,000 元** → 低于免赔额，**0 赔付**
> - 若本次住院花费 **50,000 元** → 超过免赔额 40,000 元，赔付 **40,000 × 100% = 40,000 元**
> - 若本次住院花费 **1,000,000 元** → 超过免赔额 990,000 元，赔付 **990,000 元**

#### 百万医疗险的业务系统关键字段

```sql
-- 百万医疗险扩展表（示意）
CREATE TABLE ins_health_medical_ext (
    id              BIGINT PRIMARY KEY COMMENT '主键',
    policy_id       BIGINT NOT NULL COMMENT '关联保单ID',
    
    -- 免赔额配置
    deductible_amount   DECIMAL(12,2) NOT NULL COMMENT '免赔额（元）',
    deductible_type     TINYINT NOT NULL COMMENT '免赔类型：1-绝对免赔 2-相对免赔',
    
    -- 报销比例
    reimbursement_rate_insured  DECIMAL(5,2) COMMENT '社保内费用报销比例（%）',
    reimbursement_rate_self     DECIMAL(5,2) COMMENT '自费项目报销比例（%）',
    
    -- 保额
    coverage_amount     DECIMAL(15,2) NOT NULL COMMENT '每次住院最高赔付额（元）',
    annual_limit        DECIMAL(15,2) COMMENT '年度累计最高赔付额（元，0表示不限）',
    
    -- 续保条件
    renewal_type        TINYINT NOT NULL COMMENT '续保类型：1-保证续保 2-非保证续保',
    renewal_max_age     INT COMMENT '可续保最大年龄',

    waiting_period_days INT DEFAULT 30 COMMENT '等待期（天）',
    waiting_period_cancer_days INT DEFAULT 180 COMMENT '恶性肿瘤等待期（天）',
    
    creator     VARCHAR(64) DEFAULT '' COMMENT '创建者',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updater     VARCHAR(64) DEFAULT '' COMMENT '更新者',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted     BIT(1) NOT NULL DEFAULT 0 COMMENT '是否删除',
    tenant_id   BIGINT NOT NULL DEFAULT 0 COMMENT '租户编号'
);
```

### 2.3 重大疾病险：最复杂的健康险产品

重疾险是健康险中**技术含量最高**的产品，理解重疾险是设计健康险系统的关键。

#### 什么是重大疾病？

中国保险行业协会（中保协）规定了**28种必保重大疾病**（所有重疾险产品必须包含），以及扩展的可选病种。

```
必保的6种核心重大疾病（最常见理赔病种）：
1. 恶性肿瘤（癌症）
2. 急性心肌梗塞
3. 脑中风后遗症
4. 重大器官移植术（含造血干细胞移植）
5. 冠状动脉搭桥术
6. 终末期肾病（尿毒症）

这6种病占重疾险理赔案件的约 85% 以上
```

#### 轻症 / 中症 / 重症的定义与赔付逻辑

```
重疾险的三级保障结构：

重症（Major Disease）
  ├── 定义：符合中保协28种标准重疾定义
  ├── 赔付比例：保额 × 100%（一次性给付）
  ├── 赔付后：保单继续有效（多次赔付重疾险）或终止
  └── 举例：确诊为肺癌（恶性肿瘤）→ 赔 50万

中症（Moderate Illness）★ 2021年新增
  ├── 定义：介于轻症和重症之间的疾病状态
  ├── 赔付比例：保额 × 50%（各产品不同，约30%-60%）
  ├── 中症后：保额不减少，还可继续赔付轻症和重症
  └── 举例：不符合重症标准的心肌梗塞前期 → 赔 25万

轻症（Minor Illness）
  ├── 定义：严重程度较低的特定疾病状态
  ├── 赔付比例：保额 × 30%（各产品不同，约20%-40%）
  ├── 轻症后：豁免后续保费，保单继续有效
  └── 举例：原位癌（极早期癌症）→ 赔 15万 + 豁免保费
```

#### 等待期（Waiting Period）

等待期是重疾险最重要的免责条款之一：

```
等待期规则：
─────────────────────────────────────────────────
等待期 = 保单生效后，必须等待一段时间，该期间内出险不赔

常见等待期设置：
  - 一般等待期：90天（3个月）
  - 恶性肿瘤单独等待期：180天（6个月）
  
等待期内出险处理：
  - 重症出险 → 退还全部已缴保费，保单终止（不赔保额）
  - 意外导致的出险 → 等待期内也赔（意外不受等待期限制）

为什么设置等待期？
  防止"带病投保"：明知自己已患病，立即投保后申请理赔（逆向选择）
```

#### 多次赔付重疾险

现代重疾险产品普遍支持"多次赔付"，这大大增加了系统的复杂性：

```
多次赔付规则示意（以某款产品为例）：

第1次赔付：任意重症     → 赔 100% 保额（如50万）
  ▼ 间隔期：180天
第2次赔付：特定重症     → 赔 100% 保额（再赔50万）
  ▼ 间隔期：180天
第3次赔付：特定重症     → 赔 100% 保额（再赔50万）

轻症：最多赔付3次，每次赔30%保额
中症：最多赔付2次，每次赔50%保额

注意：
  - 同一疾病不可多次赔付（如：第1次赔了肺癌，不能再赔肺癌）
  - 不同疾病组（分组设计）之间的赔付次数有限制
```

> 💻 **开发挑战**：
>
> 多次赔付的逻辑意味着系统需要维护"已赔付疾病列表"和"赔付次数计数器"。每次新的理赔申请，都需要校验：
> 1. 本次申请的疾病是否已赔付过？
> 2. 各类型（轻/中/重症）的赔付次数是否已达上限？
> 3. 距离上次赔付是否满足间隔期？
>
> 这些逻辑应该封装为**状态机 + 规则引擎**，而不是硬编码 if-else。

### 2.4 健康告知：保险系统的"地雷区"

**健康告知（Health Declaration）** 是健康险投保流程中最关键的环节，也是系统开发中最容易踩坑的部分。

#### 健康告知的法律地位

```
法律依据：《保险法》第十六条

投保人的如实告知义务：
  投保人故意或因重大过失未履行如实告知义务，
  足以影响保险人决定是否同意承保或者提高保险费率的，
  保险人有权解除合同。

保险人的不可抗辩权（2年规则）：
  合同解除权，自保险人知道有解除事由之日起，
  超过30日不行使而消灭。
  自合同成立之日起超过2年的，保险人不得解除合同；
  发生保险事故的，保险人应当承担赔偿或者给付保险金的责任。
```

#### 典型健康告知题目示例

```
健康告知问卷（示意）

1. 您是否曾被医生诊断或怀疑患有以下疾病？
   □ 恶性肿瘤（癌症）
   □ 心脏病（包括：冠心病、心肌梗塞、心力衰竭）
   □ 脑血管疾病（包括：脑中风、脑出血）
   □ 糖尿病
   □ 高血压（血压值≥140/90mmHg）
   □ 慢性肝病（包括：乙肝、丙肝、肝硬化）
   
2. 过去2年内，您是否曾因病住院治疗（精神疾病、外伤除外）？

3. 目前您的身体质量指数（BMI = 体重kg ÷ 身高m²）是否超过32？

4. 您是否目前正在接受医疗建议的治疗、用药或手术？

以上任何一题回答"是"：
  → 系统标记为"需要核保"
  → 进入人工审核流程，可能被拒保或附加除外责任
```

#### 不如实告知的后果

```
场景一：投保人故意隐瞒病情
  行为：明知自己患有糖尿病，健康告知时勾选"否"
  后果：
    ├── 保险公司查明后 → 解除保险合同，不退保费（故意不实告知）
    └── 理赔时发现   → 拒赔，并解除合同

场景二：投保人疏忽漏报
  行为：忘记告知体检报告中的"轻度脂肪肝"
  后果：
    ├── 2年内发现      → 保险公司可以解除合同，退还保费
    └── 2年后发现      → 保险公司不能解除合同（2年不可抗辩条款）
    
场景三：投保后2年理赔
  行为：2年前投保时未如实告知，2年后确诊癌症申请理赔
  后果：→ 保险公司必须赔付（超过2年不可抗辩）
         ※ 例外：如果是故意欺诈，仍可拒赔
```

#### 系统实现关键点

```java
/**
 * 健康告知行为轨迹记录
 * 监管要求：健康告知的每一步操作都必须记录，用于合规存证
 */
@Data
public class HealthDeclarationTraceDTO {
    
    /** 业务申请ID */
    private Long applicationId;
    
    /** 问题编号 */
    private String questionCode;
    
    /** 问题内容（冗余存储，防止问题被修改导致存证失效） */
    private String questionContent;
    
    /** 用户的回答：YES / NO */
    private String answer;
    
    /** 用户在该问题上停留的秒数（监管要求：证明用户确实阅读了） */
    private Integer dwellSeconds;
    
    /** 是否修改过答案（从YES改为NO，需要重点存证） */
    private Boolean wasModified;
    
    /** 修改前的答案 */
    private String previousAnswer;
    
    /** 回答时间戳 */
    private LocalDateTime answeredAt;
    
    /** 客户端IP */
    private String clientIp;
    
    /** 设备指纹 */
    private String deviceFingerprint;
}
```

> ⚠️ **监管要求**：根据《互联网保险业务监管办法》，C端在线投保时，健康告知页面必须：
> 1. **不能预填答案**（不能默认勾选"否"）
> 2. **客户逐条确认**，不能一次性全部勾选
> 3. **记录停留时长**，防止快速滑过不看
> 4. **存证保留**，保单有效期内随时可调取

### 2.5 护理险：政策背景与商业机会

护理险（Long-term Care Insurance）是一个新兴险种，背后有重要的政策背景。

#### 政策背景：中国老龄化危机

```
2023年中国老龄化数据（参考）：
  - 60岁以上人口：约2.97亿，占总人口21.1%
  - 65岁以上人口：约2.17亿，占总人口15.4%
  
失能老人问题：
  - 全国约有4500万失能半失能老人
  - 专业护理机构严重不足
  - 家庭照护成本极高（专业护工每月5,000-15,000元）
```

#### 长期护理保险（政策险）

```
长期护理保险（长护险）试点：
  ├── 2016年：国家开始在15个城市试点
  ├── 2020年：扩大到49个城市
  └── 2024年：全国推广中
  
长护险与社保的关系：
  ├── 资金来源：从医保基金划拨 + 个人缴费
  ├── 保障范围：失能人员的基本生活照料和医疗护理
  └── 赔付方式：护理服务（实物给付）或现金补贴
  
护理等级评定（6级）：
  一级（完全失能）：日常生活完全依赖他人
  二级：基本依赖他人
  三级：部分依赖
  四级：轻度依赖
  五级：基本自理（稍受损）
  六级：完全自理 ← 不在护理险保障范围
```

#### 商业护理险的系统特点

```
商业护理险 vs 普通寿险/健康险的差异：
  ├── 触发条件复杂：需要专业机构评定护理等级
  ├── 赔付持续性：符合条件后，每月/每年持续赔付（非一次性）
  └── 理赔周期长：可能持续赔付数十年
  
对系统设计的影响：
  ├── 需要支持"持续给付"记录（定期检查护理状态）
  ├── 需要对接护理评级机构的数据
  └── 财务上需要精算准备金计算
```

## 第三章：企业财产险（Commercial Property Insurance）

企业财产险是保险中介机构面向企业客户（B2B）的核心产品，单笔保费远高于个人险，佣金收入丰厚。

### 3.1 企业财产险的承保对象

```
企业财产险承保什么？

承保对象（财产）：
  ├── 建筑物：厂房、办公楼、仓库、商业用房
  ├── 机器设备：生产设备、电子设备、专业机器
  ├── 存货/库存：原材料、在制品、产成品
  ├── 装修装潢：办公室、商铺装修
  └── 其他财产：家具、办公用品等

承保风险（事故原因）：
  ├── 火灾
  ├── 爆炸
  ├── 雷击
  ├── 洪水、暴风、暴雨（需要附加险）
  ├── 盗窃（需要附加险）
  └── ...
```

---

### 3.2 财产一切险 vs 综合险：关键区别

这是企财险中最常被混淆的概念，必须掌握：

| 对比维度 | 财产一切险（All Risk） | 综合险（Comprehensive） |
|--------|---------------------|----------------------|
| **保障逻辑** | **列明除外**：除了条款明确排除的，其余都保 | **列明承保**：只保条款列出的灾害风险 |
| **保障范围** | 更广泛，覆盖大多数意外损失 | 较窄，只覆盖11种指定灾害 |
| **保费** | 较高（因为保障更全面） | 较低 |
| **理解难点** | 出险后保险公司举证"属于除外"才能拒赔 | 出险后投保人举证"属于列明灾因"才能赔 |
| **适合场景** | 高价值资产、复杂风险环境 | 预算有限、风险环境简单 |

> 📌 **一切险的例子**：
>
> 某工厂发生火灾，同时引发爆炸，造成厂房损失 200 万。
> - **财产一切险**：无需区分是"火灾"还是"爆炸"导致的损失，直接赔付。
> - **综合险**：需要分别核定"火灾损失"和"爆炸损失"，如果某项不在列明范围内则不赔。

#### 综合险覆盖的11种标准灾害

```
综合险标准承保灾因：
1. 火灾
2. 爆炸（非锅炉爆炸，锅炉需单独投保）
3. 雷击
4. 飞行物体及其他空中运行物体坠落
5. 暴风
6. 龙卷风
7. 暴雨
8. 洪水
9. 冰雹
10. 滑坡和崖崩
11. 突发性滑坡（含地面下陷）
```

---

### 3.3 保额确定：重置价值法 vs 账面价值法

保额的确定是企财险核保的核心问题，也是常见纠纷的来源。

#### 重置价值法（Replacement Value）

```
概念：以"全新同等物品的购置成本"作为保额基准

计算方法：
  假设某台生产设备：
    购置时间：2018年，购置价格：100万
    当前市场同型号新机价格：150万（因通货膨胀/设备升级）
  
  重置价值 = 150万
  → 应投保额：150万
  
特点：
  ✅ 出险后可以买到同等新设备，不会有损失
  ❌ 保费更高（因为保额更高）
  ❌ 保险公司可能要求对标准的新设备价格进行核实
```

#### 账面价值法（Book Value）

```
概念：以"资产账面净值"作为保额基准（即扣除折旧后的价值）

计算方法：
  同一台设备：
    购置价格：100万
    已使用6年，折旧年限10年，残值率10%
    
    年折旧额 = (100万 - 100×10%) ÷ 10 = 9万/年
    账面净值 = 100万 - 9万 × 6年 = 46万
  
  账面价值 = 46万
  → 应投保额：46万
  
特点：
  ✅ 保费较低
  ❌ 出险后赔付款买不了新设备（产生差额）
  ❌ 如果资产未正确折旧，容易出现"不足额投保"
```

#### 不足额投保（重要！）

```
不足额投保是企业财产险中最常见的纠纷来源：

规则：按照财产实际价值的比例赔付

举例：
  财产实际价值：200万
  投保金额（保额）：100万（只投了50%）
  实际损失：80万
  
  理赔计算：80万 × (100万 ÷ 200万) = 40万
  
  结果：只赔了40万，而不是80万！
  
常见原因：
  1. 企业为省保费，故意低报财产价值
  2. 财产价值随市场变化，但保额多年未更新
  3. 新购置财产忘记增加批单（保单变更）
  
系统设计提示：
  应提醒用户每年更新财产价值，系统可以根据资产折旧/通胀系数提供建议保额
```

### 3.4 业务中断险（Business Interruption Insurance，BI）

业务中断险是企财险的重要附加险，也是保险金融从业者最难解释的险种之一。

#### BI 的工作原理

```
场景：某食品工厂发生火灾

直接损失（由企业财产险赔付）：
  厂房修复费用：200万
  生产设备损失：100万
  存货损失：50万
  直接损失合计：350万

间接损失（由业务中断险赔付）：
  修复期间：60天
  
  期间损失的利润：
    正常月利润：50万元
    60天停产损失利润：100万元
    
  期间无法减少的固定支出：
    员工工资：30万（员工不能辞退，还得发工资）
    贷款利息：5万
    租金：10万
    固定支出合计：45万
  
  业务中断险赔付：100万（利润损失）+ 45万（固定支出）= 145万
```

#### BI 的核心参数

```
业务中断险的关键条款：
  
1. 赔偿期限（Indemnity Period）
   ├── 从财产险事故发生之日起
   ├── 到企业恢复正常经营为止
   └── 通常设置最长限额（如12个月、24个月）
   
2. 等待期（Excess Period）
   └── 类似免赔额的概念，前N天（如7天）的损失不赔
   
3. 年营业额申报（Annual Turnover Declaration）
   ├── 投保时需要申报上年度营业额
   └── 保额通常以"预期年营业额 ÷ 12 × 赔偿期月数"计算
   
4. 利润率约定
   └── 赔付的是"毛利润损失"，而非"营业额损失"
```

> 💡 **对系统设计的启示**：
>
> BI 险与财产险捆绑销售，但它们的**赔付触发条件不同**（BI 依赖于财产险理赔事故的发生）。在系统中，BI 险的理赔申请必须关联到对应的财产险理赔案件。
