## 4. 机构费率政策

> **PDF 参考：** `PDF-139`（政策管理目录）、`PDF-140`（机构费率政策）

### 4.1 功能概述

**菜单路径：** 寿险 → 政策管理 → 机构费率政策

为各机构配置从平台获得的寿险佣金费率（即下游分配费率的一种特殊形式，专针对机构层级）。

**操作入口（PDF-140）：** 点击【寿险】→【政策管理】→【机构费率政策】→【添加】→【单个新增】

### 4.2 单个新增政策批次

> **PDF 参考：** `PDF-140`（1.1~1.3）

**模版选择（PDF-140 关键细节）：**

单个新增界面有 **四个机构政策模版** 可根据需要自行选择：

| 模版 | 费用类型 | 特点 |
|---|---|---|
| 模版一 | 机构基础费率 | 可针对全部保司设置 |
| 模版二 | 机构业绩费率 | 按业绩档位计算 |
| 模版四 | 续期服务津贴 | 续期保单专属 |
| 模版五 | 机构基础费率 | 与模版一相同费用类型，配置维度不同 |

**政策基本条件（PDF-140 模版一详述）：**

| 字段名 | 必填 | 说明 |
|---|---|---|
| 结算时间 | ★ | 枚举：月结/季结/年结/活动结束当月结算 |
| 费用类型 | ★ | 依据所选模版自动带出（见上表） |
| 机构 | ★ | 多选，从机构树选择；仅归属所选组织的保单可匹配此政策，支持同时选多个组织 |
| 保险公司 | ★ | 多选（模版一可针对全部保司设置） |
| 产品名称 | 选填 | 多选（不选则代表该保司所有产品均适用） |
| 比例基础 | ★ | 枚举：保费/佣金+奖金/标保 |
| 开始时间 | ★ | 政策生效开始日期 |
| 结束时间 | 选填 | 留空或选"永久"表示永久有效 |
| 政策名称 | ★ | 自定义名称，最多50字 |

**费率明细表格（PDF-140 汇率因子规格）：**

- 缴费年限行数：最多添加 **15 行**（PDF-140 关键约束）
- 保单年度佣金率列数：最多添加 **20 列**（PDF-140 关键约束）
- 表格支持行内编辑

| 列名 | 说明 |
|---|---|
| 缴费年限/档位 | 年度范围或职级档位 |
| 首年佣金率 | 百分比，精度到 0.01% |
| 续年佣金率 | 百分比 |
| 增值服务佣金率 | 百分比（选填） |
| 有效期 | 该行费率的有效时间段 |

底部【添加一行】按钮，单行支持删除。

**保存（PDF-140 1.3）：** 核实政策限制条件、点位等信息无误后，点击保存即可。

### 4.3 批量新增政策

> **PDF 参考：** `PDF-140`

点击【添加】→【批量新增】：
1. 下载批量政策模板
2. 按模板填写政策批次信息
3. 上传模板
4. EasyExcel 解析，批量 INSERT `ins_life_policy_rate`

### 4.4 政策审批

> **PDF 参考：** `PDF-141`（政策审批章节，与上游政策审批流程相同）

**审批入口（PDF-141）：** 点击【寿险】→【政策审批】→【查询】查到政策批次后点【待审批】，也可以勾选多条政策批次点【批量审批】。

**审批状态流转：**
- 新建或修改的政策状态为 **待审批**
- 审批通过后状态为 **审批通过**，才能参与佣金计算，否则计算结果为0（PDF-141）
- 审批未通过驳回后，在对应政策中显示"未通过"，可再次编辑后重新提交审核

**Flowable 审批流：** 审批流 Key = `LIFE_POLICY_RATE_APPROVE`
- 审批通过：UPDATE `status = ACTIVE`，政策生效
- 审批拒绝：status = REJECTED，通知申请人

**版本管理：** 同一机构+险种+保司只能有一条 `ACTIVE` 的政策。新政策审批通过时，旧政策自动变更为 `EXPIRED`。

## 5. 上游政策配置

> **PDF 参考：** `PDF-139`（政策管理目录）、`PDF-141`（上游政策配置和政策审批）

### 5.1 功能概述

**菜单路径：** 寿险 → 政策管理 → 上游政策配置

配置各保司支付给平台的佣金费率（即保司→平台的收入来源）。

**操作入口（PDF-141）：** 点击【寿险】→【政策管理】→【上游政策配置】→【添加】

### 5.2 新增上游政策

> **PDF 参考：** `PDF-141`（一、新增政策批次）

| 字段名 | 必填 | 说明 |
|---|---|---|
| 政策类型 | ★ | 枚举：首期政策/续期政策 |
| 结算时间 | ★ | 枚举：月结/季结/年结/活动结束当月结算 |
| 费用类型 | ★ | 枚举：基础首期佣金/销售奖金/首期其他（注：费用类型支持自定义填写）（PDF-141） |
| 机构 | ★ | 多选，仅归属所选组织的保单可匹配此政策，支持同时选多个机构 |
| 保险公司 | ★ | 单选 |
| 产品名称 | 选填 | 多选 |
| 比例基数 | ★ | 枚举：保费/佣金+奖金/标保/FYC（**注：目前不支持首期、续期政策选比例基数"标保"计算**）（PDF-141） |
| 开始时间 | ★ | |
| 结束时间 | 选填 | 永久时留空或选"永久" |
| 业绩 | 选填 | 枚举：标保/FYC（**注：仅政策类型为首期政策时才有该条件**；如勾选业绩，下方缴费因子必须按照业绩来设置点位）（PDF-141） |
| 继续率算法 | 选填 | 枚举：年度通算/月度通算（**注：仅政策类型为续期政策时才有该条件**；如填写继续率算法，下方缴费因子需按照继续率月份与范围设置比例）（PDF-141） |
| 政策名称 | ★ | 自定义 |

**汇率因子（档位明细，PDF-141）：**
- 页面内嵌表格，最多支持 **100 条**
- 每行：起始保费档位 | 结束保费档位 | 佣金率（%）
- 支持单条删除、中间插入新行

**缴费因子（PDF-141）：**
- 根据政策勾选相关限制条件（如仅适用月缴/年缴等）
- 若勾选了"业绩"，缴费因子必须按照业绩来设置点位
- 若填写了"继续率算法"，缴费因子需按继续率月份与范围设置比例

**保存后端逻辑：**
- INSERT `ins_life_upstream_policy` + `ins_life_upstream_policy_factor`
- status = `PENDING`（待审批）

### 5.3 政策审批

> **PDF 参考：** `PDF-141`（二、政策审批）

**审批流程同机构费率政策（见 4.4），关键要点（PDF-141）：**
- 新建或修改政策状态为**待审批**
- 审批通过后状态为审批通过才能参与佣金计算
- 驳回后可再次编辑再提交

**支持撤销申请：**
- 仅 `PENDING`（待审批）状态可撤销
- 撤销后 status = `CANCELED`

**版本互斥控制：** 同保司/险种只能一条生效中的上游政策，新政策审批通过时旧政策自动失效。

## 6. 下游政策配置

> **PDF 参考：** `PDF-139`（政策管理目录）、`PDF-142`（下游政策配置和政策审批）

### 6.1 功能概述

**菜单路径：** 寿险 → 政策管理 → 下游政策配置

配置平台向业务员/机构分配的寿险佣金政策（即平台→业务员的分配规则）。

**操作入口（PDF-142）：** 点击【寿险】→【政策管理】→【下游政策配置】→【添加】

### 6.2 新增下游政策

> **PDF 参考：** `PDF-142`（一、新增政策批次）

| 字段名 | 必填 | 说明 |
|---|---|---|
| 政策类型 | ★ | 枚举：直销佣金政策/基本法额外奖励政策/代理人奖励政策 |
| 机构 | ★ | 多选，仅归属所选组织的保单可匹配此政策，支持同时选多个组织 |
| 保险公司 | ★ | 单选 |
| 产品名称 | 选填 | 多选 |
| 比例基数 | ★ | 枚举：保费/标保 |
| 开始时间 | ★ | |
| 结束时间 | 选填 | 永久时留空 |
| 政策名称 | ★ | 自定义 |

**直销政策扩展字段（PDF-142 关键细节）：**
- 政策类型=直销佣金政策时，显示【基本法选择】字段
- 选择适用的基本法（下拉，从已配置的寿险基本法中选择）

**汇率因子（PDF-142）：**
- 最多支持 **100 条**，汇率因子支持单个删除和中间新增（PDF-142 原文）
- 页面内嵌表格

**缴费因子（PDF-142）：**
- 根据政策勾选相关限制条件（适用缴费方式限制）

**后端校验（上下游利润空间校验）：**
- 保存时校验：下游分配比例不得超过对应上游政策的佣金率（避免倒挂亏损）
- 若下游率 > 上游率，弹出警告"下游佣金率高于上游，请确认是否继续"，用户二次确认才可保存

### 6.3 政策审批

> **PDF 参考：** `PDF-142`（二、政策审批）

同上游政策审批流程，Flowable 多级审批：机构负责人 → 平台管理员。

- 审批通过后生效，版本互斥控制（同保司/机构只能一条生效中的下游政策）
- 支持批量审批：勾选多条政策批次后点【批量审批】

## 7. 折标系数

> **PDF 参考：** `PDF-139`（政策管理目录）、`PDF-143`（折标系数和政策审批）

### 7.1 功能概述

**菜单路径：** 寿险 → 政策管理 → 折标系数

配置各险种/缴费方式的折标系数，用于将实际保费折算为标准保费（标保），是业绩统计和档位计算的基础。

**折标公式：** `标保 = 年度保费 × 折标系数`

**操作入口（PDF-143）：** 点击【寿险】→【政策管理】→【折标系数】→【添加】

### 7.2 新增折标系数政策

> **PDF 参考：** `PDF-143`（一、新增政策批次）

| 字段名 | 必填 | 说明 |
|---|---|---|
| 政策类型 | 只读 | 固定：折标政策（PDF-143 原文） |
| 机构 | ★ | 多选，仅归属所选组织的保单可匹配此政策，支持同时选多个组织 |
| 保险公司 | ★ | 单选 |
| 产品名称 | 选填 | 多选（不选则对应保司所有产品） |
| 折标对象 | ★ | 枚举：上游折标/下游折标/上下游折标（PDF-143 原文细节见下） |
| 开始时间 | ★ | |
| 结束时间 | 选填 | 永久时留空 |
| 政策名称 | ★ | 自定义 |

**折标对象说明（PDF-143 关键细节）：**

| 折标对象值 | 说明 |
|---|---|
| 上游折标 | 对接上游折标系数，用于向保司结算时的标保计算 |
| 下游折标 | 对接下游折标系数，用于向业务员分配时的标保计算 |
| 上下游折标 | 同时对接上下游折标系数 |

**折标明细表格（行内编辑）：**

| 列名 | 说明 |
|---|---|
| 缴费方式 | 枚举：趸缴/年缴/半年缴/季缴/月缴 |
| 折标系数 | 小数，精度4位，如 1.0000/0.1000 |

每种缴费方式单独配置折标系数，参考值：
- 趸缴：通常 = 0.1（即10年期折算）
- 年缴：通常 = 1.0

**保存后端逻辑：**
- INSERT `ins_life_std_ratio`
- status = `PENDING`（待审批）

### 7.3 政策审批

与机构费率政策审批流程相同，审批通过才生效。

## 8. 保司结算时间

> **PDF 参考：** `PDF-139`（政策管理目录）、`PDF-144`（保司结算时间设置）

### 8.1 功能概述

**菜单路径：** 寿险 → 政策管理 → 保司结算时间设置

配置每家保司的佣金结算周期和相关截止日期，用于佣金到账时间预测和结算月判定。

**注意（PDF-144 关键业务逻辑）：**
> 机构不支持多选，针对不同组织需建立不同的保司结算时间。

**应用场景（PDF-144）：**
> 回执回访日期是次月的时间，例如：回执/回访截止都设置为1日，则在下个月1日前回执回访的结算月是当月，超过该日期即归入下个月的结算月。

### 8.2 新增保司结算时间（单个新增）

> **PDF 参考：** `PDF-144`（一、新增保司结算时间）

**操作入口：** 点击【寿险】→【政策管理】→【保司结算时间设置】→【新增】

| 字段名 | 必填 | 说明 |
|---|---|---|
| 机构 | ★ | 单选（注：机构不支持多选）（PDF-144） |
| 保险公司 | ★ | 下拉，选择寿险合作保司 |
| 开始时间 | ★ | 该保司组织配置的生效开始时间 |
| 结束时间 | 选填 | 该保司组织配置的生效结束时间 |
| 犹豫期截止次月 | ★ | 根据保司规定设置（日，1~31）（PDF-144） |
| 回访截止次月 | ★ | 根据保司规定设置（日，1~31）（PDF-144） |
| 回执截止次月 | ★ | 根据保司规定设置（PDF-144，注：签收截止次月是客户回执的签收日期） |
| 签收截止次月 | ★ | 根据保司规定设置（PDF-144） |
| 状态 | ★ | 枚举：有效/无效（PDF-144） |

**保存逻辑：**
- UPSERT `ins_life_insurer_settlement_config`（已存在则更新，不存在则插入）
- 核实标星等信息无误后，确定保存（PDF-144 原文）

**在佣金预测中的应用：** 当系统计算"预计到账日期"时，读取此配置：`预计到账日 = 结算截止日 + 账期天数`

### 8.3 批量导入保司结算时间

> **PDF 参考：** `PDF-144`（二、导入保司结算时间）

**操作流程（PDF-144 详述）：**
1. 点击【寿险】→【政策管理】→【保司结算时间设置】→【导入】
2. 点击【模版下载】，下载导入模版
3. 填写模版内容（字段说明，PDF-144 关键细节）：
   - **机构代码**：点击【人管】→【组织机构管理】→【组织机构代码】模块查看（含人管超链接）
   - **保险公司代码**：见表格 Sheet4 保险公司代码
   - **签收截止日、回执截止日、犹豫期、回访截止日**：根据保司规定设置，只填写具体日期数字（例：到次月5日，就填写5）
   - **开始时间、结束时间**：选择该保司组织的生效时间（标星为必填项）
4. 信息确认无误后保存表格，点击【导入】→【选取文件】添加填好的表格后点击【导入】

**后端逻辑：**
- EasyExcel 解析
- 按机构代码+保险公司代码匹配，UPSERT 处理
- 生成导入报告

## 9. 客户维护与影像管理

> **PDF 参考：** `PDF-145`（目录）、`PDF-146`（客户查询）、`PDF-147`（客户新增）、`PDF-148`（客户导入）、`PDF-149`（客户导出）

### 9.1 功能概述

**菜单路径：** 寿险 → 客户维护/影像管理

**子菜单（PDF-145 目录，含视频 09:51）：**
- 客户查询
- 客户新增
- 客户导入
- 客户导出

寿险专用客户管理入口。数据与通用 CRM（`ins_customer` 表）共享，但寿险入口有额外的寿险专属字段，且支持影像件管理。

**重要补充（PDF-147）：** 个险保单手动填写的客户信息会**自动同步**到客户维护模块。

### 9.2 客户查询

> **PDF 参考：** `PDF-146`（客户查询）、`PDF-147`（客户信息查询章节）

**筛选条件（PDF-147 原文）：**

| 筛选项 | 类型 | 说明 |
|---|---|---|

### 9.6 导出客户名单

> **PDF 参考：** `PDF-149`（客户导出）

**操作流程（PDF-149）：**
1. 点击【寿险】→【客户维护】，通过机构、证件号、姓名筛选需要导出的客户
2. 筛选出客户之后，点击右上角的【客户导出】按钮
3. 点击导出后，有**任务列表**可以查看进度，完成后下载文件即可（PDF-149 关键细节：支持进度查看）

**支持两种格式：**
- **标准格式**：系统标准字段，EasyExcel导出，证件号脱敏

- 采用异步导出（XXL-Job 任务队列），避免大数据量超时
- 导出任务记录存入 `ins_export_task`（task_type, status, file_url, create_time, finish_time）
- 完成后提供下载链接

### 9.7 影像管理

**菜单路径：** 寿险 → 客户维护/影像管理（影像管理为客户详情页 Tab）

#### 9.7.1 影像分类树

**`ins_image_category` 表（可配置）：**

```
├── 身份证
│   ├── 正面
│   └── 反面
├── 授权书
├── 回执书
├── 健康告知书
├── 医疗报告
└── 其他
```

#### 9.7.2 上传影像

1. 进入客户详情页，点击【影像管理】Tab
2. 选择影像分类 → 点击【上传】→ 选择文件（图片或PDF，单文件 ≤ 20MB）
3. 上传到 OSS，路径：`/life-customer/{customer_id}/{category_code}/{yyyyMM}/{filename}`
4. INSERT `ins_customer_image`（customer_id, category_id, file_name, oss_url, expire_date, upload_time, operator_id）

**影像有效期管理：**
- 身份证影像有效期：上传时填写身份证有效期（`expire_date` 字段）
- XXL-Job 定时任务每天检查：距过期 ≤ 30天的影像，发站内信提醒业务员"客户XX的身份证影像将于XX日到期，请及时更新"

#### 9.7.3 查看影像

- OSS URL 使用阿里云 OSS 签名URL，有效期1小时，避免直接暴露URL
- 点击【预览】：图片直接在页面预览；PDF在新标签页打开

#### 9.7.4 删除影像

- 删除操作需有特定权限（`LIFE_IMAGE_DELETE` 权限码）
- 软删除：UPDATE `is_deleted = 1`，OSS文件保留（可通过配置决定是否物理删除）

## 附录C：保司结算时间配置业务规则（PDF-144 整理）

| 配置项 | 业务含义 |
|---|---|
| 犹豫期截止次月（日） | 客户行使犹豫期退保权的截止日，超过则不可退保 |
| 回访截止次月（日） | 业务员完成回访的截止日，影响结算月归属 |
| 回执截止次月（日） | 客户提交回执的截止日，影响结算月归属 |
| 签收截止次月（日） | 客户签收回执的截止日（与回执截止日关联） |

**结算月判定规则（PDF-144 原文逻辑）：**

以回执截止日=1日为例：
- 客户在下个月1日**前**完成回执 → 结算月 = **当月**
- 客户在下个月1日**后**完成回执 → 结算月 = **下个月**
