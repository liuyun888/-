## 第一节：车险产品结构——先把产品搞清楚

### 1.1 交强险：国家说多少就多少

**交强险（机动车交通事故责任强制保险）** 是中国唯一一个由法律（《道路交通安全法》）强制要求购买的保险。

> 💡 **通俗理解**：你开车上路，万一撞了别人，国家要求你必须有能力赔偿。交强险就是这个"最低保障"。没买交强险上路，被交警查到直接扣车。

**交强险的核心特点：**

| 特点 | 说明 |
|------|------|
| 强制性 | 未投保不得上路行驶 |
| 固定费率 | 全国统一，由原银保监会（现国家金融监督管理总局）定价 |
| 无盈利原则 | 以收定支，监管要求保险公司不得通过交强险盈利 |
| 赔付对象 | 只赔第三方（受害方），不赔自己车辆和自身损失 |

**交强险责任限额（2022年调整后）：**

| 赔偿项目 | 死亡伤残 | 医疗费用 | 财产损失 |
|----------|---------|---------|---------|
| 有责任 | 18万元 | 1.8万元 | 2000元 |
| 无责任 | 1.8万元 | 1800元 | 100元 |

**交强险费率标准（私家车示例）：**

交强险保费按车辆种类和出险记录挂钩，以6座以下私家车为例：

- 未发生有责道路交通事故：基准保费 950 元/年
- 发生 1 次：950 × 1.1 = 1045 元
- 发生 2 次：950 × 1.3 = 1235 元
- 发生 3 次及以上：950 × 1.5 = 1425 元
- 连续 3 年未出险：950 × 0.7 = 665 元
- 连续 2 年未出险：950 × 0.85 = 808 元

> ⚠️ **系统开发注意**：交强险费率由国家统一规定，不存在"比价"的概念，但**每家保险公司都必须卖**，中介系统需要记录客户在哪家保司投保了交强险，以便续保管理。

### 1.2 商业车险：这才是市场竞争的主战场

商业车险是车主**自愿**购买的险种，保险公司可以**自主定价**（在监管规定范围内）。这里才有市场竞争，才有"报价比价"的需求。

**商业车险的产品架构：**

```
商业车险
├── 主险（必选项）
│   ├── 机动车损失险（车损险）
│   ├── 第三者责任险（三者险）
│   ├── 车上人员责任险（座位险）
│   ├── 全车盗抢险
│   ├── 自燃损失险
│   └── 玻璃单独破碎险
└── 附加险（可选项）
    ├── 不计免赔特约险（"不计免赔"）
    ├── 指定修理厂险
    ├── 无法找到第三方特约险
    ├── 驾驶人意外险
    └── 驾驶证丢失险 等
```

---

### 1.3 主险六大险种详解

#### 险种一：机动车损失险（车损险）

**承保范围**：自然灾害、意外事故导致的**被保险车辆本身**的损失。

**通俗说**：你撞了电线杆，自己的车修了 1.5 万，车损险赔你这笔钱。

**几个关键概念：**
- **绝对免赔额**：每次赔付需由被保险人自行承担一定比例，通常为 15%-20%（除非额外购买"不计免赔险"）
- **折旧**：旧车零件有折旧，不是按新件价格赔付

**不赔的情形（熟记！这是合同纠纷高发区）：**
- 酒驾、无证驾驶
- 超速超载
- 车辆本身质量缺陷
- 轮胎单独损坏（没有其他损坏时）
- 驾驶员故意行为

---

#### 险种二：第三者责任险（三者险）

**承保范围**：被保险车辆发生事故，导致**第三方人员伤亡或财产损失**，由保险公司赔付。

**通俗说**：你撞了别人的车，或撞伤了行人，三者险替你赔。

> 🔑 **这是商业车险最重要的险种！** 交强险的赔付上限很低（现在只有约 20 万元），远不够大事故的赔付。三者险的保额通常选 100万、200万，甚至 500万。

**保额选择建议（这是销售场景中经常遇到的问题）：**

| 城市 | 建议最低保额 | 原因 |
|------|------------|------|
| 一线城市 | 200万 | 豪车多，行人多，一旦出事赔付金额巨大 |
| 二三线城市 | 100万 | 基本能覆盖大多数事故 |
| 农村道路 | 100万 | 速度较慢，但也需足够保障 |

---

#### 险种三：车上人员责任险（座位险）

**承保范围**：车辆发生意外，车上**乘客**（含驾驶员）发生人身伤亡。

**通俗说**：你开车出了事故，车上的朋友受伤了，座位险赔你朋友。

> 💡 **常见误区**：很多人以为三者险会赔车内乘客。错！三者险只赔"第三方"，车内人员不算第三方，需要座位险单独承保。

**按座位投保**：通常以"每座限额"计算，比如"5座，每座 5 万元"。

---

#### 险种四：全车盗抢险

**承保范围**：整车被盗窃、抢劫、抢夺。

> ⚠️ **注意**：只赔整车被盗，**车内物品被盗不赔！** 车窗被砸、车内导航被盗，这些都不在全车盗抢险承保范围内。

**投保建议**：随着新能源车普及和城市安防升级，整车被盗率已大幅下降，部分车主会放弃此险种。

---

#### 险种五：自燃损失险

**承保范围**：车辆因**自身电路、油路、供油系统等非人为外因**起火燃烧造成的损失。

**重要说明**：车损险对于火灾一般有免赔，自燃损失险是对车损险的补充。

> 💡 **新能源车时代特别注意**：新能源车的电池自燃风险比传统燃油车高，自燃险对新能源车主更有价值。

---

#### 险种六：玻璃单独破碎险

**承保范围**：车辆**玻璃**（前后挡风玻璃、侧窗玻璃）**单独破碎**（即不伴随其他车身损失）。

**通俗说**：停在路边，飞来一块石子把挡风玻璃砸了，这种情况车损险不赔（因为只有玻璃坏了），但玻璃险可以赔。

**选择建议**：分为"国产玻璃"和"原厂玻璃"两个版本，原厂版保费稍高但赔付更好。

---

### 1.4 附加险体系

主险之外，附加险可以进一步补充保障。最重要的是：

| 附加险名称 | 作用 |
|----------|------|
| 不计免赔特约险 | 免除主险的绝对免赔额，事故赔付更充分 |
| 无法找到第三方特约险 | 停车场被蹭，肇事者跑了，此险赔付本车损失 |
| 驾驶人意外险 | 驾驶员本人发生意外的人身赔付（座位险不赔驾驶员时的补充） |
| 发动机涉水损失险 | 涉水行驶导致发动机进水损坏 |
| 新能源车专属附加险 | 三电系统（电池、电机、电控）保障 |

## 第二节：商业车险费率改革历程

### 2.1 改革前的问题（2015年之前）

改革前，商业车险的费率由监管部门统一制定，保险公司**不能自主定价**，竞争主要靠**高返佣**（高手续费）来抢市场，导致：

- 优质客户（不出险）和劣质客户（频繁出险）**交同样多的保费**，不公平
- 保险公司通过虚高的手续费抢客户，**市场严重亏损**
- 消费者无法感受到"不出险有奖励"的正向激励

---

### 2.2 2015年一次费改：NCD系数横空出世

**时间**：2015年6月，原银保监会启动商业车险费率改革试点。

**核心变化**：

1. **引入 NCD 系数（无赔款优待系数）**：连续不出险可以打折，出险次数多了要涨价
2. **引入自主定价系数**：保险公司可以在一定范围内自主定价
3. **从"产品导向"转变为"风险导向"**：保费应该反映真实风险

> 📖 **NCD（No Claim Discount）** 直译为"无赔款优待"，核心思想是：**你越不出险，你的保费就越便宜**。这是保险精算中"以风险定价"最直观的体现。

---

### 2.3 2020年"二次费改"：扩大自主定价权

**时间**：2020年9月，银保监会印发《关于实施车险综合改革的指导意见》。

**"二次费改"的核心变化：**

| 改革项目 | 改革前 | 改革后 |
|---------|-------|-------|
| 交强险责任限额 | 12.2万元 | 20万元 |
| 商业险附加费用率上限 | 35% | 25% |
| 自主定价系数范围 | 0.85-1.15 | 0.65-1.35 |
| 保费总体变化 | — | 平均下降约10% |
| 产品数量 | 少量 | 大幅增加 |

**最重要的变化：取消了"代理手续费上限"的单独规定，将费用并入定价系数体系中。**

> 💡 **通俗理解**：二次费改把之前的"暗补贴"变成了"明折扣"。以前保险公司私下给中介高手续费（违规），现在通过降低保费来体现竞争。消费者的保费普遍降了，优质客户的保费降得更多。

---

### 2.4 报行合一：佣金穿透管理

**时间**：2023年开始严格执行。

**什么是"报行合一"？**

保险公司向监管机构（国家金融监督管理总局）报备产品时，必须如实报告**实际支付给中介机构的手续费率**，且实际执行不能超过报备费率。

**监管目的**：杜绝"报低用高"的套利行为。以前有些保司报备手续费率10%，实际给中介支付20%，差额通过各种"费用科目"违规输送。

**对系统设计的影响（非常重要！）：**

1. 佣金计算模块必须维护**各保司、各险种的报备费率上限**
2. 结算时系统必须**自动校验**：实际计提佣金 ≤ 报备费率上限
3. 任何超出报备费率的部分，系统应预警或拦截

```java
// 伪代码示例：报行合一校验
public void validateCommissionRate(CommissionRecord record) {
    // 从数据库获取该保司、该险种的报备费率上限
    BigDecimal maxRate = policyRateRepository.findMaxRate(
        record.getInsurerId(),
        record.getInsuranceType()
    );
    
    if (record.getActualRate().compareTo(maxRate) > 0) {
        throw new BusinessException(
            ErrorCode.COMMISSION_EXCEED_LIMIT,
            String.format("佣金率 %.2f%% 超出报备上限 %.2f%%，违反报行合一要求",
                record.getActualRate().multiply(new BigDecimal("100")),
                maxRate.multiply(new BigDecimal("100"))
            )
        );
    }
}
```

## 第三节：定价公式深度拆解

### 3.1 完整定价公式

```
商业车险保费 = 基准纯风险保费 × (1 + 附加费用率) × 自主定价系数 × NCD系数
```

我们逐个拆解每一项。

---

### 3.2 基准纯风险保费（Base Pure Premium）

**定义**：精算师基于历史赔付数据测算出的、仅用于赔付的纯保费部分。

**影响因素**：
- 车辆型号（不同车型赔付率不同）
- 车辆使用年限
- 发动机排量
- 购置价格（影响车损险基准保费）

**行业实践**：各保司会定期重新测算基准纯风险保费，更新精算模型。这就是为什么**同一辆车在不同保司报价不同**的根本原因。

---

### 3.3 附加费用率（Loading Factor）

**定义**：在纯风险保费之上，加载的经营费用（运营成本、销售费用、利润等）。

**公式变形**：`毛保费 = 纯风险保费 / (1 - 附加费用率)`

**二次费改规定**：附加费用率上限从 35% 降至 **25%**。

> 💡 **附加费用率 = 保险公司的"成本空间"**。里面包含了销售费用（即中介手续费）、管理费用、利润等。改革后压缩附加费用率，就是倒逼保险公司降低运营成本和销售费用。

---

### 3.4 自主定价系数（Independent Pricing Factor）

**定义**：保险公司在监管允许范围内，根据自身经营策略自主调整的系数。

**范围**（二次费改后）：**0.65 ～ 1.35**

**意义解读**：
- 系数 = 0.65：相当于打了 65 折（最低）
- 系数 = 1.0：不打折不加价
- 系数 = 1.35：加价 35%（通常用于高风险客户）

**保司如何决定这个系数？**

这正是保险公司定价策略的核心——每家公司会根据以下因素设定内部定价模型：

- 地区风险（不同城市赔付率差异大）
- 渠道类型（直销 vs 中介渠道系数可能不同）
- 车主历史（CRM数据沉淀后的客户分层）
- 市场竞争（抢市场时压低系数，盈利期提高系数）

> 🔑 **这就是为什么同一辆车同一条件，在不同保司的报价不同！** 每家保司的自主定价系数策略各不相同。

---

### 3.5 NCD系数（No Claim Discount，无赔款优待系数）

这是整个车险定价体系中最精妙的设计，也是系统实现最复杂的部分。

#### 3.5.1 NCD系数标准表

根据中国银保监会的规定（各地略有差异）：

| 上一年度理赔情况 | NCD系数 |
|---------------|---------|
| 连续5年及以上未出险 | 0.60（打六折）|
| 连续4年未出险 | 0.65 |
| 连续3年未出险 | 0.70 |
| 连续2年未出险 | 0.75 |
| 连续1年未出险 | 0.85 |
| 当年未出险（初次投保） | 1.00 |
| 出险1次 | 1.00（不优惠） |
| 出险2次 | 1.25（涨25%） |
| 出险3次 | 1.50（涨50%） |
| 出险4次及以上 | 1.75（涨75%） |

> 📌 **注意**：不同地区、不同保司的 NCD 表略有差异，上表是通用参考值。系统开发时应支持**按保司、按地区配置 NCD 系数表**，而非硬编码。

#### 3.5.2 NCD 系数的计算规则

**关键规则 1：只统计商业险理赔，交强险理赔单独计算**

商业险的 NCD 只看商业险出险记录，交强险有自己独立的费率浮动规则。

**关键规则 2：出险次数统计口径**

- 统计**上一保险年度**的出险次数
- 单次事故算 1 次，无论赔付多少钱
- 只有**获得赔付**的才算出险，报案后撤案不算

**关键规则 3：NCD 连续计算**

- 连续不出险年数是**累积**的
- 一旦出险，**重置**，需要重新积累
- 例：连续 4 年未出险（NCD=0.65），第 5 年出险 1 次，第 6 年 NCD = 1.00，而不是 0.65×1.00

#### 3.5.3 实际计算案例

**案例：一辆市场价 30 万的家用轿车，三者险保额 100 万**

> 假设条件：
> - 车损险基准保费：5000 元
> - 三者险（100万）基准保费：3000 元
> - 附加费用率：25%
> - 自主定价系数：0.85（某保司对该地区优质客户）
> - 该车主连续 3 年未出险（NCD = 0.70）

**计算过程：**

```
车损险保费 = 5000 × (1 + 25%) × 0.85 × 0.70
           = 5000 × 1.25 × 0.85 × 0.70
           = 5000 × 0.74375
           = 3718.75 元

三者险保费 = 3000 × (1 + 25%) × 0.85 × 0.70
           = 3000 × 1.25 × 0.85 × 0.70
           = 3000 × 0.74375
           = 2231.25 元

商业险合计 = 3718.75 + 2231.25 = 5950 元
交强险 = 950 × 0.7（连续3年未出险）= 665 元

总保费 ≈ 5950 + 665 = 6615 元
```

如果这位车主第 4 年出了 1 次险，则下一年：

```
NCD系数 = 1.00（不优惠，因为出险了）
连续未出险年数归零

车损险保费 = 5000 × 1.25 × 0.85 × 1.00 = 5312.5 元
三者险保费 = 3000 × 1.25 × 0.85 × 1.00 = 3187.5 元
商业险合计 = 5312.5 + 3187.5 = 8500 元

和之前相比多付了 8500 - 5950 = 2550 元！
```

> 💡 **这个案例告诉车主**：一次小额事故（比如 500 元），如果自己掏钱修，下一年省下来的保费 2550 元远大于这次修车费用，所以很多有经验的老司机会选择"小额事故不报险"。这个心理效应也是 NCD 设计的精妙之处——减少小额理赔，降低整体赔付率。

### 3.6 定价公式完整 Java 实现

以下是一个可以运行的完整示例，模拟商业车险报价计算引擎核心逻辑：

```java
import java.math.RoundingMode;
import java.util.ArrayList;
import java.util.List;

/**
 * 商业车险保费计算引擎 - 核心计算逻辑示例
 * 
 * 公式：保费 = 基准纯风险保费 × (1 + 附加费用率) × 自主定价系数 × NCD系数
 */
public class CarInsuranceCalculator {

    // ============================================================
    // 1. 数据模型
    // ============================================================

    /**
     * 报价请求参数
     */
    public static class QuoteRequest {
        /** 车辆型号编码（用于查询基准纯风险保费） */

        /** 三者险保额（万元） */

        /** 座位数 */
        private int seats;

        private BigDecimal perSeatAmount;
        /** 上年度商业险出险次数 */
        private int lastYearClaimCount;
        /** 连续未出险年数（0=本年度首次投保或去年出险） */
        private int consecutiveNoClaimYears;
        /** 是否需要不计免赔险 */
        private boolean needNonDeductibleCoverage;
        /** 是否需要全车盗抢险 */
        private boolean needTheftCoverage;
        /** 是否需要自燃险 */
        private boolean needSpontaneousCombustion;
        /** 是否需要玻璃险 */
        private boolean needGlassCoverage;

        // Getters and Setters
        public String getVehicleModelCode() { return vehicleModelCode; }
        public void setVehicleModelCode(String v) { this.vehicleModelCode = v; }
        public BigDecimal getPurchasePrice() { return purchasePrice; }
        public void setPurchasePrice(BigDecimal v) { this.purchasePrice = v; }
        public int getUseYears() { return useYears; }
        public void setUseYears(int v) { this.useYears = v; }
        public BigDecimal getThirdPartyLiabilityAmount() { return thirdPartyLiabilityAmount; }
        public void setThirdPartyLiabilityAmount(BigDecimal v) { this.thirdPartyLiabilityAmount = v; }
        public int getSeats() { return seats; }
        public void setSeats(int v) { this.seats = v; }
        public BigDecimal getPerSeatAmount() { return perSeatAmount; }
        public void setPerSeatAmount(BigDecimal v) { this.perSeatAmount = v; }
        public int getLastYearClaimCount() { return lastYearClaimCount; }
        public void setLastYearClaimCount(int v) { this.lastYearClaimCount = v; }
        public int getConsecutiveNoClaimYears() { return consecutiveNoClaimYears; }
        public void setConsecutiveNoClaimYears(int v) { this.consecutiveNoClaimYears = v; }
        public boolean isNeedNonDeductibleCoverage() { return needNonDeductibleCoverage; }
        public void setNeedNonDeductibleCoverage(boolean v) { this.needNonDeductibleCoverage = v; }
        public boolean isNeedTheftCoverage() { return needTheftCoverage; }
        public void setNeedTheftCoverage(boolean v) { this.needTheftCoverage = v; }
        public boolean isNeedSpontaneousCombustion() { return needSpontaneousCombustion; }
        public void setNeedSpontaneousCombustion(boolean v) { this.needSpontaneousCombustion = v; }
        public boolean isNeedGlassCoverage() { return needGlassCoverage; }
        public void setNeedGlassCoverage(boolean v) { this.needGlassCoverage = v; }
    }

    /**
     * 单项险种保费明细
     */
    public static class CoverageDetail {
        private String coverageCode;   // 险种编码
        private String coverageName;   // 险种名称
        private BigDecimal basePremium; // 基准纯风险保费
        private BigDecimal loadedPremium; // 加载附加费用后
        private BigDecimal finalPremium;  // 最终保费（含所有系数）

        public CoverageDetail(String code, String name, BigDecimal base, 
                               BigDecimal loaded, BigDecimal fin) {
            this.coverageCode = code;
            this.coverageName = name;
            this.basePremium = base;
            this.loadedPremium = loaded;
            this.finalPremium = fin;
        }

        @Override
        public String toString() {
            return String.format("%-20s | 基准:%8.2f | 加载后:%8.2f | 最终:%8.2f",
                coverageName, basePremium, loadedPremium, finalPremium);
        }
    }

    /**
     * 报价结果
     */
    public static class QuoteResult {
        private BigDecimal ncdFactor;           // NCD系数
        private BigDecimal independentFactor;   // 自主定价系数
        private BigDecimal loadingRate;         // 附加费用率
        private List<CoverageDetail> details = new ArrayList<>();
        private BigDecimal commercialTotal;     // 商业险合计
        private BigDecimal compulsoryPremium;   // 交强险
        private BigDecimal totalPremium;        // 合计保费

        // Getters and Setters (simplified)
        public void setNcdFactor(BigDecimal v) { this.ncdFactor = v; }
        public void setIndependentFactor(BigDecimal v) { this.independentFactor = v; }
        public void setLoadingRate(BigDecimal v) { this.loadingRate = v; }
        public void addDetail(CoverageDetail d) { this.details.add(d); }
        public void setCommercialTotal(BigDecimal v) { this.commercialTotal = v; }
        public void setCompulsoryPremium(BigDecimal v) { this.compulsoryPremium = v; }
        public void setTotalPremium(BigDecimal v) { this.totalPremium = v; }

        public void print() {
            System.out.println("╔══════════════════════════════════════════════════════════╗");
            System.out.println("║                    车险报价结果                          ║");
            System.out.println("╠══════════════════════════════════════════════════════════╣");
            System.out.printf("║ NCD系数: %-10.2f  自主定价系数: %-10.2f              ║%n",
                ncdFactor, independentFactor);
            System.out.printf("║ 附加费用率: %.0f%%                                          ║%n",
                loadingRate.multiply(BigDecimal.valueOf(100)));
            System.out.println("╠══════════════════════════════════════════════════════════╣");
            System.out.println("║ 险种明细                                                 ║");
            for (CoverageDetail d : details) {
                System.out.println("║  " + d);
            }
            System.out.println("╠══════════════════════════════════════════════════════════╣");
            System.out.printf("║ 商业险合计: %10.2f 元                                  ║%n", commercialTotal);
            System.out.printf("║ 交强险:     %10.2f 元                                  ║%n", compulsoryPremium);
            System.out.printf("║ 【总保费】: %10.2f 元                                  ║%n", totalPremium);
            System.out.println("╚══════════════════════════════════════════════════════════╝");
        }
    }

    // ============================================================
    // 2. NCD系数计算
    // ============================================================

    /**
     * 计算NCD系数
     * 
     * @param lastYearClaimCount      上年度出险次数
     * @param consecutiveNoClaimYears 连续未出险年数（0代表去年出险或首次投保）
     * @return NCD系数
     */
    public BigDecimal calculateNcdFactor(int lastYearClaimCount, int consecutiveNoClaimYears) {
        // 如果去年出险，先按出险次数计算上浮
        if (lastYearClaimCount > 0) {
            switch (lastYearClaimCount) {
                case 1: return new BigDecimal("1.00");
                case 2: return new BigDecimal("1.25");
                case 3: return new BigDecimal("1.50");
                default: return new BigDecimal("1.75"); // 4次及以上
            }
        }

        // 去年未出险，按连续未出险年数给予优惠
        switch (consecutiveNoClaimYears) {
            case 0: return new BigDecimal("1.00"); // 首次投保或刚出完险
            case 1: return new BigDecimal("0.85");
            case 2: return new BigDecimal("0.75");
            case 3: return new BigDecimal("0.70");
            case 4: return new BigDecimal("0.65");
            default: return new BigDecimal("0.60"); // 5年及以上，最大优惠
        }
    }

    // ============================================================
    // 3. 基准纯风险保费计算（简化版，实际需查精算费率表）
    // ============================================================

    /**
     * 计算车损险基准纯风险保费
     * 实际项目中，这里应查询数据库中的精算费率表（按车型、排量等多维度定价）
     * 这里用简化公式演示
     */
    private BigDecimal calcVehicleLossBasePremium(BigDecimal purchasePrice, int useYears) {
        // 简化公式：基准保费 = 购置价 × 基准费率
        // 基准费率随使用年限递增（旧车零件贵，维修难度大）
        BigDecimal baseRate;
        if (useYears <= 1) {
            baseRate = new BigDecimal("0.0155"); // 1.55%
        } else if (useYears <= 3) {
            baseRate = new BigDecimal("0.0165"); // 1.65%
        } else if (useYears <= 5) {
            baseRate = new BigDecimal("0.0175"); // 1.75%
        } else {
            baseRate = new BigDecimal("0.0185"); // 1.85%
        }
        // 购置价单位为万元，转换为元
        return purchasePrice.multiply(new BigDecimal("10000")).multiply(baseRate)
                            .setScale(2, RoundingMode.HALF_UP);
    }

    /**
     * 计算三者险基准纯风险保费（按保额查费率表）
     */
    private BigDecimal calcThirdPartyBasePremium(BigDecimal liabilityAmountWan) {
        // 简化的保额-费率对照（实际按地区、车型有差异）
        int amount = liabilityAmountWan.intValue();
        if (amount <= 20)       return new BigDecimal("1050.00");
        else if (amount <= 50)  return new BigDecimal("1680.00");
        else if (amount <= 100) return new BigDecimal("2450.00");
        else if (amount <= 150) return new BigDecimal("2900.00");
        else if (amount <= 200) return new BigDecimal("3300.00");
        else if (amount <= 300) return new BigDecimal("3780.00");
        else if (amount <= 500) return new BigDecimal("4400.00");
        else                    return new BigDecimal("5200.00");
    }

    /**
     * 计算座位险基准纯风险保费
     */
    private BigDecimal calcPassengerBasePremium(int seats, BigDecimal perSeatAmountWan) {
        // 每座基准保费 = 每座限额 × 费率
        BigDecimal ratePerSeat = new BigDecimal("0.012"); // 1.2% per seat
        BigDecimal perSeatBase = perSeatAmountWan.multiply(new BigDecimal("10000"))
                                                  .multiply(ratePerSeat)
                                                  .setScale(2, RoundingMode.HALF_UP);
        return perSeatBase.multiply(new BigDecimal(seats));
    }

    // ============================================================
    // 4. 核心计算入口
    // ============================================================

    /**
     * 核心报价计算方法
     * 
     * @param request       报价请求
     * @param loadingRate   附加费用率（由保司传入，二次费改后上限25%）
     * @param indepFactor   自主定价系数（由保司传入，0.65-1.35）
     */
    public QuoteResult calculate(QuoteRequest request, BigDecimal loadingRate, BigDecimal indepFactor) {
        QuoteResult result = new QuoteResult();
        result.setLoadingRate(loadingRate);
        result.setIndependentFactor(indepFactor);

        // 1. 计算NCD系数
        BigDecimal ncdFactor = calculateNcdFactor(
            request.getLastYearClaimCount(),
            request.getConsecutiveNoClaimYears()
        );
        result.setNcdFactor(ncdFactor);

        // 2. 综合系数 = (1 + 附加费用率) × 自主定价系数 × NCD系数
        BigDecimal comprehensiveFactor = BigDecimal.ONE.add(loadingRate)
                                                        .multiply(indepFactor)
                                                        .multiply(ncdFactor)
                                                        .setScale(4, RoundingMode.HALF_UP);

        BigDecimal commercialTotal = BigDecimal.ZERO;

        // 3. 计算各险种保费
        // 3.1 车损险
        BigDecimal vlBase = calcVehicleLossBasePremium(request.getPurchasePrice(), request.getUseYears());
        BigDecimal vlFinal = vlBase.multiply(comprehensiveFactor).setScale(2, RoundingMode.HALF_UP);
        BigDecimal vlLoaded = vlBase.multiply(BigDecimal.ONE.add(loadingRate)).setScale(2, RoundingMode.HALF_UP);
        result.addDetail(new CoverageDetail("VL", "机动车损失险", vlBase, vlLoaded, vlFinal));
        commercialTotal = commercialTotal.add(vlFinal);

        // 3.2 三者险
        BigDecimal tplBase = calcThirdPartyBasePremium(request.getThirdPartyLiabilityAmount());
        BigDecimal tplFinal = tplBase.multiply(comprehensiveFactor).setScale(2, RoundingMode.HALF_UP);
        BigDecimal tplLoaded = tplBase.multiply(BigDecimal.ONE.add(loadingRate)).setScale(2, RoundingMode.HALF_UP);
        result.addDetail(new CoverageDetail("TPL", 
            "第三者责任险(" + request.getThirdPartyLiabilityAmount().intValue() + "万)", 
            tplBase, tplLoaded, tplFinal));
        commercialTotal = commercialTotal.add(tplFinal);

        // 3.3 座位险（如果选了的话，这里默认都加）
        BigDecimal pasBase = calcPassengerBasePremium(request.getSeats(), request.getPerSeatAmount());
        BigDecimal pasFinal = pasBase.multiply(comprehensiveFactor).setScale(2, RoundingMode.HALF_UP);
        BigDecimal pasLoaded = pasBase.multiply(BigDecimal.ONE.add(loadingRate)).setScale(2, RoundingMode.HALF_UP);
        result.addDetail(new CoverageDetail("PAL", 
            "车上人员责任险(" + request.getSeats() + "座×" + request.getPerSeatAmount().intValue() + "万)", 
            pasBase, pasLoaded, pasFinal));
        commercialTotal = commercialTotal.add(pasFinal);

        // 3.4 不计免赔险（约为主险保费的15%）
        if (request.isNeedNonDeductibleCoverage()) {
            BigDecimal ndBase = vlFinal.add(tplFinal).multiply(new BigDecimal("0.15"))
                                      .setScale(2, RoundingMode.HALF_UP);
            result.addDetail(new CoverageDetail("ND", "不计免赔特约险", ndBase, ndBase, ndBase));
            commercialTotal = commercialTotal.add(ndBase);
        }

        // 3.5 全车盗抢险
        if (request.isNeedTheftCoverage()) {
            BigDecimal theftBase = request.getPurchasePrice().multiply(new BigDecimal("10000"))
                                          .multiply(new BigDecimal("0.0035"))
                                          .setScale(2, RoundingMode.HALF_UP);
            BigDecimal theftFinal = theftBase.multiply(comprehensiveFactor).setScale(2, RoundingMode.HALF_UP);
            result.addDetail(new CoverageDetail("THEFT", "全车盗抢险", theftBase, theftBase, theftFinal));
            commercialTotal = commercialTotal.add(theftFinal);
        }

        // 3.6 自燃险
        if (request.isNeedSpontaneousCombustion()) {
            BigDecimal scBase = new BigDecimal("320.00");
            BigDecimal scFinal = scBase.multiply(comprehensiveFactor).setScale(2, RoundingMode.HALF_UP);
            result.addDetail(new CoverageDetail("SC", "自燃损失险", scBase, scBase, scFinal));
            commercialTotal = commercialTotal.add(scFinal);
        }

        // 3.7 玻璃险
        if (request.isNeedGlassCoverage()) {
            BigDecimal glassBase = new BigDecimal("280.00");
            result.addDetail(new CoverageDetail("GLASS", "玻璃单独破碎险(原厂)", glassBase, glassBase, glassBase));
            commercialTotal = commercialTotal.add(glassBase);
        }

        // 4. 计算交强险（简化：使用连续未出险年数）
        BigDecimal compulsoryBase = new BigDecimal("950.00");
        BigDecimal compulsoryNcd;
        switch (request.getConsecutiveNoClaimYears()) {
            case 0: compulsoryNcd = BigDecimal.ONE; break;
            case 1: compulsoryNcd = new BigDecimal("1.10"); break; // 去年出险示例
            default: 
                if (request.getConsecutiveNoClaimYears() >= 3) {
                    compulsoryNcd = new BigDecimal("0.70");
                } else if (request.getConsecutiveNoClaimYears() == 2) {
                    compulsoryNcd = new BigDecimal("0.85");
                } else {
                    compulsoryNcd = BigDecimal.ONE;
                }
        }
        // 如果去年出险则上浮
        if (request.getLastYearClaimCount() == 1) {
            compulsoryNcd = new BigDecimal("1.10");
        } else if (request.getLastYearClaimCount() == 2) {
            compulsoryNcd = new BigDecimal("1.30");
        } else if (request.getLastYearClaimCount() >= 3) {
            compulsoryNcd = new BigDecimal("1.50");
        }
        BigDecimal compulsoryFinal = compulsoryBase.multiply(compulsoryNcd).setScale(2, RoundingMode.HALF_UP);

        result.setCommercialTotal(commercialTotal);
        result.setCompulsoryPremium(compulsoryFinal);
        result.setTotalPremium(commercialTotal.add(compulsoryFinal));

        return result;
    }

    // ============================================================
    // 5. main方法：运行示例
    // ============================================================

    public static void main(String[] args) {
        CarInsuranceCalculator calculator = new CarInsuranceCalculator();

        // 构建报价请求：30万轿车，使用3年，三者险100万，5座，连续3年未出险
        QuoteRequest request = new QuoteRequest();
        request.setVehicleModelCode("CAR_SEDAN_3000CC");
        request.setPurchasePrice(new BigDecimal("30"));   // 30万
        request.setUseYears(3);
        request.setThirdPartyLiabilityAmount(new BigDecimal("100")); // 100万三者险

        request.setPerSeatAmount(new BigDecimal("5")); // 每座5万
        request.setLastYearClaimCount(0);              // 去年未出险
        request.setConsecutiveNoClaimYears(3);         // 连续3年未出险
        request.setNeedNonDeductibleCoverage(true);
        request.setNeedTheftCoverage(false);
        request.setNeedSpontaneousCombustion(false);
        request.setNeedGlassCoverage(true);

        System.out.println("\n========== 保司A 报价 ==========");

        QuoteResult resultA = calculator.calculate(request, 
            new BigDecimal("0.25"), new BigDecimal("0.85"));
        resultA.print();

        System.out.println("\n========== 保司B 报价 ==========");

            new BigDecimal("0.22"), new BigDecimal("0.90"));
        resultB.print();

        System.out.println("\n========== 场景对比：若去年出险1次 ==========");
        request.setLastYearClaimCount(1);
        request.setConsecutiveNoClaimYears(0);

            new BigDecimal("0.25"), new BigDecimal("0.85"));
        resultClaim.print();
    }
```

## 第四节：承保与核保——高风险车辆的拒保逻辑

### 4.1 核保（Underwriting）是什么？

核保是保险公司在承保前对被保险标的进行风险评估的过程。简单说：**保险公司要决定"这个风险我愿不愿意保，如果保要收多少钱"。**

### 4.2 车险核保的主要维度

| 核保维度 | 判断要素 | 系统处理方式 |
|---------|---------|------------|
| 车辆性质 | 家用 / 营运 / 特种车辆 | 投保页面强制选择，营运车辆费率另计 |
| 车辆改装 | 是否改装发动机、车身结构 | 用户填报 + 行驶证OCR核验 |
| 使用地区 | 车辆注册地/主要使用地 | 不同地区赔付率差异大 |
| 驾驶人情况 | 主要驾驶人年龄、驾龄、出险记录 | 与保司数据库对接查询 |
| 车龄 | 超过一定年限的旧车 | 系统自动计算，达到阈值提示 |
| 历史出险 | 连续多年高频出险 | NCD系数体现 + 特殊标记 |

### 4.3 常见拒保场景（系统必须处理）

```java
/**
 * 车险承保资格校验
 */
public class CarInsuranceUnderwritingChecker {

    /**
     * 承保资格校验结果
     */
    public enum CheckResult {
        ACCEPTED,           // 正常承保
        ACCEPT_WITH_NOTE,   // 可以承保，但需要注意
        REFER_TO_MANUAL,    // 需要人工核保
        DECLINED            // 拒保
    }

    public static class UnderwritingDecision {
        private CheckResult result;
        private String reason;
        private String code;

        public UnderwritingDecision(CheckResult result, String code, String reason) {
            this.result = result;
            this.code = code;
            this.reason = reason;
        }

        @Override
        public String toString() {
            return String.format("[%s] 代码:%s - %s", result, code, reason);
        }
    }

    /**
     * 执行核保校验
     */
    public UnderwritingDecision check(CarInfo carInfo) {
        // 规则1：营运车辆（出租车、网约车、货车等）——拒保商业险主险
        if (carInfo.isCommercialUse() && !carInfo.isCommercialPolicyAvailable()) {
            return new UnderwritingDecision(CheckResult.DECLINED, "UW_001",
                "营运车辆不在本保司承保范围内，建议联系专属营运车险渠道");
        }

        // 规则2：车龄超过15年——拒保
        if (carInfo.getVehicleAgeYears() > 15) {
            return new UnderwritingDecision(CheckResult.DECLINED, "UW_002",
                "车辆使用年限超过15年，无法提供商业险报价");
        }

        // 规则3：已改装车辆（非合规改装）——人工核保
        if (carInfo.isModified()) {
            return new UnderwritingDecision(CheckResult.REFER_TO_MANUAL, "UW_003",
                "车辆存在改装记录，需提交改装合格证书，由人工核保审核");
        }

        // 规则4：车辆已被抵押且抵押人未同意——拒保（风险转移问题）
        if (carInfo.isMortgaged() && !carInfo.isMortgageeConsented()) {
            return new UnderwritingDecision(CheckResult.DECLINED, "UW_004",
                "车辆抵押状态下需抵押权人书面同意方可投保");
        }

        // 规则5：车辆已报废或被盗
        if (carInfo.isWrittenOff() || carInfo.isStolenReported()) {
            return new UnderwritingDecision(CheckResult.DECLINED, "UW_005",
                "车辆处于报废或盗窃报案状态，无法承保");
        }

        // 规则6：上年度出险超过5次——人工核保
        if (carInfo.getLastYearClaimCount() >= 5) {
            return new UnderwritingDecision(CheckResult.REFER_TO_MANUAL, "UW_006",
                "上年度出险次数过多（" + carInfo.getLastYearClaimCount() + "次），需人工评估承保风险");
        }

        // 规则7：高价值车辆（超过200万）——人工核保
        if (carInfo.getPurchasePrice().compareTo(new BigDecimal("200")) > 0) {
            return new UnderwritingDecision(CheckResult.REFER_TO_MANUAL, "UW_007",
                "豪华车辆（购置价超200万），需人工核保确认承保条件");
        }

        return new UnderwritingDecision(CheckResult.ACCEPTED, "OK", "核保通过，可正常承保");
    }
```

## 第五节：对系统设计的影响——业务驱动架构

这是本篇最重要的一节，也是从"学保险知识"到"设计保险系统"的关键桥梁。

### 5.1 为什么需要并发请求多家保司？

**业务背景**：

在中介平台的使用场景中，一个业务员为客户报价时，需要对比**多家保险公司**的价格，给客户选择最合适的方案。

**这带来了技术挑战：**

| 挑战 | 原因 | 解决方案 |
|------|------|---------|
| 每家保司接口不同 | 各家保司的API协议、参数格式、鉴权方式各异 | 适配器模式（Adapter Pattern）统一封装 |
| 响应速度慢 | 保司API平均响应时间 2-8 秒 | 并发调用，设置超时，聚合结果 |
| 部分保司不响应 | API宕机、网络超时、限流 | 降级处理，返回已有结果 |
| 结果格式不统一 | 各保司返回字段名称、格式不同 | 统一结果模型 + 结果转换层 |

**系统架构设计：**

```
                    ┌─────────────────────────────┐
                    │        报价聚合服务           │
                    │    (ins-quote microservice)  │
                    └──────────────┬──────────────┘
                                   │ 并发调用
              ┌───────────┬────────┴────────┬───────────┐
              ▼           ▼                 ▼           ▼
        ┌──────────┐ ┌──────────┐   ┌──────────┐ ┌──────────┐
        │ 平安保险  │ │ 人保财险 │   │ 太平洋   │ │ 国寿财险 │
        │ Adapter  │ │ Adapter  │   │ Adapter  │ │ Adapter  │
        └──────────┘ └──────────┘   └──────────┘ └──────────┘
              │           │                 │           │
              ▼           ▼                 ▼           ▼
        平安API      人保API           太平洋API    国寿API
```

**核心代码示例：并发报价聚合**

```java
package com.example.insurance.quote;

import java.util.*;
import java.util.concurrent.*;
import java.math.BigDecimal;

/**
 * 多保司并发报价聚合器
 */
public class MultiInsurerQuoteAggregator {

    private final ExecutorService executor;
    private final Map<String, InsurerQuoteAdapter> adapters;
    
    // 单次报价请求的超时时间（毫秒）
    private static final long QUOTE_TIMEOUT_MS = 5000;

    public MultiInsurerQuoteAggregator() {
        // 使用有界线程池，避免保司接口慢时线程耗尽
        this.executor = new ThreadPoolExecutor(
            10, 30, 60, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(100),
            new ThreadFactory() {
                int count = 0;
                public Thread newThread(Runnable r) {
                    return new Thread(r, "insurer-quote-" + (++count));
                }
            },
            new ThreadPoolExecutor.CallerRunsPolicy()
        );
        this.adapters = initAdapters();
    }

    private Map<String, InsurerQuoteAdapter> initAdapters() {
        Map<String, InsurerQuoteAdapter> map = new LinkedHashMap<>();
        map.put("PICC",    new MockInsurerAdapter("人保财险", 800, new BigDecimal("0.87")));
        map.put("CPIC",    new MockInsurerAdapter("太平洋保险", 1200, new BigDecimal("0.82")));
        map.put("PING_AN", new MockInsurerAdapter("平安产险", 600, new BigDecimal("0.90")));
        map.put("LIFAN",   new MockInsurerAdapter("利安保险", 3000, null)); // 模拟超时
        map.put("NCI",     new MockInsurerAdapter("国寿财险", 900, new BigDecimal("0.85")));
        return map;
    }

    /**
     * 并发向多家保司发起报价请求，聚合返回结果
     */
    public AggregatedQuoteResult aggregateQuotes(CarQuoteRequest request) {
        long startTime = System.currentTimeMillis();

        // 创建CompletableFuture列表
        List<CompletableFuture<InsurerQuoteResult>> futures = new ArrayList<>();
        
        for (Map.Entry<String, InsurerQuoteAdapter> entry : adapters.entrySet()) {
            String insurerCode = entry.getKey();
            InsurerQuoteAdapter adapter = entry.getValue();
            
            CompletableFuture<InsurerQuoteResult> future = CompletableFuture
                .supplyAsync(() -> {
                    try {
                        return adapter.getQuote(insurerCode, request);
                    } catch (Exception e) {
                        // 单家保司失败不影响整体
                        return InsurerQuoteResult.failed(insurerCode, e.getMessage());
                    }
                }, executor)
                .orTimeout(QUOTE_TIMEOUT_MS, TimeUnit.MILLISECONDS)
                .exceptionally(ex -> {
                    String reason = ex.getCause() instanceof TimeoutException 
                        ? "接口超时（>" + QUOTE_TIMEOUT_MS + "ms）" 
                        : ex.getMessage();
                    return InsurerQuoteResult.failed(insurerCode, reason);
                });
            
            futures.add(future);
        }

        // 等待所有保司响应（或超时）
        CompletableFuture<Void> allDone = CompletableFuture.allOf(
            futures.toArray(new CompletableFuture[0])
        );
        
        try {
            allDone.get(QUOTE_TIMEOUT_MS + 500, TimeUnit.MILLISECONDS);
        } catch (Exception e) {
            // 整体超时，已完成的保司结果依然可用
        }

        // 收集结果
        List<InsurerQuoteResult> results = new ArrayList<>();
        for (CompletableFuture<InsurerQuoteResult> f : futures) {
            if (f.isDone() && !f.isCompletedExceptionally()) {
                try {
                    results.add(f.get());
                } catch (Exception ignored) {}
            }
        }

        long elapsed = System.currentTimeMillis() - startTime;
        
        return new AggregatedQuoteResult(results, elapsed);
    }

    // ============================================================
    // 内部数据模型和适配器
    // ============================================================

    /** 报价请求 */
    public static class CarQuoteRequest {
        String plateNumber;
        String vehicleModel;
        int consecutiveNoClaimYears;
        // ... 其他字段
    }

    /** 单家保司报价结果 */
    public static class InsurerQuoteResult {
        String insurerCode;
        String insurerName;
        boolean success;
        BigDecimal totalPremium;
        BigDecimal commercialPremium;
        BigDecimal compulsoryPremium;
        long responseTimeMs;
        String failReason;

        public static InsurerQuoteResult failed(String code, String reason) {
            InsurerQuoteResult r = new InsurerQuoteResult();
            r.insurerCode = code;
            r.success = false;
            r.failReason = reason;
            return r;
        }
    }

    /** 聚合报价结果 */
    public static class AggregatedQuoteResult {
        List<InsurerQuoteResult> results;
        long totalElapsedMs;
        int successCount;
        int failCount;

        public AggregatedQuoteResult(List<InsurerQuoteResult> results, long elapsed) {
            this.results = results;
            this.totalElapsedMs = elapsed;
            this.successCount = (int) results.stream().filter(r -> r.success).count();
            this.failCount = results.size() - successCount;
        }

        public void print() {
            System.out.println("\n====== 聚合报价结果 ======");
            System.out.printf("总耗时: %dms | 成功: %d家 | 失败/超时: %d家%n%n",
                totalElapsedMs, successCount, failCount);
            
            // 按总保费升序排列（成功的在前）
            results.sort((a, b) -> {
                if (a.success && b.success) return a.totalPremium.compareTo(b.totalPremium);
                if (a.success) return -1;
                return 1;
            });

            System.out.printf("%-10s %-14s %12s %10s %10s%n",
                "保司代码", "保司名称", "总保费(元)", "商业险", "交强险");
            System.out.println("-".repeat(60));
            
            for (InsurerQuoteResult r : results) {
                if (r.success) {
                    System.out.printf("%-10s %-14s %12.2f %10.2f %10.2f%n",
                        r.insurerCode, r.insurerName, 
                        r.totalPremium, r.commercialPremium, r.compulsoryPremium);
                } else {
                    System.out.printf("%-10s %-14s %12s%n",
                        r.insurerCode, "（" + r.failReason + "）", "—");
                }
            }
        }
    }

    /** 适配器接口 */
    interface InsurerQuoteAdapter {
        InsurerQuoteResult getQuote(String insurerCode, CarQuoteRequest request);
    }

    /** Mock适配器（模拟保司API响应） */
    static class MockInsurerAdapter implements InsurerQuoteAdapter {
        private final String name;
        private final long mockDelayMs;
        private final BigDecimal factor; // null表示接口不可用

        MockInsurerAdapter(String name, long delayMs, BigDecimal factor) {
            this.name = name;
            this.mockDelayMs = delayMs;
            this.factor = factor;
        }

        @Override
        public InsurerQuoteResult getQuote(String code, CarQuoteRequest req) {
            try {
                Thread.sleep(mockDelayMs);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            
            if (factor == null) {
                throw new RuntimeException("保司API服务不可用");
            }

            InsurerQuoteResult result = new InsurerQuoteResult();
            result.insurerCode = code;
            result.insurerName = name;
            result.success = true;
            result.responseTimeMs = mockDelayMs;
            
            // 模拟报价计算
            BigDecimal commercial = new BigDecimal("8000").multiply(factor);
            result.commercialPremium = commercial;
            result.compulsoryPremium = new BigDecimal("665");
            result.totalPremium = commercial.add(result.compulsoryPremium);
            return result;
        }
    }

    public static void main(String[] args) throws InterruptedException {
        MultiInsurerQuoteAggregator aggregator = new MultiInsurerQuoteAggregator();
        
        CarQuoteRequest request = new CarQuoteRequest();
        request.plateNumber = "沪A12345";
        request.consecutiveNoClaimYears = 3;
        
        System.out.println("开始向5家保司并发请求报价...");
        AggregatedQuoteResult result = aggregator.aggregateQuotes(request);
        result.print();
        
        aggregator.executor.shutdown();
    }
}
```

### 5.2 为什么费率每天都可能变化？

这是很多初次接触保险系统的开发者感到困惑的问题：**为什么报价不能简单缓存？**

**原因分析：**

| 变化来源 | 变化频率 | 举例 |
|---------|---------|------|
| 保司调整自主定价系数 | 随时（有时每天） | 年底冲业绩，临时降系数抢市场 |
| 特定地区赔付率超标 | 月度 | 某城市暴雨季，暂停或提高该地区报价 |
| 险种停售或新推出 | 不定期 | 新能源车专属附加险上线 |
| 渠道手续费调整 | 月度 | 保司按月调整中介渠道手续费 |
| 精算模型更新 | 年度 | 年度精算重测，基准纯风险保费更新 |
| 监管政策变化 | 不定期 | 报行合一新规，系数范围调整 |

**系统设计应对方案：**

```java
/**
 * 报价缓存策略示例
 * 
 * 不能永久缓存，但也不能每次都实时调用保司API（太慢）
 * 采用短时间TTL缓存 + 智能失效策略
 */
public class QuoteCacheStrategy {
    
    // 不同场景的缓存TTL
    private static final int REALTIME_QUOTE_TTL_SECONDS = 0;     // 实时报价：不缓存
    private static final int COMPARISON_QUOTE_TTL_SECONDS = 300; // 比价场景：缓存5分钟
    private static final int RENEW_REMINDER_TTL_SECONDS = 3600;  // 续保提醒：缓存1小时

    /**
     * 获取报价（带缓存）
     * 
     * @param request     报价请求
     * @param scene       使用场景
     * @param cacheClient Redis客户端（伪代码）
     */
    public AggregatedQuoteResult getQuoteWithCache(
            CarQuoteRequest request, 
            QuoteScene scene,
            Object cacheClient) {
        
        int ttl = getTtl(scene);
        
        if (ttl == 0) {
            // 实时场景：直接调用保司API
            return callInsurerApis(request);
        }
        
        // 生成缓存Key（包含影响报价的所有关键因素）
        String cacheKey = buildCacheKey(request);
        
        // 尝试从缓存读取
        AggregatedQuoteResult cached = getFromCache(cacheClient, cacheKey);
        if (cached != null) {
            cached.setFromCache(true);
            return cached;
        }
        
        // 缓存未命中，调用保司API
        AggregatedQuoteResult result = callInsurerApis(request);
        
        // 写入缓存（仅当所有保司都有响应时才缓存）
        if (result.getSuccessCount() == result.getTotalCount()) {
            putToCache(cacheClient, cacheKey, result, ttl);
        }
        
        return result;
    }

    /**
     * 生成缓存Key的重要性：
     * Key必须包含所有影响报价的因素，否则可能返回错误的缓存数据
     */
    private String buildCacheKey(CarQuoteRequest request) {
        return String.format("quote:car:%s:%d:%s:%s",
            request.plateNumber,
            request.consecutiveNoClaimYears,
            request.vehicleModel,
            // 注意：同一辆车今天和明天的报价可能不同，需要加日期
            java.time.LocalDate.now().toString()
        );
    }

    private int getTtl(QuoteScene scene) {
        switch (scene) {
            case REALTIME_INVEST: return REALTIME_QUOTE_TTL_SECONDS;
            case PRICE_COMPARISON: return COMPARISON_QUOTE_TTL_SECONDS;
            case RENEW_REMINDER: return RENEW_REMINDER_TTL_SECONDS;
            default: return COMPARISON_QUOTE_TTL_SECONDS;
        }
    }

    enum QuoteScene {
        REALTIME_INVEST,    // 客户即将投保，需要实时价格
        PRICE_COMPARISON,   // 比价参考，可以接受短时间内的价格
        RENEW_REMINDER      // 续保提醒邮件/短信，价格仅供参考
    }

    // 伪代码方法（实际项目中用Redis）
    private AggregatedQuoteResult callInsurerApis(CarQuoteRequest req) { return null; }
    private AggregatedQuoteResult getFromCache(Object cache, String key) { return null; }
    private void putToCache(Object cache, String key, AggregatedQuoteResult r, int ttl) {}
}
```

## 第六节：知识串联——从业务规则到系统决策

学完这篇，让我们完整梳理一下：**每一个业务规则，对应了系统的什么设计决策**。

| 业务规则 | 系统设计决策 | 涉及微服务 |
|---------|------------|---------|
| 交强险固定费率，但各家保司都要卖 | 系统需要记录客户在哪家保司买了交强险，便于续保提醒 | ins-order |
| 商业险自主定价，各家保司系数不同 | 必须并发请求多家保司，不能用固定费率表 | ins-quote |
| NCD系数影响报价 | 必须从保司API获取客户历史出险记录（或客户手动输入并留存证明） | ins-quote |
| 费率随时可能调整 | 报价结果设置短TTL（5分钟）缓存，不能永久缓存；投保前必须重新获取实时报价 | ins-quote |
| 营运车辆需特殊核保 | 投保表单中必须有"车辆性质"字段，且后台有核保规则引擎 | ins-quote, ins-order |
| 报行合一：手续费不能超报备上限 | 佣金模块必须维护各保司报备费率，结算时自动校验 | ins-commission |
| 不同保司API协议各异 | 使用适配器模式，每家保司独立一个Adapter，统一上层接口 | ins-quote |
| 报价结果要存档 | 每次报价都写入数据库（ins_quote_result），保司给的报价快照不可修改 | ins-quote |

读完本篇，带着以下问题继续思考：
