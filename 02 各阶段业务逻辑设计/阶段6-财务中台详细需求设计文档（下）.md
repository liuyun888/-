## 四、BI报表模块

### 4.1 经营看板

#### 4.1.1 功能概述

实时展示平台核心业务经营指标，支持首页大屏和PC后台看板两种视图，数据每5分钟刷新一次（WebSocket推送），关键指标支持与上期环比展示。

#### 4.1.2 功能入口

PC管理后台 → 财务中台 → BI报表 → 经营看板。

#### 4.1.3 页面布局

**顶部时间筛选栏：**
- 时间维度切换：今日 / 本周 / 本月 / 本季度 / 本年 / 自定义（日期范围，最长365天）
- 默认展示本月数据
- 点击"刷新"按钮手动刷新

**第一行：核心指标卡片（4个）**

每个卡片包含：指标名称、当期数值、环比箭头（↑红色/↓绿色）+ 环比百分比。

| 指标名称 | 计算逻辑 |
|---------|---------|
| 总保费（元） | 所选时段内已成交订单的保费总和 |
| 佣金收入（元） | 所选时段内已结算佣金总额 |
| 新增保单数 | 所选时段内新增订单数 |
| 活跃业务员数 | 所选时段内有出单记录的业务员去重数量 |

**第二行：趋势折线图（ECharts）**

- 标题：近12个月保费/佣金趋势
- X轴：月份（最近12个月）
- Y轴：金额（万元）
- 双折线：保费折线（蓝色）+ 佣金折线（橙色）
- hover展示详细数值
- 右上角：切换 月度/季度/年度

**第三行：双图布局**

- 左图：险种保费占比饼图（ECharts donut图），各险种色块，hover展示险种名和占比
- 右图：TOP10业务员产能柱状图（按当期保费排序，横向柱状图），展示姓名+保费金额

**第四行：渠道/团队保费热力表**

- 行：一级团队
- 列：月份（近6月）
- 单元格：该团队该月保费（颜色深浅代表高低）

#### 4.1.4 后端数据接口

**接口1（页面加载）：** GET `/finance/bi/dashboard?timeType=month&startDate=&endDate=`

响应包含所有看板数据（KPI卡片、图表数据）。

**接口2（WebSocket推送）：** 建立WebSocket连接 `/ws/finance/dashboard`，后端每5分钟推送最新KPI卡片数据（仅推送实时数据，减少传输量）。

**后端SQL优化要点：**
- 统计数据提前通过定时任务（每小时）聚合写入统计宽表 `finance_stat_daily`，避免实时大表聚合。
- 当日数据则实时查询+Redis缓存（TTL 5分钟）。

```sql
-- 日统计宽表
CREATE TABLE finance_stat_daily (
  id            BIGINT PRIMARY KEY AUTO_INCREMENT,
  stat_date     DATE NOT NULL COMMENT '统计日期',
  stat_type     VARCHAR(20) COMMENT 'daily/monthly/quarterly',
  insurer_id    BIGINT COMMENT '保险公司（可为0表示全部）',
  team_id       BIGINT COMMENT '团队ID（可为0表示全部）',
  product_type  VARCHAR(50) COMMENT '险种（可为ALL）',
  policy_count  INT DEFAULT 0 COMMENT '保单数',
  premium_amount DECIMAL(16,2) DEFAULT 0 COMMENT '保费',
  commission_amount DECIMAL(16,2) DEFAULT 0 COMMENT '佣金',
  agent_count   INT DEFAULT 0 COMMENT '活跃业务员数',
  creator       BIGINT,
  create_time   DATETIME DEFAULT CURRENT_TIMESTAMP,
  tenant_id     BIGINT,
  UNIQUE KEY uk_stat (stat_date, stat_type, insurer_id, team_id, product_type, tenant_id)
) ENGINE=InnoDB COMMENT='财务统计宽表';
```

### 4.2 保费统计

#### 4.2.1 功能概述

多维度分析保费数据，支持按险种、保险公司、时间、业务员、团队等维度交叉分析，支持钻取查看明细。

#### 4.2.2 功能入口

PC管理后台 → 财务中台 → BI报表 → 保费统计。

#### 4.2.3 页面功能

**筛选栏：**
- 统计维度（单选必填）：险种 / 保险公司 / 团队 / 业务员
- 时间范围（日期范围，必填，最长1年）
- 险种（多选，可选）
- 保险公司（多选，可选）
- 团队（多选，可选）

点击"查询"按钮生成报表。

**图表区（查询后展示）：**

- 上方：柱状图，X轴为所选维度的分类，Y轴为保费金额（万元），支持排序（从高到低）
- 下方：明细汇总表格

**汇总表格字段（以"险种"维度为例）：**

| 险种名称 | 保单数 | 保费合计 | 占比 | 环比 | 佣金合计 | 佣金率 |
|---------|-------|---------|-----|-----|---------|-------|
| 车险 | 1,234 | ¥5,678,900 | 45.2% | ↑3.5% | ¥567,890 | 10.0% |
| 意外险 | 567 | ... | ... | ... | ... | ... |

表格底部：合计行（加粗）。

**钻取功能：** 点击任一行，弹出该维度的月度明细折线图（近12月趋势）。

**导出功能：** 右上角"导出Excel"按钮，导出当前查询结果（EasyExcel）。

### 4.6 自定义报表

#### 4.6.1 功能概述

允许财务/运营人员自行选择维度和指标，动态生成数据报表，无需开发介入。支持保存常用报表配置。

#### 4.6.2 功能入口

PC管理后台 → 财务中台 → BI报表 → 自定义报表。

#### 4.6.3 自定义报表页面

**页面分为左中右三区：**

**左侧：维度/指标选择区**

维度列表（可多选拖拽）：
- 时间维度：年/月/日/周
- 业务维度：保险公司、险种大类、险种小类、团队、业务员
- 地区维度：省份、城市

指标列表（可多选）：
- 保单数、保费金额、佣金金额、佣金率、个税金额、实发金额、活跃人数

**中间：报表配置区**

- 行维度：拖入1-2个维度字段
- 列维度：拖入0-1个维度字段（构成交叉表）
- 指标：勾选展示哪些指标列
- 时间范围：必填，日期选择器
- 筛选条件：可选，如只看某保险公司、某险种

点击"生成报表"按钮，触发查询。

**右侧：报表展示区**

展示动态生成的数据表格，支持：
- 列排序
- 合计行（自动计算数字列的合计）
- 分页（每页50条）

表格右上角：
- **导出Excel**：EasyExcel动态表头导出
- **保存配置**：弹出输入框，填写报表名称，保存当前配置（存入`finance_custom_report_config`表）
- **我的报表**：查看已保存的报表配置，点击可快速加载

#### 4.6.4 后端动态SQL实现

1. 前端提交`ReportQueryDTO`（包含行维度列表、列维度、指标列表、时间范围、筛选条件）。
2. 后端通过`ReportSqlBuilder`服务，根据参数动态构建SQL：
   - SELECT子句：根据指标列表构建聚合函数（SUM/COUNT/AVG）
   - GROUP BY子句：根据维度列表构建分组字段
   - WHERE子句：根据时间范围和筛选条件构建
   - 使用MyBatis的`@SelectProvider`或手动拼接（特别注意SQL注入防护：字段名必须从白名单校验，不允许直接传入）
3. 白名单机制：所有可选维度、指标均有对应的数据库字段映射枚举，前端传入枚举Key，后端映射为SQL字段，拒绝不在枚举中的字段。

```sql
-- 自定义报表配置表
CREATE TABLE finance_custom_report_config (
  id            BIGINT PRIMARY KEY AUTO_INCREMENT,
  report_name   VARCHAR(100) NOT NULL COMMENT '报表名称',
  config_json   TEXT NOT NULL COMMENT '配置JSON（维度、指标、筛选等）',
  creator       BIGINT,
  create_time   DATETIME DEFAULT CURRENT_TIMESTAMP,
  updater       BIGINT,
  update_time   DATETIME,
  deleted       TINYINT DEFAULT 0,
  tenant_id     BIGINT
) ENGINE=InnoDB COMMENT='自定义报表配置';
```

## 五、监管报表模块

### 5.1 业务台账

#### 5.1.1 功能概述

按监管要求生成标准格式的业务台账表，记录保险中介机构的保费收入、佣金收入等，用于向银保监会（金融监管总局）报送。

#### 5.1.2 功能入口

PC管理后台 → 财务中台 → 监管报表 → 业务台账。

#### 5.1.3 台账列表页

筛选：报表年月（月份选择器）、报表状态（待生成/已生成/已上报/已归档）。

列表字段：

| 字段 | 说明 |
|------|------|
| 台账编号 | REG+年月+序号 |
| 报表名称 | 如"2025年01月保险中介业务统计台账" |
| 报表类型 | 月报/季报/年报 |
| 报表月份 | |
| 生成状态 | |
| 上报状态 | |
| 归档状态 | |
| 操作 | 生成 / 预览 / 下载 / 标记已上报 / 归档 |

#### 5.1.4 生成台账

点击"生成"按钮（或系统定时自动触发，每月1日凌晨自动生成上月台账）：

**后端生成逻辑：**

1. 查询上月已完成对账和结算的所有数据。
2. 按监管报表固定格式填充数据：
   - 表一：保险代理业务基本情况表
     - 字段：业务类型（车险/非车险）、保单数量、保费收入（元）、手续费收入（元）、手续费率
   - 表二：人员情况表
     - 字段：从业人员类别、期初人数、本期新增、本期减少、期末人数
   - 表三：客户投诉情况（如有）
3. 使用EasyExcel模板填充（预置好格式的xlsx模板），生成最终文件存OSS。
4. 更新台账记录状态为"已生成"。

**预览功能：** 点击"预览"，在新标签页打开OSS文件（xlsx在线预览，需接入Office在线预览服务，或下载后查看）。

#### 5.1.5 标记已上报

点击"标记已上报"，弹出对话框：

| 字段 | 类型 | 是否必填 | 说明 |
|------|------|---------|------|
| 上报日期 | 日期选择器 | 必填 | |
| 上报方式 | 单选 | 必填 | 监管系统在线上报 / 纸质报送 |
| 上报凭证 | 文件上传 | 非必填 | 上传截图等证明文件 |
| 备注 | 文本 | 非必填 | |

确认后更新上报状态、记录信息。

### 5.2 数据上报

#### 5.2.1 功能概述

部分监管机构（如各地银保局）要求通过API接口直接上报数据，本模块实现对接监管数据上报接口，自动推送数据。

#### 5.2.2 功能入口

PC管理后台 → 财务中台 → 监管报表 → 数据上报。

#### 5.2.3 上报配置

首次使用需在"系统配置"中配置监管接口参数：
- 监管机构名称
- 接口地址（URL）
- 认证方式（API Key / OAuth2 / 数字证书）
- 认证参数（加密存储）
- 数据格式（JSON/XML/固定格式报文）

#### 5.2.4 上报记录列表

筛选：上报月份、上报机构、上报状态（成功/失败/待上报）。

列表字段：

| 字段 | 说明 |
|------|------|
| 上报批次号 | RPT+年月日+序号 |
| 上报机构 | 监管机构名称 |
| 上报月份 | |
| 数据条数 | 本次上报数据量 |
| 上报状态 | 成功/失败/部分成功 |
| 上报时间 | |
| 响应信息 | 监管系统返回的响应码和描述 |
| 操作 | 查看请求/响应详情 / 重试 |

#### 5.2.5 上报逻辑

**定时自动上报：** 每月5日凌晨3点，XXL-Job触发上报任务。

**手动触发：** 在列表上方点击"立即上报"按钮，选择上报月份和机构，确认执行。

**后端处理：**
1. 查询目标月份需上报的业务数据。
2. 按监管接口规范组装报文。
3. 调用HTTP接口发送，记录请求报文和响应报文（加密存储，防止数据泄露）。
4. 解析响应，成功则更新上报状态；失败则记录错误，推送告警通知。
5. 对于失败的记录，支持手动点击"重试"按钮重新上报。

### 5.3 报表归档

#### 5.3.1 功能概述

统一管理各类报表文件的归档，支持按月、按类型检索，确保报表可追溯、可审计。

#### 5.3.2 功能入口

PC管理后台 → 财务中台 → 监管报表 → 报表归档。

#### 5.3.3 归档列表页

筛选：报表类型（下拉：对账报表/结算报表/税务申报/监管台账）、时间范围、归档状态。

列表字段：

| 字段 | 说明 |
|------|------|
| 报表名称 | |
| 报表类型 | |
| 所属月份 | YYYY-MM |
| 文件大小 | KB/MB |
| 归档时间 | |
| 归档人 | |
| 操作 | 下载 / 删除（需权限） |

**归档操作（手动）：**

点击"归档"按钮（在各报表生成页），将已生成的文件标记为归档状态，系统记录归档人和时间，文件在OSS中移动到归档目录（路径：finance/archive/年/月/报表类型/文件名）。

**自动归档规则：**
- 定时任务每月1日将上月所有已上报的监管台账自动归档。
- 归档后的文件在OSS中启用"不可删除"（Object Lock），保存期限7年（符合金融监管要求）。

#### 5.3.4 数据库表设计

```sql
-- 报表归档表
CREATE TABLE finance_report_archive (
  id              BIGINT PRIMARY KEY AUTO_INCREMENT,
  report_name     VARCHAR(200) NOT NULL COMMENT '报表名称',
  report_type     VARCHAR(50) COMMENT '报表类型：reconcile/settlement/tax/regulatory',
  report_month    VARCHAR(7) COMMENT '所属月份',
  file_name       VARCHAR(200) COMMENT '文件名',
  file_url        VARCHAR(500) COMMENT 'OSS文件路径',
  file_size       BIGINT COMMENT '文件大小（字节）',
  archive_type    TINYINT DEFAULT 0 COMMENT '0手动归档 1自动归档',
  source_id       BIGINT COMMENT '来源记录ID',
  source_type     VARCHAR(50) COMMENT '来源类型',
  creator         BIGINT,
  create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
  updater         BIGINT,
  update_time     DATETIME,
  deleted         TINYINT DEFAULT 0,
  tenant_id       BIGINT
) ENGINE=InnoDB COMMENT='报表归档管理';
```
