## 一、模块总览

### 1.1 模块定位

寿险模块是保险中介平台 V13 新增的核心业务模块，专为寿险（人身险）业务线提供完整的数字化支撑。与车险、非车险相比，寿险具有以下独特性：

- **保单生命周期长**：缴费期通常 10-30 年，需要专项续期管理和孤儿单管理机制
- **关系人复杂**：一张保单涉及投保人、被保人、受益人三种角色
- **监管要求严**：需满足保监局数据报送、回访合规、健康告知等要求
- **财务结算专属**：FYC（首年佣金）、RYC（续年佣金）、折标系数等寿险专有财务概念
- **数据回传要求**：需将保单数据定期回传保险公司或监管系统

### 1.3 微服务职责划分

| 微服务 | 职责范围 |
|--------|---------|
| `intermediary-module-ins-order` | 寿险保单 CRUD、回访管理、保全维护、孤儿单管理、理赔管理、C端投保流程订单 |
| `intermediary-module-ins-life` | 寿险专属的续期跟踪、数据回传、财务结算、监管报表、政策配置、系统配置、产品管理 |

### 1.4 技术约束

- **数据库规范**：无物理外键，应用层维护引用完整性；表前缀 `ins_life_`；Schema `db_ins_life`
- **敏感数据**：身份证号、银行卡号 AES-256 加密存储，展示时脱敏（前4后4中间***）
- **标准审计字段**：所有表含 `creator`、`create_time`、`updater`、`update_time`、`deleted`、`tenant_id`
- **接口规范**：PC后台接口前缀 `/admin-api/ins/life/**`，App/C端前缀 `/app-api/ins/life/**`
- **异步处理**：数据回传、Excel导出等大批量操作通过 RocketMQ 异步处理

### 2.1 个险保单录入

**菜单路径**：寿险 → 保单管理 → 个险保单 → 保单录入  
**对应 PDF**：PDF-110

#### 2.1.1 整体流程

多步骤表单，分四步完成：基本信息 → 关系人信息 → 险种信息 → 缴费信息。每步点击【下一步】校验当前步骤必填项，最后一步点击【提交】完成录入。

#### 2.1.2 第一步：基本信息

| 字段名 | 必填 | 控件 | 校验规则 |
|--------|------|------|---------|
| 险种 | ★ | 下拉枚举 | 枚举：寿险/重疾/医疗/意外/年金/万能险 |
| 保险公司 | ★ | 下拉 | 仅显示已配置的寿险合作保司 |
| 产品名称 | ★ | 联动下拉 | 联动险种+保司过滤，仅显示上架产品 |
| 保单号/投保单号 | ★ | 文本输入 | 同保司+保单号唯一性校验（后端） |
| 起保日期 | ★ | 日期选择器 | — |
| 到期日期 | ★ | 日期选择器 | 须 > 起保日期 |
| 缴费方式 | ★ | 单选 | 枚举：趸缴/年缴/半年缴/季缴/月缴 |
| 缴费期间（年） | ★ | 数字输入 | 趸缴固定1年；其他缴费方式手动输入 |
| 年度保费 | ★ | 金额输入 | 精度两位小数，大于0 |
| 续期保费 | 选填 | 金额输入 | — |
| 件数状态 | 选填 | 下拉 | 枚举：新单/续保/转保 |
| 保单服务人 | ★ | 人员选择 | 默认当前操作人，管理员可修改 |
| 签单日期 | 选填 | 日期选择器 | 默认等于起保日期 |
| 承保日期 | 选填 | 日期选择器 | — |
| 是否复效保单 | 选填 | 开关 | — |
| 备注 | 选填 | 文本域 | 最多500字 |

**后端校验逻辑**：保单号+保险公司组合唯一性校验（查 `ins_policy_life` 表）

#### 2.1.3 第二步：关系人信息（投保人 & 被保人）

> **关键交互**（来自 PDF-110）：被保人选择客户后，必须点击【添加】，下方生成一行客户信息才算添加成功。

**投保人信息**（必须有且仅有一个）：

| 字段名 | 必填 | 说明 |
|--------|------|------|
| 投保人姓名 | ★ | 可点击【选择客户】从客户库搜索带入 |
| 证件类型 | ★ | 枚举：身份证/护照/港澳通行证 |
| 证件号码 | ★ | AES-256加密存储，展示时脱敏（前4后4中间***） |
| 出生日期 | ★ | 可从身份证号自动解析 |
| 性别 | ★ | 可从身份证解析 |
| 联系电话 | 选填 | 脱敏展示 131****1111 |
| 与被保人关系 | ★ | 枚举：本人/配偶/子女/父母/其他 |

**被保人信息**（至少一个，支持多个）：

字段与投保人类似，额外增加：受益人类型（法定/指定）、受益人信息（指定受益人时必填）。

**自动客户匹配逻辑**：提交时按证件号精确匹配 `ins_customer` 表；匹配则更新 `ins_policy_life.customer_id`；未匹配则自动创建新客户后关联。

#### 2.1.4 第三步：险种信息

> **关键交互**（来自 PDF-110）：险种填写完成后必须点击【保存险种】，下方生成一行险种信息才算保存完成。支持多个险种（主险+多个附险）。

| 字段名 | 必填 | 说明 |
|--------|------|------|
| 险种分类 | ★ | 枚举：主险/附加险 |
| 险种名称 | ★ | 关联产品库 |
| 产品代码 | ★ | 自动带出 |
| 保额 | ★ | 金额 |
| 险种保费 | ★ | 该险种对应保费 |
| 保险期间 | 选填 | 如"终身"/"20年" |
| 缴费期 | 选填 | — |

#### 2.1.5 第四步：缴费信息

| 字段名 | 必填 | 说明 |
|--------|------|------|
| 上游标准保费 | ★ | 用于上游结算计算基础 |
| 下游首年标准保费 | 选填 | 用于下游结算 |
| 下游续期标准保费 | 选填 | — |
| 上游折标系数 | 选填 | 百分比，默认100% |
| 下游首年折标系数 | 选填 | — |
| 价值保费 | 选填 | — |

**提交后端处理**：
1. INSERT `ins_policy_life`（保单主表）
2. INSERT `ins_policy_life_insured`（投保人/被保人记录，`record_type`：0=投保人，1=被保人，2=受益人）
3. INSERT `ins_policy_life_coverage`（险种信息）
4. 触发佣金预计算（RocketMQ 异步消息）
5. 保单状态初始为 `UNDERWRITING`（核保中）

### 2.3 批量导入保单

**对应 PDF**：PDF-112

**操作流程**：
1. 点击【批量导入】→ 点击【下载模版】
2. 按模版填写保单数据（模版前9行不能删除或修改，带 ※ 的列为必填项）
3. 填好后点击【批量导入】→ 选取文件导入
4. 后端 EasyExcel 解析，逐行校验；有问题时弹出报错窗口，点击【导出校验结果】排查
5. 校验通过后批量 INSERT `ins_policy_life` 及关联表

**校验规则**：
- 保单号+保司唯一性
- 必填字段不为空
- 日期格式、金额格式合法
- 险种、保司枚举值合法

**处理结果**：展示成功条数/失败条数/失败明细（含行号+原因）

### 2.8 保单状态变更

**对应 PDF**：PDF-120（目录）、PDF-121（状态变更）

保单状态枚举及可变更路径：

```
UNDERWRITING（核保中）→ ACTIVE（生效）
ACTIVE（生效）→ LAPSED（中止/失效）
ACTIVE（生效）→ CANCELLED（退保）
ACTIVE（生效）→ MATURE（满期）
LAPSED（中止）→ ACTIVE（复效）
任意状态 → REJECTED（核保拒绝）
```

**状态变更操作**：
1. 在保单列表点击【状态变更】
2. 弹出变更弹窗，选择目标状态、填写变更原因、变更日期
3. 某些状态变更需上传证明材料（如退保需上传退保申请书）
4. 保存后：UPDATE `ins_policy_life.policy_status`、INSERT `ins_policy_status_log`（记录变更历史）

### 2.9 保单修改与删除

**对应 PDF**：PDF-122（修改）、PDF-123（删除）、PDF-124（批量影像件）

#### 2.9.1 保单修改

**核心字段变更需审批**（如起保日期、保额、被保人）：
1. 点击【编辑】，修改字段后提交
2. 检测到核心字段变更，弹出确认框"检测到核心字段变更，需提交审批"
3. 创建 Flowable 审批工单（`LIFE_POLICY_CHANGE_APPROVE`）

**继续率计算逻辑**：

```
R13 = 满足条件的保单数（起保后第13个月仍有效且已续缴）
```

SQL实现思路：取基准月往前推12个月的起保保单为分母，在分母中找到第13次缴费记录已存在的为分子。

**明细查询与导出**（来自 PDF-161）：点击某节点展开保单明细列表，支持筛选，点击【导出】自动跳转到导出任务列表下载。

### 12.4 寿险投保流程（C端）

投保流程分四步，顶部步骤条（已完成为绿色勾，当前高亮）：

```
第1步：健康告知 → 第2步：填写信息 → 第3步：缴费确认 → 第4步：支付&核保
```

**投保草稿机制**：`POST /app-api/ins/life/order/start` 创建草稿 `ins_life_order_draft`，返回 `draft_id`。草稿24小时未完成自动过期（XXL-Job 清理）。离开页面弹出"确认放弃本次投保？"提示。

#### 12.4.1 第1步：健康告知问卷

**问卷加载**：`GET /app-api/ins/life/product/{product_id}/health-notice`

**问卷渲染规则**：
- 每题提供「是」「否」两个互斥单选按钮
- 选「是」时红色高亮提示"注意：此选项可能影响承保结论"，并动态展开追问题目
- 选「否」时追问题目折叠隐藏
- 必须全部作答，未作答题目红色边框高亮
- 底部确认勾选框："本人已如实告知，并了解如未如实告知可能影响理赔"（必须勾选才可提交）

**提交与智能核保**：`POST /app-api/ins/life/order/health-notice`

**后端核保规则引擎结果**：

| 结果 | 处理 |
|------|------|
| PASS（通过） | 进入第2步 |
| MANUAL_REVIEW（人工核保） | 提示"需人工核保，审核周期1-3个工作日"，可选择继续（提交人工审核）或放弃 |
| REJECT（拒绝） | 弹出拒绝说明，推荐其他适合产品 |

#### 12.4.2 第2步：填写投保信息

分为三个 Section：投保人信息、被保人信息、受益人信息（可选）。

**OCR 辅助**：身份证 OCR 识别，自动填充姓名、证件号、性别、出生日期。

**投保人信息**：

| 字段 | 必填 | 校验 |
|------|------|------|
| 姓名 | ★ | — |
| 证件类型 | ★ | 枚举 |
| 证件号 | ★ | AES 加密存储，格式校验 |
| 出生日期 | ★ | 可从身份证解析 |
| 性别 | ★ | 可从身份证解析 |
| 手机号 | ★ | 11位手机号格式 |
| 与被保人关系 | ★ | 枚举：本人/配偶/父母/子女/其他 |

**被保人信息**：字段同投保人（去掉"与被保人关系"）。若投保人与被保人相同，勾选【与投保人相同】复选框，自动同步数据，被保人字段只读。

**受益人信息**（可选择指定受益人）：填写受益人姓名★、证件号★、与被保人关系★、受益比例（多受益人时必填，合计须等于100%）。

#### 12.4.3 第3步：缴费方式确认

**展示内容**：
- 产品名称、险种
- 被保人信息摘要（姓名脱敏、年龄、性别）
- 保费明细（年缴保费/月缴保费，由费率引擎实时计算）
- 缴费方式选择（产品支持的缴费方式）
- 缴费期限（年期）
- 首次缴费日（默认今天，可选择未来30天内的日期）
- 总保费合计（缴费期×年度保费）

**后端接口**：`GET /app-api/ins/life/order/draft/{draft_id}/payment-confirm`（返回保费计算结果）

#### 12.4.4 第4步：支付与核保

**支付方式**：微信支付（小程序内调起）/ 支付宝 / 银行卡。

**提交投保接口**：`POST /app-api/ins/life/order/submit`

**后端处理流程**：
1. 校验草稿状态和所有步骤数据完整性
2. 创建正式订单 `ins_life_order`（status=PENDING_PAYMENT）
3. 调起支付（framework 支付中台）
4. 支付成功回调 → UPDATE status=PENDING_UNDERWRITING → RocketMQ 推送核保消息 → 更新为 UNDERWRITING
5. 核保完成后（保司回调或人工审核）→ 若通过则 UPDATE status=ACTIVE → 创建 `ins_policy_life` 记录

**核保等待页**：
- 展示保单预览信息
- "核保中"动画提示
- 每3秒轮询核保状态（`GET /app-api/ins/life/order/{order_no}/underwriting-status`）
- 核保通过：跳转"投保成功"页，展示保单号、生效日期
- 核保附加条件（加费/除外）：弹窗展示条件详情，【接受条件继续】【拒绝并退款】两个按钮
- 核保拒绝：展示拒绝原因，推荐其他产品

### 12.5 我的寿险保单（C端）

**入口**：小程序底部 Tab「我的」→ 在已有Tab旁新增「寿险」Tab（左右滑动切换）

**筛选 Tab**：全部 / 生效中 / 核保中 / 已失效

**保单卡片展示字段**：险种图标、产品名称、保险公司、被保人姓名（脱敏：仅姓+*，如"张*"）、保障额度（如"保额100万"）、缴费状态标签、下次缴费日。

**缴费状态标签**：

| 标签 | 触发条件 |
|------|----------|
| 🟢 保障中 | `status=ACTIVE` 且 `next_payment_date > 今日` |
| 🟡 待缴费 | `status=ACTIVE` 且 `next_payment_date` 在7天内（含今日） |
| 🔴 已逾期 | `status=ACTIVE` 且 `next_payment_date < 今日` |
| ⚪ 核保中 | `status=UNDERWRITING` |
| ⚫ 已失效 | `status IN (CANCELLED, LAPSED)` |

**逾期缴费提醒弹窗**：若存在逾期保单，页面加载后自动弹出一次（Session 变量控制），提供【立即补缴】【稍后再说】两个按钮。

**保单详情页**：
- 展示完整保单信息、险种、关系人、缴费历史
- 操作按钮（根据保单状态动态显示）：【续期缴费】【查看条款】【申请保全】【报案】
- 保全申请：`POST /app-api/ins/life/policy/{policy_id}/preservation/apply`
- 理赔报案：`POST /app-api/ins/life/policy/{policy_id}/claim/report`

### 12.6 续期缴费提醒（C端）

**提醒触发**（XXL-Job 每日执行）：
1. 查询 `next_payment_date` 在未来30天内的 ACTIVE 保单
2. 推送微信模板消息（标题"您的保单即将到期，请及时续缴"）
3. 推送包含保单号、产品名称、到期日、缴费金额、【立即缴费】跳转链接

**消息中心**：用户可在"我的"→"消息中心"查看全部提醒记录。
