## 一、产品详情模块

产品详情模块是用户了解保险产品的核心页面,需要全面展示产品信息、提供试算工具、展示用户评价,帮助用户做出购买决策。

### 4.1 产品信息展示

**开发工时**: 前端1.5天 + 后端1天

#### 4.1.1 功能概述

全面展示保险产品的详细信息,包括产品介绍、保障内容、投保规则、条款说明等,帮助用户充分了解产品,做出明智的购买决策。

#### 4.1.2 核心业务逻辑

**产品信息结构**:

1. **基础信息展示**:

   - 产品名称:如"平安e生保·长期医疗险2024版"
   - 产品标签:热销、新品、爆款、限时优惠等
   - 保险公司:公司logo + 名称 + 备案号
   - 销量数据:已售XXX份(动态更新)
   - 评分评价:5星评分 + 评价数量
   - 产品有效期:销售有效期,如"2024-01-01至2024-12-31"

2. **核心卖点**:

   - 3-5个核心亮点,如"最高600万保额"、"0免赔额"、"保证续保20年"
   - 图标+文字形式展示
   - 突出产品差异化优势
   - 支持后台可配置

3. **保费信息**:

   - 起始保费:如"30元/年起"
   - 保费计算器入口:点击进入试算页面
   - 分期支付说明:支持月缴、季缴、年缴
   - 优惠活动:首月0元、限时折扣等

4. **保障内容**:

   - 基础保障

     :

     - 保障项目列表:如住院医疗、特殊门诊、门诊手术
     - 保额限额:每项保障的保额或赔付比例
     - 免赔额说明:如年度免赔额1万元
     - 等待期:如30天等待期

   - 可选保障

     :

     - 附加险种:如质子重离子、特效药、住院津贴
     - 费用说明:附加险的额外费用
     - 勾选方式:用户可自主选择是否添加

   - 责任免除

     :

     - 不保事项列表
     - 重点提示用户注意
     - 展开/收起形式,避免页面过长

5. **投保规则**:

   - 投保年龄:如0-60周岁
   - 职业类别:1-4类职业可投保
   - 健康要求:健康告知要求说明
   - 等待期:如意外无等待期,疾病30天等待期
   - 犹豫期:如15天犹豫期,全额退保
   - 保障期限:1年、至70岁、终身等
   - 缴费期限:趸交、5年、10年、20年、至60岁等

6. **投保须知**:

   - 如实告知义务
   - 投保流程说明
   - 保单生效时间
   - 退保规则
   - 理赔流程概述
   - 常见问题解答(FAQ)

7. **条款文件**:

   - 保险条款PDF:可在线预览和下载
   - 产品说明书:详细的产品说明文档
   - 健康告知书:健康告知问卷模板
   - 投保须知:投保注意事项文档
   - 文件版本号和备案号

**页面布局设计**:

1. **顶部区域**:
   - 产品主图轮播:3-5张产品宣传图
   - 产品名称和标签
   - 保险公司信息
   - 销量和评分
2. **核心卖点区**:
   - 3-5个核心卖点横向排列
   - 图标+文字形式
   - 底色区分,突出重点
3. **保费区**:
   - 起始保费显示
   - 立即试算按钮
   - 优惠活动提示
4. **详情Tab页**:
   - Tab1:产品详情(保障内容、投保规则)
   - Tab2:投保须知(投保流程、退保规则)
   - Tab3:理赔指南(理赔流程、所需材料)
   - Tab4:条款文件(PDF文件列表)
   - Tab5:常见问题(FAQ折叠展开)
5. **底部操作区**:
   - 收藏按钮
   - 分享按钮
   - 在线客服按钮
   - 立即投保按钮(固定底部)

**动态内容加载**:

1. **异步加载策略**:
   - 首屏快速加载:基础信息、核心卖点、保费信息
   - 懒加载内容:用户评价、详细条款、FAQ等
   - 图片懒加载:滚动到可视区域再加载
2. **缓存策略**:
   - 产品基础信息缓存30分钟
   - 销量数据实时查询
   - 评分评价缓存10分钟

#### 4.1.4 接口设计

**查询产品详情**:

- 路径: GET /api/product/detail/{productId}
- 参数: productId(产品ID)
- 返回: 产品基础信息、详情内容、标签、保费范围、统计数据

**查询产品保障内容**:

- 路径: GET /api/product/coverage/{productId}
- 参数: productId(产品ID)
- 返回: 基础保障列表、可选保障列表、责任免除列表

**查询投保规则**:

- 路径: GET /api/product/rules/{productId}
- 参数: productId(产品ID)
- 返回: 投保年龄、职业类别、健康要求、等待期、犹豫期等

**获取条款文件**:

- 路径: GET /api/product/clause/{productId}
- 参数: productId(产品ID)
- 返回: 条款文件列表(文件名、URL、版本号、备案号)

**记录浏览**:

- 路径: POST /api/product/view
- 参数: productId(产品ID), userId(用户ID,可选)
- 返回: 浏览记录ID
- 说明: 异步记录,更新浏览量,用于推荐算法

#### 4.1.5 技术实现要点

1. **数据缓存策略**:
   - 产品基础信息:Redis缓存30分钟,key=`product:detail:{productId}`
   - 保障内容:Redis缓存1小时,key=`product:coverage:{productId}`
   - 统计数据:Redis实时更新,每5分钟同步到MySQL
   - 条款文件URL:永久缓存,版本变更时清除
2. **性能优化**:
   - 分步加载:首屏加载基础信息,Tab内容懒加载
   - 图片优化:CDN加速,WebP格式,懒加载
   - 数据预热:热门产品提前加载到缓存
   - 接口合并:基础信息和保障内容可合并为一个接口减少请求
3. **安全控制**:
   - 产品下架后不允许访问详情
   - 销售期外产品显示"即将上线"或"已售罄"
   - 条款文件URL加签名防盗链
   - 敏感信息脱敏(如被保险人姓名)
4. **数据一致性**:
   - 销量、浏览量使用Redis计数器,异步同步到MySQL
   - 评分计算:每次新增评价后重新计算平均分
   - 缓存与数据库双写一致性:先更新DB,再删除缓存
5. **埋点统计**:
   - 页面浏览:记录productId、userId、来源页面、停留时长
   - 按钮点击:试算、投保、收藏、分享、客服等
   - 滚动深度:记录用户浏览到的内容区域
   - 跳出率:用户未操作直接离开的比例

#### 4.1.6 注意事项

- 产品信息必须与保险公司备案信息一致
- 保障内容描述清晰,避免歧义和误导
- 免责条款必须明确提示,不得隐藏
- 条款文件必须可下载,不得仅提供在线预览
- 健康告知要求必须详细列出,不得模糊描述
- 产品下架或售罄后及时更新状态
- 价格变动时同步更新所有展示位置

### 4.2 保费试算

**开发工时**: 前端2天 + 后端2天

#### 4.2.1 功能概述

根据用户输入的投保信息(年龄、性别、保额、缴费期限等),实时计算保费,支持多种险种的试算,提供保费对比功能,帮助用户选择最优方案。

#### 4.2.2 核心业务逻辑

**试算参数输入**:

1. **通用参数**:
   - 被保险人年龄:可输入或选择,0-100岁
   - 被保险人性别:男/女,影响保费
   - 保额:根据产品规则设置可选范围,如10万-100万
   - 保障期限:1年、至60岁、至70岁、终身等
   - 缴费期限:趸交、5年、10年、20年、至60岁等
   - 保障内容:基础保障+可选附加险
2. **车险特有参数**:
   - 车辆品牌型号:下拉选择或VIN码识别
   - 购车价格:影响车损险保费
   - 首次登记日期:影响车龄系数
   - 行驶城市:影响地区系数
   - 上年出险次数:影响NCD系数(无赔款优待)
   - 险种组合:交强险、车损险、三者险、座位险等
3. **健康险特有参数**:
   - 社保情况:有社保/无社保,影响保费和赔付比例
   - 既往病史:是否有既往病史(影响核保)
   - 职业类别:1-6类,高危职业加费或拒保
   - 有无医保:影响免赔额和赔付比例
4. **寿险特有参数**:
   - 吸烟情况:吸烟/不吸烟,影响保费
   - BMI指数:身高体重,影响核保
   - 家族病史:是否有重大疾病家族史

**保费计算逻辑**:

1. **基础保费计算**:

   ```
   基础保费 = 基准保费 × 年龄系数 × 性别系数 × 保额系数 × 缴费期限系数
   ```

   - 基准保费:产品的基础费率,由精算部门提供
   - 年龄系数:年龄越大系数越高,如0岁=0.5, 30岁=1.0, 60岁=3.0
   - 性别系数:男性通常高于女性,如男=1.2, 女=1.0
   - 保额系数:保额越高系数越高,但非线性增长
   - 缴费期限系数:期限越长每年保费越低,如趸交=1.0, 20年=0.6

2. **附加险保费**:

   - 用户勾选的附加险种分别计算
   - 附加险保费 = 附加险基准保费 × 年龄系数 × 保额系数
   - 总保费 = 基础保费 + Σ附加险保费

3. **优惠折扣**:

   - 首年优惠:如首年8折
   - 团购优惠:如5人团购9折
   - 优惠券抵扣:满减券、折扣券
   - 会员折扣:根据会员等级折扣

4. **车险特殊计算**:

   - NCD系数(无赔款优待):
     - 连续0次出险:0.7折
     - 连续1年未出险:0.85折
     - 1次出险:原价
     - 2次出险:1.25倍
     - 3次及以上:1.5-2倍
   - 车型系数:根据车型风险等级
   - 地区系数:不同城市费率不同
   - 车龄系数:新车系数低,老车系数高

**试算结果展示**:

1. **保费明细**:
   - 基础保费:XXX元/年
   - 附加险保费:XXX元/年(展开显示各项)
   - 优惠金额:-XXX元
   - 应付总额:XXX元
2. **缴费方式对比**:
   - 年缴:XXX元/年,共缴XX年,总计XXX元
   - 季缴:XXX元/季,共缴XX季,总计XXX元
   - 月缴:XXX元/月,共缴XX月,总计XXX元
   - 标注推荐方式
3. **保障收益说明**:
   - 保障额度:XXX万元
   - 缴费总额:XXX万元
   - 保障杠杆:XXX倍(保额/总保费)
   - IRR收益率:X.XX%(仅寿险显示)
4. **智能推荐**:
   - 根据用户年龄、收入、家庭情况推荐最优方案
   - 如"建议选择20年缴费,每年负担更轻"
   - 如"建议增加重疾多次赔付,保障更全面"

**试算记录保存**:

- 每次试算自动保存记录
- 用户可查看历史试算记录
- 支持对比不同方案
- 可直接从试算记录进入投保流程

#### 4.2.5 技术实现要点

1. **费率计算引擎**:
   - 使用规则引擎(如Drools)或自研计算引擎
   - 费率表JSON存储,灵活配置各维度系数
   - 支持费率版本管理,历史费率可追溯
   - 计算过程记录日志,便于审计和调试
2. **性能优化**:
   - 费率表缓存到Redis,key=`rate:{productId}:{version}`
   - 试算结果缓存5分钟,相同参数直接返回缓存
   - 异步保存试算记录,不阻塞用户
   - 批量查询历史记录时使用分页和索引
3. **精度控制**:
   - 金额使用BIGINT存储,单位为分,避免浮点误差
   - 计算过程使用BigDecimal,保留6位小数
   - 最终结果四舍五入到元
   - 优惠金额计算后与总保费对比,避免负数
4. **参数校验**:
   - 年龄范围:0-100岁,超出范围提示"不在承保年龄范围"
   - 保额范围:根据产品规则校验,如10万-500万
   - 缴费期限:不得超过保障期限
   - 附加险:检查是否可与主险搭配
5. **异常处理**:
   - 费率表不存在:提示"产品费率未配置,请联系客服"
   - 计算失败:记录错误日志,返回友好提示
   - 优惠券不适用:提示"优惠券不适用于该产品"
   - 网络超时:前端重试3次,超时提示"试算超时,请重试"

#### 4.2.6 注意事项

- 试算结果仅供参考,实际保费以核保为准
- 车险保费受上年出险次数影响,需如实填写
- 健康险试算不考虑健康告知,核保时可能加费或拒保
- 优惠活动有时效性,下单时需重新计算
- 试算记录保存30天,过期自动删除
- 费率调整时需通知用户重新试算

### 4.3 用户评价

**开发工时**: 前端1天 + 后端1天

#### 4.3.1 功能概述

用户购买保险后可对产品进行评价,包括文字评价、星级评分、晒单图片等,帮助其他用户了解产品真实使用体验,提升购买信心。

#### 4.3.2 核心业务逻辑

**评价权限**:

1. **评价资格**:
   - 必须是已投保用户
   - 保单状态为"已生效"或"已理赔"
   - 每个保单只能评价1次
   - 购买后180天内可评价
2. **评价入口**:
   - 保单详情页"去评价"按钮
   - 订单列表"评价"入口
   - 系统推送评价提醒(投保后7天、30天)

**评价内容**:

1. **星级评分**(必填):
   - 总体评分:1-5星
   - 分项评分(可选):
     - 产品保障:1-5星
     - 理赔速度:1-5星(仅理赔过的用户)
     - 服务态度:1-5星
     - 性价比:1-5星
2. **文字评价**(可选):
   - 最少10字,最多500字
   - 支持表情符号
   - 敏感词过滤(脏话、广告、竞品等)
   - 禁止包含联系方式(手机号、微信号)
3. **图片上传**(可选):
   - 最多9张图片
   - 图片格式:jpg、png、jpeg
   - 单张最大5MB
   - 自动压缩和水印
   - 敏感图片识别(腾讯云内容安全)
4. **标签选择**(可选):
   - 预设标签:保障全面、理赔快、性价比高、服务好、值得推荐
   - 负面标签:理赔慢、条款复杂、性价比低
   - 最多选择5个标签

**评价审核**:

1. **自动审核**:
   - 敏感词过滤:替换为***或直接拦截
   - 广告识别:包含联系方式、竞品名称的评价标记为待审核
   - 重复内容检测:同一用户多次提交相同内容标记为刷评
   - 图片鉴黄:使用腾讯云图片内容安全API
2. **人工审核**:
   - 自动审核未通过的评价进入人工审核队列
   - 审核时间:1-3个工作日
   - 审核结果:通过、拒绝(注明原因)、修改后通过
   - 拒绝理由:广告、辱骂、不实描述、与产品无关等
3. **审核状态**:
   - 待审核:提交后初始状态
   - 审核通过:前端展示
   - 审核拒绝:通知用户并说明原因
   - 已删除:用户主动删除或后台删除

**评价展示**:

1. **排序规则**:
   - 默认排序:综合排序(有用度 + 时间 + 评分)
   - 最新评价:按评价时间倒序
   - 高分优先:按评分从高到低
   - 有图优先:有图片的评价排前面
   - 已理赔优先:理赔过的用户评价更有参考价值
2. **评价筛选**:
   - 全部评价
   - 好评(4-5星)
   - 中评(3星)
   - 差评(1-2星)
   - 有图评价
   - 已理赔评价
3. **评价卡片信息**:
   - 用户头像+昵称(脱敏)
   - 星级评分
   - 评价时间
   - 评价内容(折叠显示,点击展开)
   - 评价图片(点击放大)
   - 评价标签
   - 有用数(点赞数)
   - 商家回复(如有)

**评价互动**:

1. **点赞(有用)**:
   - 用户可对评价点"有用"
   - 同一用户对同一评价只能点1次
   - 有用数影响排序权重
   - Redis计数器实时更新
2. **商家回复**:
   - 后台运营人员可回复评价
   - 回复内容显示在评价下方
   - 每条评价只能回复1次
   - 回复后通知评价用户
3. **追加评价**:
   - 首次评价后60天内可追加
   - 追加内容显示在原评价下方
   - 追加后不可再次追加

**评价统计**:

1. **评分统计**:
   - 总体平均分:所有评价的平均分
   - 各星级占比:5星XX%, 4星XX%, ...
   - 分项评分:产品保障、理赔速度、服务态度、性价比
2. **标签统计**:
   - 高频标签:保障全面(XX人), 理赔快(XX人), ...
   - 标签云展示
3. **评价趋势**:
   - 近30天评价数
   - 近30天平均分
   - 评分趋势图(折线图)

#### 4.4.1 功能概述

**收藏操作**:

   - 再次点击"收藏"按钮,图标变为空心
   - 提示"已取消收藏"
   - 收藏数-1
3. **收藏限制**:
   - 未登录用户:提示"请先登录"
   - 已收藏产品:提示"已在收藏夹中"
   - 收藏上限:最多收藏200个产品
   - 产品下架:仍可在收藏夹查看,但不可投保

**收藏夹管理**:

1. **收藏列表**:
   - 显示所有收藏的产品
   - 产品卡片:产品图、名称、保费、评分、收藏时间
   - 产品状态:在售、即将上线、已下架

   - 默认排序:按收藏时间倒序
   - 按价格:从低到高、从高到低
   - 按评分:从高到低
   - 按销量:从高到低
3. **筛选功能**:
   - 按分类:车险、健康险、寿险、意外险等
   - 按状态:在售、已下架
   - 按价格区间:自定义价格范围
4. **批量操作**:
   - 批量取消收藏:勾选后点击"删除"
   - 批量对比:勾选2-5个产品进行对比
   - 批量分享:分享收藏夹给好友

**收藏提醒**:

1. **价格变动提醒**:
   - 收藏产品降价时,推送通知
   - 提醒内容:"您收藏的XX产品降价啦,立省XX元"
   - 提醒方式:App推送、短信(可选)、站内消息
2. **库存提醒**:
   - 收藏产品即将售罄时,提醒用户
   - 提醒内容:"XX产品仅剩XX份,抓紧下单"
3. **上新提醒**:
   - 收藏产品有新版本或类似产品上线时提醒
   - 提醒内容:"XX产品新版本上线,保障更全面"
4. **活动提醒**:
   - 收藏产品参与促销活动时提醒
   - 提醒内容:"XX产品限时8折,仅限今日"

**收藏分析**:

1. **用户画像**:
   - 根据收藏产品分析用户需求
   - 如收藏多个重疾险,推荐相关产品
   - 如收藏多个车险,标记为有车用户
2. **产品推荐**:
   - 基于收藏历史推荐相似产品
   - 基于其他用户的收藏推荐"看了又看"
   - 推荐互补产品,如收藏医疗险推荐重疾险

#### 4.4.3 数据库设计

**收藏表 (tb_product_favorite)**:

```
主键: favorite_id BIGINT
关联字段:
```

## 二、车险投保模块

车险投保模块是C端商城的重要业务模块,涉及车辆信息采集、险种选择、在线报价、核保、支付等环节,流程复杂,需要与保险公司接口对接。

### 5.1 车辆信息录入

**开发工时**: 前端1.5天 + 后端2天

#### 5.1.1 功能概述

用户通过手动输入或VIN码识别的方式录入车辆信息,包括车牌号、品牌型号、购车价格、首次登记日期等,为后续报价和核保提供基础数据。

#### 5.1.2 核心业务逻辑

**车辆信息录入方式**:

1. **手动输入**:
   - 车牌号:7位车牌号,如"京A12345"
   - 车牌省份:下拉选择,如"京、沪、粤"等
   - 车牌城市:根据省份联动显示城市代码
   - 车辆品牌:下拉选择,如"大众、丰田、本田"
   - 车系:根据品牌联动显示车系,如"大众-迈腾、帕萨特"
   - 车型:根据车系联动显示具体车型,如"2024款 迈腾 380TSI 豪华型"
   - 购车价格:新车购置价,影响车损险保费
   - 首次登记日期:影响车龄,影响保费系数
2. **VIN码识别**(推荐):
   - VIN码输入:17位车架号,如"LSGGH54U9ES123456"
   - 扫描识别:拍照识别行驶证上的VIN码
   - 自动匹配:调用车辆数据库,匹配车型、排量、购车价等
   - 信息确认:用户确认识别结果,可手动修改
3. **行驶证识别**(OCR):
   - 拍照上传:拍摄行驶证正副页
   - OCR识别:提取车牌号、VIN码、品牌型号、注册日期等
   - 信息校验:与车辆数据库对比校验
   - 一键导入:识别结果一键填充到表单

**车辆信息校验**:

1. **车牌号校验**:
   - 格式校验:正则表达式验证,如^[京津沪渝冀豫云辽黑湘皖鲁新苏浙赣鄂桂甘晋蒙陕吉闽贵粤青藏川宁琼使领][A-Z][A-Z0-9]{5}$
   - 重复校验:检查是否已投保(同一车牌号同一保险公司一年内只能投保一次)
   - 黑名单校验:检查是否在黑名单(如套牌车、违规车)
2. **VIN码校验**:
   - 格式校验:17位字母数字组合,不含I、O、Q
   - 校验位算法:根据VIN码前16位计算第9位校验位
   - 数据库匹配:调用车辆数据库接口验证VIN码真实性
3. **车型匹配校验**:
   - 品牌车系车型三级联动
   - 车型代码唯一性校验
   - 购车价格范围校验:与官方指导价对比,差异>20%提示异常
4. **日期校验**:
   - 首次登记日期不得晚于当前日期
   - 车龄不得超过15年(部分产品限制)
   - 保险起期不得早于当前日期

**车辆数据库对接**:

1. **数据源**:
   - 中保研数据库:官方车型库,数据最全
   - 公安部车辆数据:VIN码查询车辆信息
   - 保险公司车型库:各保险公司自有车型库
2. **接口调用**:
   - 根据VIN码查询车型信息
   - 根据品牌车系查询车型列表
   - 根据车型代码查询详细参数(排量、座位数、购车价等)
3. **数据缓存**:
   - 常用车型缓存:热门车型信息缓存到Redis,有效期7天
   - VIN码查询缓存:查询结果缓存24小时
   - 车型列表缓存:品牌车系车型树缓存1天

**上年投保信息**:

1. **上年保险公司**:
   - 下拉选择,如"人保、平安、太保、国寿"等
   - 影响NCD系数(无赔款优待)
2. **上年出险次数**:
   - 选择或输入,0-5次
   - 0次出险:享受最大折扣(7折)
   - 1次出险:无折扣
   - 2次及以上:保费上浮25%-100%
3. **上年商业险保费**:
   - 用于对比本次报价
   - 计算续保优惠
4. **上年交强险到期日**:
   - 判断是否脱保
   - 脱保超过3个月需重新验车

**新能源车特殊处理**:

1. **能源类型**:
   - 燃油车、纯电动、插电混动、增程式
   - 影响车损险费率
2. **电池信息**:
   - 电池品牌:如宁德时代、比亚迪
   - 电池容量:kWh
   - 电池质保:影响保障方案
3. **专属保障**:
   - 三电系统(电池、电机、电控)
   - 充电桩损失险
   - 充电安全责任险

### 5.2 险种选择

**开发工时**: 前端1.5天 + 后端1天

#### 5.2.1 功能概述

用户根据自身需求选择车险险种组合,包括交强险、商业险主险和附加险,系统提供智能推荐方案,帮助用户快速选择合适的保障。

#### 5.2.2 核心业务逻辑

**险种类别**:

1. **交强险**(必选):

   - 全称:机动车交通事故责任强制保险
   - 性质:国家强制购买
   - 保障内容:
     - 死亡伤残赔偿限额:18万元
     - 医疗费用赔偿限额:1.8万元
     - 财产损失赔偿限额:0.2万元
   - 保费:固定保费,根据车型和座位数确定
     - 6座以下家用车:950元/年(首次),连续3年无出险降30%至665元
     - 6座及以上家用车:1100元/年(首次),最低770元

2. **商业险主险**(可选):

   a) **车损险**:

   - 保障内容:保障自己车辆的损失
   - 保额:按车辆实际价值或新车购置价确定
   - 赔偿范围:碰撞、倾覆、火灾、爆炸、外界物体坠落、自然灾害等
   - 新增责任:地震、玻璃单独破碎、自燃、发动机涉水、不计免赔、无法找到第三方
   - 推荐指数:★★★★★(强烈推荐)

   b) **第三者责任险**(三者险):

   - 保障内容:保障造成第三方的人身伤亡和财产损失
   - 保额:20万、50万、100万、150万、200万、300万可选
   - 推荐保额:
     - 一线城市:200万-300万
     - 二三线城市:100万-200万
     - 四线及以下:50万-100万
   - 推荐指数:★★★★★(强烈推荐)

   c) **车上人员责任险**(座位险):

   - 保障内容:保障车上人员(司机+乘客)的伤亡
   - 保额:
     - 司机:1万、2万、5万、10万可选
     - 乘客:每座1万、2万、5万可选
   - 推荐指数:★★★☆☆(建议购买,也可用意外险替代)

3. **商业险附加险**(可选):

   (以下附加险已整合进车损险,无需单独购买):

   - 玻璃单独破碎险:挡风玻璃、车窗玻璃单独破损
   - 自燃损失险:车辆自燃造成的损失
   - 发动机涉水损失险:发动机进水造成的损失
   - 车身划痕损失险:无明显碰撞痕迹的划痕(新车推荐)
   - 不计免赔险:取消事故责任免赔率

   (以下为独立附加险):

   - 机动车损失保险无法找到第三方特约险:肇事逃逸时由保险公司承担本应由第三方承担的30%
   - 修理期间费用补偿险:车辆维修期间的交通费、租车费补偿
   - 车上货物责任险:车上货物损失赔偿
   - 法定节假日限额翻倍险:节假日出险时三者险保额翻倍

**保障方案推荐**:

1. **经济型**(适合老旧车辆、驾驶熟练):
   - 交强险
   - 第三者责任险50万
   - 不含车损险
   - 适用场景:车龄>5年,车辆价值<5万
   - 预估保费:1500-2000元/年
2. **标准型**(适合大多数车主):
   - 交强险
   - 车损险(含玻璃、自燃、涉水、不计免赔)
   - 第三者责任险100万
   - 车上人员责任险(司机5万+乘客每座1万)
   - 适用场景:普通家用车,车龄<5年
   - 预估保费:3500-5000元/年
3. **尊享型**(适合新车、豪华车):
   - 交强险
   - 车损险(含全部附加责任)
   - 第三者责任险200万-300万
   - 车上人员责任险(司机10万+乘客每座5万)
   - 划痕险
   - 修理期间费用补偿险
   - 适用场景:新车、豪华车、新手司机
   - 预估保费:6000-10000元/年

**险种选择交互**:

1. **快速选择**:
   - 显示3种推荐方案卡片
   - 点击卡片一键选中该方案
   - 可在此基础上调整
2. **自定义选择**:
   - 列表展示所有险种
   - 勾选框选择险种
   - 三者险、座位险等可调整保额

3. **智能推荐**:
   - 根据车辆价值、车龄、用户画像推荐
   - 如新车推荐尊享型,老车推荐经济型
   - 如用户历史购买过高保额,推荐类似方案
4. **险种说明**:
   - 每个险种附带"?"图标
   - 点击查看详细说明

**保费计算规则**:

1. **交强险保费**:
   - 固定保费,根据车型座位数
   - NCD系数:
     - 首次投保或上年赔付:1.0倍
     - 上年未赔付:0.9倍
     - 连续2年未赔付:0.8倍
     - 连续3年及以上未赔付:0.7倍
   - 违章系数:酒驾等严重违章上浮30%
2. **商业险保费**:
   - 基础保费 = 基准保费 × NCD系数 × 自主渠道系数 × 交通违法系数
   - 基准保费:根据车型、保额、车龄等查询费率表
   - NCD系数:
     - 上年0次出险:0.7倍
     - 上年1次不涉及死亡:1.0倍
     - 上年2次:1.25倍
     - 上年3次:1.5倍
     - 上年4次:1.75倍
     - 上年5次及以上:2.0倍
   - 自主渠道系数:一般为0.85-1.0,电商渠道可低至0.8
   - 交通违法系数:无严重违章为1.0,有违章1.1-1.3

#### 5.2.4 接口设计

**查询险种列表**:

- 路径: GET /api/car/insurance/typeList
- 参数: category(险种类别,可选)
- 返回: 险种列表(包含保障说明、保额选项等)

**查询推荐方案**:

- 路径: GET /api/car/insurance/recommendPlans
- 参数: vehicleId(车辆ID)
- 返回: 推荐方案列表(经济型、标准型、尊享型)

**计算保费**:

- 路径: POST /api/car/insurance/calculatePremium
- 参数: vehicleId(车辆ID), insuranceTypes(选择的险种和保额)
- 返回: 各险种保费明细、总保费

**保存险种选择**:

- 路径: POST /api/car/insurance/savePlan
- 参数: vehicleId(车辆ID), insuranceTypes(选择的险种和保额), planType(方案类型)
- 返回: 选择记录ID

#### 5.2.5 技术实现要点

1. **费率计算**:
   - 费率表:与各保险公司对接,获取最新费率数据
   - 系数计算:NCD系数、自主渠道系数、违章系数等
   - 组合优惠:批量购买、老客户续保等优惠
   - 实时更新:保费随险种选择实时计算
2. **智能推荐算法**:
   - 车辆画像:车龄、车价、品牌等级
   - 用户画像:驾龄、出险历史、消费能力
   - 协同过滤:相似用户的选择偏好
   - 规则引擎:根据业务规则推荐,如新车必推车损险
3. **数据缓存**:
   - 险种列表:缓存1小时,key=`insurance:type:list`
   - 推荐方案:缓存30分钟,key=`insurance:plan:{vehicleId}`
   - 费率数据:缓存1天,key=`insurance:rate:{insurerCode}`
4. **异常处理**:
   - 费率数据不存在:提示"暂无报价,请联系客服"
   - 计算超时:前端loading,后端异步计算
   - 保险公司接口异常:使用历史数据或默认费率

#### 5.2.6 注意事项

- 交强险为强制险种,必须购买
- 车损险保额不得超过车辆实际价值
- 三者险保额根据当地经济水平和车辆用途选择
- 附加险需在主险基础上购买,不可单独购买
- 险种选择后可随时调整,以最终下单时为准
