### 2.1 寿险产品列表（C端）

> **📎 PDF 参考**：
> - `PDF-168`：寿险-产品管理-目录（确认险种分类枚举）
> - `PDF-170`：寿险-产品管理-寿险产品管理（产品录入字段：名称、险种、保险公司、保障期限、缴费期）
> - `PDF-084`：非车目录（C端产品列表组件为复用基准）

#### 2.1.1 页面入口

商城首页底部 Tab 栏或顶部导航新增「寿险」Tab（与"车险""非车险"Tab 并列，图标风格保持统一）。点击后进入寿险产品列表页。该页面**复用非车险 C端产品列表组件（参见 `PDF-084`）**，在此基础上扩展寿险专有字段和筛选器。

#### 2.1.2 列表展示逻辑

页面加载时调用 `GET /app-api/ins/life/product/page`，默认分页 10 条，按热销权重降序排列。

每张产品卡片展示以下信息：

| 展示字段 | 数据来源字段 | 补充说明 |
|---------|------------|---------|
| 险种图标 | `ins_life_product.category_icon` | OSS图片URL，宽高 48×48px |
| 保险公司Logo | 关联 `ins_insurer.logo_url` | 宽高 32×32px，右上角展示 |
| 产品名称 | `ins_life_product.product_name` | 超出2行截断，完整名称显示在产品详情 |
| 亮点文案 | `ins_life_product.highlight_list` | JSON数组，最多展示3条，每条不超过20字，超出截断加"…" |
| 参考保费 | `ins_life_product.reference_premium_desc` | 纯文案字段，不实时计算，如"30岁男性，20年期，月缴¥328起" |
| 保障期限 | `ins_life_product.coverage_period` | 如"保至终身"/"保至70岁" |
| 热销角标 | `ins_life_product.is_hot = 1` | 橙色"热销"角标，产品卡片右上角绝对定位 |

**空数据处理**：若筛选后列表为空，展示空状态图+文案"暂无符合条件的产品，换个条件试试"，并显示「清空筛选」按钮重置条件。

**下拉加载更多**（采用下拉加载分页，非翻页）：
- 首次加载第1页（`page_no=1, page_size=10`）。
- 滚动至列表底部时，自动触发加载下一页，内容追加到列表末尾。
- 加载中展示骨架屏；加载完毕显示"已经到底了"。

#### 2.1.3 筛选条件

> **📎 PDF 参考**：`PDF-170` 寿险产品管理（险种分类、投保年龄、缴费期均来自产品配置）

列表页顶部提供筛选栏：

| 筛选字段 | 说明 | 入参字段名 | 枚举值 |
|---------|------|-----------|------|
| 险种分类 | 寿险产品大类 | `category_code` | `CRITICAL`（重疾）/ `LIFE`（寿险）/ `MEDICAL`（医疗）/ `ACCIDENT`（意外）/ `SAVING`（储蓄） |
| 年龄段 | 被保人适合年龄 | `age_range` | `AGE_0_17` / `AGE_18_40` / `AGE_41_60` / `AGE_60_PLUS` |
| 缴费期 | 缴费年限筛选 | `payment_period` | `SINGLE`（趸缴）/ `Y10`（10年）/ `Y20`（20年）/ `Y30`（30年）/ `WHOLE_LIFE`（终身） |
| 保障额度 | 保额区间筛选 | `coverage_amount_range` | `LT10`（10万以下）/ `W10_50`（10-50万）/ `W50_100`（50-100万）/ `GT100`（100万以上） |

**筛选交互规则**：
- 选择任一条件后防抖 300ms 实时刷新（无需点击查询）。
- 同一维度支持**多选**，多值取**并集**；不同维度之间取**交集**。
- 顶部展示已选条件 Tag，点击 Tag 上的 × 可单独取消；「重置」按钮清空所有筛选。
- 切换 Tab 时所有筛选条件重置为默认（全部不选）。

#### 2.1.4 后端接口逻辑

**接口**：`GET /app-api/ins/life/product/page`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| `category_code` | `String[]` | 否 | 险种分类，支持多值 |
| `age_range` | `String[]` | 否 | 年龄段，支持多值 |
| `payment_period` | `String[]` | 否 | 缴费期，支持多值 |
| `coverage_amount_range` | `String[]` | 否 | 保障额度区间，支持多值 |
| `page_no` | `Integer` | 是 | 页码，从1开始 |
| `page_size` | `Integer` | 是 | 每页条数，默认10，最大20 |

**处理步骤**：

1. 基础查询条件：`status = 1`，`deleted = 0`。

2. **筛选条件转换规则**（MyBatis-Plus 动态 SQL）：

   **年龄段筛选** → 转换为对产品投保年龄字段的范围重叠判断：

   | age_range 值 | 对应 SQL 条件 |
   |------------|-------------|
   | `AGE_0_17` | `min_insure_age <= 17` |
   | `AGE_18_40` | `min_insure_age <= 40 AND max_insure_age >= 18` |
   | `AGE_41_60` | `min_insure_age <= 60 AND max_insure_age >= 41` |
   | `AGE_60_PLUS` | `max_insure_age >= 60` |

   多个年龄段之间取 OR（并集），示例：
   ```sql
   AND (
     (min_insure_age <= 40 AND max_insure_age >= 18)
     OR (max_insure_age >= 60)
   )
   ```

   **缴费期筛选** → JSON 字段包含查询：
   ```sql
   AND JSON_CONTAINS(payment_period_list, '"10年"')
   -- 多值 OR 拼接
   ```

   **保障额度筛选** → 范围重叠判断（单位：元）：

   | coverage_amount_range | 对应 SQL 条件 |
   |-----------------------|-------------|
   | `LT10` | `min_coverage_amount < 100000` |
   | `W10_50` | `min_coverage_amount < 500000 AND max_coverage_amount >= 100000` |
   | `W50_100` | `min_coverage_amount < 1000000 AND max_coverage_amount >= 500000` |
   | `GT100` | `max_coverage_amount >= 1000000` |

3. **排序**：`ORDER BY is_hot DESC, sort_weight DESC`。

4. **返回字段**（VO）：`product_id`、`product_name`、`category_code`、`category_icon`、`highlight_list`、`reference_premium_desc`、`coverage_period`、`is_hot`、`insurer_name`、`insurer_logo_url`。

5. **Redis 缓存**：
   - Key：`ins:life:product:list:{md5(全部筛选参数 + page_no)}`
   - TTL：5 分钟
   - 主动清除时机：PC 后台产品上下架、修改权重、修改分类时触发缓存清除（`@CacheEvict` 注解或手动 `redisTemplate.delete`）。

6. **异常处理**：
   - 枚举参数非法：返回 HTTP 400 + 错误码 `PARAM_INVALID`。
   - 数据库超时：返回 HTTP 500，前端展示"加载失败，点击重试"。

#### 2.2.3 收藏功能接口逻辑

**接口**：`POST /app-api/ins/life/product/favorite`（需登录）

请求体：`{"product_id": 12345}`

后端逻辑（Toggle 模式）：
1. 查询 `ins_member_favorite`，条件：`member_id = 当前用户` AND `product_id = {product_id}` AND `product_type = 'LIFE'` AND `deleted = 0`：
   - **记录不存在**：`INSERT` 新收藏记录 → 返回 `{"is_favorite": true}`。
   - **记录已存在**：`UPDATE deleted = 1`（逻辑删除，取消收藏）→ 返回 `{"is_favorite": false}`。
2. 清除 Redis 缓存：Key `ins:member:favorite:{member_id}`。

**未登录时的前端处理**：检测到 `SecurityFrameworkUtils.getLoginUserId() == null` 时，前端弹出微信授权弹窗，授权成功后自动补发收藏请求（登录回调中携带原始 `product_id`）。

**`ins_member_favorite` 表**（所属 `intermediary-module-member`，非车险/寿险通用）：

#### 2.2.4 用户评价接口逻辑

**接口**：`GET /app-api/ins/life/product/{id}/reviews`（无需登录）

处理逻辑：
1. 查询 `ins_life_product_review`，条件：`product_id = {id}` AND `status = 1`（已发布）AND `deleted = 0`。
2. 按 `create_time` 降序排列，分页返回（默认 `page_size=10`）。
3. **昵称脱敏**：取姓名/昵称第一个字 + `**`（如"张**"）；若昵称为英文则取首字母 + `**`。
4. 返回字段：`nickname_masked`、`rating`（1-5星）、`content`（最多200字）、`create_time`（格式 `yyyy-MM-dd`）、`reply`（商家回复，可为空）。

**`ins_life_product_review` 表**：

```sql
CREATE TABLE `ins_life_product_review` (
  `id`          BIGINT       NOT NULL AUTO_INCREMENT,
  `product_id`  BIGINT       NOT NULL COMMENT '产品ID',
  `member_id`   BIGINT       COMMENT '评价用户ID',
  `policy_id`   BIGINT       COMMENT '关联保单ID（投保成功后可评价）',
  `nickname`    VARCHAR(50)  COMMENT '昵称（明文存储，返回时脱敏）',
  `rating`      TINYINT      NOT NULL COMMENT '评分：1-5星',
```

#### 2.3.4 后端接口逻辑

**接口**：`POST /app-api/ins/life/plan/request`

**请求体**：
```json
{
  "age": 30,
  "gender": "M",
```

**Step 1：参数校验**

| 参数 | 规则 | 错误码 |
|-----|------|-------|
| `age` | 整数，18-65 | `PARAM_AGE_OUT_OF_RANGE` |
| `gender` | M 或 F | `PARAM_GENDER_INVALID` |

频率限制检查（见 §2.3.2）。

**Step 2：简版推荐产品计算**

```sql
SELECT id, product_name, reference_premium_desc, highlight_list, coverage_period
FROM ins_life_product
WHERE status = 1
  AND deleted = 0
  AND min_insure_age <= #{age}
  AND max_insure_age >= #{age}
  AND min_coverage_amount <= #{expectedCoverage * 10000}
  AND max_coverage_amount >= #{expectedCoverage * 10000}
ORDER BY is_hot DESC, sort_weight DESC
LIMIT 3
```

**Step 3：同步返回推荐结果**（不等待 PDF 生成）：

```json
{
  "lead_id": 9001,
  "recommended_products": [
    {
      "product_id": 100,
      "product_name": "XX守护百万重疾险",
      "reference_premium_desc": "30岁男性，20年期，月缴¥328起",
      "highlight_list": ["保额最高300万", "重疾轻症均赔"],
      "coverage_period": "保至终身"
    }
  ],
  "email_sent": true
}
```

**Step 4：写入潜在客户记录**（`ins_life_plan_lead` 表）：
- `member_id`：`SecurityFrameworkUtils.getLoginUserId()`，未登录为 `null`。
- `open_id`：从微信授权会话取，未授权为 `null`。
- `agent_id`：Redis 归因缓存取（Key `ins:trace:member:{member_id}` 或 `ins:trace:openid:{open_id}`）。
- `recommended_ids`：JSON 数组，记录推荐的产品ID列表。

**Step 5：异步触发 PDF 生成**（仅当 `email != null`）：

向 RocketMQ 发送消息：
- Topic：`ins_life_plan_pdf_generate`
- Tag：`GENERATE`
- Body：`{lead_id, age, gender, expected_coverage, email, recommended_product_ids, member_id}`

**PDF 生成消费者**（`InsLifePlanPdfConsumer`，位于 `intermediary-module-ins-life`）：

```
幂等保护：
  先检查 ins_life_plan_lead.pdf_status，若 != PENDING(0) 则跳过
  Redis 分布式锁：Key=ins:pdf:lock:{lead_id}，TTL=10分钟

处理流程：
  1. 更新 pdf_status = GENERATING(1)
  2. Feign 调用 InsAiPlanApi.generatePlanPdf()
     入参：{age, gender, expected_coverage, recommended_product_ids}
     返回：PDF字节流 或 OSS临时上传地址
  3. 上传PDF至OSS（调用 intermediary-module-infra FileApi）
  4. 更新 ins_life_plan_lead：pdf_url=OSS地址，pdf_status=DONE(2)
  5. 发送邮件（携带PDF附件，主题："您的[产品名]保障规划书"）
  6. 发送站内信（ins_member_message，message_type=PLAN_PDF_READY）
  7. 若 agent_id 不为空，触发业务员通知（§2.3.5）

异常处理：
  任一步骤失败 → pdf_status=FAILED(3)，retry_count+1
  若 retry_count < 3：RocketMQ 延迟重试消息（延迟5分钟）
  若 retry_count >= 3：写入告警日志，触发企业微信 Webhook 通知运维
```

**邮件发送失败不影响主流程**：PDF 仍写入数据库，用户可通过"我的消息"中心找到计划书下载链接。

### 3.2 数据安全与脱敏规则

| 数据类型 | 存储 | 展示 |

1. 展示"该产品已下架"提示页（不跳转 404）。
2. 自动调用 `GET /app-api/ins/life/product/page?category_code={原险种}&page_size=3` 推荐3个同类在售产品。
3. 展示推荐产品卡片，引导用户查看相似产品。
