## 1. 个险保单录入

> **对应 PDF：`[PDF-110]`**

### 1.1 入口与整体流程

菜单路径：寿险 → 保单管理 → 个险保单 → 保单录入

点击【保单录入】按钮，右侧或新页面弹出多步骤表单（基本信息 → 关系人信息 → 险种信息 → 缴费信息），每步骤完成后点【下一步】，最后一步点【提交】完成录入。

> **`[PDF-110]` 原文摘要：** "点击【保单录入】打开填写界面，标星内容为必填项。关系人信息可以直接点【选择客户】，选择系统中已维护好的客户信息，点击保存，也可以直接填写客户信息。"

### 1.2 第一步：基本信息

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 险种 | ★必填 | 下拉枚举：寿险/重疾/医疗/意外/年金/万能险，从 `ins_life_product_type` 枚举加载 |
| 保险公司 | ★必填 | 下拉，从已配置的寿险合作保司中选取（需在产品-合作保司管理授权） |
| 产品名称 | ★必填 | 联动险种+保司，下拉选择已上架产品，`ins_life_product` 表中 `status=1` 的记录 |
| 保单号/投保单号 | ★必填 | 手动输入；后端去重校验：同一保司+保单号不允许重复，重复则返回错误"保单号已存在" |
| 起保日期（保险起期） | ★必填 | 日期选择器 |
| 到期日期（保险止期） | ★必填 | 日期选择器；后端校验：到期日期须 > 起保日期 |
| 缴费方式 | ★必填 | 枚举：趸缴/年缴/半年缴/季缴/月缴 |
| 缴费期间 | ★必填 | 数字输入，单位年；趸缴时固定为1年，其他缴费方式手动输入 |
| 年度保费（首期保费） | ★必填 | 金额，精度两位小数，单位元 |
| 续期保费 | 选填 | 金额，精度两位小数 |
| 缴费频率 | 选填 | 一般与缴费方式联动自动带出（年=1次/年，月=12次/年等） |
| 件数状态（业务类型） | 选填 | 枚举：新单/续保/转保 |
| 保单服务人（录入业务员） | ★必填 | 默认当前操作人；管理员可修改为其他在职业务员 |
| 保单销售人 | 选填 | 销售员，可与服务人不同 |
| 签单日期/投保日期 | 选填 | 默认等于起保日期 |
| 承保日期 | 选填 | 保司实际承保日期 |
| 组织名称/组织代码 | 选填 | 关联机构信息 |
| 是否复效保单 | 选填 | 布尔值 |
| 备注 | 选填 | 文本，最多500字 |

**后端校验逻辑（第一步提交时）：**
- 保单号 + 保险公司组合唯一性校验（查 `ins_policy_life` 表）
- 起保日期/到期日期格式校验
- 年度保费 > 0

### 1.3 第二步：关系人信息（投保人 & 被保人）

> **`[PDF-110]` 原文摘要：** "关系人信息可以直接点【选择客户】，选择系统中已维护好的客户信息，点击保存，也可以直接填写客户信息。被保人选择客户后，需要点击添加，在下方出现一行客户信息，才是添加被保人成功。"

**投保人（必须有且仅有一个）：**

点击【选择客户】弹出客户搜索弹窗 → 输入姓名/证件号搜索 → 选中后自动填充如下字段；也可手动填写。

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 投保人姓名 | ★必填 | |
| 证件类型 | ★必填 | 枚举：身份证/护照/港澳通行证 |
| 证件号码 | ★必填 | 存储时 AES-256 加密（复用系统已有加密工具类）；展示时脱敏（前4后4，中间***） |
| 出生日期 | ★必填 | 身份证号可自动解析带出 |
| 性别 | ★必填 | 单选，可从身份证解析 |
| 联系电话 | 选填 | 脱敏展示 131****1111 |
| 年龄 | 选填 | 可由出生日期自动计算 |
| 住址 | 选填 | |
| 证件是否长期有效 | 选填 | 布尔值 |
| 证件有效期 | 条件选填 | 证件非长期有效时填写 |

**被保人（至少一个，可多个）：**

点击【添加被保人】按钮，出现一行被保人录入区。每个被保人字段与投保人类似，额外增加：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 与投保人关系 | ★必填 | 枚举：本人/配偶/子女/父母/其他 |
| 受益人类型 | 选填 | 枚举：法定受益人/指定受益人 |
| 受益人信息 | 条件必填 | 若受益人类型=指定受益人，则受益人姓名、证件号为必填 |

> ⚠️ **关键交互（来自 `[PDF-110]`）：** 被保人选择客户后，**必须点击【添加】**，下方出现一行客户信息才算添加成功。前端需强制校验此操作，避免用户填了不点添加就提交。

**自动客户匹配逻辑（后端）：**
- 被保人/投保人提交时，按证件号（解密后精确匹配）查询 `ins_customer` 表
- 匹配到则更新 `ins_policy_life.customer_id`；未匹配则自动创建新客户记录（`ins_customer` 插入新行），再关联

### 1.4 第三步：险种信息

> **`[PDF-110]` 原文摘要：** "点击添加险种，选择产品名称，填写产品详情，完成后需要点击保存险种（注：下方生成一行险种信息，才是保存完成）"

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 险种分类（主/附加险标志） | ★必填 | 枚举：主险/附加险 |
| 险种名称 | ★必填 | 关联产品库选择 |
| 产品代码 | ★必填 | 自动带出 |
| 保额 | ★必填 | 金额 |
| 险种保费 | ★必填 | 该险种对应保费 |
| 保险期间 | 选填 | 如终身/20年等 |
| 缴费期 | 选填 | |

> ⚠️ **关键交互（来自 `[PDF-110]`）：** 险种填写完成后**必须点击【保存险种】**，下方生成一行险种信息才算保存完成。支持多个险种（主险+多个附险）。

### 1.5 第四步：缴费信息

> **`[PDF-110]` 原文摘要：** "缴费信息标星项必填，所有信息都填好后，点提交即保存成功。"

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 上游标准保费 | ★必填 | 用于上游结算计算基础 |
| 下游首年标准保费 | 选填 | 用于下游结算 |
| 下游续期标准保费 | 选填 | |
| 上游折标系数 | 选填 | 百分比，默认100% |
| 下游首年折标系数 | 选填 | |
| 价值保费 | 选填 | |
| 产品代码 | 选填 | |

点击【提交】按钮后：
1. 后端完整校验所有步骤数据
2. 事务内执行：
   - INSERT `ins_policy_life`（保单主表）
   - INSERT `ins_policy_life_insured`（被保人/受益人表，可多行）
   - INSERT `ins_policy_attachment`（附件表）
   - 创建回执记录 `ins_policy_receipt`（`receipt_status=0`，待回执）
   - 触发 MQ 消息（`InsOrderProducer.publishPolicyCreated`），异步触发佣金预计算
3. 返回成功，前端弹出"录入成功"提示，跳转到保单列表

### 1.6 附件上传（保单详情页）

支持分类上传以下类型的附件，每类最多 5 个文件，文件大小限制单个 ≤ 20MB：
- 保单首页
- 缴费通知单
- 健康告知书
- 体检报告
- 其他材料

上传方式：点击对应分类的【上传】按钮 → 选择文件 → 前端直传 OSS，获取 OSS 路径后提交。

## 2. 个险保单查询与导出

> **对应 PDF：`[PDF-111]`（保单查询）、`[PDF-113]`（保单导出）**

### 2.1 搜索条件

> **`[PDF-111]` 原文摘要：** "点击【查询】可根据筛选条件查到系统内已存在的保单。筛选出的保单在页面最下方显示汇总的总保费。若系统设置了对应产品的折标政策，并且在保单录入时匹配上了，就会一起汇总上下游标准保费。默认一页展示 10 条保单，可以选择更多条进行展示。"

列表页顶部展示筛选区，支持以下条件（均为选填，支持组合查询）：

| 筛选项 | 类型 | 说明 |
|---|---|---|
| 日期类型 | 下拉 | 投保日期/承保日期/起保日期/到期日期，控制下方日期区间的查询基准 |
| 投保日期范围 | 日期区间 | 与"日期类型"联动 |
| 保单类型 | 下拉 | 全部/个险/团险 |
| 机构选择 | 下拉树 | 可按机构层级筛选 |
| 保险公司 | 下拉多选 | |
| 产品名称 | 下拉搜索 | 可多个代码分号隔开 |
| 保单号 | 文本输入 | 支持多个保单号用逗号分隔 |
| 投保单号 | 文本输入 | |
| 人员类型 | 下拉 | 投保人/被保人/保单服务人/保单销售人 |
| 人员姓名/证件号 | 文本输入 | 与"人员类型"联动 |
| 保单状态 | 下拉多选 | 有效/失效/减额缴清/退保/延期缴费/保单贷款/中止/终止 |
| 结算状态 | 下拉 | 未结算/已结算 |
| 是否复效保单 | 下拉 | 全部/是/否 |
| 组织名称 | 文本 | 模糊匹配 |
| 组织代码 | 文本 | |

点击【查询】发起 POST 请求，分页返回（默认10条/页，可切换20/50/100/200条）。

**列表底部展示汇总行：**
- 总保费合计
- 上游标准保费合计
- 下游标准保费合计（若有折标政策）

### 2.2 列表字段与操作按钮

> **`[PDF-111]` 原文摘要：** "每个保单后面的操作按钮，根据保单状态不同可操作的按钮不同：
> - 已承保未回执或未回访的保单显示：明细，去回访/去回执，保单日志
> - 中止/终止的保单显示：明细，保单日志
> - 退保的保单显示：明细，已退保，保单日志"

默认显示列：保单号/投保单号、机构名称、保单销售人员、保单服务人员、投保日期、承保日期、保险公司、险种、年度保费、缴费方式、保单状态、操作。

**操作列按保单状态动态渲染：**

| 保单状态 | 可用操作按钮 |
|---|---|
| 已承保-待回执 | 明细、去回执、保单日志 |
| 已承保-待回访 | 明细、去回访、保单日志 |
| 中止 / 终止 | 明细、保单日志 |
| 退保 | 明细、已退保（灰显）、保单日志 |
| 其他正常状态 | 明细、修改、保单日志 |

- **明细**：点击跳转保单详情页，查看险种、关系人信息等完整信息
- **去回执**：进入回执填写页，填写回执日期完成回执操作
- **去回访**：进入回访填写页，填写回访记录
- **修改**：进入保单编辑页（见第9节）
- **保单日志**：查看该保单的所有操作变更记录

### 2.3 自定义列设置

点击列表右上角【列设置】按钮 → 弹出字段选择弹窗 → 用户勾选需要显示的列 → 点保存 → 前端将配置写入用户偏好（接口：`/admin-api/ins/life/policy/column-config`，按当前用户 ID 存储）。

可选字段包括（分组展示）：
- **保单基本信息**：销售机构名称、保单销售团队、保单销售人员、保单销售人员代码、保单销售人员工号、服务人员代码、服务人员、保单服务人员工号、保单服务人员手机号、投保单号、保单号、保单年度、投保日期、承保日期、生效日期、保单类型、是否自保件
- **险种信息**：保险公司、产品名称、主附险标志、保额、首期保费、续期保费、缴费年期、缴费方式、上游标准保费、上游折标系数(%)、下游首年标准保费、下游续期标准保费、产品代码、价值保费、保险期间
- **投保人信息**：投保人姓名、电话、证件类型、证件号、年龄、出生日期、住址、证件是否长期有效、证件有效期、教育程度、政治面貌
- **被保人信息**：被保人姓名、证件类型、证件号码等

### 2.4 批量导出

> **`[PDF-113]` 原文摘要：** "点击【导出】可以导出当前页的保单详情。点击【+自定义模板】，可以对已有的自定义模板编辑或禁用，根据自己的需求增加模板"

【导出】按钮功能：
1. 点击展开导出方式选择：**默认模板** 或 **自定义模板**
2. 导出范围：导出当前查询条件下所有记录（不受分页限制）
3. 数量 ≤ 2000条：同步生成 Excel，直接下载
4. 数量 > 2000条：异步任务，使用 EasyExcel 分批写入，任务完成后写入 `ins_export_task` 表，跳转到【导出任务列表】，可操作【下载】或【删除】
5. 导出字段中证件号**脱敏**（仅显示首4后4，中间用***代替）

**自定义模板管理：**
- 点击【+自定义模板】进入模板配置页
- 可为模板命名，然后在"未选字段"中勾选需要的字段
- 字段分组同 2.3 节的列设置字段分组
- 配置保存后，导出时可选择该模板
- 支持对已有模板【编辑】或【禁用】

## 3. 批量导入保单

> **对应 PDF：`[PDF-112]`**

### 3.1 操作流程

> **`[PDF-112]` 原文摘要：** "点击【批量导入】-【模版下载】下载一个系统模板，按照系统模板提示填写保单信息，标星内容是必填的，其余内容选填。红色角标点击会弹出提示信息，按照示例格式进行填写。"

1. 点击列表页【批量导入】按钮 → 弹出导入向导弹窗
2. 点击【模版下载】下载系统标准 Excel 模板（模板内不同险种可共用同一模板）
3. 用户在本地填写 Excel：
   - 标星（`*`）内容为必填，其余选填
   - 红色角标有填写提示说明，按示例格式填写
   - **注意：模板内任何示例信息都不能删除，从第6行开始填写实际数据**
4. 返回系统点击【选取文件】上传（限制：xlsx格式，≤5000行，文件大小≤20MB）
5. 上传后开始导入，若有错误则弹出报错窗口
6. 点击【下载校验文件】查看具体错误行和错误原因
7. 根据提示修改原表格后重新导入

### 3.2 后端导入逻辑

1. 读取 Excel（EasyExcel）按行解析，每行映射到 `InsLifePolicyImportDTO`
2. 逐行校验：必填字段为空、保单号格式、日期格式、金额格式
3. 对校验通过的行：
   - 查重：保单号+保险公司已存在则标记为"重复"，不插入
   - 自动匹配客户（同录入逻辑，按证件号精确匹配）
   - 批量 INSERT，每批500条（MyBatis Plus `saveBatch`），使用事务
4. 全部处理完毕后生成导入报告：
   - 成功N条、失败N条（含原因）、重复N条
   - 报告写入 `ins_import_log` 表
5. 下载校验文件：原Excel最后列追加"校验不通过数据列（请修改完删除此列）"，标注每行错误说明

### 3.3 导入模板字段（核心字段）

模板表头（标星为必填）：保单号*、险种*、保险公司*、产品名称*、起保日期*、到期日期*、年度保费（首期保费）*、缴费方式*、缴费期间*、投保单号、承保日期、投保人姓名*、投保人证件号*、被保人姓名*、被保人证件号*、被保人出生日期*、被保人性别*、保单服务人员（工号）、保单销售人员（工号）、上游标准保费、下游首年标准保费、是否复效保单

## 4. 保单核对

> **对应 PDF：`[PDF-114]`**

### 4.1 功能说明

将保司下发的对账数据与系统保单数据进行比对，找出差异。支持单条手动触发和批量上传比对。

### 4.2 操作流程

1. 菜单：寿险 → 保单管理 → 个险保单 → 保单核对
2. 点击【导入保司数据】→ 选择保司 → 下载该保司的对账模板 → 填写后上传
3. 上传完成后，点击【开始核对】，后端异步执行比对（数据量大时可能耗时，完成后发站内信通知）
4. 核对完成后展示差异报告页面

### 4.3 差异分类与处理

后端比对逻辑：以保单号 + 被保人证件号（AES解密后比对）为匹配键，逐行比对：

| 差异类型 | 判断逻辑 | 展示颜色 |
|---|---|---|
| 系统有/保司无 | 系统中存在该保单，保司数据中没有 | 橙色 |
| 保司有/系统无 | 保司数据中存在，系统中不存在 | 红色 |
| 金额差异 | 两边都有该保单，但年度保费不一致 | 黄色 |
| 完全一致 | 两边数据一致 | 绿色 |

每条差异记录提供三个处理按钮：
- **接受保司**：以保司数据为准，更新系统保单数据（需记录变更日志）
- **保留系统**：标记该差异为"已处理-保留系统"，不修改数据
- **手动修正**：打开编辑弹窗，手动修改数据后保存

核对记录归档到 `ins_policy_reconcile` 表。核对报告可导出为 Excel。

## 8. 保单状态变更

> **对应 PDF：`[PDF-121]`**

### 8.1 操作入口

菜单：寿险 → 保单管理 → 保单修改 → 状态变更

在保单列表操作列点击【状态变更】按钮，弹出状态变更弹窗。

> **`[PDF-120]` 目录原文：** "保单修改模块包含：状态变更、保单修改、保单删除、批量影像件"

### 8.2 状态流转规则

状态机（`ins_policy_life.policy_status`）：

```
有效（ACTIVE）
  ├── → 失效（LAPSE）          触发：欠费未续缴
  ├── → 减额缴清（REDUCED_PAID）  触发：客户申请停缴，保额等比例缩减
  ├── → 延期缴费（GRACE_PERIOD）  触发：宽限期内
  ├── → 保单贷款（LOAN）         触发：客户以保单现金价值贷款
  ├── → 退保（SURRENDERED）     触发：客户申请退保
  ├── → 中止（SUSPENDED）       触发：约定条件触发中止
  └── → 终止（TERMINATED）      触发：保险期届满
```

非有效状态不可再变更（除非特殊审批）。

### 8.3 变更表单字段

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 目标状态 | ★必填 | 下拉，只展示当前状态可流转到的目标状态 |
| 变更原因 | ★必填 | 文本，最多500字 |
| 生效日期 | ★必填 | 默认今天，可选未来日期（不可选过去日期） |
| 证明材料 | 条件必填 | 退保时必须上传，其他状态选填 |

### 8.4 后端处理逻辑

1. 校验状态流转合法性（不合法返回错误）
2. UPDATE `ins_policy_life.policy_status` = 目标状态，`status_change_date` = 生效日期
3. INSERT `ins_policy_status_log`（old_status, new_status, change_reason, effective_date, operator_id, change_time）
4. 更新续期管理队列（`ins_life_renewal_track`）：若保单失效/退保，将对应续期记录标记为"已结束"
5. **退保特殊处理**：发送 MQ 消息，触发佣金回收计算
6. 非退保变更不影响已计算佣金

### 11.2 孤儿单池列表（保单分配页）

菜单：寿险 → 孤儿单管理 → 保单分配

展示所有未分配（`UNASSIGNED`）或归属离职业务员的保单，列表字段：保单号、被保人、险种、保司、年度保费、原业务员（已离职）、入池时间、状态。

筛选：险种、保司、原保单服务人员（离职人员姓名/工号）、入池日期范围、状态（未分配/已分配）。

### 11.3 分配操作

**单个分配：**

> **`[PDF-133]` 原文摘要：** "选择分配后的保单服务人员，填写收益继承比例，点击【确定】即可完成保单分配。注：此处收益继承比例会影响到被分配人员的佣金。"

1. 点击某行的【分配】按钮
2. 弹出分配弹窗，字段：
   - 接手业务员（保单服务人员）★：下拉搜索，只显示本机构在职业务员
   - **收益继承比例** ★：填写百分比（此处设置会影响被分配人员的佣金计算）
   - 分配备注：文本，选填
3. 点确定后：
   - UPDATE `ins_policy_life.agent_id` = 新业务员ID
   - UPDATE `ins_life_orphan.status = ASSIGNED`、`assigned_agent_id` = 新业务员ID、`assigned_time` = now()
   - INSERT `ins_life_orphan_log`（policy_id, from_agent_id, to_agent_id, inherit_ratio, operator_id, assign_time）

**批量分配：**

> **`[PDF-133]` 原文摘要：** "可以根据查询条件全选，批量进行分配保单。选择之前的保单服务人员，选中要分配的保单，点击分配。"

1. 根据查询条件筛选目标保单，可全选当前查询结果
2. 勾选多个孤儿单（同一机构下）
3. 点击【批量分配】按钮
4. 弹窗选择接手业务员 + 收益继承比例
5. 后端循环处理，每条保单单独记录分配轨迹
6. 全部处理在同一个事务中（任意失败则回滚）

### 11.4 分配轨迹查询

> **`[PDF-134]` 原文摘要：** "根据查询条件查询保单的分配轨迹，例如输入保单号可以查询到该保单的操作记录。导出：将查询的保单明细操作导出，操作导出后跳转到任务列表，可操作【下载】或【删除】。"

菜单：寿险 → 孤儿单管理 → 分配轨迹

筛选：保单号、被保人姓名、原业务员、接手业务员、分配时间范围。

列表展示 `ins_life_orphan_log` 所有记录，每条记录可查看完整的分配历史（一个保单可能被多次分配）。

**新增保单分配入口（来自 `[PDF-134]`）：**
- 在分配轨迹页也提供【新增】按钮
- 点击后跳转至保单分配模块，对保单进行具体分配操作

**导出功能（来自 `[PDF-134]`）：**
- 点击【导出】将查询的保单分配明细导出
- 导出后自动跳转到**导出任务列表**
- 在任务列表可操作：
  - 【下载】：下载生成的 Excel 文件，打开可查看分配详情
  - 【删除】：删除此导出批次记录
