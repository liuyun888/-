### 5.3 在线报价

**开发工时**: 前端1天 + 后端2天

#### 5.3.1 功能概述

基于用户选择的车辆信息和险种方案,实时调用保险公司接口进行在线报价,展示各保险公司的保费明细和优惠信息,帮助用户对比选择最优方案。

#### 5.3.2 核心业务逻辑

**报价流程**:

1. 1. **报价触发**:

   - • 用户完成车辆信息录入和险种选择
   - • 点击"立即报价"按钮触发
   - • 系统校验:车辆信息完整性、险种选择合理性
   - • 校验通过后进入报价环节

2. 2. **多家公司报价**:

   - • 并发调用3-5家保险公司报价接口
   - • 保险公司范围:人保、平安、太保、国寿、太平洋等
   - • 报价请求参数:
     - • 车辆信息:车牌号、VIN码、品牌车型、购车价格
     - • 投保人信息:姓名、身份证、联系方式
     - • 险种信息:交强险、商业险各险种及保额
     - • 上年投保:上年保险公司、出险次数
   - • 超时处理:单家公司超时5秒则跳过
   - • 失败重试:失败自动重试1次

3. 3. **报价结果处理**:

   - • 接收各保险公司返回的报价
   - • 解析保费明细:
     - • 交强险保费
     - • 车损险保费
     - • 三者险保费
     - • 其他附加险保费
     - • 总保费(含税)
   - • 保费对比:按总保费从低到高排序
   - • 优惠信息:新客优惠、续保优惠、平台优惠等

4. 4. **报价展示**:

   - • 列表展示各保险公司报价
   - • 每个报价卡片包含:
     - • 保险公司logo和名称
     - • 总保费(大字突出)
     - • 保费明细(可展开查看)
     - • 优惠标签(如"新客立减100元")
     - • "立即投保"按钮
   - • 最优报价标记:"推荐"或"最低价"角标

**保费计算逻辑**:

1. 1. **交强险保费**:

   - • 基础保费(6座以下950元,6座以上1100元)

   - • NCD系数:

     ```
     首次投保或上年赔付: 1.0
     上年未赔付: 0.9
     连续2年未赔付: 0.8
     连续3年及以上未赔付: 0.7
     ```

   - • 违章系数:酒驾等严重违章上浮10%-30%

   - • 最终保费 = 基础保费 × NCD系数 × 违章系数

2. 2. **商业险保费**:

   - • 车损险保费:

     ```
     基准保费 = 车辆购置价 × 基准费率 + 固定保费
     NCD系数:根据上年出险次数(0次0.7,1次1.0,2次1.25...)
     车型系数:不同车型风险不同,系数0.8-1.5
     车龄系数:车龄越高系数越低,新车1.0,5年车0.85
     最终保费 = 基准保费 × NCD系数 × 车型系数 × 车龄系数 × 渠道系数
     ```

   - • 三者险保费:

     ```
     基准保费:根据保额和地区费率表查询
     例如:北京地区,100万保额基准保费约1800元
     NCD系数:同车损险
     最终保费 = 基准保费 × NCD系数 × 渠道系数
     ```

   - • 附加险保费:

     ```
     各附加险有独立的基准保费和系数
     例如:划痕险基准保费=车价×0.002×保额系数
     ```

3. 3. **优惠折扣**:

   - • 渠道系数:线上渠道0.85,线下1.0
   - • 平台补贴:首单立减100元,老客户返现50元
   - • 保险公司优惠:各公司自有优惠活动
   - • 组合优惠:购买多个险种享受折扣
   - • 总保费 = 各险种保费之和 - 优惠金额

**保费明细**:

报价结果需展示详细的保费构成,包括:

```
交强险: 665.00元
  └─ 基础保费: 950.00元
  └─ NCD系数折扣: -285.00元(0.7倍)

商业险: 3,245.00元
  ├─ 车损险: 1,850.00元
  │   └─ 基准保费: 2,200.00元
  │   └─ NCD折扣: -350.00元(0.7倍)
  │   └─ 车龄折扣: -100.00元
  ├─ 三者险(100万): 1,260.00元
  │   └─ 基准保费: 1,800.00元
  │   └─ NCD折扣: -540.00元(0.7倍)
  └─ 座位险(司机5万+乘客1万×4): 135.00元

平台优惠: -100.00元(新客专享)

总保费: 3,810.00元
```

**报价缓存**:

1. 1. **缓存策略**:

   - • 相同车辆+险种方案的报价缓存30分钟
   - • Redis Key: `car:quote:{vehicleId}:{planHash}`
   - • planHash: 险种方案的MD5值
   - • 缓存命中率统计,优化报价体验

2. 2. **缓存更新**:

   - • 用户修改险种选择时清除缓存
   - • 车辆信息变更时清除该车辆所有报价缓存
   - • 定时任务每小时清理过期缓存

**异常处理**:

1. 1. **保险公司接口异常**:

   - • 超时:5秒超时后跳过该公司
   - • 报价失败:显示"该公司暂时无法报价"
   - • 降级方案:使用历史平均保费估算
   - • 错误日志:记录接口调用异常,供运营排查

2. 2. **数据异常**:

   - • 保费为0或异常高:标记异常,不展示
   - • 险种缺失:提示"该公司不支持部分险种"
   - • 数据校验:保费合理性校验,超出阈值则告警

3. 3. **网络异常**:

   - • 前端重试:报价失败自动重试1次
   - • 降级提示:全部失败时提示"网络异常,请稍后再试"
   - • 客服引导:提供在线客服入口

#### 5.3.5 技术实现要点

1. 1. **并发报价**:

   - • 使用CompletableFuture异步并发调用保险公司接口
   - • 线程池配置:核心线程5,最大线程10
   - • 超时控制:单个接口超时5秒,总超时20秒
   - • 结果聚合:收集所有报价结果,失败的跳过

2. 2. **接口对接**:

   - • 各保险公司提供统一的报价接口规范
   - • 请求加密:使用保险公司提供的公钥加密敏感信息
   - • 签名验证:对请求参数进行签名,防篡改
   - • 响应解析:统一解析各公司的报价响应格式

3. 3. **保费计算**:

   - • 费率表:预先从保险公司获取费率表,存储到数据库
   - • 本地计算:优先使用本地费率表计算,减少接口调用
   - • 实时校验:本地计算结果与接口返回结果对比,误差>5%告警
   - • 降级方案:接口异常时使用本地费率表估算

4. 4. **性能优化**:

   - • 报价缓存:相同条件报价缓存30分钟
   - • 预报价:用户选择险种时预先触发报价,不等用户点击
   - • 接口限流:限制单个用户每分钟报价次数<10次
   - • 数据库优化:报价记录定期归档,只保留近3个月数据

5. 5. **监控告警**:

   - • 报价成功率:低于90%触发告警
   - • 接口响应时间:超过3秒告警
   - • 保费异常:保费为0或超高告警
   - • 每日报表:统计各保险公司报价成功率和平均保费

#### 5.3.6 注意事项

- • 报价有效期通常为24小时,超时需重新报价
- • 保费会随时间、政策调整,以最终投保时为准
- • 部分保险公司可能不支持某些险种,需前端提示
- • 报价不等于核保通过,最终需核保确认
- • 敏感信息(如身份证)需加密传输和存储
- • 遵守保险监管要求,不得虚假报价或误导用户

### 5.4 投保信息填写

**开发工时**: 前端1.5天 + 后端1天

#### 5.4.1 功能概述

用户选定报价后,需填写投保人、被保险人、车主等信息,以及上传必要的证件材料,为核保和出单做准备。

#### 5.4.2 核心业务逻辑

**信息采集**:

1. 1. **投保人信息**(必填):

   - • 姓名:中文姓名,2-4个字符
   - • 证件类型:身份证(默认)、护照、军官证
   - • 证件号码:
     - • 身份证:18位,支持OCR识别
     - • 护照:8-9位字母数字组合
   - • 手机号:11位,接收保单和理赔信息
   - • 邮箱:用于接收电子保单
   - • 地址:省市区+详细地址

2. 2. **被保险人信息**:

   - • 与投保人关系:本人、配偶、父母、子女、其他
   - • 如非本人,需填写被保险人姓名和证件号
   - • 企业投保:需上传营业执照

3. 3. **车主信息**:

   - • 与投保人关系:本人、配偶、父母、子女、其他
   - • 如非本人,需填写车主姓名和证件号
   - • 单位车辆:需上传单位证明材料

4. 4. **联系信息**:

   - • 常用地址:省市区三级联动选择
   - • 详细地址:街道门牌号
   - • 电子邮箱:用于接收保单
   - • 备用手机:可选

**证件材料上传**:

1. 1. **投保人证件**:

   - • 身份证正反面照片
   - • 支持拍照上传或相册选择
   - • OCR自动识别:姓名、证件号、地址、有效期
   - • 图片要求:清晰、完整、无遮挡,大小<5MB
   - • 格式要求:JPG、PNG

2. 2. **行驶证**:

   - • 行驶证正副页照片
   - • OCR识别:车牌号、VIN码、品牌型号、注册日期
   - • 自动校验:与录入的车辆信息对比

3. 3. **其他材料**(视情况):

   - • 营业执照(企业投保)
   - • 单位证明(单位车辆)
   - • 关系证明(投保人非车主时)

**信息校验**:

1. 1. **姓名校验**:

   - • 中文姓名:2-4个汉字,不含特殊字符
   - • 少数民族姓名:支持中间点"·"
   - • 英文姓名:仅护照可用,2-20个字母

2. 2. **身份证校验**:

   - • 格式校验:18位,前17位数字,最后一位数字或X
   - • 校验位算法:根据前17位计算第18位校验位
   - • 出生日期:提取并校验格式和范围
   - • 年龄限制:投保人年龄18-70岁
   - • 地区码:前6位为有效行政区划代码

3. 3. **手机号校验**:

   - • 格式:11位,1开头,第二位3-9
   - • 黑名单:检查是否在黑名单
   - • 重复校验:同一手机号不可重复投保

4. 4. **关系校验**:

   - • 投保人与被保险人关系合理性
   - • 投保人与车主关系合理性

**智能填充**:

### 5.5 核保流程

**开发工时**: 前端0.5天 + 后端2天

#### 5.5.1 功能概述

投保信息提交后,系统调用保险公司核保接口,对投保申请进行审核,判断是否接受承保、是否需要加费或除外责任,最终给出核保结论。

#### 5.5.2 核心业务逻辑

**核保流程**:

1. 1. **自动核保**:

   - • 用户提交投保信息后自动触发
   - • 调用保险公司核保接口
   - • 传递参数:
     - • 车辆信息:车牌、VIN码、品牌车型、车龄、违章记录
     - • 投保人信息:姓名、证件号、年龄、职业、信用记录
     - • 历史投保:上年出险次数、赔付金额
     - • 险种方案:投保的险种和保额
   - • 核保规则引擎自动判断

2. 2. **核保条件判断**:

   - • **车辆条件**:
     - • 车龄:一般不超过15年
     - • 违章记录:严重违章(酒驾、肇事逃逸)拒保
     - • 出险历史:上年出险5次以上可能拒保
     - • 黑名单车辆:套牌车、事故车拒保
   - • **人员条件**:
     - • 年龄:投保人18-70岁
     - • 职业:高危职业(如爆破工)可能拒保或加费
     - • 信用:征信不良可能拒保
     - • 黑名单:曾骗保、恶意拒赔的拒保
   - • **历史理赔**:
     - • 赔付率:上年赔付率>120%可能拒保
     - • 重大案件:涉及死亡案件谨慎承保
     - • 频繁出险:短期内多次出险拒保

3. 3. **核保结果**:

   - • **通过**:
     - • 自动出单
     - • 发送通知:短信+APP推送
     - • 跳转支付页面
   - • **加费承保**:
     - • 提示:由于XX原因,保费上浮XX%
     - • 用户确认:是否接受加费
     - • 接受:继续支付,拒绝:取消投保
   - • **除外承保**:
     - • 提示:XX责任除外,不予承保
     - • 用户确认:是否接受除外
     - • 接受:签署除外协议,继续支付
   - • **拒保**:
     - • 提示:由于XX原因,无法承保
     - • 建议:推荐其他保险公司或产品
     - • 记录:拒保记录留档

4. 4. **人工核保**:

   - • 自动核保无法判断的案件转人工
   - • 触发条件:
     - • 车辆或人员信息异常
     - • 出险次数过多但未达拒保标准
     - • 保额过高(如三者险>500万)
   - • 核保员审核:
     - • 查看详细资料
     - • 调取历史记录
     - • 联系用户核实
     - • 给出核保意见:通过、加费、除外、拒保
   - • 核保时限:1-2个工作日

**核保通知**:

1. 1. **短信通知**:

   - • 核保通过:"您的车险投保申请已通过核保,请尽快支付"
   - • 核保拒绝:"抱歉,您的投保申请未通过核保,原因:XX"

2. 2. **APP推送**:

   - • 实时推送核保结果
   - • 点击查看详情

3. 3. **站内信**:

   - • 保存核保通知记录
   - • 用户可随时查看

**风控规则**:

系统内置风控规则引擎,自动识别高风险投保:

1. 1. **异常投保行为**:

   - • 短期内多次投保同一车辆
   - • 投保后立即出险(骗保嫌疑)
   - • 投保金额异常高
   - • IP地址异常(境外IP)

2. 2. **黑名单检查**:

   - • 黑名单车辆库:套牌车、事故车
   - • 黑名单人员库:骗保人员、老赖
   - • 征信黑名单:失信被执行人

3. 3. **数据对比**:

   - • 公安部车辆数据:核实车辆真实性
   - • 保险行业数据:查询历史投保和理赔记录
   - • 征信数据:查询信用状况

   - • 历史记录缓存:用户历史投保和理赔记录缓存

5. 5. **监控告警**:

   - • 核保成功率:低于95%告警
   - • 核保时长:超过5分钟告警
   - • 拒保率异常:突然上升告警
   - • 风控拦截:高风险案件实时告警

#### 5.5.6 注意事项

- • 核保时间通常为数秒到数分钟,复杂案件可能需1-2天
- • 核保结果最终以保险公司为准
- • 拒保不影响用户信用,可向其他公司投保
- • 加费和除外需用户明确确认,不可强制
- • 核保数据需留档备查,保存期限5年以上

### 5.6 支付下单

**开发工时**: 前端1天 + 后端2天

#### 5.6.1 功能概述

核保通过后,用户选择支付方式完成支付,系统调用保险公司出单接口生成电子保单,完成整个投保流程。

#### 5.6.2 核心业务逻辑

**支付流程**:

1. 1. **订单生成**:

   - • 核保通过后自动生成支付订单
   - • 订单信息:
     - • 订单号:唯一,格式YYYYMMDDHHmmss+6位随机数
     - • 商品名称:车险保费(交强险+商业险)
     - • 支付金额:核保通过的保费
     - • 订单有效期:30分钟
   - • 订单状态:待支付

2. 2. **支付方式选择**:

   - • **微信支付**:
     - • H5支付:跳转微信支付页面
     - • 小程序支付:唤起微信支付
     - • 扫码支付:展示付款码
   - • **支付宝支付**:
     - • H5支付:跳转支付宝
     - • APP支付:唤起支付宝
   - • **银行卡支付**:
     - • 快捷支付:绑定银行卡
     - • 网银支付:跳转网银
   - • **余额支付**(企业用户):
     - • 账户余额抵扣
     - • 余额不足可组合支付

3. 3. **支付请求**:

   - • 调用支付渠道接口
   - • 传递参数:
     - • 订单号
     - • 支付金额
     - • 商品描述
     - • 回调URL
   - • 返回支付参数:
     - • 微信:prepay_id, sign等
     - • 支付宝:orderString
   - • 前端唤起支付

4. 4. **支付回调**:

   - • 支付成功后,支付平台回调系统
   - • 验证回调签名:防篡改
   - • 更新订单状态:已支付
   - • 调用出单接口:向保险公司发起出单请求
   - • 返回支付成功页面

5. 5. **出单流程**:

   - • 调用保险公司出单接口
   - • 传递投保数据:车辆、人员、险种、保费等
   - • 保险公司返回:
     - • 保单号
     - • 电子保单PDF
     - • 生效时间
   - • 保存保单信息到数据库
   - • 发送保单:短信+邮件+APP

6. 6. **支付失败处理**:

   - • 支付超时:订单30分钟未支付自动取消
   - • 支付失败:提示重新支付或更换支付方式
   - • 余额不足:提示充值或组合支付

**支付安全**:

1. 1. **订单防篡改**:

   - • 订单金额加密:前端不可修改
   - • 签名验证:所有请求验证签名
   - • 幂等性:相同订单号不可重复支付

2. 2. **支付密码**(高额订单):

   - • 订单金额>5000元需输入支付密码
   - • 支付密码6位数字
   - • 连续错误5次锁定

3. 3. **风控检查**:

   - • 异常支付行为:短时间大量支付
   - • 可疑IP:境外IP或代理IP
   - • 风险设备:异常设备指纹
   - • 盗刷检测:银行卡盗刷识别

**支付通知**:

1. 1. **支付成功通知**:

   - • 短信:"您的车险保单已生成,保单号:XXX,生效时间:XXX"
   - • APP推送:"支付成功,保单已生成"
   - • 邮件:发送电子保单PDF

2. 2. **保单配送**:

   - • 电子保单:立即发送到邮箱和APP
   - • 纸质保单:可选快递配送(部分地区)
   - • 发票:电子发票或纸质发票

**订单管理**:

1. 1. **订单查询**:

   - • 用户可查询所有订单
   - • 订单状态:待支付、已支付、已取消、已退款
   - • 订单详情:保单信息、支付信息

2. 2. **订单取消**:

   - • 待支付订单可取消
   - • 已支付订单需联系客服退款

3. 3. **订单重试**:

   - • 支付失败可重新支付
   - • 订单有效期内可多次尝试

#### 5.6.5 技术实现要点

1. 1. **支付集成**:

   - • 统一支付网关:封装各支付渠道
   - • 支付工厂模式:根据支付方式选择不同实现
   - • 支付参数签名:SHA256或RSA签名
   - • 回调验签:验证支付平台回调的真实性
   - • 幂等性保证:相同订单号不可重复支付

2. 2. **出单对接**:

   - • 调用保险公司出单接口
   - • 请求参数:完整的投保数据
   - • 接口加密:敏感数据加密传输
   - • 异步出单:出单可能需要数秒到数分钟
   - • 出单回调:保险公司出单完成后回调
   - • 重试机制:出单失败自动重试3次

3. 3. **定时任务**:

   - • 订单超时关闭:每分钟扫描超时未支付订单,自动关闭
   - • 出单状态同步:每小时同步保险公司出单状态
   - • 保单有效期提醒:保单到期前30天、15天、7天、3天提醒续保

4. 4. **消息通知**:

   - • 短信通知:支付成功、保单生成
   - • 邮件通知:发送电子保单PDF
   - • APP推送:实时推送订单状态
   - • 站内信:保存通知记录

5. 5. **数据安全**:

   - • 订单加密:订单金额等敏感字段加密
   - • 支付密码:BCrypt加密存储
   - • 日志脱敏:支付日志脱敏处理
   - • 审计日志:记录所有支付操作

6. 6. **性能优化**:

   - • 订单缓存:订单信息缓存到Redis
   - • 异步出单:出单请求异步处理,提升响应速度
   - • 分库分表:订单表按月分表
   - • 读写分离:查询走从库

#### 5.6.6 注意事项

- • 支付金额需与核保金额一致,不可篡改
- • 支付成功后不可修改或取消,只能退保
- • 电子保单与纸质保单具有同等法律效力
- • 保单生效时间一般为支付后的次日零时
- • 部分地区要求48小时后生效,需提前告知用户
- • 支付数据需留档备查,保存期限5年以上
- • 遵守支付监管要求,防范洗钱等风险

## 二、非车险投保模块

非车险包括健康险、寿险、意外险、财产险等,投保流程与车险不同,需要采集被保人健康信息、受益人信息,并进行健康告知和电子签名。

### 6.1 被保人信息

**开发工时**: 前端1天 + 后端1天

#### 6.1.1 功能概述

采集被保险人的基本信息、健康状况、职业信息等,为核保和理赔提供依据。被保人可以是本人,也可以是他人(如父母、配偶、子女)。

#### 6.1.2 核心业务逻辑

**被保人信息采集**:

1. 1. **基本信息**:

   - • 姓名:中文姓名,2-10个字符
   - • 性别:男/女
   - • 出生日期:YYYY-MM-DD
   - • 年龄:根据出生日期自动计算
   - • 证件类型:身份证、护照、出生证明(儿童)
   - • 证件号码:
     - • 身份证:18位
     - • 护照:8-9位
     - • 出生证明:字母+数字
   - • 婚姻状况:未婚、已婚、离异、丧偶
   - • 学历:小学、初中、高中、大专、本科、硕士、博士
   - • 年收入:选择区间,如<5万、5-10万、10-20万、>50万

2. 2. **联系信息**:

   - • 手机号:11位,接收保单和理赔通知
   - • 邮箱:用于接收电子保单
   - • 通讯地址:省市区+详细地址
   - • 紧急联系人:姓名、关系、电话

3. 3. **职业信息**:

   - • 工作单位:公司名称,可选
   - • 职业类别:
     - • 一类:办公室文员、教师、公务员等(低风险)
     - • 二类:外勤业务员、司机等(中低风险)
     - • 三类:工厂工人、装修工等(中风险)
     - • 四类:高空作业、电工等(中高风险)
     - • 五类:矿工、爆破工等(高风险)
     - • 六类:拒保职业(如特技演员、试飞员)
   - • 具体职业:选择或手动填写
   - • 职业风险系数:影响保费,高风险职业保费更高

4. 4. **健康信息**(健康险/寿险):

   - • 身高:单位cm
   - • 体重:单位kg
   - • BMI:自动计算,BMI = 体重(kg) / 身高(m)²
   - • 吸烟情况:是/否,每天吸烟量
   - • 饮酒情况:是/否,每周饮酒量
   - • 运动习惯:选择频率
   - • 健康状况:自评,良好/一般/较差

5. 5. **与投保人关系**:

   - • 本人:投保人即被保人
   - • 配偶:需提供结婚证
   - • 父母:需提供户口本或出生证明
   - • 子女:需提供出生证明或户口本
   - • 其他:需说明关系并提供证明

**年龄限制**:

不同险种对被保人年龄有不同要求:

- • 重疾险:一般0-60岁
- • 医疗险:一般0-65岁
- • 寿险:一般18-60岁
- • 意外险:一般0-80岁
- • 儿童险:0-17岁

超过年龄限制无法投保,系统需提前校验并提示。

**职业限制**:

1. 1. **职业风险等级**:

   - • 1-4类:可正常承保
   - • 5类:加费承保或限制保额
   - • 6类:拒保

2. 2. **高危职业特殊处理**:

   - • 需人工核保
   - • 可能需要额外体检
   - • 保费上浮20%-50%
   - • 部分责任除外

**健康指标判断**:

1. 1. **BMI指数**:

   - • 偏瘦(<18.5):可能需补充说明
   - • 正常(18.5-24):正常承保
   - • 超重(24-28):可能加费
   - • 肥胖(>28):需健康告知或体检

2. 2. **吸烟饮酒**:

   - • 不吸烟不饮酒:正常承保
   - • 少量吸烟/饮酒:可正常承保
   - • 大量吸烟/饮酒:加费或需体检

**证件上传**:

1. 1. **必传证件**:

   - • 被保人身份证正反面
   - • OCR识别:姓名、证件号、出生日期
   - • 自动校验:与手填信息对比

2. 2. **关系证明**(非本人投保):

   - • 配偶:结婚证
   - • 父母:户口本或出生证明
   - • 子女:出生证明或户口本

3. 3. **其他材料**(视情况):

   - • 体检报告(高保额或年龄较大)
   - • 病历资料(有既往病史)
   - • 收入证明(高保额寿险)

#### 6.1.5 技术实现要点

1. 1. **年龄计算**:

   - • 根据出生日期自动计算年龄
   - • 精确到天:当前日期 - 出生日期
   - • 周岁计算:满365天为1周岁
   - • 实时更新:定时任务每日更新年龄

2. 2. **BMI计算**:

   - • 公式:BMI = 体重(kg) / [身高(m)]²
   - • 自动计算:输入身高体重后实时计算
   - • 分类标准:
     - • <18.5:偏瘦
     - • 18.5-23.9:正常
     - • 24-27.9:超重
     - • ≥28:肥胖

3. 3. **职业匹配**:

   - • 职业库:预置数千种职业
   - • 模糊搜索:支持关键词搜索职业
   - • 智能推荐:根据行业和岗位推荐
   - • 风险评估:自动评估职业风险等级

4. 4. **数据回填**:

   - • 历史数据:之前投保过,自动回填信息
   - • OCR识别:上传身份证自动填充
   - • 默认值:部分字段设置合理默认值
   - • 用户确认:回填数据需用户确认

5. 5. **数据加密**:

   - • 敏感字段加密:身份证号、手机号等
   - • AES加密:对称加密,密钥存储在配置中心
   - • 脱敏显示:前端显示时中间位用*代替
   - • 日志脱敏:记录日志时自动脱敏

6. 6. **数据校验**:

   - • 前端校验:实时校验,立即反馈
   - • 后端校验:二次校验,防绕过
   - • 业务校验:年龄、职业、BMI等业务规则
   - • 完整性校验:必填字段检查

### 6.2 受益人信息

**开发工时**: 前端0.5天 + 后端0.5天

#### 6.2.1 功能概述

指定保险金的受益人,包括生存受益人和身故受益人,明确保险金的领取权利,避免理赔纠纷。

#### 6.2.2 核心业务逻辑

**受益人类型**:

1. 1. **生存受益人**:

   - • 适用产品:年金险、分红险等
   - • 被保人生存期间领取保险金
   - • 通常为被保人本人
   - • 部分产品可指定其他人

2. 2. **身故受益人**:

   - • 适用产品:寿险、意外险等
   - • 被保人身故后领取保险金
   - • 可指定一人或多人
   - • 可指定受益顺序和受益比例

**受益人选择**:

1. 1. **法定受益人**:

   - • 默认选项,无需指定具体人员
   - • 受益顺序:
     - • 第一顺序:配偶、子女、父母
     - • 第二顺序:兄弟姐妹、祖父母、外祖父母
   - • 同一顺序继承人平分
   - • 优点:简单,无需提供证明材料
   - • 缺点:可能引发继承纠纷

2. 2. **指定受益人**:

   - • 用户主动指定,明确受益人
   - • 可指定1-5人
   - • 需填写:姓名、性别、出生日期、证件号、与被保人关系
   - • 需上传:受益人身份证,关系证明(如结婚证、户口本)
   - • 优点:明确受益权,避免纠纷
   - • 缺点:需准备材料,流程稍复杂

**受益顺序**:

指定多个受益人时,可设置受益顺序:

1. 1. **第一受益人**:

   - • 优先受益
   - • 可指定1-5人
   - • 可设置受益比例,如配偶50%、子女各25%
   - • 比例之和必须为100%

2. 2. **第二受益人**:

   - • 第一受益人全部身故时受益
   - • 可指定1-5人
   - • 同样可设置受益比例

**受益比例**:

1. 1. **等额分配**:

   - • 受益人平均分配保险金
   - • 例如:3个受益人,各33.33%

2. 2. **自定义比例**:

   - • 用户自定义比例
   - • 例如:配偶60%,父母各20%
   - • 总和必须为100%

**受益人信息**:

1. 1. **基本信息**:

   - • 姓名:中文姓名
   - • 性别:男/女
   - • 出生日期:YYYY-MM-DD
   - • 证件类型:身份证、护照等
   - • 证件号码:18位身份证或护照号
   - • 与被保人关系:配偶、子女、父母、兄弟姐妹等

2. 2. **联系信息**:

   - • 手机号:用于理赔通知
   - • 邮箱:可选
   - • 地址:可选

3. 3. **证明材料**:

   - • 受益人身份证正反面
   - • 关系证明:
     - • 配偶:结婚证
     - • 子女:出生证明或户口本
     - • 父母:户口本或出生证明
     - • 其他:需说明关系并提供证明

**受益人变更**:

1. 1. **变更场景**:

   - • 受益人身故
   - • 受益人放弃受益权
   - • 投保人与受益人关系变化(如离婚)
   - • 投保人主动变更

2. 2. **变更流程**:

   - • 投保人提交变更申请
   - • 需被保人书面同意(如被保人非投保人)
   - • 提供新受益人信息和证明材料
   - • 保险公司审核通过后生效

3. 3. **变更限制**:

   - • 部分产品不可变更
   - • 已发生保险事故不可变更
   - • 变更后重新计算等待期

**特殊情况处理**:

1. 1. **受益人早于被保人身故**:

   - • 其受益份额按法定继承处理
   - • 或由其他受益人按比例分配

2. 2. **受益人与被保人同时身故**:

   - • 推定受益人后死亡
   - • 保险金作为被保人遗产处理

3. 3. **受益人故意造成被保人死亡**:

   - • 丧失受益权
   - • 份额由其他受益人分配

4. 4. **未成年受益人**:

   - • 保险金由监护人代为管理
   - • 需提供监护关系证明

### 6.3 健康告知

**开发工时**: 前端1.5天 + 后端1.5天

#### 6.3.1 功能概述

用户如实告知被保人的健康状况、既往病史、就诊记录等,保险公司据此评估承保风险,决定是否承保、是否加费或除外。健康告知是健康险、寿险的核心环节,关系到理赔权益。

#### 6.3.2 核心业务逻辑

**健康告知原则**:

1. 1. **如实告知义务**:

   - • 投保人、被保人必须如实回答所有问题
   - • 不得隐瞒、虚报健康状况
   - • 违反告知义务可能导致拒赔或解除合同

2. 2. **有限告知**:

   - • 只需回答问卷中的问题
   - • 未问及的健康状况无需主动告知
   - • 问卷设计应全面、清晰

**健康告知问卷**:

问卷内容因产品而异,一般包括:

1. 1. **基本健康状况**:

   - • 是否患有高血压、糖尿病、心脏病等常见慢性病?
   - • 是否有重大疾病病史(癌症、肝硬化、肾衰竭等)?
   - • 是否有遗传性疾病家族史?
   - • 是否有传染病(乙肝、艾滋病等)?

2. 2. **就诊记录**:

   - • 过去2年内是否住院治疗?
   - • 过去1年内是否连续服药超过30天?
   - • 是否曾被诊断为XX疾病?(列举具体疾病)
   - • 是否有待查明的症状或体征?

3. 3. **体检异常**:

   - • 近期体检是否有异常指标?
   - • 是否有以下异常:
     - • 血压≥140/90mmHg
     - • 血糖≥6.1mmol/L(空腹)
     - • 肝功能异常(转氨酶升高)
     - • 肾功能异常(肌酐升高)
     - • 尿蛋白阳性
     - • 肿瘤标志物升高
     - • 心电图异常

4. 4. **女性专属问题**:

   - • 是否怀孕?孕周?
   - • 是否有妇科疾病(子宫肌瘤、卵巢囊肿等)?
   - • 是否有乳腺疾病?

5. 5. **儿童专属问题**:

   - • 出生是否早产或低体重?
   - • 是否有先天性疾病?
   - • 发育是否正常?

6. 6. **其他问题**:

   - • 是否有残疾或功能障碍?
   - • 是否正在接受治疗或康复?
   - • 近期是否计划手术?

**答题方式**:

1. 1. **是/否选择**:

   - • 大部分问题为是否题
   - • 选"是"需补充说明详情

2. 2. **详情补充**:

   - • 疾病名称:确诊的疾病名称
   - • 诊断时间:首次确诊时间
   - • 治疗情况:是否治疗、治疗方式、治疗效果
   - • 当前状态:是否痊愈、是否复发、是否需要持续用药
   - • 就诊医院:三甲医院优先
   - • 上传资料:诊断证明、病历、检查报告等

3. 3. **智能提示**:

   - • 关键词提示:输入症状关键词,匹配相关疾病
   - • 常见病解释:鼠标悬停显示疾病说明
   - • 案例参考:提供类似情况的处理方式

**健康告知结果**:

1. 1. **标准体承保**:

   - • 所有问题均回答"否"
   - • 或虽有异常但不影响承保
   - • 正常保费,无除外

2. 2. **加费承保**:

   - • 存在轻微健康异常
   - • 保费上浮10%-50%
   - • 用户可选择接受或放弃

3. 3. **除外承保**:

   - • 存在特定疾病或异常
   - • 该疾病及相关并发症除外不保
   - • 其他保障正常
   - • 例如:甲状腺结节除外,其他保障正常

4. 4. **延期承保**:

   - • 当前健康状况不明确
   - • 建议治疗或观察一段时间后再投保
   - • 例如:肺部阴影待查,建议3个月后复查再投保

5. 5. **拒保**:

   - • 健康风险过高,无法承保
   - • 例如:癌症患者、严重心脏病患者

**核保审核**:

1. 1. **自动核保**:

   - • 所有问题回答"否"→ 自动通过
   - • 部分问题回答"是"但符合预设规则→ 自动加费或除外
   - • 90%以上的投保可自动核保,秒级出结果

2. 2. **人工核保**:

   - • 复杂病史无法自动判断→ 转人工
   - • 核保员查看详细资料
   - • 必要时要求补充材料或体检
   - • 1-5个工作日出结果

**补充告知**:

1. 1. **智能核保**:

   - • 针对特定健康问题的专项问卷
   - • 例如:乙肝智能核保、结节智能核保
   - • 根据回答自动给出核保结论
   - • 结论示例:"符合条件,加费20%承保"

2. 2. **人工预核保**:

   - • 上传病历资料
   - • 核保员预先评估
   - • 给出大致核保意见
   - • 不影响其他公司投保

**材料上传**:

如有健康异常,需上传:

- • 诊断证明:医院开具的诊断证明书
- • 病历资料:住院病历、门诊病历
- • 检查报告:影像学报告、化验单、病理报告
- • 用药记录:处方、用药清单
- • 体检报告:近期完整体检报告
