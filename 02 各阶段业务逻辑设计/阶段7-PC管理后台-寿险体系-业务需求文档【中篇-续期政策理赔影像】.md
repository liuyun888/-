## 1. 续期跟踪

### 1.1 功能概述

菜单：寿险 → 续期管理 → 续期跟踪

展示即将到期续缴的寿险保单清单，供业务员记录跟进情况，提升续期率。

### 1.2 续期跟踪列表

**列表数据来源：** 查询 `ins_policy_life` 中 `policy_status = ACTIVE` 且根据缴费方式计算出下次缴费日期在指定范围内的保单。

续期周期计算逻辑（`InsLifeRenewalService.calcNextDueDate`）：
- 年缴：`start_date + N年`（N = 当前是第几年）
- 半年缴：`start_date + N*6个月`
- 季缴：`start_date + N*3个月`
- 月缴：`start_date + N月`

**默认展示范围：** 未来60天内到期的保单（可通过筛选条件调整）。

**筛选条件：**

| 筛选项 | 类型 |
|---|---|
| 录入业务员 | 下拉搜索 |
| 险种 | 下拉多选 |
| 保险公司 | 下拉多选 |
| 缴费到期日期区间 | 日期范围选择器 |
| 跟进状态 | 枚举：未跟进/跟进中/已缴费/已拒缴/放弃 |

**列表字段：** 保单号、被保人、险种、保司、年度保费、缴费方式、下次缴费日、距到期天数、跟进状态、最后跟进时间、操作。

### 1.3 续期跟进记录录入

点击列表某行的【跟进】按钮，弹出跟进记录录入弹窗：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 联系时间 | ★必填 | 默认当前时间 |
| 联系方式 | ★必填 | 枚举：电话/微信/短信/上门 |
| 联系结果 | ★必填 | 枚举：无人接听/已联系-准备续缴/已联系-犹豫中/已联系-明确拒缴/已缴费 |
| 备注 | 选填 | 最多500字 |
| 下次跟进计划 | 选填 | 日期时间 |

保存后 INSERT `ins_life_renewal_track`（policy_id, contact_time, contact_type, contact_result, note, next_plan_time, operator_id）。

同时 UPDATE `ins_life_renewal.last_follow_time` 和 `follow_status`（若结果=已缴费则更新 status=已缴费）。

### 1.4 批量导入续期记录

场景：保司下发续期清单（含已缴费/未缴费状态），批量导入到系统。

操作：点击【批量导入】→ 下载续期导入模板（字段：保单号、续期状态、实缴金额、实缴日期） → 上传 → 后端按保单号匹配更新 `ins_life_renewal_track` 记录。

### 1.5 续期统计看板

两个核心指标（SQL聚合计算）：
- **继续率** = 当期实际缴费保单数 / 当期到期应续缴保单数 × 100%
- **断缴率** = 未缴费保单数 / 到期应续缴保单数 × 100%

按时间维度（月/季）和险种/保司维度筛选展示，数据展示在看板卡片+趋势折线图。

## 2. 数据回传

### 2.1 功能概述

菜单：寿险 → 数据回传

将系统中的寿险保单数据、续期数据按保司要求格式回传到保险公司或监管系统，满足合规监管要求。

### 2.2 保司接口配置（交互平台）

菜单：寿险 → 数据回传 → 交互平台

为每家保司配置数据回传接口：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 保险公司 | ★必填 | 下拉选择已配置的寿险合作保司 |
| 传输方式 | ★必填 | 枚举：API接口/SFTP文件传输 |
| 接口地址 | 条件必填 | 传输方式=API时必填，HTTP/HTTPS URL |
| SFTP地址 | 条件必填 | 传输方式=SFTP时必填，含IP/端口/路径 |
| 认证方式 | ★必填 | 枚举：API-KEY/用户名密码/证书 |
| AppID/用户名 | ★必填 | |
| AppKey/密码 | ★必填 | 加密存储（AES-256） |
| 数据格式 | ★必填 | 枚举：JSON/XML/CSV/固定格式Excel |
| 回传数据类型 | ★必填 | 多选：保单数据/续期数据/理赔数据 |
| 是否启用 | ★必填 | 开关，默认关闭 |

保存到 `ins_life_data_return_config` 表。

点击【测试连接】按钮：后端尝试连接对应接口（API则发一个探测请求，SFTP则尝试建立连接），返回"连接成功"或"连接失败（含错误原因）"。

### 2.3 执行数据回传

菜单：寿险 → 数据回传 → 数据回传

**手动触发回传：**

1. 选择保司、数据类型、回传时间段（起止日期）
2. 点击【开始回传】按钮
3. 后端异步执行（RocketMQ 异步，避免超时）：
   - 从 `ins_policy_life` 查询指定时间段内的保单数据
   - 按对应保司的配置格式进行数据转换（适配器模式：`InsLifeDataReturnAdapter` 接口，每个保司有独立实现类）
   - API方式：HTTP POST 到保司接口
   - SFTP方式：生成文件，SFTP上传
   - 记录回传结果到 `ins_life_data_return_log`（insurer_id, data_type, date_range, total_count, success_count, fail_count, status, error_detail, return_time）
4. 完成后发站内信通知操作人结果

**补传功能：** 针对某次失败的回传记录，点击【重试】按钮重新执行该批次回传。

**回传结果列表：** 展示 `ins_life_data_return_log` 表记录，状态：成功/失败/部分失败/待处理。

### 2.4 异常数据排查工具

对于"失败"或"部分失败"的回传记录，点击【查看详情】：
- 展示每条失败数据的保单号、错误字段、错误原因（从保司返回的错误信息解析）
- 支持下载失败数据明细 Excel
- 支持针对单条失败数据手动修正后重传

## 3. 理赔管理

### 3.1 功能概述

菜单：寿险 → 理赔管理

记录和跟进寿险理赔案件全生命周期。

### 3.2 新增理赔案件

点击【新增案件】按钮，弹出录入弹窗：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 保单号 | ★必填 | 输入后联查带出：被保人姓名、险种、保司、保额 |
| 出险时间 | ★必填 | 日期时间选择器 |
| 出险类型 | ★必填 | 枚举：身故/全残/重大疾病/轻症/住院/意外 |
| 出险经过 | ★必填 | 文本，最多2000字 |
| 赔案号 | 选填 | 保司分配的理赔流水号，可后续补录 |
| 已提交材料 | 选填 | 多选枚举：死亡证明/医疗报告/身份证/银行卡/其他 |
| 负责业务员 | ★必填 | 默认保单归属业务员，可修改 |

保存 INSERT `ins_claim_record`（policy_id, claim_no, accident_time, accident_type, accident_desc, case_no, submitted_docs, agent_id, status='PROCESSING', create_time）

### 3.3 案件跟进记录

在案件详情页，点击【添加跟进】：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 跟进时间 | ★必填 | 默认当前时间 |
| 处理进展 | ★必填 | 文本 |
| 保司补充材料要求 | 选填 | 记录保司要求补交的材料清单 |
| 附件 | 选填 | 上传相关材料，OSS存储 |

INSERT `ins_claim_follow_record`（claim_id, follow_time, progress, doc_require, attachment_urls, operator_id）

### 3.4 赔付结果录入

案件跟进到结案时，点击【录入赔付结果】：

| 字段名 | 是否必填 | 说明 |
|---|---|---|
| 赔付金额 | ★必填 | BigDecimal，精度2位小数 |
| 赔付日期 | ★必填 | 日期 |
| 赔付账号 | ★必填 | 收款银行卡/账号信息 |
| 赔付结果说明 | 选填 | 文本 |

保存后：
- UPDATE `ins_claim_record.status = SETTLED`、`claim_amount`、`claim_date`
- 若出险类型=身故，联动保单状态变更为 `LAPSE`（失效），并发站内信给业务员

### 3.5 案件查询

筛选：保单号、被保人姓名、案件状态（处理中/已结案/已拒赔）、出险类型、时间段。

### 3.6 统计看板

- 总案件数（本月/年）
- 平均处理天数（从开案到结案天数平均值）
- 总赔付金额
- 拒赔率
以上指标按险种/保司可下钻。

## 4. 机构费率政策

### 4.1 功能概述

菜单：寿险 → 政策管理 → 机构费率政策

为各机构配置从平台获得的寿险佣金费率（即下游分配费率的一种特殊形式，专针对机构层级）。

### 4.2 新增政策批次

点击【添加】→【单个新增】，弹出政策配置弹窗：

**政策基本条件（每个模板相同）：**

| 字段名 | 必填 | 说明 |
|---|---|---|
| 结算时间 | ★ | 枚举：月结/季结/年结/活动结束当月结算 |
| 费用类型 | ★ | 枚举：机构基础费率（模板1/5）/机构业绩费率（模板2）/续期服务津贴（模板4） |
| 机构 | ★ | 多选，从机构树选择，选中的机构才能匹配此政策 |
| 保险公司 | ★ | 多选 |
| 产品名称 | 选填 | 多选（不选则代表该保司所有产品均适用） |
| 比例基数 | ★ | 枚举：保费/标保/佣金+奖金 |
| 开始时间 | ★ | 政策生效开始日期 |
| 结束时间 | 选填 | 留空表示永久有效 |
| 政策名称 | ★ | 自定义名称，最多50字 |

**费率明细表格（页面内嵌可编辑表格）：**

| 列名 | 说明 |
|---|---|
| 职级/档位 | 业务员/机构职级，如初级/中级/高级 |
| 首年佣金率 | 百分比，精度到0.01% |
| 续年佣金率 | 百分比 |
| 增值服务佣金率 | 百分比（选填） |
| 有效期 | 该行费率的有效时间段 |

表格支持行内编辑，底部【添加一行】按钮，单行支持删除。

### 4.3 批量新增政策

点击【添加】→【批量新增】：下载批量政策模板 → 填写 → 上传。EasyExcel解析，批量 INSERT `ins_life_policy_rate`。

### 4.4 政策审批

新建/修改政策后状态为`待审批`，需进入 Flowable 审批流（`LIFE_POLICY_RATE_APPROVE`）：
- 审批通过：UPDATE `status = ACTIVE`，政策生效
- 审批拒绝：status = REJECTED，通知申请人

**版本管理：** 同一机构+险种+保司只能有一条 `ACTIVE` 的政策。新政策审批通过时，旧政策自动变更为 `EXPIRED`。

## 5. 上游政策配置

### 5.1 功能概述

菜单：寿险 → 政策管理 → 上游政策配置

配置各保司支付给平台的佣金费率（即保司→平台的收入来源）。

### 5.2 新增上游政策

点击【添加】，弹出配置弹窗：

| 字段名 | 必填 | 说明 |
|---|---|---|
| 政策类型 | ★ | 枚举：首期政策/续期政策 |
| 结算时间 | ★ | 枚举：月结/季结/年结/活动结束当月结算 |
| 费用类型 | ★ | 枚举：基础首期佣金/销售奖金/首期其他（支持自定义输入） |
| 机构 | ★ | 多选，限制哪些机构的保单可匹配此政策 |
| 保险公司 | ★ | 单选 |
| 产品名称 | 选填 | 多选 |
| 比例基数 | ★ | 枚举：保费/佣金+奖金/标保/FYC（注：首期/续期政策不支持选标保作为比例基数） |
| 开始时间 | ★ | |
| 结束时间 | 选填 | 永久时留空 |
| 政策名称 | ★ | |

**汇率因子（档位明细）：**
- 页面内嵌表格，最多支持100条
- 每行：起始保费档位 | 结束保费档位 | 佣金率（%）
- 支持单条删除、中间插入新行

**缴费因子：**
- 勾选适用的缴费方式限制条件（如仅适用月缴/年缴等）

点击【保存】→ INSERT `ins_life_upstream_policy` + `ins_life_upstream_policy_factor`，status = `PENDING`（待审批）。

### 5.3 政策审批

同机构费率政策审批流程，审批通过后生效，版本互斥（同保司/险种只能一条生效中的上游政策）。

支持【撤销申请】：仅 `PENDING` 状态可撤销，撤销后 status = `CANCELED`。

## 6. 下游政策配置

### 6.1 功能概述

菜单：寿险 → 政策管理 → 下游政策配置

配置平台向业务员/机构分配的寿险佣金政策（即平台→业务员的分配规则）。

### 6.2 新增下游政策

点击【添加】，弹出配置弹窗：

| 字段名 | 必填 | 说明 |
|---|---|---|
| 政策类型 | ★ | 枚举：直销佣金政策/基本法额外奖励政策/代理人奖励政策 |
| 机构 | ★ | 多选 |
| 保险公司 | ★ | 单选 |
| 产品名称 | 选填 | 多选 |
| 比例基数 | ★ | 枚举：保费/标保 |
| 开始时间 | ★ | |
| 结束时间 | 选填 | |
| 政策名称 | ★ | |

**直销政策扩展字段：**
- 选择适用的基本法（下拉，从已配置的寿险基本法中选择）

**汇率因子（同上游政策）：** 档位+佣金率表格，最多100条，支持删除/中间新增。

**缴费因子：** 勾选适用缴费方式限制。

**后端校验（上下游利润空间校验）：**
- 保存时校验：下游分配比例不得超过对应上游政策的佣金率（避免倒挂亏损）
- 若下游率 > 上游率，弹出警告"下游佣金率高于上游，请确认是否继续"，用户二次确认才可保存

### 6.3 政策审批

同上游政策，Flowable 多级审批：机构负责人 → 平台管理员。审批通过生效，版本互斥控制。

## 7. 折标系数

### 7.1 功能概述

菜单：寿险 → 政策管理 → 折标系数

配置各险种/缴费方式的折标系数，用于将实际保费折算为标准保费（标保），是业绩统计和档位计算的基础。

折标公式：`标保 = 年度保费 × 折标系数`

### 7.2 新增折标系数政策

点击【添加】，弹出配置弹窗：

| 字段名 | 必填 | 说明 |
|---|---|---|
| 政策类型 | 只读 | 固定：折标政策 |
| 机构 | ★ | 多选 |
| 保险公司 | ★ | 单选 |
| 产品名称 | 选填 | 多选（不选则对应保司所有产品） |
| 折标对象 | ★ | 枚举：上游折标/下游折标/上下游折标 |
| 开始时间 | ★ | |
| 结束时间 | 选填 | |
| 政策名称 | ★ | |

**折标明细表格（行内编辑）：**

| 列名 | 说明 |
|---|---|
| 缴费方式 | 枚举：趸缴/年缴/半年缴/季缴/月缴 |
| 折标系数 | 小数，精度4位，如 1.0000/0.1000 |

每种缴费方式单独配置折标系数，趸缴通常=0.1（即10年期折算），年缴通常=1.0。

保存后 INSERT `ins_life_std_ratio`，同样需要审批流，审批通过才生效。

## 8. 保司结算时间

### 8.1 功能概述

菜单：寿险 → 政策管理 → 保司结算时间设置

配置每家保司的佣金结算周期和结算日，用于佣金到账时间预测。

### 8.2 配置项

| 字段名 | 必填 | 说明 |
|---|---|---|
| 保险公司 | ★ | 下拉，选择寿险合作保司 |
| 结算周期 | ★ | 枚举：月结/季结/年结 |
| 结算日 | ★ | 数字，月结时填1-28（代表每月几号结算）；季结时填每季末月的具体日期 |

| 备注 | 选填 | |

保存 UPSERT `ins_life_insurer_settlement_config`（已存在则更新，不存在则插入）。

**批量设置：** 支持导出当前配置列表到 Excel，修改后重新导入批量更新。

**在佣金预测中的应用：** 当系统计算"预计到账日期"时，读取此配置：`预计到账日 = 结算日 + 账期天数`。

## 9. 客户维护与影像管理

### 9.1 功能概述

菜单：寿险 → 客户维护/影像管理

寿险专用客户管理入口。数据与通用 CRM（`ins_customer` 表）共享，但寿险入口有额外的寿险专属字段，且支持影像件管理。

### 9.2 客户查询

筛选条件：姓名（模糊）、证件号（精确）、保单号（关联查询）、标签。

列表字段：姓名、证件号（脱敏）、手机号（脱敏）、关联保单数、最近保单日期、影像文件数量、操作。

### 9.3 新增寿险客户

点击【新增客户】，弹出录入弹窗：

**基础字段（与通用 CRM 共用 `ins_customer` 表）：**
姓名★、证件类型★、证件号★（AES加密存储）、出生日期★、性别★、手机号、邮箱、地址

**寿险专属字段（存入 `ins_customer_life_ext` 扩展表）：**

| 字段名 | 必填 | 说明 |
|---|---|---|
| 健康告知历史 | 选填 | 文本，记录曾申报的健康状况 |
| 既往症 | 选填 | 文本，已知既往疾病描述 |
| 职业类型 | 选填 | 枚举：1类职业/2类职业/3类职业/4类职业（影响投保核保） |
| 是否高净值客户 | 选填 | 开关 |
| 来源渠道 | 选填 | 枚举：业务员开发/转介绍/活动获取/C端自注册 |

保存时：
1. 检查证件号是否已存在 `ins_customer`（按解密后证件号匹配）
2. 已存在则提示"该客户已存在，是否合并？"
3. 不存在则 INSERT `ins_customer` + INSERT `ins_customer_life_ext`

### 9.4 批量导入客户

点击【批量导入】→ 选择导入格式（标准格式/保司导出格式）：

**保司导出格式适配：** 不同保司导出的客户 Excel 列名和顺序不同，系统需维护字段映射表（`ins_insurer_import_field_mapping`）：保司名称 + 保司字段名 → 系统字段名。

导入逻辑：
1. 按已配置的字段映射表进行列名转换
2. 逐行去重检查（按证件号）
3. 批量 INSERT（每批500条），已存在的客户跳过（记录到日志）
4. 生成导入报告

### 9.5 导出客户名单

支持两种格式：
- **标准格式**：系统标准字段，EasyExcel导出，证件号脱敏
- **自定义格式**：用户勾选需要导出的字段，系统按选择生成动态表头

### 9.6 影像管理

**影像分类树（`ins_image_category` 表，可配置）：**
```
├── 身份证
│   ├── 正面
│   └── 反面
├── 授权书
├── 回执书
├── 健康告知书
├── 医疗报告
└── 其他
```

**为客户上传影像：**
1. 进入客户详情页，点击【影像管理】Tab
2. 选择影像分类 → 点击【上传】→ 选择文件（图片或PDF，单文件≤20MB）
3. 上传到 OSS，路径：`/life-customer/{customer_id}/{category_code}/{yyyyMM}/{filename}`
4. INSERT `ins_customer_image`（customer_id, category_id, file_name, oss_url, expire_date, upload_time, operator_id）

**影像有效期管理：**
- 身份证影像有效期：上传时填写身份证有效期（`expire_date` 字段）
- XXL-Job 定时任务每天检查：距过期 ≤ 30天的影像，发站内信提醒业务员"客户XX的身份证影像将于XX日到期，请及时更新"

**查看影像：**
- OSS URL 直接访问（使用阿里云 OSS 签名URL，有效期1小时，避免直接暴露URL）
- 点击【预览】：图片直接在页面预览；PDF在新标签页打开

**删除影像：**
- 删除操作需有特定权限（`LIFE_IMAGE_DELETE` 权限码）
- 软删除：UPDATE `is_deleted = 1`，OSS 文件保留（可通过配置决定是否物理删除）
