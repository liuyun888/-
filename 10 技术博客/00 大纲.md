## 第003篇：车险的前世今生——从商业车险改革到报价引擎

**副标题**：了解费率形成机制，才能设计好报价系统

### 知识点清单
- **车险产品结构**：
  - 交强险：强制购买，固定费率，由国家统一定价
  - 商业车险：自主定价，主险+附加险组合
- **主险六大险种**：
  - 机动车损失险（车损险）
  - 第三者责任险（三者险）
  - 车上人员责任险（座位险）
  - 全车盗抢险
  - 自燃损失险
  - 玻璃单独破碎险
- **商业车险费率改革历程**：
  - 2015年费改：引入NCD系数（无赔款优待）
  - 2020年"二次费改"：取消折扣优惠区间，扩大自主定价系数
  - 报行合一：保险公司手续费必须与报备费率一致
- **定价公式**：`保费 = 基准纯风险保费 × (1 + 附加费用率) × 自主定价系数 × NCD系数`
- **NCD系数详解**：连续几年未出险可以打多少折？出险一次上浮多少？
- **承保核保**：高风险车辆（改装车、营运车）的拒保逻辑
- **对系统设计的影响**：为什么需要并发请求多家保司？费率为什么每天都可能变化？

## 第005篇：寿险的复杂性——为什么寿险保单需要独立的微服务模块

**副标题**：一张保单管理30年，技术系统如何应对？

### 知识点清单
- **寿险产品类型**：
  - 普通寿险（定期寿险、终身寿险）
  - 两全保险：储蓄+保障二合一
  - 分红保险：红利不确定，监管要求特殊说明
  - 万能保险：保费灵活，账户价值每月结算
  - 投连险：与基金挂钩，高风险高收益
- **寿险的关键业务节点**（这些都需要对应的系统功能）：
  - 投保 → 核保 → 承保 → 生效 → 回访（监管要求）
  - 犹豫期退保（通常10个工作日内，可全额退保）
  - 宽限期：保费缴纳延迟60天内，保单依然有效
  - 保单贷款：以保单现金价值为抵押
  - 减保：降低保额，部分退还保费
  - 保全变更：受益人变更、联系方式变更等
  - 孤儿单处理：业务员离职后的保单管理
  - 年度续期提醒
  - 满期给付
  - 理赔
- **寿险财务特殊性**：
  - 续期佣金（RYC）：保单持续有效，业务员每年继续获得佣金
  - 继续率考核：续保率低会影响业务员职级

## 第008篇：保险佣金机制——"基本法"是什么？为什么难以实现？

**副标题**：FYC、RYC、Override、Bonus，四种佣金类型的完整解析

### 知识点清单
- **佣金术语完全解析**：
  - **FYC**（First Year Commission）：首年佣金，新单承保后按保费比例发放
  - **RYC**（Renewal Year Commission）：续期佣金，长险每年续期后发放
  - **Override**（管理津贴）：上级从下级业绩中提取的管理佣金
  - **Bonus**（绩效奖金）：季度/年度达标奖励
  - **ORD**（Overall Rate of Distribution）：综合手续费率
- **基本法的核心逻辑**：
  - 职级体系：业务员→主管→部经理→区域总监
  - 晋升条件：FYP达标 + 团队人数 + 持续月数
  - 维持条件：不达标则降级
- **多级分润举例**：
  ```
  业务员出一单保费10000元，FYC率25% = 2500元佣金
  主管Override率5% = 125元管理津贴
  部经理Override率3% = 75元管理津贴
  ```
- **特殊佣金规则**：
  - 分红险佣金分期发放（首年70%，余下5年各6%）
  - 退保回收：犹豫期内退保，佣金全额追回
  - 保单失效：续期未缴费导致保单失效，停止发放RYC
- **"报行合一"政策的影响**：
  - 2023年开始严格执行
  - 实际手续费率不能超过向监管报备的费率
  - 对系统的要求：佣金率上限校验 + 报备费率维护
- **为什么佣金系统是平台的"命脉"**：业务员留存与佣金透明度的关系

  - 手机号：`phone_hash VARCHAR(64)` + `phone_encrypted VARCHAR(200)`

  ```java
  // 核心代码结构
  List<CompletableFuture<QuoteResult>> futures = enabledInsurers.stream()

          .supplyAsync(() -> doQuote(insurer, request), quoteThreadPool)
          .orTimeout(3, TimeUnit.SECONDS)
          .exceptionally(e -> QuoteResult.fail(insurer, e.getMessage())))
  ```

  ```
  阶段1: 精确匹配（保单号完全一致 + 保费金额一致）
  阶段2: 模糊匹配（保单号一致 + 保费金额在±0.01元误差内）
  阶段3: 未匹配项进入人工处理队列
  ```
- **差异分类处理**：
  - 保司多报：系统无此保单 → 核查后补录或标记异常
  - 保司漏报：系统有但保司未结算 → 催促保司补报
  - 金额差异：佣金金额不一致 → 核查费率差异原因
- **"报行合一"合规校验**：`实际费率 ≤ 报备费率`
- **对账报告生成**：汇总差异，Excel/PDF格式导出

  - 存的不是视频本身，而是视频文件的SHA256哈希值
  - 哈希值上链后，任何对视频的篡改都会导致哈希值变化

- **蚂蚁链接入流程**：
  1. 开通蚂蚁链存证服务
  2. 获取AppId、AppKey
  3. 构建存证请求（文件哈希 + 业务标识 + 时间戳）
  4. 调用存证API
  5. 返回：区块链交易ID（txId）+ 区块高度 + 时间戳
  6. 保存到数据库 `ins_blockchain_evidence`
- **存证证书生成**：
  - PDF格式：含文件哈希、TxId、区块高度、时间戳
  - 在线验证URL：可以通过URL在蚂蚁链验证存证有效性
- **多文件存证**：
  - 每个双录视频单独存证
  - 行为轨迹JSON打包后存证
  - 所有存证关联到同一保单ID
- **存证失败处理**：
  - 区块链网络偶发故障 → 进入重试队列（RocketMQ延迟消息）
  - 最多重试5次，仍失败则人工处理

  - 年缴保单：每年到期自动发送缴费提醒
  - 月缴保单：每月发送缴费提醒（可配置提前天数）
  - 宽限期处理：60天宽限期内保单状态为"宽限期"
  - 宽限期过后未缴费：保单失效，进入失效管理流程
- **孤儿单管理**：
  - 触发条件：业务员离职/被取消代理资格
  - 自动转移规则：按地区 + 险种 + 业绩平衡分配
  - 孤儿单佣金：转移后RYC归属规则

## 第095篇：寿险理赔系统——身故/重疾/残疾理赔的差异化处理

**副标题**：寿险理赔比车险复杂10倍，系统如何支撑

### 知识点清单
- **寿险理赔的复杂性**：
  - 身故理赔：需要死亡证明、受益人身份核验、法定继承权
  - 重疾理赔：需要诊断证明、病历、出院总结
  - 残疾理赔：需要残疾鉴定、等级评定
  - 住院医疗：需要住院病历、发票清单
- **理赔材料管理**：
  - 动态材料清单：根据理赔类型自动显示所需材料
  - 材料上传：App拍照上传，OCR自动提取关键信息
  - 材料审核：核查完整性和真实性
- **理赔金额计算**：
  - 重疾理赔：固定赔付（保额×赔付比例）
  - 医疗险理赔：实报实销，扣除免赔额 + 按比例报销
  - 多次赔付险：每次赔付后剩余保额
- **理赔审批流程（Flowable）**：
  - 小额理赔（<5万）：系统自动审批
  - 中额（5-50万）：主管审批
  - 大额（>50万）：总监 + 合规双审批
- **代位追偿**：第三方责任事故中，保司赔付后向责任方追偿的系统支持

## 第098篇：保险系统的压测方案——模拟1万人同时在线投保

**副标题**：JMeter压测场景设计 + 性能瓶颈定位 + 优化方案

### 知识点清单
- **压测指标定义**：
  - TPS（Transactions Per Second）：每秒事务数
  - RT（Response Time）：响应时间
  - P99：99%的请求在N毫秒内响应
  - 错误率：应 < 0.1%
- **保险系统的关键压测场景**：
  - 报价接口：并发100，P99 < 3000ms
  - 投保提交：并发50，P99 < 2000ms
  - 保单列表查询：并发200，P99 < 500ms
  - 双录视频上传：并发20，成功率 > 99%
- **JMeter场景设计**：
  - 用户行为模拟：登录 → 浏览产品 → 发起报价 → 投保
  - 数据准备：预先准备1000个测试账号
  - 阶梯递增：从10并发增加到100并发
- **性能瓶颈定位工具**：
  - SkyWalking：慢请求链路追踪
  - Arthas：在线定位热点代码
  - MySQL slow query log：慢查询分析
- **常见优化方案**：
  - 增加缓存命中率（报价结果缓存）
  - SQL优化（增加索引、优化JOIN）
  - 增加服务实例（水平扩容）
